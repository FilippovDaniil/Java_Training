## 📚 О репозитории

Это **учебный** репозиторий: не библиотека и не приложение, а **тренажёр**. В каждой теме — объяснение и набор задач-каркасов с пометками `// TODO`. **Готовых решений нет** — код пишет студент. Прогресс идёт по нарастающей: от `System.out.println` до микросервисов, от пузырьковой сортировки до Ахо-Корасик, от одного класса до гексагональной архитектуры с DDD.

Все три раздела построены по одной переносимой методике — [`LEARNING-METHODOLOGY.md`](LEARNING-METHODOLOGY.md):

> **Знание дробится на маленькие темы; каждая тема = короткая теория + набор задач с нарастающей сложностью; задачи даются без готовых решений; каждый результат изолирован, дописывается по частям и объективно проверяется.**

## 🧩 Анатомия задачи (единая для всех разделов)

Каждая задача — это **условие + пустой каркас**, который дописывает студент. Условие лежит в JavaDoc-шапке точки входа:

```java
/**
 * Задача NN — Тема NN: <название>
 *
 * ЗАДАНИЕ:        что нужно сделать (1–3 предложения).
 * ПРИМЕР / ВЫВОД: как выглядит правильный результат.
 * ТРЕБОВАНИЯ:     ограничения и обязательные условия.
 * ПОДСКАЗКА:      направление, но НЕ ответ.
 */
public class TaskNN {
    public static void main(String[] args) {
        // TODO: ваш код
    }
}
```

**Две оси нарастающей сложности:** внутри темы (задачи `01 → 07`, где 7-я — мини-проект) и между темами (от азов к продвинутому уровню).

### Структура темы

```
<раздел>/
├── README.md                  # карта раздела
└── module-NN-topic/           # одна папка на тему
    ├── theory.md              # объяснение + схемы + «подводные камни» + «что дальше»
    └── practice/
        ├── Task01.java        # задача из одного класса
        ├── ...
        └── Task07/            # задача с несколькими типами → отдельная папка
            ├── Task07.java    #   точка входа + условие в JavaDoc
            └── ...            #   вспомогательные типы — каждый своим файлом
```

> **Конвенция:** один top-level класс — файл `TaskNN.java`; несколько типов — папка `TaskNN/`. Пакет не объявляется (default package).

## ▶️ Как запускать

Большинство задач Core Java и весь `algorithms-course` / `patterns-architecture` собираются напрямую компилятором:

```bash
# задача из одного файла
javac -encoding UTF-8 TaskNN.java
java TaskNN

# задача-папка (все файлы вместе)
javac -encoding UTF-8 -d out module-NN-.../practice/TaskNN/*.java
java -cp out TaskNN
```

Темы со Spring / Hibernate / Testcontainers и тест-классы (JUnit) запускаются через **Gradle** или прямо из **IntelliJ IDEA** (▶ рядом с `main` или тест-классом).

> ⚙️ **Окружение:** JDK 17 LTS · IntelliJ IDEA (Community достаточно) · Docker — для модулей контейнеризации `java-course`.

## 🚀 Как учиться

1. **Идите сверху вниз** — темы выстроены по нарастанию сложности.
2. **Сначала теория** (`theory.md`): разберите примеры, схемы и блок «подводные камни».
3. **Потом практика** (`Task01 → Task07`): пишите сами, подглядывая в подсказку, а не в ответ.
4. **Ведите сквозные проекты** — к финалу из мелких задач собирается реальная система.
5. **Проверяйте объективно** — компиляция, запуск, тесты, ожидаемый вывод в шапке задачи.

## 📐 Принципы методики

| # | Принцип | Зачем |
|---|---------|-------|
| 1 | Атомарная единица = одна тема | Пройти за один присест, не утонув |
| 2 | Теория → практика в каждой теме | Сразу закрепляем прочитанное |
| 3 | Фиксированный набор задач (5–7) | Предсказуемый ритм, рост сложности |
| 4 | Задача без готового решения | Учится тот, кто пишет сам |
| 5 | Последняя задача — мини-проект | Синтез темы в одно целое |
| 6 | Изоляция и самодостаточность | Любую задачу можно сделать/проверить отдельно |
| 7 | Сквозные проекты | Изолированные темы собираются в реальную систему |
| 8 | Объективная проверяемость | Компиляция/запуск/тест дают честное «да/нет» |

Полное описание и чек-лист переноса методики на другой язык/предмет — в [`LEARNING-METHODOLOGY.md`](LEARNING-METHODOLOGY.md).
