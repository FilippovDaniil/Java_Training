# Алгоритмы и структуры данных на Java

Третий раздел репозитория — рядом с [`../java-course`](../java-course/README.md) и [`../patterns-architecture`](../patterns-architecture/README.md). Если основной курс ведёт **от синтаксиса до production-микросервиса**, а второй развивает **архитектурное мышление**, то этот раздел строит **алгоритмический фундамент**: анализ сложности → базовые структуры → сортировки и поиск → деревья/хеши/кучи → графы → динамическое программирование, строки и сжатие.

Построен строго по [`../LEARNING-METHODOLOGY.md`](../LEARNING-METHODOLOGY.md): **маленькая тема → короткая теория → задачи без готовых решений → проверяемый результат → следующая тема.**

- **25 атомарных тем**, сквозная нумерация `01 → 25` (Big O → продвинутые алгоритмы и сжатие).
- На каждую тему — `theory.md` и **5–7 задач** (`Task01 → Task07`), седьмая — мини-проект сквозного проекта Data-Cruncher.
- **Готовых решений нет:** условие в JavaDoc-шапке + каркас с `// TODO`. Код пишет ученик.
- **Один сквозной проект** — Data-Cruncher — проходит через весь раздел (см. ниже).

> Статус генерации тем (что уже готово, что нет) — в [`PROGRESS.md`](PROGRESS.md). Ссылки на `theory.md` ниже резолвятся по мере наполнения батчами.

## 📁 Структура раздела

```
algorithms-course/
├── README.md                 # этот файл — навигация по разделу
├── PROGRESS.md               # статус генерации тем по батчам + чек-лист всех задач
└── module-NN-topic-name/      # одна папка на тему
    ├── theory.md              # объяснение + схемы + «подводные камни» + «что дальше»
    └── practice/              # задачи для самостоятельного решения
        ├── Task01.java        # простая задача из одного класса
        ├── ...
        └── Task07/            # задача со множеством типов → отдельная папка
            ├── Task07.java    #   точка входа (main/тест) + условие в JavaDoc
            ├── DataCruncher.java  # вспомогательные типы — каждый своим файлом
            └── ...
```

### Конвенции (унаследованы от `java-course` и `patterns-architecture`)

- Задача в один класс — файл `TaskNN.java`. Несколько top-level типов — папка `TaskNN/`, точка входа `TaskNN.java`, остальные типы — отдельными файлами.
- **Пакет не объявляется** (default package). Файлы внутри папки компилируются вместе.
- Условие задачи — в **JavaDoc-шапке** точки входа (это «шапка задачи» из методики, §3).

```
# задача-папка (все файлы вместе):
javac -encoding UTF-8 -d out module-NN-.../practice/TaskNN/*.java
java -cp out TaskNN

# задача из одного файла:
javac -encoding UTF-8 TaskNN.java
java TaskNN
```

## 🧵 Сквозной проект — Data-Cruncher

«Аналитическая система обработки данных». Эволюционирует от простого консольного анализатора чисел до системы, которая:

- читает большие наборы данных (файлы, генераторы);
- выполняет алгоритмические задачи (поиск, сортировка, сжатие, кластеризация, поиск путей);
- визуализирует время выполнения и потребление памяти (для изучения сложности).

На каждой новой теме студент добавляет в проект соответствующий алгоритм или структуру (задача 07), улучшая производительность и функциональность. К теме 25 Data-Cruncher умеет: загружать данные → анализировать (поиск, сортировка, графы, DP) → сжимать результаты.

| Тема | Что добавляется в Data-Cruncher (задача 07) |
|------|---------------------------------------------|
| 01 | `ComplexityAnalyzer` — прогон алгоритмов с замерами времени/памяти |
| 02 | загрузка больших массивов из CSV + `rotate` |
| 03 | LRU-кэш последних операций (список + хеш-карта) |
| 04 | очередь задач + калькулятор выражений в RPN |
| 05 | генератор комбинаций параметров анализа |

| 10 | линейная сортировка для целочисленных данных с известным диапазоном |
| 11 | поиск паттернов в загруженных текстовых файлах |

## 🧩 Анатомия задачи (методика, §3)

В каждой задаче — только условие и каркас. Шапка в JavaDoc точки входа:

```java
/**
 * Задача NN — Тема NN: <название>
 *
 * ЗАДАНИЕ:
 *   что нужно сделать (1–3 предложения).
 *
 * ПРИМЕР / ВЫВОД:
 *   как выглядит правильный результат.
 *
 * ТРЕБОВАНИЯ:
 *   ограничения и обязательные условия (например: «без java.util.Arrays.sort»).
 *
 * ПОДСКАЗКА:
 *   направление, но НЕ ответ.
 */
public class TaskNN {
    public static void main(String[] args) {
        // TODO: ваш код
    }
}
```

**Правило:** готового решения нет. Есть только то, что помогает начать.

## ✅ Объективная проверяемость (методика, §10, §12)

| Тип темы | Темы | Как проверять |
|----------|------|---------------|
| Алгоритм/структура | большинство | компиляция + запуск + ожидаемый вывод в шапке |
| Замеры производительности | 01, 05, 07, 08, 09, 13, 15, 19 | бенчмарк-вывод (время для разных n) + ожидаемая тенденция |
| Корректность на крайних случаях | поиск, графы, DP | `assert` / контрольные примеры в `main` (проверяют интерфейс, не содержат решения) |

> Замеры времени зависят от железа — проверяется **тенденция** (линейный медленнее логарифмического, n² хуже n log n), а не абсолютные миллисекунды.
