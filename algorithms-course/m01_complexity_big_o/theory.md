# Тема 01. Анализ сложности: Big O, Θ, Ω

Прежде чем писать алгоритмы, надо научиться **оценивать их стоимость** — иначе невозможно сравнить два решения и выбрать лучшее. Сложность — это не «сколько секунд», а **как растёт число операций при росте входных данных**. Алгоритм за 0.1 с на 100 элементах может работать часы на миллионе — и Big O покажет это заранее, без запуска.

```
        Как растёт число операций T(n) при росте n
  T(n)
   |                                   n²  (квадратичный)
   |                              /
   |                         /
   |                    /            n·log n
   |               /            -------
   |          /        -----
   |      /   ----                    n   (линейный)
   |   /--        -----------------
   | /--------------                  log n (логарифмический)
   |--------------------------------- 1   (константа)
   +--------------------------------------► n
```

---

## 1. Три обозначения: O, Θ, Ω

| Нотация | Смысл | Аналогия |
|---------|-------|----------|
| **O(f(n))** | **верхняя** граница: «не хуже, чем» | «приду не позже 18:00» |
| **Ω(f(n))** | **нижняя** граница: «не лучше, чем» | «раньше 17:00 не приду» |
| **Θ(f(n))** | **точная** граница: и сверху, и снизу | «приду ровно в 17:30 ± чуть» |

На практике чаще всего говорят про **O** — про худший случай. Когда верхняя и нижняя границы совпадают, корректнее писать **Θ**, но в обиходе пишут O.

> Big O **отбрасывает константы и младшие слагаемые**: `3n² + 5n + 100` → O(n²). При больших n квадратичное слагаемое «съедает» всё остальное.

---

## 2. Как считать сложность по коду

**Правило 1. Последовательные блоки — складываются, берётся максимум.**
```java
for (int i = 0; i < n; i++) { ... }   // O(n)
for (int j = 0; j < n; j++) { ... }   // O(n)
// итого O(n) + O(n) = O(2n) = O(n)
```

**Правило 2. Вложенные циклы — перемножаются.**
```java
for (int i = 0; i < n; i++)        // n раз
    for (int j = 0; j < n; j++)    //   × n раз
        sum++;                     // → O(n²)
```

**Правило 3. Деление пополам на каждом шаге — это O(log n).**
```java
while (n > 1) n /= 2;   // O(log n): сколько раз n делится на 2 до единицы
```

**Правило 4. Цикл, зависящий от другой переменной, — не всегда O(n²).**
```java
for (int i = 0; i < n; i++)
    for (int j = i; j < n; j++)    // i, i+1, ... — в сумме n+(n-1)+...+1 = n(n+1)/2
        ...                        // → O(n²) (но с константой 1/2)
```

## 3. Сложность рекурсии (Master Theorem — упрощённо)

Рекурсия `T(n) = a·T(n/b) + f(n)`: задача дробится на `a` подзадач размера `n/b`, плюс работа `f(n)` на «склейку».

| Пример | a | b | f(n) | Итог |
|--------|---|---|------|------|
| Бинарный поиск | 1 | 2 | O(1) | **O(log n)** |
| Сортировка слиянием | 2 | 2 | O(n) | **O(n log n)** |
| Наивный Фибоначчи `fib(n-1)+fib(n-2)` | 2 | — | O(1) | **O(2ⁿ)** (дерево вызовов) |

Грубое правило: нарисуй **дерево вызовов**, посчитай работу на каждом уровне и число уровней. Бинарный поиск — одна ветка, глубина log n → O(log n). Наивный Фибоначчи — ветвление надвое на каждом шаге → экспонента.

---

## 4. Сложность по **памяти**

Считается так же, но для **дополнительной** памяти (помимо входа).

```java
int sum(int[] a) {            // память O(1): пара переменных
    int s = 0;
    for (int x : a) s += x;
    return s;
}

int factorial(int n) {        // память O(n): n кадров на стеке вызовов!
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}
```

> Рекурсия глубины `d` тратит **O(d)** памяти на стек вызовов, даже если каждый кадр крошечный. Глубокая рекурсия → `StackOverflowError`.

---

## 5. Шпаргалка: типичные сложности

| O | Название | Пример | n=10 | n=1000 |
|---|----------|--------|------|--------|
| O(1) | константа | доступ по индексу `a[i]` | 1 | 1 |
| O(log n) | логарифм | бинарный поиск | ~3 | ~10 |
| O(n) | линейная | проход по массиву | 10 | 1000 |
| O(n log n) | линейно-логарифм. | хорошие сортировки | ~33 | ~10⁴ |
| O(n²) | квадрат | вложенные циклы | 100 | 10⁶ |
| O(2ⁿ) | экспонента | перебор подмножеств | 1024 | ∞ (нереально) |

## ⚠️ Подводные камни

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| Путать «время работы» и «сложность» | «у меня быстро на моих данных» | сложность — про рост при больших n, не про секунды |
| Считать константы значимыми | спор «O(2n) vs O(n)» | константы отбрасываются: O(2n)=O(n) |
| Забыть про память рекурсии | `StackOverflowError` на больших n | глубина рекурсии = доп. память O(глубины) |
| `String +=` в цикле | «линейный» цикл на деле O(n²) | каждый `+=` копирует строку; используйте `StringBuilder` |
| `list.contains()` в цикле по списку | «один цикл», но внутри ещё поиск | `ArrayList.contains` = O(n) → весь цикл O(n²) |
