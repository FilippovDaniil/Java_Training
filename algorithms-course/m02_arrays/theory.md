# Тема 02. Массивы

Массив — самая базовая структура данных: непрерывный блок памяти, где элементы лежат подряд. Именно поэтому **доступ по индексу — O(1)** (адрес = начало + индекс × размер элемента), а **вставка/удаление в середине — O(n)** (надо сдвигать хвост). Понимание этого компромисса — ключ ко всем остальным структурам.

```
Массив [10, 20, 30, 40, 50]   индексы 0..4
        ▲                       ▲
   a[0]=10 (O(1))          доступ по любому индексу — мгновенный

Вставка 99 в индекс 1:  надо сдвинуть [20,30,40,50] вправо → O(n)
[10, _, 20, 30, 40, 50]
     99
```

---

## 1. Статический массив (`int[]`)

Размер фиксирован при создании и **не меняется**. Быстрый, но негибкий.

```java
int[] a = new int[5];     // [0,0,0,0,0], размер 5 навсегда
a[2] = 30;                // O(1)
int x = a[2];             // O(1)
int len = a.length;       // поле, не метод; O(1)
```

| Операция | Сложность | Почему |
|----------|-----------|--------|
| доступ `a[i]` | O(1) | прямой адрес |
| изменение `a[i]=v` | O(1) | прямой адрес |
| поиск значения | O(n) | надо просмотреть все |
| «вставка» в середину | O(n) | сдвиг хвоста (вручную) |

---

## 2. Динамический массив (`ArrayList`)

Оборачивает обычный массив и **автоматически растёт**, когда заполняется. Внутри — тот же `Object[]`, плюс поле `size`.

```java
List<Integer> list = new ArrayList<>();
list.add(10);        // амортизированно O(1)
list.get(0);         // O(1)
list.add(1, 99);     // вставка в середину — O(n)
list.remove(0);      // удаление с начала — O(n) (сдвиг)
```

### Амортизированная O(1) для `add`

Когда внутренний массив заполнен, `ArrayList` создаёт **новый вдвое больше** и копирует элементы (O(n)). Но это происходит редко: при удвоении на N вставок суммарно копий ~2N → **в среднем O(1) на вставку**.

```
ёмкость:  растёт 1 → 2 → 4 → 8 → 16 ...
копий при росте:    1 + 2 + 4 + 8 = 15 ≈ N для N=16 элементов
```

> Если бы рост был «+1 каждый раз», то на N вставок было бы 1+2+...+N = O(n²) копий — катастрофа. **Удвоение** превращает это в O(n) суммарно.

---

## 3. `System.arraycopy` — быстрое копирование

Нативный метод копирования участка массива — намного быстрее ручного цикла. Используется внутри `ArrayList`, сдвигов, сортировки вставками.

```java
// скопировать src[srcPos..] в dst[dstPos..], length элементов
System.arraycopy(src, srcPos, dst, dstPos, length);

// сдвинуть хвост вправо на 1 (освободить место под вставку в index):
System.arraycopy(a, index, a, index + 1, size - index);
```

---

## 4. Приёмы in-place (без доп. памяти)

Многие операции над массивом можно делать **на месте**, память O(1):

- **reverse** — два указателя с краёв навстречу, меняем местами;
- **циклический сдвиг** — через тройной reverse;
- **removeDuplicates** в отсортированном — «медленный/быстрый» указатель.

```java
// reverse in-place
int l = 0, r = a.length - 1;
while (l < r) { int t = a[l]; a[l] = a[r]; a[r] = t; l++; r--; }
```

## ⚠️ Подводные камни

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| Рост массива «+1» | O(n²) на заполнении | удваивайте ёмкость |
| `a.length()` вместо `a.length` | не компилируется | у массива `length` — поле, без скобок |
| Сдвиг вручную в цикле | медленно, легко ошибиться | `System.arraycopy` |
| `ArrayList.remove(int)` в цикле вперёд | пропуск элементов | идите с конца или используйте `Iterator.remove` |
| Выход за границы | `ArrayIndexOutOfBoundsException` | проверяйте `0 <= i < length` |
