# Тема 03. Связные списки

Связный список — это цепочка **узлов**, где каждый узел хранит значение и **ссылку на следующий**. В отличие от массива, элементы не лежат подряд в памяти. Это меняет компромиссы с точностью до наоборот: **вставка/удаление в начале — O(1)**, но **доступ по индексу — O(n)** (надо идти от головы).

```
Односвязный список:
 head
  │
  ▼
┌────┬───┐   ┌────┬───┐   ┌────┬───┐
│ 10 │ ●─┼──▶│ 20 │ ●─┼──▶│ 30 │ ✕ │   (✕ = null)
└────┴───┘   └────┴───┘   └────┴───┘
 узел        узел         хвост (tail)
```

---

## 1. Узел (Node)

```java
class Node {
    int value;
    Node next;          // ссылка на следующий узел (null у последнего)
    Node(int value) { this.value = value; }
}
```

Список — это просто ссылка на первый узел (`head`). Зная `head`, можно дойти до любого, переходя по `next`.

---

## 2. Массив vs связный список

| Операция | Массив | Связный список |
|----------|--------|----------------|
| доступ по индексу | **O(1)** | O(n) |
| вставка в начало | O(n) (сдвиг) | **O(1)** |
| вставка в конец | O(1)* (амортиз.) | O(1) (если есть `tail`) / O(n) без него |
| удаление из начала | O(n) (сдвиг) | **O(1)** |
| память на элемент | компактно | + ссылка(и) на узел |
| локальность кэша | отличная | плохая (узлы разбросаны) |

> Вывод: массив хорош для **доступа по индексу и итерации**; список — для **частых вставок/удалений с концов**. На практике `ArrayList` чаще быстрее `LinkedList` из-за кэша процессора, даже там, где «по теории» список выигрывает.

---

## 3. Базовые операции (односвязный)

```java
// addFirst — O(1)
Node n = new Node(value);
n.next = head;
head = n;

// проход по списку — O(n)
for (Node cur = head; cur != null; cur = cur.next)
    System.out.print(cur.value + " ");
```

`addLast` за O(1) требует хранить ещё и ссылку `tail`. Без неё — O(n) (дойти до конца).

---

## 4. Два указателя: «черепаха и заяц»

Мощный приём для списков: два указателя движутся с разной скоростью.

- **Середина списка за один проход:** `slow` шагает на 1, `fast` — на 2. Когда `fast` дошёл до конца, `slow` — ровно в середине.
- **Обнаружение цикла (алгоритм Флойда):** если `fast` догонит `slow` — есть цикл; если `fast` дойдёт до `null` — цикла нет.

```
slow→ ●   fast→ ●
шаг:  ●→●         ●→●→●
Если зациклено — рано или поздно slow == fast.
```

---

## 5. Двусвязный список

Каждый узел хранит ссылки и на следующий (`next`), и на предыдущий (`prev`). С хранением `head` и `tail` это даёт `addLast`/`removeLast` за **O(1)** — то, чего не умеет односвязный.

```java
class DNode { int value; DNode prev, next; }
```

Цена — лишняя ссылка в каждом узле и аккуратность при перевязывании `prev/next`. Двусвязный список — основа `java.util.LinkedList` и LRU-кэша (задача 07).

---

## ⚠️ Подводные камни

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| Потеря ссылки при перевязке | часть списка «отвалилась» | сохраните `next` во временную переменную ДО переназначения |
| Не обновили `tail` | `addLast` ломается после удаления | поддерживайте `head` И `tail` согласованно |
| `NullPointerException` на пустом списке | падение на `head.next` | проверяйте `head == null` в начале |
| Цикл в списке → бесконечный проход | программа зависает | алгоритм Флойда для детекции |
| Доступ по индексу в цикле | «один цикл», но O(n²) | не используйте `get(i)` в цикле по списку |
