# Тема 12. Двоичные деревья поиска (BST) и обходы

Переходим от линейных структур к **иерархическим**. Двоичное дерево поиска (BST) хранит элементы так, что для каждого узла: всё в **левом** поддереве **меньше** его, всё в **правом** — **больше**. Это инвариант позволяет искать, вставлять и удалять за **O(h)**, где h — высота. На сбалансированном дереве h ≈ log n → O(log n). Но если вставлять отсортированные данные, дерево **вырождается в список** → O(n) (это лечат темой 13).

```
        8              Инвариант BST:
       / \             всё слева < узла < всё справа
      3   10           поиск 6: 6<8 идём влево, 6>3 вправо, 6=6 нашли
     / \    \
    1   6    14        in-order обход даёт ОТСОРТИРОВАННУЮ последовательность:
       / \             1 3 4 6 7 8 10 14
      4   7
```

---

## 1. Узел и вставка

```java
class Node {
    int value;
    Node left, right;
    Node(int value) { this.value = value; }
}

Node insert(Node root, int value) {
    if (root == null) return new Node(value);
    if (value < root.value) root.left = insert(root.left, value);
    else if (value > root.value) root.right = insert(root.right, value);
    // == : дубликаты обычно игнорируют
    return root;
}
```

Поиск симметричен: идём влево/вправо, пока не найдём или не упрёмся в `null`.

---

## 2. Четыре обхода

| Обход | Порядок | Применение |
|-------|---------|-----------|
| **pre-order** (КЛП) | узел → лево → право | копирование дерева, сериализация |
| **in-order** (ЛКП) | лево → узел → право | **отсортированный** вывод BST |
| **post-order** (ЛПК) | лево → право → узел | удаление дерева, вычисление выражений |
| **level-order** (BFS) | по уровням сверху вниз | поиск по слоям, кратчайший путь в дереве |

```java
void inorder(Node n) {
    if (n == null) return;
    inorder(n.left);
    visit(n.value);        // между поддеревьями
    inorder(n.right);
}
```

**Level-order** — это BFS через очередь (см. тему 18): кладём корень, достаём узел, печатаем, кладём его детей.

---

## 3. Удаление узла — три случая

Самая хитрая операция. Удаляем узел `x`:

1. **Нет детей** (лист) — просто отвязать.
2. **Один ребёнок** — заменить узел его единственным ребёнком.
3. **Два ребёнка** — заменить значением **преемника** (минимум в правом поддереве — самый левый узел справа), затем удалить преемника (у него детей ≤ 1).

```
Удаляем 8 (два ребёнка):       преемник = 10 (мин в правом поддереве)
        8                              10
       / \           →                / \
      3   10                         3   14
           \
            14
```

---

## 4. Сложности

| Операция | Сбалансированное | Вырожденное (худшее) |
|----------|------------------|----------------------|
| поиск/вставка/удаление | O(log n) | O(n) |
| in-order обход | O(n) | O(n) |
| память | O(n) | O(n) |

> **Вырождение** — главная проблема обычного BST. Вставка 1,2,3,4,5 по порядку даёт «бамбук» (фактически связный список) с O(n) операциями. Решение — самобалансирующиеся деревья (AVL, красно-чёрные) в теме 13.

---

## ⚠️ Подводные камни

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| Вставка отсортированных данных | дерево-«бамбук», O(n) | балансировка (тема 13) |
| Удаление с двумя детьми «в лоб» | нарушен инвариант BST | замена преемником/предшественником |
| Рекурсия без `return new Node` | вставка «теряется» | возвращайте поддерево и присваивайте |
| Глубокая рекурсия на вырожденном дереве | `StackOverflowError` | итеративные обходы / балансировка |
| Сравнение по `==` для дубликатов | дубликаты ломают логику | решите политику (игнор/счётчик) |
