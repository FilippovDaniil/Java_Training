# Тема 17. Представление графов

Граф — это вершины (vertices) и связи между ними (рёбра, edges). Им моделируют дороги, соцсети, зависимости задач, состояния игры. Прежде чем запускать алгоритмы (BFS/DFS, кратчайшие пути, MST), надо выбрать **способ хранения** графа — он определяет, какие операции дёшевы, а какие дороги, и сколько памяти уходит.

```
Неориентированный граф:        Список смежности:        Матрица смежности:
                                 0: [1, 2]                  0 1 2 3
   0 --- 1                       1: [0, 2]                0 0 1 1 0
   |  /                          2: [0, 1, 3]             1 1 0 1 0
   2 --- 3                       3: [2]                   2 1 1 0 1
                                                          3 0 0 1 0
```

---

## 1. Ориентированные vs неориентированные

- **Неориентированный**: ребро (u,v) ходит в обе стороны. В списке смежности добавляем v к u И u к v.
- **Ориентированный (орграф)**: ребро u→v только в одну сторону. Добавляем только v к u.

---

## 2. Список смежности (adjacency list)

Для каждой вершины — список её соседей. Самое частое представление, особенно для **разреженных** графов (рёбер мало).

```java
List<List<Integer>> adj = new ArrayList<>();
for (int i = 0; i < n; i++) adj.add(new ArrayList<>());
adj.get(u).add(v);          // ребро u->v
adj.get(v).add(u);          // для неориентированного — и обратно
```

| Операция | Список смежности |
|----------|------------------|
| Память | **O(V + E)** |
| Перебор соседей u | O(deg(u)) — оптимально |
| Есть ли ребро (u,v)? | O(deg(u)) |
| Добавить ребро | O(1) |

---

## 3. Матрица смежности (adjacency matrix)

Двумерный массив `m[u][v]` = 1, если есть ребро (или вес ребра). Хороша для **плотных** графов и когда часто спрашивают «есть ли ребро (u,v)?».

```java
int[][] m = new int[n][n];
m[u][v] = 1;                // ребро u->v
m[v][u] = 1;                // для неориентированного
```

| Операция | Матрица смежности |
|----------|-------------------|
| Память | **O(V²)** (даже если рёбер мало) |
| Есть ли ребро (u,v)? | **O(1)** |
| Перебор соседей u | O(V) — даже если соседей мало |
| Добавить ребро | O(1) |

> Правило: **разреженный** граф (E ≈ V) → список смежности; **плотный** (E ≈ V²) или нужны частые проверки «есть ребро?» → матрица.

---

## 4. Взвешенные графы

Рёбра несут вес (расстояние, стоимость, время). В списке смежности храним не просто соседа, а пару (сосед, вес):

```java
record Edge(int to, int weight) {}
List<List<Edge>> adj;       // adj.get(u) -> список Edge
```

В матрице — `m[u][v] = вес` (а отсутствие ребра кодируют, например, бесконечностью).

---

## 5. Базовые операции и свойства

- **Степень вершины** (degree) — число инцидентных рёбер. В орграфе различают in-degree и out-degree.
- **Транспонирование орграфа** — развернуть все рёбра (u→v становится v→u). Нужно, например, для поиска компонент сильной связности.
- **Двудольность** — можно ли раскрасить вершины в 2 цвета так, чтобы концы каждого ребра были разного цвета (проверяется BFS/DFS).

---

## ⚠️ Подводные камни

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| Неориентированное ребро добавлено в одну сторону | «пропадают» связи | добавляйте (u,v) И (v,u) |
| Матрица для разреженного большого графа | O(V²) памяти, `OutOfMemory` | список смежности |
| Список смежности и частые «есть ли ребро?» | O(deg) на каждый запрос | матрица или `Set` соседей |
| Самопетли/кратные рёбра не учтены | неверные степени/обходы | договоритесь о политике заранее |
| Вершины не от 0 до V−1 | выход за границы массива | нормализуйте/мапьте идентификаторы |
