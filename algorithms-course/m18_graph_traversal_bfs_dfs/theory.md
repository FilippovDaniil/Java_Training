# Тема 18. Обход графов: BFS и DFS

Два фундаментальных способа систематически посетить все достижимые вершины. **BFS** (поиск в ширину) идёт «слоями» от старта, используя очередь, и находит кратчайший путь в **невзвешенном** графе. **DFS** (поиск в глубину) уходит вглубь по одной ветке, используя рекурсию или стек, и лежит в основе поиска циклов, топологической сортировки, компонент связности. Обоим нужен массив `visited`, чтобы не зациклиться.

```
Граф:  0 - 1 - 3        BFS от 0 (по слоям):     DFS от 0 (вглубь):
       |   |            слой0: 0                  0 -> 1 -> 3 -> (назад) -> 2
       2 - 4            слой1: 1, 2               порядок посещения зависит
                        слой2: 3, 4              от порядка соседей
```

---

## 1. BFS — поиск в ширину (очередь)

Посещаем стартовую вершину, потом всех её соседей, потом соседей соседей — «волнами». Очередь хранит «фронт» обхода.

```java
Queue<Integer> q = new ArrayDeque<>();
boolean[] visited = new boolean[n];
q.add(start); visited[start] = true;
while (!q.isEmpty()) {
    int u = q.poll();
    for (int v : adj.get(u))
        if (!visited[v]) { visited[v] = true; q.add(v); }
}
```

**Ключевое свойство:** в невзвешенном графе BFS находит **кратчайший путь** (по числу рёбер) от старта до всех вершин. Запоминая `parent[v]`, можно восстановить сам путь.

---

## 2. DFS — поиск в глубину (рекурсия или стек)

Идём вглубь, пока есть непосещённые соседи; упёрлись — откатываемся.

```java
void dfs(int u) {
    visited[u] = true;
    for (int v : adj.get(u))
        if (!visited[v]) dfs(v);
}
```

Итеративная версия — через явный стек (на случай глубоких графов и `StackOverflowError`):
```java
Deque<Integer> stack = new ArrayDeque<>();
stack.push(start);
while (!stack.isEmpty()) {
    int u = stack.pop();
    if (visited[u]) continue;
    visited[u] = true;
    for (int v : adj.get(u)) if (!visited[v]) stack.push(v);
}
```

---

## 3. Применения

| Задача | Инструмент |
|--------|-----------|
| Кратчайший путь в невзвешенном графе | **BFS** (по слоям) |
| Компоненты связности | BFS или DFS от каждой непосещённой |
| Обнаружение цикла | DFS с цветовой маркировкой |
| Топологическая сортировка | DFS + стек, или Кан (BFS по in-degree) |
| Проверка двудольности | BFS/DFS 2-раскраской (тема 17) |
| Достижимость (можно ли дойти из s в t) | любой обход |

---

## 4. Обнаружение цикла в орграфе: три цвета

DFS с маркировкой вершин тремя состояниями:
- **0 (белый)** — не посещена;
- **1 (серый)** — в текущем рекурсивном стеке (обрабатывается);
- **2 (чёрный)** — полностью обработана.

Если во время DFS встречаем **серую** вершину — нашли цикл (ребро назад в текущий путь).

```
если сосед белый -> рекурсия;
если сосед серый -> ЦИКЛ!
если сосед чёрный -> игнорируем (уже обработан)
```

> В **неориентированном** графе цикл ищут проще: при DFS встретили посещённого соседа, который не является родителем — цикл.

---

## 5. Топологическая сортировка (для DAG)

Линейный порядок вершин, где для каждого ребра u→v вершина u идёт раньше v. Существует **только для ациклических** орграфов (DAG).
- **через DFS:** post-order, затем развернуть (добавлять вершину в стек после обработки всех потомков);
- **алгоритм Кана:** убирать вершины с in-degree 0, обновляя степени (BFS-подобно). Подробно — в теме 21.

## ⚠️ Подводные камни

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| Нет `visited` | бесконечный цикл | помечайте посещённые |
| Помечаем visited при ИЗВЛЕЧЕНИИ из очереди (BFS) | вершина попадает в очередь дважды | помечайте при ДОБАВЛЕНИИ |
| DFS на большом графе рекурсией | `StackOverflowError` | итеративный DFS (стек) |
| Обход только от одной вершины в несвязном графе | посещены не все | запускать от каждой непосещённой |
| Цикл в неориентированном: считать родителя | ложный цикл | исключать ребро к непосредственному родителю |
