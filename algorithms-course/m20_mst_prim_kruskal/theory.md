# Тема 20. Минимальное остовное дерево: Прим и Крускал

**Остовное дерево** (spanning tree) связного взвешенного графа — подмножество рёбер, которое соединяет все вершины без циклов (ровно V−1 ребро). **Минимальное** остовное дерево (MST) — то, у которого суммарный вес рёбер наименьший. Задача: «соединить все точки минимальной ценой» — кабели, дороги, кластеризация. Два классических жадных алгоритма: **Крускал** (через DSU, тема 16) и **Прим** (через кучу, тема 15).

```
Граф:          Одно из MST (вес 1+2+2 = 5):
   0 --3-- 1       0       1
   |  \    |       |       |
   1   2   2       1       2
   |    \  |       |       |
   2 --4-- 3       2 --2-- 3
Берём самые дёшевые рёбра, не создающие цикл.
```

---

## 1. Крускал — сортировка рёбер + DSU

Жадно: рассматриваем рёбра **в порядке возрастания веса**; добавляем ребро в MST, **если оно соединяет разные компоненты** (иначе — цикл). Принадлежность к компонентам отслеживает DSU (тема 16) с операциями `find`/`union`.

```java
sort(edges по весу);
DSU dsu = new DSU(V);
int total = 0, used = 0;
for (Edge e : edges) {
    if (dsu.find(e.u) != dsu.find(e.v)) {   // концы в разных компонентах -> нет цикла
        dsu.union(e.u, e.v);
        total += e.w; used++;
        if (used == V - 1) break;            // дерево готово
    }
}
```

**Сложность:** O(E log E) (доминирует сортировка рёбер). Хорош для **разреженных** графов.

---

## 2. Прим — наращивание дерева + куча

Жадно: начинаем с одной вершины и **наращиваем дерево**, каждый раз добавляя самое дешёвое ребро, ведущее из дерева наружу (к ещё не включённой вершине). «Самое дешёвое ребро на границе» эффективно выдаёт min-heap (тема 15).

```
дерево = {start}
повторять V-1 раз:
  выбрать минимальное ребро (внутри дерева -> снаружи)
  добавить его и новую вершину в дерево
```

**Ленивая версия:** кладём в кучу все рёбра из добавленной вершины; при извлечении пропускаем те, что ведут в уже включённую вершину. Сложность O(E log E). Прим (с массивом ключей) хорош для **плотных** графов — O(V²).

---

## 3. Сравнение

| | Крускал | Прим |
|---|---------|------|
| Идея | сортируем рёбра, склеиваем компоненты | растим одно дерево от вершины |
| Структура | **DSU** (тема 16) | **куча** (тема 15) |
| Сложность | O(E log E) | O(E log V) (ленивый) / O(V²) (плотный) |
| Лучше для | разреженных графов | плотных графов |
| Граф должен быть связным | для полного MST — да (иначе остовный лес) | да |

> Оба **жадные** и оба корректны благодаря свойству разреза: минимальное ребро, пересекающее любой разрез графа, безопасно входит в некоторое MST.

---

## 4. Свойства MST

- Ровно **V−1 ребро**, без циклов, связывает все вершины.
- Если все веса **различны** — MST **единственно**. При равных весах MST может быть несколько (одинакового суммарного веса).
- **Второе по минимальности** остовное дерево отличается от MST ровно одним ребром (идея: для каждого небазового ребра попробовать заменить им максимальное ребро на пути в MST).

---

## 5. Применения

- Проектирование сетей (минимум кабеля/дороги, соединяющих все узлы).
- Кластеризация (single-linkage): удалить k−1 самых дорогих рёбер MST → k кластеров.
- Аппроксимации (например, для метрической задачи коммивояжёра).

## ⚠️ Подводные камни

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| Крускал без DSU (проверка цикла обходом) | медленно, O(V·E) | DSU с find/union |
| Не остановились после V−1 рёбер | лишняя работа | break при used == V−1 |
| Прим: не пропускаем устаревшие рёбра кучи | неверный/медленный | пропускать рёбра в уже включённую вершину |
| Несвязный граф, ждём дерево | получится лес, не дерево | проверять связность / трактовать как остовный лес |
| Сравнение рёбер по весу неверно | неоптимальное дерево | сортировать/сравнивать строго по весу |
