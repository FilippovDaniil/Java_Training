# Тема 23. DP на подотрезках и многомерное DP

Развиваем DP темы 22 на более сложные состояния. **DP на подотрезках** (interval DP): состояние — отрезок `[l, r]`, ответ собирается из меньших отрезков (умножение цепочки матриц, палиндромы). **Многомерное DP**: состояние — несколько координат (две позиции в строке/сетке). Главная трудность здесь — придумать правильное состояние и **порядок** заполнения (по длине отрезка, по диагонали, по строкам).

```
Interval DP (по возрастанию длины отрезка):
  длина 1: [0,0] [1,1] [2,2] ...   (база)
  длина 2: [0,1] [1,2] ...
  длина 3: [0,2] ...
  ответ:   [0, n-1]                 (весь массив/строка)
```

---

## 1. LIS — наибольшая возрастающая подпоследовательность

Две реализации:

**O(n²):** `dp[i]` = длина LIS, заканчивающейся в i. `dp[i] = 1 + max(dp[j])` по всем `j < i` с `a[j] < a[i]`.

**O(n log n):** поддерживаем массив `tails`, где `tails[k]` — наименьший возможный «хвост» возрастающей подпоследовательности длины k+1. Для каждого элемента бинарным поиском находим позицию и обновляем.
```
для x: позиция = первый tails >= x;
если такого нет -> добавить x в конец (LIS удлинилась);
иначе tails[позиция] = x (улучшили хвост)
длина LIS = размер tails
```

> `tails` НЕ является самой LIS, но его длина равна длине LIS. Это классический приём «бинпоиск + жадность».

---

## 2. Умножение цепочки матриц

Дана последовательность матриц; их произведение можно расставить скобками по-разному, и число скалярных умножений зависит от расстановки. Найти минимальное.

Interval DP: `dp[l][r]` = мин число умножений для перемножения матриц `l..r`. Перебираем точку разбиения `k`:
```
dp[l][r] = min по k из (dp[l][k] + dp[k+1][r] + cost(l,k,r))
```
Заполняем **по возрастанию длины отрезка**. O(n³).

---

## 3. Самая длинная палиндромная подстрока

`dp[l][r]` = является ли подстрока `s[l..r]` палиндромом. Переход: `s[l]==s[r] && dp[l+1][r-1]`. Заполняем по возрастанию длины; запоминаем самый длинный палиндром. O(n²).

```
"babad": палиндромы "bab" и "aba" (длина 3)
dp[l][r] = (s[l]==s[r]) и (внутренность dp[l+1][r-1] — палиндром)
```

---

## 4. Многомерное DP: сетка

**Уникальные пути в сетке с препятствиями:** `dp[i][j]` = число путей в клетку (i,j), двигаясь вправо/вниз. `dp[i][j] = dp[i-1][j] + dp[i][j-1]` (0 для препятствий). База — первая строка/столбец.

**Треугольник (минимальный путь):** `dp[i][j]` = мин сумма пути от вершины до (i,j). Удобно считать снизу вверх.

---

## 5. Word break

Можно ли разбить строку на слова из словаря. `dp[i]` = можно ли разбить префикс `s[0..i)`. `dp[i] = true`, если существует `j < i`, такое что `dp[j]` истинно и `s[j..i)` — слово из словаря. Одномерное DP по длине префикса.

## ⚠️ Подводные камни

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| Interval DP не по длине отрезка | используется непосчитанный отрезок | внешний цикл — по длине (len = 1..n) |
| LIS O(n log n): не тот вариант бинпоиска | считает LДP вместо LIS / неверно | для строгого возрастания — lower_bound |
| Сетка: не обработаны препятствия | неверное число путей | в препятствии dp=0, не суммировать |
| Палиндром: внутренность для len≤2 | выход за границы | базы для длин 1 и 2 отдельно |
| Путаница i/j (строки/столбцы) | неверный ответ | зафиксируй смысл осей |
