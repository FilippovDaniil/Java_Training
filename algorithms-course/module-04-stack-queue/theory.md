# Тема 04. Стек и очередь

Стек и очередь — две фундаментальные абстракции, отличающиеся только **порядком выдачи** элементов. Они не про хранение, а про **дисциплину доступа**. На них держатся обходы графов, рекурсия (стек вызовов), вычисление выражений, планировщики задач.

```
СТЕК (LIFO — Last In, First Out)      ОЧЕРЕДЬ (FIFO — First In, First Out)
   push ↓   ↑ pop                          enqueue →  ┌──┬──┬──┐ → dequeue
        ┌──┐                                          │  │  │  │
        │ 3│ ← вершина (top)                add сзади  └──┴──┴──┘  берём спереди
        │ 2│                                «как стопка тарелок»  «как очередь в кассу»
        │ 1│
        └──┘
 «как стопка тарелок»
```

---

## 1. Стек (LIFO)

Добавление и удаление — только с одного конца (вершины).

| Операция | Смысл | Сложность |
|----------|-------|-----------|
| `push(x)` | положить на вершину | O(1) |
| `pop()` | снять с вершины | O(1) |
| `peek()` | посмотреть вершину | O(1) |
| `isEmpty()` | пуст ли | O(1) |

Реализации: на массиве (индекс вершины) или на связном списке (`addFirst`/`removeFirst`).

```java
Deque<Integer> stack = new ArrayDeque<>();   // рекомендуемый стек в Java
stack.push(1); stack.push(2);
stack.pop();    // 2
```

> ⚠️ Старый класс `java.util.Stack` наследует `Vector`, синхронизирован и считается легаси. Для стека используйте `ArrayDeque`.

---

## 2. Очередь (FIFO)

Добавление с одного конца (хвост), удаление — с другого (голова).

| Операция | Смысл | Сложность |
|----------|-------|-----------|
| `enqueue(x)` / `offer` | в хвост | O(1) |
| `dequeue()` / `poll` | из головы | O(1) |
| `peek()` | посмотреть голову | O(1) |

```java
Queue<Integer> q = new ArrayDeque<>();
q.offer(1); q.offer(2);
q.poll();   // 1
```

---

## 3. `ArrayDeque` — и стек, и очередь

`ArrayDeque` (двусторонняя очередь) — это кольцевой буфер на массиве. Умеет добавлять/удалять с обоих концов за O(1), поэтому работает и как стек (`push`/`pop`), и как очередь (`offer`/`poll`). В большинстве задач это лучший выбор.

---

## 4. Кольцевой буфер (circular buffer)

Очередь фиксированного размера на массиве: два индекса `head` и `tail` ходят по кругу через `% capacity`. Когда `tail` доходит до конца — «оборачивается» в начало. Память O(capacity), все операции O(1), без сдвигов.

```
capacity=5, добавили 3, сняли 1:
индексы:  0   1   2   3   4
         [_] [b] [c] [d] [_]
              ▲head        ▲tail (следующая позиция для вставки)
tail = (tail + 1) % capacity
```

---

## 5. Применения

- **Стек** — проверка скобок, обратная польская запись (RPN), DFS, откат (undo), стек вызовов.
- **Очередь** — BFS (тема 18), буферизация, планировщики, очередь задач (задача 07).
- **Очередь с приоритетом** — выдаёт не «первый пришёл», а «самый приоритетный»; наивно — O(n) на вставку/извлечение, эффективно — на куче (тема 15).

---

## ⚠️ Подводные камни

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| `pop`/`poll` на пустом | `NoSuchElementException` или `null` | проверяйте `isEmpty()` |
| `Stack`/`Vector` ради стека | медленно, легаси | `ArrayDeque` |
| Очередь на массиве со сдвигом | `dequeue` за O(n) | кольцевой буфер с `% capacity` |
| Перепутать `add`/`offer` поведение при заполнении | исключение vs `false` | `offer` возвращает `false`, `add` бросает |
| `null` в `ArrayDeque` | `NullPointerException` | `ArrayDeque` запрещает `null` |
