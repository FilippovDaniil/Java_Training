# Тема 06. Линейный и бинарный поиск

Поиск элемента — базовая операция. Линейный поиск работает на любых данных за O(n). Но если данные **отсортированы**, бинарный поиск находит элемент за O(log n), отбрасывая половину кандидатов на каждом шаге. Это первый алгоритм курса с логарифмической сложностью — и шаблон «разделяй и властвуй» в чистом виде.

```
Бинарный поиск 7 в [1, 3, 5, 7, 9, 11, 13]:
шаг 1: lo=0 hi=6 mid=3 -> a[3]=7 == 7  найдено за 1 сравнение!

Поиск 11:
шаг 1: mid=3 a[3]=7  < 11 -> ищем справа, lo=4
шаг 2: lo=4 hi=6 mid=5 a[5]=11 == 11  найдено
log2(7) ≈ 3 сравнения максимум вместо 7.
```

---

## 1. Линейный поиск — O(n)

Просто идём по массиву. Работает всегда, сортировка не нужна.

```java
int indexOf(int[] a, int target) {
    for (int i = 0; i < a.length; i++)
        if (a[i] == target) return i;
    return -1;
}
```

Полезные вариации: индекс **первого** вхождения (вернуть сразу), индекс **последнего** (идти с конца или запоминать).

---

## 2. Бинарный поиск — O(log n)

**Требует отсортированного массива.** Держим границы `lo`, `hi`; смотрим середину; половина отбрасывается.

```java
int binarySearch(int[] a, int target) {
    int lo = 0, hi = a.length - 1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;     // НЕ (lo+hi)/2 — защита от переполнения
        if (a[mid] == target) return mid;
        else if (a[mid] < target) lo = mid + 1;
        else hi = mid - 1;
    }
    return -1;
}
```

> ⚠️ `mid = (lo + hi) / 2` может **переполнить int** при больших индексах. Пишите `lo + (hi - lo) / 2`.

`Arrays.binarySearch(a, key)` делает это из коробки, но возвращает «точку вставки» (отрицательное число) при отсутствии — об этом ниже.

---

## 3. Поиск границ (lower/upper bound)

Когда в массиве есть **дубликаты**, обычный бинарный поиск вернёт *какой-то* индекс. Часто нужна **левая граница** (первое вхождение) или **правая** (последнее). Приём: не останавливаться при совпадении, а сужать границу дальше.

```java
// первое вхождение (левая граница):
while (lo < hi) {
    int mid = lo + (hi - lo) / 2;
    if (a[mid] < target) lo = mid + 1;
    else hi = mid;            // не mid-1: возможно, mid и есть ответ
}
// lo — первый индекс, где a[lo] >= target
```

Это же — основа `Arrays.binarySearch`: при отсутствии ключа он возвращает `-(точка вставки) - 1`.

---

## 4. Бинарный поиск по ОТВЕТУ

Бинарный поиск работает не только по массиву, но и по **диапазону значений**, если есть монотонное свойство «да/нет». Пример — `sqrt(x)`: ищем число r в [0, x], такое что r² ≈ x. Свойство «r² <= x» монотонно → бинпоиск по вещественному диапазону до нужной точности.

```
ищем sqrt(2): lo=0, hi=2
mid=1.0   1²=1 < 2 -> lo=1.0
mid=1.5   2.25 > 2 -> hi=1.5
... сужаем, пока hi-lo > 1e-6
```

---

## 5. Нетривиальные применения

- **Сдвинутый отсортированный массив** (rotated): `[4,5,6,7,0,1,2]`. Одна из половин всегда отсортирована — определяем какая и решаем, где искать.
- **2D-матрица**, где строки и столбцы отсортированы: старт из правого верхнего угла, шаг влево (если больше) или вниз (если меньше) → O(n+m).

---

## ⚠️ Подводные камни

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| Бинпоиск на неотсортированном | неверный результат | сначала сортировка (или линейный поиск) |
| `(lo + hi) / 2` | переполнение int | `lo + (hi - lo) / 2` |
| `lo <= hi` vs `lo < hi` | зацикливание или промах границы | следите за инвариантом и обновлением границ |
| `hi = mid - 1` при поиске границы | пропуск ответа | для границ `hi = mid` (без -1) |
| `==` для double в sqrt | никогда не сойдётся | сравнивайте по точности `hi - lo > eps` |
