# Тема 07. Простые сортировки: пузырёк, выбором, вставками

Три классические сортировки за **O(n²)**. Они медленные на больших данных, но важны: на них понимают идею сортировки, стабильности и адаптивности. А сортировка **вставками** реально применяется — как «добивка» в быстрых сортировках на маленьких подмассивах (тема 09) и на почти отсортированных данных.

```
Пузырёк: «тяжёлые» всплывают в конец за проходы
[5 3 8 1]→[3 5 1 8]→[3 1 5 8]→[1 3 5 8]   (соседние свопы)

Выбором: ищем минимум, ставим на место
[5 3 8 1]→ min=1 → [1 | 3 8 5]→ min=3 → [1 3 | 8 5]→[1 3 5 8]

Вставками: вставляем каждый элемент в уже отсортированную часть
[5 |3 8 1]→[3 5 |8 1]→[3 5 8 |1]→[1 3 5 8]
```

---

## 1. Пузырьковая сортировка (bubble)

Многократно проходим по массиву, меняя местами соседние элементы, если они в неверном порядке. После k-го прохода k наибольших элементов «всплыли» в конец.

**Оптимизация (ранний выход):** если за проход не было ни одного обмена — массив уже отсортирован, выходим. Это даёт O(n) на уже отсортированных данных.

```java
for (int i = 0; i < n - 1; i++) {
    boolean swapped = false;
    for (int j = 0; j < n - 1 - i; j++)
        if (a[j] > a[j+1]) { swap(a, j, j+1); swapped = true; }
    if (!swapped) break;        // ранний выход
}
```

---

## 2. Сортировка выбором (selection)

На каждом шаге находим **минимум** в неотсортированной части и ставим его в начало этой части. Всегда O(n²) сравнений — **не адаптивна**. Зато делает минимум обменов (ровно n-1).

```java
for (int i = 0; i < n - 1; i++) {
    int min = i;
    for (int j = i + 1; j < n; j++) if (a[j] < a[min]) min = j;
    swap(a, i, min);
}
```

---

## 3. Сортировка вставками (insertion)

Берём элемент и «вставляем» его на место в уже отсортированную левую часть, сдвигая большие элементы вправо. **Адаптивна**: на почти отсортированных данных близка к O(n).

```java
for (int i = 1; i < n; i++) {
    int key = a[i], j = i - 1;
    while (j >= 0 && a[j] > key) { a[j+1] = a[j]; j--; }
    a[j+1] = key;
}
```

---

## 4. Сравнение

| Сортировка | Лучший | Средний | Худший | Стабильна? | Адаптивна? | Обменов |
|-----------|--------|---------|--------|-----------|-----------|---------|
| Пузырёк (с ранним выходом) | O(n) | O(n²) | O(n²) | да | да | много |
| Выбором | O(n²) | O(n²) | O(n²) | нет* | нет | мало (n-1) |
| Вставками | O(n) | O(n²) | O(n²) | да | да | сдвиги |

- **Стабильность** — равные элементы сохраняют относительный порядок. Важно при сортировке по нескольким ключам.
- **Адаптивность** — на почти отсортированных данных работает быстрее.

> Вывод: для малых n (десятки элементов) или почти отсортированных данных — **вставки**. Для больших — переходим к O(n log n) (темы 08–09).

## ⚠️ Подводные камни

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| Границы во вложенном цикле пузырька | выход за массив или лишние проходы | `j < n - 1 - i` |
| Пузырёк без раннего выхода | O(n²) даже на отсортированном | флаг `swapped` |
| Сравнение `>` vs `>=` в insertion | теряется стабильность | для стабильности — строгое `>` |
| «Сортировка» через сравнение по `==` для double | редко срабатывает | сравнивайте порядок, а не равенство |
| Выводы о скорости по одному запуску | шум JIT/железа | усредняйте, прогревайте |

## 🔗 Связь с другими темами

- **Вставки** — основа гибридных сортировок (тема 09: переключение на insertion для малых подмассивов).
- **Стабильность** критична для поразрядной сортировки (тема 10).
- O(n²) → мотивация перейти к «разделяй и властвуй» (темы 08–09).
- Идея сравнений/обменов восходит к анализу сложности (тема 01).
