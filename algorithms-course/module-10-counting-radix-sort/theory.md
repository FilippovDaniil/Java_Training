# Тема 10. Counting Sort и Radix Sort

Все сортировки из тем 07–09 основаны на **сравнении** элементов, и для них доказан предел — **O(n log n)**. Но если про данные известно больше (это целые числа в ограниченном диапазоне), можно сортировать **без сравнений** — распределением по «корзинам» — за **линейное** время O(n + k). Это Counting Sort и построенный на нём Radix Sort.

```
Counting Sort массива [2,5,2,1,5,2] (значения 0..5):
1) считаем частоты:   value: 0 1 2 3 4 5
                      count: 0 1 3 0 0 2
2) префиксные суммы -> позиции вставки
3) раскладываем элементы по позициям -> [1,2,2,2,5,5]
```

---

## 1. Counting Sort — O(n + k)

Где `n` — число элементов, `k` — размер диапазона значений. Шаги:
1. Посчитать, сколько раз встречается каждое значение (массив `count` размера k).
2. Превратить `count` в **префиксные суммы** — это даёт конечные позиции.
3. Пройти исходный массив (обычно **справа налево** для стабильности) и расставить элементы.

```java
int[] count = new int[k];
for (int x : a) count[x]++;                 // частоты
for (int i = 1; i < k; i++) count[i] += count[i-1];   // префиксные суммы
int[] out = new int[a.length];
for (int i = a.length - 1; i >= 0; i--)     // справа налево -> стабильность
    out[--count[a[i]]] = a[i];
```

**Ограничение:** эффективен только когда `k` (диапазон) сопоставим с `n`. Для значений 0..10⁹ массив `count` нереален.

---

## 2. Отрицательные числа — сдвиг

Counting Sort индексирует `count` значением элемента, а индекс не может быть отрицательным. Решение: **сдвиг** на минимум. Работаем со значением `x - min`, диапазон `k = max - min + 1`.

```
[-3, 1, -1, 0, 2]  min=-3  ->  индексы 0..5 (значение+3)
```

---

## 3. Radix Sort (LSD) — O(d · (n + k))

Сортирует целые **поразрядно**, от младшего разряда к старшему (LSD — Least Significant Digit). Каждый разряд сортируется **стабильным** Counting Sort. После прохода по всем `d` разрядам массив полностью отсортирован.

```
Числа: 170 045 075 090 002 802 024 066
по единицам: 170 090 002 802 024 045 075 066
по десяткам: 802 002 024 045 066 170 075 090
по сотням:   002 024 045 066 075 090 170 802  ← отсортировано
Ключ: стабильность Counting Sort сохраняет порядок предыдущих разрядов.
```

Для `int` удобно сортировать не по десятичным цифрам, а по группам **бит** (например 4 прохода по 8 бит) — `k=256`, `d=4`.

---

## 4. Сравнение с сортировками сравнением

| | Counting / Radix | Quick / Merge |
|---|------------------|---------------|
| Время | O(n + k) / O(d·(n+k)) | O(n log n) |
| На чём работает | целые / ключи фикс. длины | любые сравнимые |
| Память | O(n + k) | O(1)–O(n) |
| Обходит ли предел O(n log n) | да (не сравнивает) | нет |

> Radix Sort может обогнать quicksort на больших массивах целых/строк фиксированной длины. Но он не универсален: для произвольных объектов со сравнением остаются Quick/Merge.

---

## 5. Стабильность — почему она критична для Radix

Radix Sort работает **только** если сортировка каждого разряда **стабильна**: иначе порядок, установленный предыдущими (младшими) разрядами, разрушится. Поэтому в основе именно Counting Sort (стабильный), а не, скажем, quicksort.

---

## ⚠️ Подводные камни

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| Counting на большом диапазоне | огромный `count`, `OutOfMemory` | применять только при k ~ n |
| Забыт сдвиг для отрицательных | `ArrayIndexOutOfBounds` (отриц. индекс) | индексировать `x - min` |
| Проход слева направо в Counting | теряется стабильность | идти справа налево |
| Radix по старшему разряду первым (LSD) | неверный результат | LSD — от младшего к старшему |
| Нестабильная сортировка разряда в Radix | разрушается порядок | только стабильный Counting |
