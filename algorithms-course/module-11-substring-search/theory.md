# Тема 11. Поиск подстроки: наивный, КМП, Бойер-Мур

Задача: найти все вхождения паттерна `P` (длины m) в тексте `T` (длины n). Наивный алгоритм проверяет каждую позицию за O(n·m). «Умные» алгоритмы — КМП и Бойер-Мур — используют структуру самого паттерна, чтобы **не возвращаться назад** по тексту и пропускать заведомо безнадёжные позиции, достигая O(n+m).

```
Текст:    A B A B A B C A B A B
Паттерн:  A B A B C
Наивный сдвигает паттерн на 1 при каждом несовпадении — много лишней работы.
КМП/Бойер-Мур по несовпадению сдвигают сразу на несколько позиций.
```

---

## 1. Наивный поиск — O(n·m)

Для каждой стартовой позиции `i` в тексте сравниваем паттерн посимвольно. При несовпадении — сдвиг на 1.

```java
for (int i = 0; i + m <= n; i++) {
    int j = 0;
    while (j < m && T.charAt(i + j) == P.charAt(j)) j++;
    if (j == m) found(i);     // полное совпадение в позиции i
}
```

Худший случай (например `T="aaaa...a"`, `P="aaab"`) — действительно O(n·m).

---

## 2. КМП (Кнут-Моррис-Пратт) — O(n + m)

Идея: при несовпадении мы уже знаем часть текста (совпавший префикс паттерна) — не нужно сравнивать её заново. **Префикс-функция** (failure function) `pi[j]` = длина наибольшего собственного префикса P[0..j], который одновременно является суффиксом.

```
P = A B A B C
pi= 0 0 1 2 0
При несовпадении на позиции j откатываем j = pi[j-1] вместо j=0.
```

```java
// префикс-функция
int[] pi = new int[m];
for (int i = 1, k = 0; i < m; i++) {
    while (k > 0 && P.charAt(i) != P.charAt(k)) k = pi[k-1];
    if (P.charAt(i) == P.charAt(k)) k++;
    pi[i] = k;
}
```

Текст просматривается **один раз** без возвратов — O(n) после O(m) на префикс-функцию.

---

## 3. Бойер-Мур — на практике часто быстрее

Сравнивает паттерн **справа налево** и использует эвристику **стоп-символа** (bad character): при несовпадении сдвигает паттерн так, чтобы несовпавший символ текста встал напротив его последнего вхождения в паттерне (или за паттерн, если символа в нём нет). На «обычных» текстах часто пропускает большие куски → **сублинейно** в среднем.

```
Текст:   ... X ...        несовпавший символ X отсутствует в паттерне
Паттерн:     [P P P P]  → сдвигаем паттерн ЦЕЛИКОМ за X
```

Полный Бойер-Мур использует ещё эвристику «хорошего суффикса»; в задачах ниже ограничимся стоп-символом.

---

## 4. Сравнение

| Алгоритм | Препроцессинг | Поиск | Когда хорош |
|----------|---------------|-------|-------------|
| Наивный | — | O(n·m) | короткие тексты, разовый поиск |
| КМП | O(m) | O(n) | гарантированная линейность, маленький алфавит |
| Бойер-Мур | O(m + алфавит) | O(n/m) в среднем, O(n·m) худший | большой алфавит, длинный паттерн |

> На практике для естественных текстов Бойер-Мур (и его вариант Бойера-Мура-Хорспула) — один из самых быстрых. КМП ценен гарантией O(n) и простотой.

---

## ⚠️ Подводные камни

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| Откат `j=0` вместо `pi[j-1]` в КМП | теряется линейность (по сути наивный) | использовать префикс-функцию |
| Префикс-функция: `k = pi[k]` вместо `pi[k-1]` | неверные значения | откат именно `pi[k-1]` |
| Бойер-Мур: сдвиг может быть ≤ 0 | зацикливание | сдвиг = max(1, рассчитанный) |
| Пустой паттерн | деление/индексация падает | оговорить поведение для m=0 |
| Поиск только первого вхождения, когда нужны все | пропуск вхождений | после совпадения продолжить (для КМП: j = pi[j-1]) |
