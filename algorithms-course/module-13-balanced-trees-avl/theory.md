# Тема 13. Сбалансированные деревья: AVL (и обзор красно-чёрных)

Обычный BST вырождается в список при «плохом» порядке вставок → O(n). **Самобалансирующиеся** деревья после каждой вставки/удаления восстанавливают баланс **поворотами**, удерживая высоту ~log n и гарантируя **O(log n)** в худшем случае. AVL — исторически первое такое дерево и самое наглядное; красно-чёрные деревья (основа `TreeMap`/`TreeSet` в Java) — их «расслабленный» промышленный вариант.

```
Вставка 1,2,3 в обычный BST     →  AVL делает поворот:
   1                                    2
    \         дисбаланс!               / \
     2        (фактор = -2)    →      1   3   высота log n
      \
       3
```

---

## 1. Фактор баланса

Для каждого узла: **balance = высота(левое) − высота(правое)**. AVL-инвариант: |balance| ≤ 1 для всех узлов. Если после вставки где-то |balance| = 2 — делаем поворот.

```java
int height(Node n) { return n == null ? 0 : n.height; }
int balance(Node n) { return n == null ? 0 : height(n.left) - height(n.right); }
```

Каждый узел хранит свою **высоту**, чтобы не пересчитывать её обходом.

---

## 2. Четыре случая дисбаланса и повороты

| Случай | Условие | Лечение |
|--------|---------|---------|
| **LL** | balance > 1 и вставка в ЛЕВОЕ-ЛЕВОЕ | правый поворот |
| **RR** | balance < −1 и вставка в ПРАВОЕ-ПРАВОЕ | левый поворот |
| **LR** | balance > 1 и вставка в ЛЕВОЕ-ПРАВОЕ | левый у ребёнка, потом правый |
| **RL** | balance < −1 и вставка в ПРАВОЕ-ЛЕВОЕ | правый у ребёнка, потом левый |

```
Правый поворот вокруг y:        Левый поворот вокруг x:
      y           x                 x               y
     / \         / \               / \             / \
    x   C  →    A   y             A   y     →      x   C
   / \             / \               / \         / \
  A   B           B   C             B   C       A   B
```

---

## 3. Повороты — суть

Поворот меняет местами узел с ребёнком, **сохраняя инвариант BST** (порядок), но уменьшая высоту перекошенной стороны. После поворота обязательно **пересчитать высоты** задействованных узлов (сначала нижний, потом верхний).

```java
Node rotateRight(Node y) {
    Node x = y.left, B = x.right;
    x.right = y; y.left = B;
    updateHeight(y); updateHeight(x);   // порядок важен!
    return x;                            // новый корень поддерева
}
```

---

## 4. Вставка с балансировкой

```
insert(node, value):
  1. обычная вставка BST (рекурсивно)
  2. обновить height(node)
  3. b = balance(node)
  4. если |b| > 1 — определить случай (LL/RR/LR/RL) и повернуть
  5. вернуть (возможно новый) корень поддерева
```

Удаление аналогично: обычное удаление BST, затем на обратном пути пересчёт высот и повороты.

---

## 5. AVL vs красно-чёрные (обзор)

| | AVL | Красно-чёрные |
|---|-----|---------------|
| Баланс | строгий (|bf|≤1) | слабее (путь ≤ 2× кратчайшего) |
| Высота | ≤ 1.44 log n | ≤ 2 log n |
| Поиск | быстрее (жёстче баланс) | чуть медленнее |
| Вставка/удаление | больше поворотов | меньше поворотов |
| Где | БД-индексы, частый поиск | `TreeMap`/`TreeSet`, частые вставки |

> Java использует красно-чёрные деревья в `TreeMap`/`TreeSet` и в «корзинах» `HashMap` при длинных цепочках (тема 14). AVL изучаем как самый понятный механизм балансировки.

---

## ⚠️ Подводные камни

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| Не обновили height после поворота | неверный баланс, поломка | updateHeight(нижний), потом (верхний) |
| Перепутан случай LR/RL | дерево не балансируется | проверяйте balance ребёнка, не только узла |
| Поворот не возвращает новый корень | поддерево «теряется» | rotate возвращает новый корень, присваивайте |
| height у null считается как 0/-1 непоследовательно | off-by-one в балансе | договоритесь: height(null)=0, листа=1 |
| Балансировка только при вставке, не при удалении | дисбаланс после delete | балансируйте и в delete |
