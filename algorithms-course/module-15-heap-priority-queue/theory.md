# Тема 15. Куча (Priority Queue) на основе массива

Куча (heap) — структура, дающая **мгновенный доступ к минимуму (или максимуму)** и вставку/извлечение за **O(log n)**. Это не дерево поиска: куча упорядочена лишь «вертикально» (родитель ≤ детей для min-heap), а не «горизонтально». Зато она компактно живёт в массиве и идеальна, когда нужно многократно доставать «самый приоритетный» элемент: Дейкстра (19), Прим (20), heapsort, планировщики.

```
min-heap (родитель <= детей):       Хранение в массиве (по уровням):
          1                          index:  0  1  2  3  4  5
        /   \                        value: [1, 3, 2, 7, 4, 5]
       3     2                       для i: дети = 2i+1, 2i+2; родитель = (i-1)/2
      / \   /
     7   4 5
```

---

## 1. Двоичная куча в массиве

Полное двоичное дерево хранится в массиве **без ссылок**: для узла с индексом `i`:
- левый ребёнок: `2i + 1`
- правый ребёнок: `2i + 2`
- родитель: `(i - 1) / 2`

**Свойство кучи (min-heap):** значение узла ≤ значений его детей. Минимум всегда в корне (индекс 0).

---

## 2. Две операции восстановления порядка

### siftUp (всплытие) — после вставки в конец
Новый элемент кладём в конец массива и «поднимаем», меняя с родителем, пока он меньше родителя.

```java
void siftUp(int i) {
    while (i > 0 && a[(i-1)/2] > a[i]) { swap(i, (i-1)/2); i = (i-1)/2; }
}
```

### siftDown (погружение) — после извлечения корня
Минимум (корень) меняем с последним элементом, удаляем последний, и «топим» новый корень, меняя с **меньшим** из детей.

```java
void siftDown(int i) {
    while (2*i+1 < size) {
        int child = 2*i+1;
        if (child+1 < size && a[child+1] < a[child]) child++;   // меньший ребёнок
        if (a[i] <= a[child]) break;
        swap(i, child); i = child;
    }
}
```

---

## 3. Операции и сложности

| Операция | Как | Сложность |
|----------|-----|-----------|
| `peek` (min) | вернуть a[0] | O(1) |
| `insert` | в конец + siftUp | O(log n) |
| `extractMin` | a[0] ↔ a[last], удалить, siftDown | O(log n) |
| `heapify` (построить из массива) | siftDown от середины к началу | **O(n)** |

> **heapify за O(n)**, а не O(n log n)! Если строить кучу, вставляя по одному (n × siftUp), будет O(n log n). Но если взять готовый массив и применить siftDown ко всем «внутренним» узлам с конца — суммарно O(n) (нижние уровни многочисленны, но дёшевы).

---

## 4. Heapsort — сортировка кучей за O(n log n)

1. `heapify` массива в **max-heap** за O(n).
2. n раз: поменять корень (максимум) с последним элементом, уменьшить размер кучи, `siftDown` корня.

In-place (O(1) доп. памяти!), всегда O(n log n), но **нестабильна** и хуже quicksort по кэшу.

```
max-heap: максимум в корне -> меняем с концом -> «отрезаем» -> восстанавливаем
повторяем: массив сортируется с конца.
```

---

## 5. Применения

- **Очередь с приоритетом** — извлекать самое срочное (планировщики, события).
- **k наибольших/наименьших** — держать кучу размера k за O(n log k).
- **Дейкстра/Прим** (19–20) — извлекать вершину с минимальной оценкой.
- **decreaseKey** — уменьшить ключ и siftUp (нужно для Дейкстры).

> `java.util.PriorityQueue` — это бинарная min-heap. Но у неё нет эффективного `decreaseKey` — в Дейкстре это обходят повторной вставкой.

## ⚠️ Подводные камни

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| Индексы детей/родителя перепутаны | куча «ломается» | дети 2i+1/2i+2, родитель (i-1)/2 |
| siftDown выбирает не меньшего ребёнка | нарушено свойство кучи | сравнивать детей, брать меньшего (min-heap) |
| Построение вставками вместо heapify | O(n log n) вместо O(n) | siftDown от (size/2−1) к 0 |
| extractMin без siftDown нового корня | порядок нарушен | после замены корня — siftDown(0) |
| Куча ≠ BST (ждут отсортированный обход) | неверные ожидания | куча упорядочена только «родитель–дети» |
