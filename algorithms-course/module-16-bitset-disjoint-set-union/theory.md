# Тема 16. BitSet и Disjoint Set Union (Union-Find)

Две специализированные структуры, каждая решает свой узкий класс задач очень эффективно. **BitSet** — компактное множество целых: один бит на элемент, экономия памяти в 64 раза против `boolean[]`... вернее против `Boolean[]`, и быстрые битовые операции над множествами. **DSU (Union-Find)** — отвечает на вопрос «лежат ли два элемента в одной группе?» и умеет объединять группы — почти за O(1). Это фундамент Крускала (тема 20) и кластеризации.

```
BitSet (биты в long[]):  множество {1, 3, 4, 64}
  long[0] биты: ...0 1 1 0 1 0  (биты 1,3,4 установлены)
  long[1] бит 0: 1               (бит 64 -> слово 64/64=1, позиция 64%64=0)

DSU: элементы 0..6, объединения (0-1),(2-3),(3-4)
  группы: {0,1}  {2,3,4}  {5}  {6}
  find(2)==find(4)? да (одна группа). find(0)==find(5)? нет.
```

---

## 1. BitSet вручную (на `long[]`)

Каждый `long` хранит 64 бита. Для бита номер `i`:
- слово: `i >> 6` (т.е. `i / 64`);
- позиция в слове: `i & 63` (т.е. `i % 64`).

```java
void set(int i)   { words[i >> 6] |=  (1L << (i & 63)); }
void clear(int i) { words[i >> 6] &= ~(1L << (i & 63)); }
boolean get(int i){ return (words[i >> 6] & (1L << (i & 63))) != 0; }
```

`nextSetBit(from)` — найти следующий установленный бит; полезно для итерации по множеству. Битовые операции над двумя BitSet (AND/OR/XOR) дают пересечение/объединение/симметрическую разность множеств за O(n/64).

> Экономия: 1 млн флагов в `boolean[]` ≈ 1 МБ, в BitSet ≈ 125 КБ. Плюс быстрые массовые операции.

---

## 2. DSU (Disjoint Set Union / Union-Find)

Хранит разбиение элементов на непересекающиеся множества. Три операции:
- `makeSet(x)` — создать одноэлементное множество;
- `find(x)` — вернуть «представителя» (корень) множества x;
- `union(x, y)` — объединить множества, содержащие x и y.

Представление — массив `parent[]`: каждый элемент ссылается на родителя; корень ссылается на себя. `find` идёт по родителям до корня.

```java
int find(int x) { while (parent[x] != x) x = parent[x]; return x; }
void union(int x, int y) { parent[find(x)] = find(y); }
```

Без оптимизаций деревья могут выродиться → `find` за O(n). Спасают две эвристики.

---

## 3. Две эвристики → почти O(1)

### Сжатие пути (path compression)
Во время `find` «подвешиваем» все пройденные узлы прямо к корню — следующие вызовы будут мгновенными.
```java
int find(int x) { return parent[x] == x ? x : (parent[x] = find(parent[x])); }
```

### Объединение по рангу/размеру (union by rank/size)
Меньшее дерево подвешиваем к большему — так высота растёт медленно.

Вместе эти две эвристики дают амортизированную сложность **O(α(n))**, где α — обратная функция Аккермана (≤ 4 для любых мыслимых n) → практически **O(1)**.

---

## 4. Применения DSU

- **Компоненты связности** графа (сколько «островов»): union по рёбрам, число различных корней = число компонент.
- **Алгоритм Крускала** (тема 20): добавляем ребро, если его концы в разных множествах (нет цикла).
- **Кластеризация**: объединяем «похожие» элементы в группы.
- **Динамическая связность**: быстро отвечать «связаны ли a и b?» по мере добавления связей.

---

## 5. Решето Эратосфена на битах

Классическое применение BitSet: пометить составные числа до N. Идём по простым p, помечаем p², p²+p, ... как составные. Битовый массив экономит память при больших N.

## ⚠️ Подводные камни

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| `1 << i` вместо `1L << i` для long | биты выше 31 не ставятся | используйте `1L` |
| Отрицательный сдвиг/индекс | мусор в битах | проверяйте границы i |
| DSU без сжатия пути | `find` деградирует до O(n) | path compression |
| `union(x,y)` без `find` обоих | объединяет не корни | `parent[find(x)] = find(y)` |
| Сравнение `find(a)==find(b)` после изменения структуры | устаревший корень | всегда вызывайте `find` заново |
