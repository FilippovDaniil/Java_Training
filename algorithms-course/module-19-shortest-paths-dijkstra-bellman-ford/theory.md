# Тема 19. Кратчайшие пути: Дейкстра и Беллман-Форд

BFS находит кратчайший путь, только когда все рёбра «равны». Во **взвешенном** графе (рёбра несут стоимость) нужны другие алгоритмы. **Дейкстра** — для неотрицательных весов, быстрый (O((V+E) log V) с кучей). **Беллман-Форд** — для любых весов, умеет находить **отрицательные циклы**, но медленнее (O(V·E)). Это основа навигации, маршрутизации, логистики.

```
Граф с весами:        Кратчайшие расстояния от 0 (Дейкстра):
   0 --4-- 1            dist[0]=0
   |      /             dist[1]=3  (через 2: 1+2)
   1     2              dist[2]=1
   |    /               dist[3]=4  (0->2->1->3 или 0->2->3)
   2 --1-- 3
```

---

## 1. Дейкстра — жадный, с приоритетной очередью

Поддерживаем `dist[]` — лучшую известную стоимость от старта. На каждом шаге берём **ещё не финализированную вершину с минимальным dist** (это даёт min-heap, тема 15) и «релаксируем» её рёбра: если через неё до соседа дешевле — обновляем.

```java
dist[start] = 0;
PriorityQueue<int[]> pq = new PriorityQueue<>((a,b) -> a[1] - b[1]); // [вершина, dist]
pq.add(new int[]{start, 0});
while (!pq.isEmpty()) {
    int[] top = pq.poll(); int u = top[0], d = top[1];
    if (d > dist[u]) continue;                 // устаревшая запись — пропустить
    for (Edge e : adj.get(u))
        if (dist[u] + e.w < dist[e.to]) {       // релаксация
            dist[e.to] = dist[u] + e.w;
            pq.add(new int[]{e.to, dist[e.to]});
        }
}
```

> **Почему не работает с отрицательными рёбрами:** Дейкстра «финализирует» вершину, считая, что более дешёвого пути уже не найдётся. Отрицательное ребро может это нарушить.

**Сложность:** O((V + E) log V) с бинарной кучей. У `PriorityQueue` нет `decreaseKey`, поэтому добавляют новую запись и пропускают устаревшие (`d > dist[u]`).

---

## 2. Восстановление пути

Помимо `dist[]` храним `prev[v]` — из какой вершины пришли в v по оптимальному пути. Путь до t: идём `t -> prev[t] -> ... -> start`, затем разворачиваем (как в BFS, тема 18).

---

## 3. Беллман-Форд — для любых весов

Делает **V−1 итераций**, на каждой релаксируя **все рёбра**. После V−1 проходов все кратчайшие пути найдены (путь не длиннее V−1 рёбер). Дополнительный V-й проход: если что-то ещё улучшается — есть **отрицательный цикл**.

```java
for (int i = 0; i < V - 1; i++)
    for (Edge (u,v,w) : edges)
        if (dist[u] + w < dist[v]) dist[v] = dist[u] + w;   // релаксация всех рёбер
// V-я итерация: если ещё релаксируется -> отрицательный цикл
```

**Сложность:** O(V·E) — медленнее Дейкстры, но универсальнее.

---

## 4. Сравнение

| | Дейкстра | Беллман-Форд |
|---|----------|--------------|
| Веса | только **неотрицательные** | любые (в т.ч. отрицательные) |
| Отрицательный цикл | не обрабатывает | **обнаруживает** |
| Сложность | O((V+E) log V) | O(V·E) |
| Структура | приоритетная очередь | простые проходы по рёбрам |
| Когда | дороги, расстояния (веса ≥ 0) | курсы валют, графы с долгами |

Для **плотных** графов Дейкстру иногда реализуют без кучи, выбирая минимум за O(V) — итого O(V²); это выгоднее, когда E ≈ V².

---

## 5. Вариации

- **Кратчайший путь с ограничением на число рёбер** — модификация Беллмана-Форда: ровно k итераций релаксации (k рёбер максимум).
- **A\*** — Дейкстра с эвристикой (направляет поиск к цели); вне рамок темы.
- **Флойд-Уоршелл** — все пары кратчайших путей за O(V³); упоминается как альтернатива.

## ⚠️ Подводные камни

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| Дейкстра на отрицательных рёбрах | неверные расстояния | Беллман-Форд |
| Не пропускать устаревшие записи в PQ | лишняя работа / неверно | `if (d > dist[u]) continue;` |
| `dist` не инициализирован бесконечностью | релаксация ломается | `Integer.MAX_VALUE` (осторожно с переполнением!) |
| Переполнение при `dist[u] + w` | отрицательные суммы из-за MAX_VALUE | проверять `dist[u] != INF` перед сложением |
| Беллман-Форд: < V−1 итераций | не сходится | ровно V−1 проходов |
