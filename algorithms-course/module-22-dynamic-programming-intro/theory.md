# Тема 22. Введение в динамическое программирование (DP)

Динамическое программирование — это «умный перебор»: задача разбивается на **перекрывающиеся подзадачи**, ответы на которые **запоминаются**, чтобы не считать одно и то же дважды. Там, где наивная рекурсия даёт экспоненту (наивный Фибоначчи, тема 05), DP даёт полином. Это не отдельный алгоритм, а **приём**: распознать подзадачи, выразить ответ через них (рекуррента) и вычислить в правильном порядке.

```
Наивный fib(5):                  DP fib(5):
       fib(5)                    каждое fib(k) считается ОДИН раз
      /      \                   и сохраняется:
   fib(4)   fib(3)               fib: [0,1,1,2,3,5]
   /  \      /  \                 idx: 0 1 2 3 4 5
fib(3) fib(2)...   ← fib(3) считается дважды!
экспонента                       O(n)
```

---

## 1. Два признака, что задача — на DP

1. **Перекрывающиеся подзадачи** — одни и те же подзадачи встречаются многократно (иначе хватит «разделяй и властвуй»).
2. **Оптимальная подструктура** — оптимальное решение задачи строится из оптимальных решений подзадач.

Если оба есть — почти наверняка DP.

---

## 2. Два стиля: мемоизация vs табуляция

### Мемоизация (top-down) — рекурсия + кэш
Пишем естественную рекурсию, но перед вычислением проверяем кэш; результат сохраняем.
```java
Integer[] memo;
int f(int n) {
    if (n < 2) return n;
    if (memo[n] != null) return memo[n];
    return memo[n] = f(n-1) + f(n-2);
}
```

### Табуляция (bottom-up) — цикл, заполняем таблицу
Решаем подзадачи **снизу вверх**, от базовых к целевой.
```java
int[] dp = new int[n+1];
dp[0] = 0; dp[1] = 1;
for (int i = 2; i <= n; i++) dp[i] = dp[i-1] + dp[i-2];
```

| | Мемоизация | Табуляция |
|---|-----------|-----------|
| Стиль | рекурсия + кэш | цикл по таблице |
| Считает только нужные подзадачи | да | все (в заданном диапазоне) |
| Риск | `StackOverflow` на глубине | нет |
| Память можно ужать | сложнее | да (хранить последние строки) |

---

## 3. Рецепт решения DP-задачи

1. **Состояние:** что значит `dp[i]` (или `dp[i][j]`)? Сформулируй словами.
2. **База:** чему равны тривиальные состояния.
3. **Переход (рекуррента):** как `dp[i]` выражается через меньшие состояния.
4. **Порядок вычисления:** такой, чтобы при счёте `dp[i]` всё нужное уже посчитано.
5. **Ответ:** в каком состоянии лежит финальный результат.

---

## 4. Классические задачи (в практике темы)

| Задача | Состояние dp | Переход |
|--------|--------------|---------|
| Лестница (1/2 шага) | dp[i] = число способов дойти до i | dp[i] = dp[i-1] + dp[i-2] |
| Рюкзак 0/1 | dp[i][w] = макс ценность из первых i при лимите w | взять / не взять предмет i |
| LCS | dp[i][j] = длина LCS префиксов | совпали символы → +1, иначе max |
| Левенштейн | dp[i][j] = расстояние правок | min(вставка, удаление, замена) |
| Размен монет | dp[s] = мин монет на сумму s | min по монетам: dp[s-coin]+1 |

---

## 5. Связь с рекурсией и графами

DP — это рекурсия без повторного счёта. Подзадачи образуют **DAG зависимостей** (тема 21): порядок вычисления = топологический порядок этого DAG. Поэтому «DP на DAG» (longest path, число путей) — частный случай общей идеи.

---

## ⚠️ Подводные камни

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| Неверно задано состояние | переход «не сходится» | сначала чётко сформулируй, что хранит dp[i] |
| Забыта база | мусор/исключения | инициализируй тривиальные состояния |
| Неверный порядок заполнения | используются непосчитанные ячейки | считай от базы к цели |
| Наивная рекурсия без кэша | экспонента | добавь мемоизацию |
| Перепутаны индексы строк/предметов в рюкзаке | неверный ответ | dp[i][w]: i — предметы, w — вес |
