# Тема 24. Алгоритмы на строках: префиксные деревья (Trie) и Ахо-Корасик

**Trie** (бор, префиксное дерево) — структура для хранения множества строк, где общие префиксы хранятся один раз. Поиск слова и проверка префикса — за O(длины слова), независимо от размера словаря. На Trie строятся автодополнение и словарные проверки. **Ахо-Корасик** надстраивает над бором «суффиксные ссылки» и ищет **сразу множество паттернов** в тексте за один проход — O(длина текста + сумма длин паттернов + число совпадений).

```
Trie для {"cat","car","card","dog"}:
        (root)
        /     \
       c       d
       |       |
       a       o
      / \      |
     t*  r     g*       (* = конец слова)
         |
         d*
Префикс "ca" ведёт к ветке cat/car/card.
```

---

## 1. Узел Trie и операции

Каждый узел — массив/мапа детей по символам + флаг «здесь кончается слово».

```java
class TrieNode {
    TrieNode[] children = new TrieNode[26];   // для 'a'..'z'
    boolean isWord;
}
```

| Операция | Как | Сложность |
|----------|-----|-----------|
| `insert(word)` | спускаемся, создавая узлы | O(L) |
| `search(word)` | спускаемся; в конце проверяем isWord | O(L) |
| `startsWith(prefix)` | спускаемся; узел существует? | O(L) |

(L — длина слова; от числа слов в словаре сложность НЕ зависит.)

---

## 2. Автодополнение и подсчёт

- **Автодополнение:** дойти по префиксу до узла, затем обойти поддерево (DFS), собирая все слова.
- **Подсчёт слов с префиксом:** хранить в каждом узле счётчик «сколько слов проходит через него».
- **Удаление:** снять флаг isWord; при желании — подчистить «висячие» узлы без детей.

> **Сжатое префиксное дерево (radix tree):** склеивает цепочки из узлов с одним ребёнком в одно ребро-строку — экономит память. Используется в маршрутизации, IP-таблицах.

---

## 3. Ахо-Корасик — поиск множества паттернов

Задача: найти в тексте все вхождения МНОГИХ паттернов сразу. Наивно — запускать поиск каждого паттерна (медленно). Ахо-Корасик строит автомат:

1. **Бор** из всех паттернов.
2. **Суффиксные ссылки** (fail links): из узла-состояния `v` ведут в самый длинный собственный суффикс пути `v`, который тоже есть в боре. Аналог префикс-функции КМП (тема 11), но для множества строк.
3. **Прогон текста:** идём по автомату; на несовпадении — по суффиксной ссылке (как откат в КМП); в каждом состоянии собираем совпавшие паттерны (через «выходные» ссылки).

```
Суффиксные ссылки строятся BFS по бору (как уровни дерева).
Поиск: один проход по тексту, переходы по автомату — O(n + совпадения).
```

**Сложность:** O(сумма длин паттернов) на построение + O(длина текста + число совпадений) на поиск.

---

## 4. Когда что применять

| Задача | Структура |
|--------|-----------|
| Автодополнение, словарь по префиксу | Trie |
| Один паттерн в тексте | КМП / Бойер-Мур (тема 11) |
| Много паттернов одновременно | **Ахо-Корасик** |
| Фильтр запрещённых слов | Trie или Ахо-Корасик |

---

## ⚠️ Подводные камни

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| `search` не проверяет isWord | префикс принимается за слово | в конце проверяйте флаг конца слова |
| Индекс ребёнка: `c - 'a'` для не-a..z | выход за границы | ограничьте алфавит или используйте Map |
| Ахо-Корасик: суффиксные ссылки строят DFS | неверные ссылки | стройте BFS (по уровням бора) |
| Забыты «выходные» ссылки | пропуск вложенных совпадений | по fail-ссылкам собирать все слова-суффиксы |
| Удаление физически рвёт общие префиксы | теряются другие слова | удаляйте только флаг/чистите аккуратно |
