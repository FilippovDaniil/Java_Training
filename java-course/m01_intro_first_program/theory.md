# Модуль 01. Вводное занятие. Команды и первая программа

## Что такое Java

**Java** — язык программирования и платформа. Главная идея — «*написал один раз — запускай везде*» (Write Once, Run Anywhere). Исходный код компилируется не в машинный код конкретного процессора, а в промежуточный **байт-код**, который выполняет виртуальная машина.

### JDK, JRE, JVM — три кита

```
+-----------------------------------------+
|  JDK (Java Development Kit)              |  ← всё для разработки
|  +---------------------------------+    |
|  |  JRE (Java Runtime Environment) |    |  ← всё для запуска
|  |  +---------------------------+  |    |
|  |  |  JVM (Java Virtual Machine)| |    |  ← исполняет байт-код
|  |  +---------------------------+  |    |
|  |  + стандартные библиотеки       |    |
|  +---------------------------------+    |
|  + компилятор javac, отладчик и т.д.    |
+-----------------------------------------+
```

| Аббревиатура | Что это | Зачем |
|--------------|---------|-------|
| **JVM** | Виртуальная машина | исполняет байт-код `.class` |
| **JRE** | JVM + библиотеки | запуск готовых программ |
| **JDK** | JRE + инструменты (`javac`) | разработка программ |

Для разработки нужен **JDK** (версия 11 или выше).

## Как работает программа: от исходника до запуска

```
   Hello.java          Hello.class           вывод в консоль
  (исходный код)  --▶  (байт-код)      --▶   "Hello, World!"
                  javac              java
                (компиляция)      (выполнение JVM)
```

1. Пишем исходный код в файл с расширением `.java`.
2. Компилятор `javac` превращает его в байт-код `.class`.
3. Команда `java` запускает байт-код на JVM.

## Первая программа

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}
```

Разберём по частям:

| Элемент | Значение |
|---------|----------|
| `public class HelloWorld` | объявление класса. **Имя класса = имя файла** (`HelloWorld.java`) |
| `public static void main(String[] args)` | точка входа — отсюда начинается выполнение |
| `System.out.println(...)` | печать строки в консоль с переводом строки |
| `;` | каждая инструкция заканчивается точкой с запятой |
| `{ }` | фигурные скобки задают границы блока кода |

> **Правило:** в одном файле может быть только один `public`-класс, и его имя обязано совпадать с именем файла.

## Команды компиляции и запуска

```
javac HelloWorld.java     # компиляция → создаст HelloWorld.class
java HelloWorld           # запуск (без расширения .class!)
```

В IntelliJ IDEA эти шаги выполняются автоматически по нажатию ▶.

## Вывод в консоль: `print` vs `println`

```java
System.out.print("A");      // печатает без перевода строки
System.out.print("B");      // → AB
System.out.println("C");    // печатает и переводит строку
System.out.println("D");    // → C\nD\n
```

- `println` («print line») — печатает аргумент и **переходит на новую строку**.
- `print` — печатает **без** перехода на новую строку.
- Спецсимвол `\n` внутри строки — тоже перевод строки.

## Комментарии

```java
// однострочный комментарий — до конца строки

/*
   многострочный
   комментарий
*/

/**
 * JavaDoc-комментарий — для документации классов и методов.
 */
```

Комментарии **игнорируются** компилятором, но критически важны для людей.

## Подводные камни новичка

| Ошибка | Причина | Как избежать |
|--------|---------|--------------|
| `class X is public, should be in X.java` | имя класса ≠ имя файла | переименуйте файл или класс |
| `cannot find symbol` | опечатка в имени метода/переменной | проверьте регистр букв |
| забыли `;` | каждая инструкция требует точку с запятой | следите за концом строк |
| `Main method not found` | нет `public static void main(String[] args)` | проверьте сигнатуру точно |

> Java **чувствительна к регистру**: `System` ≠ `system`, `Main` ≠ `main`.
