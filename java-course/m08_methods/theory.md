# Модуль 08. Методы (функции)

**Метод** — это именованный блок кода, который можно вызывать многократно. Методы убирают дублирование и разбивают программу на понятные части.

## Анатомия метода

```java
//  модификатор  возвращаемый_тип  имя(параметры)
    public static int            sum(int a, int b) {
        return a + b;     // возвращаемое значение
    }
```

| Часть | Значение |
|-------|----------|
| `public` | модификатор доступа (видимость) |
| `static` | метод принадлежит классу, а не объекту (см. [модуль 12](../m12_classes_static/theory.md)) |
| `int` | тип возвращаемого значения |
| `sum` | имя метода (camelCase, глагол) |
| `(int a, int b)` | список параметров |
| `return` | возвращает результат и завершает метод |

> Пока все наши методы — `static`, потому что вызываются из `static main` без объекта.

---

## Вызов метода

```java
public static void main(String[] args) {
    int result = sum(3, 5);     // 3 и 5 — аргументы
    System.out.println(result); // 8
}
```

- **Параметры** — переменные в объявлении метода (`int a, int b`).
- **Аргументы** — конкретные значения при вызове (`3, 5`).

---

## Методы без возврата: `void`

`void` означает «метод ничего не возвращает».

```java
public static void greet(String name) {
    System.out.println("Привет, " + name + "!");
    // return не обязателен
}
```

---

## Возврат значения и `return`

```java
public static boolean isEven(int n) {
    return n % 2 == 0;    // вернёт true или false
}
```

> `return` немедленно завершает метод. Код после `return` не выполнится.
> Метод с типом возврата (не `void`) **обязан** вернуть значение по всем путям.

---

## Передача по значению (pass-by-value)

Java **всегда передаёт копию значения** аргумента.

```java
public static void increment(int x) {
    x++;   // меняется только локальная копия
}

int a = 5;
increment(a);
System.out.println(a);   // 5 — НЕ изменилось!
```

> ⚠️ Для **массивов и объектов** копируется **ссылка**, поэтому изменения элементов массива внутри метода видны снаружи:
> ```java
> public static void fillZero(int[] arr) { arr[0] = 0; }  // изменит исходный массив
> ```

---

## Методы с массивами

```java
public static int maxOf(int[] arr) {
    int max = arr[0];
    for (int x : arr) if (x > max) max = x;
    return max;
}
```

---

## Область видимости (scope)

Переменные, объявленные внутри метода (**локальные**), существуют только внутри него.

```java
public static void a() {
    int x = 10;   // x виден только внутри a()
}
// здесь x недоступен
```

## Рекурсия — метод вызывает сам себя

Рекурсивный метод состоит из двух частей:
1. **Базовый случай** — условие остановки.
2. **Рекурсивный шаг** — вызов себя с «меньшей» задачей.

```java
public static long factorial(int n) {
    if (n <= 1) return 1;            // базовый случай
    return n * factorial(n - 1);     // рекурсивный шаг
}
```

Как разворачивается `factorial(4)`:

```
factorial(4) = 4 * factorial(3)
             = 4 * 3 * factorial(2)
             = 4 * 3 * 2 * factorial(1)
             = 4 * 3 * 2 * 1 = 24
```

> ⚠️ Без базового случая — бесконечная рекурсия и `StackOverflowError`.
> Любую рекурсию можно переписать циклом; выбирайте по читаемости.

---

## Зачем нужны методы

| Без методов | С методами |
|-------------|------------|
| дублирование кода | переиспользование |
| длинный `main` | короткие именованные блоки |
| трудно тестировать | каждый метод проверяется отдельно |

---

## Подводные камни

| Ошибка | Объяснение |
|--------|-----------|
| `missing return statement` | не все ветки возвращают значение |
| изменение примитива внутри метода | не влияет на оригинал (pass-by-value) |
| рекурсия без базового случая | `StackOverflowError` |
| вызов нестатического из `static` | пока делаем все методы `static` |
