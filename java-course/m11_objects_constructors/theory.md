# Модуль 11. Объекты: конструкторы, инкапсуляция, equals/hashCode

Это ключевой ООП-модуль. Учимся правильно создавать объекты и защищать их данные.

## Конструктор

**Конструктор** — специальный метод, который вызывается при создании объекта (`new`). Он инициализирует поля. Имя конструктора = имя класса, типа возврата нет.

```java
class Book {
    String title;
    String author;
    int year;

    // конструктор
    Book(String title, String author, int year) {
        this.title = title;     // this.title — поле объекта
        this.author = author;   // title (без this) — параметр
        this.year = year;
    }
}
```

```java
Book b = new Book("1984", "Оруэлл", 1949);  // вызов конструктора
```

### Ключевое слово `this`

`this` — ссылка на **текущий объект**. Нужна, чтобы отличить поле от параметра с тем же именем:

```java
this.title = title;   // поле = параметр
```

### Конструктор по умолчанию

Если **не** написать ни одного конструктора, Java добавит пустой автоматически. Но как только вы объявите свой конструктор — пустой исчезает (нужно добавлять вручную при необходимости).

### Перегрузка конструкторов и `this(...)`

Можно объявить несколько конструкторов с разными параметрами. Один может вызвать другой через `this(...)`:

```java
Book(String title) {
    this(title, "Неизвестен", 0);   // вызов другого конструктора
}
```

## Инкапсуляция

**Инкапсуляция** — сокрытие внутренних данных и предоставление доступа через методы. Поля делают `private`, доступ — через **геттеры** и **сеттеры**.

```java
class Account {
    private double balance;   // скрыто извне

    public double getBalance() {      // геттер — чтение
        return balance;
    }

    public void setBalance(double balance) {   // сеттер — запись
        if (balance < 0) {                       // ВАЛИДАЦИЯ!
            System.out.println("Баланс не может быть отрицательным");
            return;
        }
        this.balance = balance;
    }
}
```

### Зачем инкапсуляция

| Без инкапсуляции | С инкапсуляцией |
|------------------|-----------------|
| `account.balance = -1000;` — любой может испортить данные | сеттер проверит и отклонит |
| нельзя изменить логику без правки всех мест | вся логика в одном методе |

| Модификатор | Видимость |
|-------------|-----------|
| `private` | только внутри класса |
| (без модификатора) | внутри пакета |
| `protected` | пакет + потомки |
| `public` | везде |

## Метод `toString()`

По умолчанию печать объекта даёт что-то вроде `Book@1b6d3586`. Переопределите `toString()`, чтобы получить читаемое представление:

```java
@Override
public String toString() {
    return "Book{title='" + title + "', year=" + year + "}";
}
```

```java
System.out.println(b);   // Book{title='1984', year=1949}
```

> `@Override` — аннотация, сообщающая компилятору, что мы переопределяем метод родителя (`Object`).

## `equals()` и `hashCode()`

По умолчанию `equals` сравнивает **ссылки** (как `==`). Чтобы сравнивать объекты **по содержимому**, переопределите `equals` и **обязательно** `hashCode`.

```java
@Override
public boolean equals(Object o) {
    if (this == o) return true;
    if (o == null || getClass() != o.getClass()) return false;
    Book book = (Book) o;
    return year == book.year && title.equals(book.title);
}

@Override
public int hashCode() {
    return Objects.hash(title, year);   // из java.util.Objects
}
```

### Контракт equals/hashCode

```
Если a.equals(b) == true  ⟹  a.hashCode() == b.hashCode()
```

> 🔑 **Правило:** переопределяете `equals` — обязательно переопределяйте `hashCode`. Иначе объекты будут некорректно работать в `HashSet`/`HashMap` (см. [модули 14](../m14_collections_set_iterator/theory.md), [15](../m15_map_collections_framework/theory.md)).

IDEA генерирует `equals`/`hashCode`/`toString` автоматически: `Alt + Insert`.

## Подводные камни

| Ошибка | Объяснение |
|--------|-----------|
| забыли `this` при совпадении имён | параметр присваивается сам себе |
| `equals` без `hashCode` | поломка в коллекциях на хешах |
| `public` поля вместо `private` | нарушение инкапсуляции |
| сравнение объектов через `==` | сравнивает ссылки, не содержимое |
| свой конструктор «убил» пустой | добавьте `new` с аргументами или объявите пустой конструктор |
