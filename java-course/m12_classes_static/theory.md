# Модуль 12. Классы и `static`

`static` означает «принадлежит **классу**, а не конкретному объекту». Статический член существует в единственном экземпляре, общий для всех объектов.

## Статические поля

Обычное поле принадлежит каждому объекту отдельно. Статическое — **общее для всех**.

```java
class Counter {
    static int totalCount = 0;   // одно на весь класс
    int id;                      // у каждого объекта своё

    Counter() {
        totalCount++;            // увеличиваем общий счётчик
        id = totalCount;         // присваиваем индивидуальный номер
    }
}
```

```java
new Counter();   // totalCount = 1
new Counter();   // totalCount = 2
new Counter();   // totalCount = 3
System.out.println(Counter.totalCount);  // 3 — обращение через ИМЯ КЛАССА
```

```
        Counter.totalCount = 3   (одно на класс)
        ┌──────────┬──────────┬──────────┐
объекты:│ id = 1   │ id = 2   │ id = 3   │  (своё у каждого)
        └──────────┴──────────┴──────────┘
```

## Статические методы

Вызываются через имя класса, **без создания объекта**. Внутри `static`-метода нельзя обращаться к нестатическим полям/методам (у них нет «своего» объекта).

```java
class MathUtils {
    static int max(int a, int b) {
        return a > b ? a : b;
    }
}

int m = MathUtils.max(3, 7);   // 7 — без new
```

> Все методы класса `Math` (`Math.sqrt`, `Math.abs`), `Integer.parseInt`, и метод `main` — статические.

| | Обычный (instance) | Статический (static) |
|--|--------------------|----------------------|
| Принадлежит | объекту | классу |
| Вызов | `obj.method()` | `Class.method()` |
| Доступ к полям объекта | да | нет |
| Когда применять | поведение конкретного объекта | утилиты, не зависящие от состояния |

## Статические константы

```java
class Circle {
    static final double PI = 3.14159;   // общая константа
}
double area = Circle.PI * r * r;
```

`static final` — классический способ объявить именованную константу.

## Блоки инициализации

```java
class Config {
    static int version;
    int instanceField;

    static {                  // СТАТИЧЕСКИЙ блок — выполняется ОДИН раз
        version = 1;          // при первой загрузке класса
        System.out.println("Класс загружен");
    }

    {                         // блок инициализации ЭКЗЕМПЛЯРА —
        instanceField = 10;   // выполняется при КАЖДОМ создании объекта
    }
}
```

Порядок выполнения при `new`:
```
static-блок (1 раз) → блок экземпляра → конструктор
```

## Вложенные классы

Класс можно объявить **внутри** другого класса.

### Статический вложенный класс

```java
class Outer {
    static class Nested {       // не зависит от объекта Outer
        void hello() { System.out.println("Я вложенный"); }
    }
}

Outer.Nested n = new Outer.Nested();
```

### Внутренний (inner) класс

```java
class Outer {
    int x = 5;
    class Inner {                // привязан к объекту Outer
        void show() { System.out.println(x); }  // имеет доступ к полям Outer
    }
}

Outer outer = new Outer();
Outer.Inner inner = outer.new Inner();
```

| Вид | Доступ к полям внешнего объекта | Создание |
|-----|-------------------------------|----------|
| `static class` | нет | `new Outer.Nested()` |
| inner `class` | да | `outer.new Inner()` |

> Вложенные классы группируют тесно связанную логику. Подробнее (анонимные классы) — в [модуле 23](../m23_oop_overloading_overriding_abstract/theory.md).

## Утилитный класс (паттерн)

Класс только со `static`-методами и приватным конструктором (чтобы нельзя было создать объект):

```java
class StringUtils {
    private StringUtils() {}    // запрет создания

    static boolean isBlank(String s) {
        return s == null || s.trim().isEmpty();
    }
}
```

## Подводные камни

| Ошибка | Объяснение |
|--------|-----------|
| доступ к нестатическому полю из `static` | у статического метода нет объекта |
| `static`-поле как «глобальная переменная» | общее состояние; меняется всеми — осторожно |
| создание объекта ради `static`-метода | вызывайте через имя класса |
| изменение `static` поля в многопотоке | требует синхронизации (см. [модуль 25](../m25_multithreading_concurrency/theory.md)) |
