# Модуль 36. Устройство сети. Сетевая модель

> ⚠️ Это **теоретический** модуль — практических задач нет.

Сеть позволяет компьютерам обмениваться данными. Чтобы упорядочить это, придумали **сетевые модели** — слои, каждый из которых решает свою задачу.

## Модель OSI (7 уровней)

Эталонная модель взаимодействия открытых систем. Данные при отправке идут сверху вниз, при получении — снизу вверх.

```
 +----------------------------------------------+
 | 7. Прикладной (Application)  HTTP, FTP, DNS    |  ← программы
 | 6. Представления (Presentation) шифрование,    |
 |    кодировки                                   |
 | 5. Сеансовый (Session)  управление сессиями    |
 | 4. Транспортный (Transport)  TCP, UDP, порты   |  ← доставка
 | 3. Сетевой (Network)  IP, маршрутизация         |  ← адресация
 | 2. Канальный (Data Link)  MAC, Ethernet         |
 | 1. Физический (Physical)  кабели, сигналы       |  ← «железо»
 +----------------------------------------------+
```

| Уровень | Что делает | Пример |
|---------|-----------|--------|
| 7 Прикладной | взаимодействие с приложением | HTTP, SMTP, DNS |
| 6 Представления | формат, шифрование | TLS, JPEG |
| 5 Сеансовый | управление соединениями | сессии |
| 4 Транспортный | надёжная/быстрая доставка | TCP, UDP |
| 3 Сетевой | адресация и маршрутизация | IP |
| 2 Канальный | передача в локальной сети | Ethernet, MAC |
| 1 Физический | биты по среде | кабель, Wi-Fi |

> Мнемоника (снизу вверх, англ.): **P**lease **D**o **N**ot **T**hrow **S**ausage **P**izza **A**way.

## Модель TCP/IP (4 уровня) — практическая

В реальности интернет использует упрощённую модель TCP/IP:

```
 OSI                      TCP/IP
 7,6,5  Прикладной    -▶  Прикладной  (HTTP, DNS, FTP)
 4      Транспортный  -▶  Транспортный (TCP, UDP)
 3      Сетевой       -▶  Интернет    (IP)
 2,1    Канальный/физ -▶  Сетевой доступ (Ethernet)
```

---

## IP-адреса и порты

- **IP-адрес** — адрес устройства в сети.
  - IPv4: `192.168.1.10` (4 байта).
  - IPv6: `2001:0db8:...` (16 байт, для исчерпания IPv4).
- **Порт** — номер «двери» приложения на устройстве (0–65535).

```
   192.168.1.10 : 8080
   +-- IP ----+   +port+
   какой компьютер  какое приложение
```

| Порт | Сервис |
|------|--------|
| 80 | HTTP |
| 443 | HTTPS |
| 22 | SSH |
| 53 | DNS |
| 5432 | PostgreSQL |

## TCP vs UDP

| | TCP | UDP |
|--|-----|-----|
| Соединение | устанавливается (handshake) | без соединения |
| Надёжность | гарантирует доставку и порядок | без гарантий |
| Скорость | медленнее (накладные расходы) | быстрее |
| Применение | web, файлы, почта | видео, игры, DNS |

### TCP «рукопожатие» (3-way handshake)

```
Клиент ---- SYN ----▶ Сервер
Клиент ◀-- SYN-ACK -- Сервер
Клиент ---- ACK ----▶ Сервер
        соединение установлено
```

## DNS — система доменных имён

DNS переводит понятное имя (`example.com`) в IP-адрес (`93.184.216.34`).

```
Браузер: "где example.com?"
   |
   ▼
DNS-сервер --▶ "93.184.216.34"
   |
   ▼
Браузер подключается к 93.184.216.34:443
```

> Это как телефонная книга интернета: имя → номер (IP).

## Топологии сети

```
  Звезда (star)         Шина (bus)           Кольцо (ring)
      ●                ●-●-●-●-●                ● - ●
    / | \              (общая линия)           |     |
   ●  ●  ●                                      ● - ●
  (через центр)                              (по кругу)
```

| Топология | Плюс | Минус |
|-----------|------|-------|
| Звезда | отказ узла не валит сеть | зависит от центра |
| Шина | просто, дёшево | обрыв линии = отказ |
| Кольцо | равномерная нагрузка | разрыв ломает кольцо |

## Дополнительные понятия

| Термин | Значение |
|--------|----------|
| **MAC-адрес** | физический адрес сетевой карты |
| **NAT** | трансляция адресов (много устройств за одним внешним IP) |
| **Маршрутизатор (router)** | соединяет сети, выбирает путь пакетов |
| **Коммутатор (switch)** | соединяет устройства в локальной сети |
| **Шлюз (gateway)** | точка выхода из локальной сети |
| **Брандмауэр (firewall)** | фильтрует трафик по правилам |
| **Пакет** | блок данных с адресами отправителя/получателя |

## Где это в Java

- Низкий уровень: `java.net.Socket`, `ServerSocket` (TCP), `DatagramSocket` (UDP).
- Высокий уровень: `java.net.http.HttpClient` (HTTP — см. [модуль 38](../m38_http_protocol/theory.md)).

## Глоссарий

| Термин | Значение |
|--------|----------|
| Протокол | набор правил обмена данными |
| Пакет/датаграмма | единица передачи данных |
| Хост | устройство в сети |
| Клиент/сервер | запрашивающий / отвечающий |
| Латентность | задержка передачи |
| Пропускная способность | объём данных в единицу времени |
