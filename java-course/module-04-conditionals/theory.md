# Модуль 04. Условный оператор: `if`, `switch`, тернарный оператор

Программа должна принимать решения в зависимости от данных. Для этого есть **условные операторы**.

## Логический тип и операторы сравнения

Условие — это выражение типа `boolean` (`true`/`false`).

| Оператор | Значение | Пример | Результат |
|----------|----------|--------|-----------|
| `==` | равно | `5 == 5` | `true` |
| `!=` | не равно | `5 != 3` | `true` |
| `>` `<` | больше / меньше | `5 > 3` | `true` |
| `>=` `<=` | больше/меньше или равно | `5 >= 5` | `true` |

### Логические операторы

| Оператор | Значение | Пример |
|----------|----------|--------|
| `&&` | И (оба истинны) | `age >= 18 && age < 65` |
| `\|\|` | ИЛИ (хотя бы одно) | `day == 6 \|\| day == 7` |
| `!` | НЕ (отрицание) | `!isEmpty` |

> `&&` и `||` «ленивые»: если результат уже ясен по левому операнду, правый не вычисляется.

## Оператор `if / else if / else`

```java
int temp = 15;

if (temp < 0) {
    System.out.println("Мороз");
} else if (temp < 15) {
    System.out.println("Прохладно");
} else if (temp < 25) {
    System.out.println("Тепло");
} else {
    System.out.println("Жара");
}
```

Логика выполнения:

```
        ┌─ temp < 0 ?  ──да──▶ "Мороз"
        │      │ нет
        ├─ temp < 15 ? ──да──▶ "Прохладно"
        │      │ нет
        ├─ temp < 25 ? ──да──▶ "Тепло"
        │      │ нет
        └────────────────────▶ "Жара"
```

> Проверки идут сверху вниз; срабатывает **первая** истинная ветка, остальные пропускаются.
> Если в блоке одна инструкция, скобки `{}` необязательны, но **ставить их — хорошая практика**.

## Тернарный оператор `? :`

Сокращённая форма `if/else` для присваивания значения.

```java
int a = 10, b = 20;
int max = (a > b) ? a : b;   // если a>b → a, иначе → b
```

Читается так: `условие ? значение_если_true : значение_если_false`.

```java
String parity = (n % 2 == 0) ? "чётное" : "нечётное";
```

> Используйте тернарник для **коротких** выражений. Сложную логику оставьте обычному `if`.

## Оператор `switch`

Удобен, когда нужно сравнить **одну переменную** с множеством конкретных значений.

```java
int day = 3;
switch (day) {
    case 1:
    case 2:
    case 3:
    case 4:
    case 5:
        System.out.println("Будний день");
        break;
    case 6:
    case 7:
        System.out.println("Выходной");
        break;
    default:
        System.out.println("Нет такого дня");
}
```

> ⚠️ **`break` обязателен!** Без него выполнение «проваливается» в следующий `case` (fall-through).
> `default` — необязательная ветка «во всех остальных случаях».

### Современный `switch` (Java 14+) — стрелочный синтаксис

```java
String type = switch (day) {
    case 1, 2, 3, 4, 5 -> "Будний";
    case 6, 7 -> "Выходной";
    default -> "Ошибка";
};
```

Здесь `break` не нужен, нет проваливания, можно возвращать значение.

---

## `if` vs `switch` — что выбрать

| Ситуация | Лучше |
|----------|-------|
| диапазоны (`x > 10`) | `if` |
| сравнение с конкретными значениями | `switch` |
| сложные логические условия (`&&`, `\|\|`) | `if` |
| много веток по одной переменной | `switch` |

---

## Подводные камни

| Ошибка | Объяснение |
|--------|-----------|
| `if (x = 5)` | `=` это присваивание; для сравнения нужно `==` |
| забыли `break` в `switch` | произойдёт проваливание в следующий case |
| сравнение строк через `==` | для строк используйте `.equals()` |
| `if (a < b < c)` | так нельзя; пишите `a < b && b < c` |
