# Модуль 10. Типы данных. Знакомство с ООП

**ООП (объектно-ориентированное программирование)** — это способ моделировать программу как набор взаимодействующих **объектов**. Это ключевая тема курса.

## Примитивные и ссылочные типы

В Java два вида типов:

| | Примитивные | Ссылочные |
|--|-------------|-----------|
| Примеры | `int`, `double`, `boolean`, `char` | `String`, массивы, любые объекты |
| Хранят | само значение | **ссылку** (адрес) на объект |
| Значение по умолчанию | `0`, `false` и т.п. | `null` |

```
Примитив:                    Ссылочный тип:
┌──────┐                    ┌──────┐      ┌─────────────┐
│ int  │                    │ ref  │ ───▶ │  объект в   │
│  5   │                    │ ●────│      │   heap      │
└──────┘                    └──────┘      └─────────────┘
 значение прямо в            переменная хранит адрес,
 переменной                  объект — в куче (heap)
```

## Что такое класс и объект

- **Класс** — это «чертёж», шаблон. Описывает, какие данные (**поля**) и действия (**методы**) есть у объектов.
- **Объект** — конкретный экземпляр класса, созданный по чертежу.

```
   Класс "Cat" (чертёж)          Объекты (экземпляры)
  ┌────────────────────┐        ┌──────────┐ ┌──────────┐
  │ поле: name         │  new   │ name="Том"│ │ name="Рыжик"│
  │ метод: meow()      │ ─────▶ │          │ │          │
  └────────────────────┘        └──────────┘ └──────────┘
```

### Простейший класс

```java
class Cat {
    String name;       // поле (характеристика)
    int age;

    void meow() {      // метод (поведение)
        System.out.println(name + " говорит: Мяу!");
    }
}
```

---

## Создание объекта — оператор `new`

```java
public class Main {
    public static void main(String[] args) {
        Cat cat = new Cat();   // создали объект
        cat.name = "Том";      // обращаемся к полю через точку
        cat.age = 3;
        cat.meow();            // вызываем метод → "Том говорит: Мяу!"
    }
}
```

```
Cat cat = new Cat();
     │         │
   ссылка    создаём объект в heap
```

## `null` — «ссылка в никуда»

`null` означает, что ссылочная переменная **не указывает ни на какой объект**.

```java
Cat cat = null;
cat.meow();    // ❌ NullPointerException — объекта нет!
```

> 💥 `NullPointerException` (NPE) — одна из самых частых ошибок. Возникает при обращении к полю/методу через `null`.
> Проверяйте: `if (cat != null) cat.meow();`

## Наследование — переиспользование через `extends`

**Наследование** позволяет создать класс на основе существующего: подкласс получает все поля и методы родителя и может добавить свои.

```java
class Animal {                 // родитель (суперкласс)
    String name;
    void eat() {
        System.out.println(name + " ест");
    }
}

class Dog extends Animal {     // потомок наследует name и eat()
    void bark() {
        System.out.println(name + " лает: Гав!");
    }
}

class Cat extends Animal {
    void meow() {
        System.out.println(name + " мяукает");
    }
}
```

```
            Animal
          (name, eat)
           /       \
        Dog         Cat
      (bark)       (meow)
```

```java
Dog dog = new Dog();
dog.name = "Рекс";
dog.eat();    // унаследовано от Animal → "Рекс ест"
dog.bark();   // собственный метод → "Рекс лает: Гав!"
```

## Несколько классов в одном файле

Для учебных задач можно объявить несколько классов в одном `.java`-файле. **`public`-класс должен совпадать с именем файла**, остальные — без `public`:

```java
public class Task05 {          // имя файла Task05.java
    public static void main(String[] args) { ... }
}

class Animal { ... }           // вспомогательные классы — без public
class Dog extends Animal { ... }
```

## Подводные камни

| Ошибка | Объяснение |
|--------|-----------|
| `NullPointerException` | обращение к методу/полю через `null` |
| забыли `new` | переменная остаётся `null` |
| `==` для объектов | сравнивает ссылки, а не содержимое (см. [модуль 11](../module-11-objects-constructors/theory.md)) |
| два `public`-класса в файле | разрешён только один, совпадающий с именем файла |
