# Тема 22. Антипаттерны и рефакторинг

Паттерны (темы 04–21) показывают, **как надо**. Антипаттерны — типовые **плохие** решения, которые сперва кажутся удобными, а потом разрушают систему. Узнавать их и уметь рефакторить — навык не менее важный, чем знание паттернов.

```
Антипаттерны кода/дизайна
  Anemic Domain Model — данные без поведения (мешок геттеров/сеттеров)
  God Object          — класс, который делает всё
  Primitive Obsession — примитивы вместо Value Object
  Feature Envy        — метод «завидует» данным чужого класса
  Long Parameter List — слишком много аргументов
  (+ Lava Flow, Golden Hammer, Spaghetti, Singleton-абьюз)
```

---

## 1. Anemic Domain Model — анемичная модель

Объект домена содержит только данные (поля + геттеры/сеттеры), а вся логика — снаружи, в «сервисах». Объектно-ориентированный код вырождается в процедурный.

```java
// ❌ анемично: Order — мешок данных, логика снаружи
class Order { private List<Item> items; List<Item> getItems(){...} }
class OrderService { long total(Order o){ /* считает, лазая в чужие данные */ } }

// ✅ богатая модель: поведение там, где данные (Information Expert, тема 14)
class Order {
    private final List<Item> items;
    public long total() { /* сам считает */ }   // данные и логика вместе
}
```

**Лечение:** перенести поведение в сущность (Information Expert, тема 14; Aggregate, тема 18).

---

## 2. God Object — божественный объект

Один класс знает и делает слишком много (тысячи строк, десятки несвязанных обязанностей). Нарушает SRP и High Cohesion.

```java
// ❌ God Object
class OrderManager { void calc(){} void render(){} void save(){} void email(){} void log(){} }

// ✅ разнести по связным классам (SRP, тема 02; High Cohesion, тема 14)
class OrderCalculator {} class OrderRenderer {} class OrderRepository {} class Mailer {}
```

**Лечение:** выделить связные обязанности в отдельные классы.

---

## 3. Primitive Obsession — одержимость примитивами

Важные понятия домена представлены примитивами (`long money`, `String email`), а не типами. Теряется валидация, легко перепутать аргументы.

```java
// ❌ что есть что? легко перепутать местами
void transfer(long amount, String from, String to);

// ✅ Value Object (тема 18): тип несёт смысл и валидацию
void transfer(Money amount, AccountId from, AccountId to);
```

**Лечение:** ввести Value Object.

---

## 4. Feature Envy — «зависть к данным»

Метод одного класса больше работает с данными **другого** класса, чем со своими. Признак, что метод не на своём месте.

```java
// ❌ метод Order лезет в кишки Customer
class Order { String customerLabel(Customer c){ return c.getFirstName()+" "+c.getLastName(); } }

// ✅ метод там, где данные
class Customer { String label(){ return firstName + " " + lastName; } }
```

**Лечение:** переместить метод в класс, чьими данными он пользуется (Information Expert).

---

## 5. Long Parameter List — длинный список параметров

Много аргументов → трудно читать и легко перепутать. Часто это спрятанный объект.

```java
// ❌
Reservation book(String guest, String room, int year, int month, int day, int nights, boolean breakfast);

// ✅ параметр-объект / Builder (тема 05)
Reservation book(BookingRequest request);
```

**Лечение:** ввести параметр-объект (или Builder).

## Прочие известные антипаттерны (узнавать)

| Антипаттерн | Суть | Лечение |
|-------------|------|---------|
| **Lava Flow** | «застывший» мёртвый код, который боятся трогать | покрыть тестами и удалить ненужное |
| **Golden Hammer** | «один инструмент на всё» (всё — микросервис, всё — Singleton) | выбирать средство под задачу |
| **Spaghetti Code** | запутанные связи, нет структуры | слои/модули (темы 15, 23) |
| **Singleton-абьюз** | глобальное состояние повсюду | DI (тема 16) вместо жёсткого Singleton |
| **Copy-Paste** | дублирование вместо переиспользования | выделить общий метод/класс (DRY) |

---

## ⚠️ Подводные камни рефакторинга

| Ошибка | Симптом | Лечение |
|--------|---------|---------|
| Рефакторинг без тестов | «починили» — сломали другое | сперва характеризующие тесты, потом рефактор |
| Большой рефакторинг «за раз» | долго, рискованно, не мёржится | маленькими безопасными шагами |
| Борьба с симптомом, не причиной | антипаттерн возвращается | устранять корень (распределение обязанностей) |
| «Чистка» ради чистки | риск без выгоды | рефакторить там, где меняешь код/есть боль |
