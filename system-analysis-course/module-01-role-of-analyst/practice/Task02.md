# Задача 02 — Модуль 01: Роль системного аналитика

## ЗАДАНИЕ
Составьте карту артефактов системного аналитика по фазам жизненного цикла: какой артефакт появляется на какой фазе, кому он нужен и какое решение помогает принять.

## ВХОДНЫЕ ДАННЫЕ
Фазы ЖЦ: Инициация → Выявление требований → Анализ и моделирование → Проектирование решения → Спецификация → Разработка → Тестирование/приёмка → Внедрение/сопровождение.
Список артефактов (вперемешку): глоссарий, модель данных (ER), карта стейкхолдеров, use cases, критерии приёмки, ТЗ/SRS, спецификация API (OpenAPI), план миграции, матрица трассировки, контекстная диаграмма, BPMN-модель процесса, пользовательская документация.

## ТРЕБОВАНИЯ
- Каждый артефакт привяжите минимум к одной фазе (можно к нескольким, если он эволюционирует).
- Для каждого артефакта укажите **потребителя** (кому нужен) и **какое решение он помогает принять**.
- Добавьте минимум 2 артефакта, которых нет в списке, но которые вы считаете нужными.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: `Фаза | Артефакт | Потребитель | Какое решение помогает принять`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Все 12 артефактов распределены по фазам.
- [ ] У каждого артефакта указан потребитель и назначение (не «для галочки»).
- [ ] Глоссарий и контекстная диаграмма — на ранних фазах; план миграции и пользовательская документация — на поздних.
- [ ] Добавлено ≥2 собственных артефакта с обоснованием.
- [ ] Матрица трассировки связана минимум с двумя другими артефактами (требование ↔ тест).

## ПОДСКАЗКА
Артефакт без потребителя — это артефакт ради артефакта. Если не можете назвать, кому и зачем он нужен, — возможно, на этой фазе он не нужен.
