# Задача 07 — Модуль 01: Роль системного аналитика

## ЗАДАНИЕ
Проведите gap-анализ компетенций по тексту вакансии middle+ системного аналитика: что требуется, что из этого вы уже умеете, где пробелы, и составьте план их закрытия.

## ВХОДНЫЕ ДАННЫЕ
Фрагмент вакансии «Системный аналитик (middle+)»:
> Требования: опыт сбора и формализации требований; уверенное знание SQL; проектирование REST API, опыт с OpenAPI/Swagger; UML (use case, sequence, class), BPMN; понимание микросервисной архитектуры и интеграций (очереди, Kafka); опыт описания нефункциональных требований; умение работать с распределёнными системами (идемпотентность, согласованность); навык декомпозиции и приоритизации; коммуникация со стейкхолдерами.

## ТРЕБОВАНИЯ
- Разбейте требования вакансии на отдельные компетенции.
- Оцените свой уровень по каждой: «знаю/умею / частично / пробел» (честно).
- Для каждого пробела укажите, **какой модуль этого курса** его закрывает.
- Составьте приоритизированный план: с чего начать и почему.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: `Компетенция | Мой уровень | Модуль курса | Приоритет (1-3)`. Ниже — план на 3 шага.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Из текста вакансии выделено ≥10 отдельных компетенций.
- [ ] Каждая оценена по шкале (без «всё знаю»).
- [ ] Каждый пробел сопоставлен с конкретным модулем курса (по номеру).
- [ ] Приоритизация обоснована (например: базовое раньше продвинутого, частое раньше редкого).
- [ ] План содержит конкретные первые 3 шага.

## ПОДСКАЗКА
Карта модулей курса — в `../../README.md`. Хороший gap-анализ честен: пробел, который вы признали, можно закрыть; пробел, который вы спрятали, всплывёт на собеседовании.
