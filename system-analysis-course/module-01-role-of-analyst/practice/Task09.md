# Задача 09 — Модуль 01: Роль системного аналитика

## ЗАДАНИЕ
Сравните, как работает системный аналитик в двух контекстах: классическая waterfall/ГОСТ-команда (госзаказ) против продуктовой Scrum-команды. Покажите, что меняется в артефактах, ритме работы и коммуникации.

## ВХОДНЫЕ ДАННЫЕ
Два проекта одинакового домена (учёт обращений граждан):
- **Проект 1:** госзаказ, фиксированный объём, приёмка по ГОСТ, ТЗ согласуется заранее, изменения через формальные допсоглашения.
- **Проект 2:** продуктовая команда внутри компании, Scrum, двухнедельные спринты, требования уточняются по ходу, частые релизы.

## ТРЕБОВАНИЯ
- Сравните по осям: основные артефакты, момент фиксации требований, объём документации, работа с изменениями, ритм взаимодействия с командой, главные риски роли.
- Сформулируйте, какие навыки СА выходят на первый план в каждом контексте.
- Объясните, почему **один и тот же** человек должен менять манеру работы при переходе между контекстами.

## ФОРМАТ РЕЗУЛЬТАТА
Сравнительная таблица по осям (Waterfall/ГОСТ vs Scrum-продукт). Ниже — вывод: 3–4 предложения о том, что переносимо между контекстами, а что нет.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Сравнение идёт по ≥5 осям.
- [ ] Для waterfall отмечены: ранняя фиксация требований, большой объём ТЗ, формальные изменения.
- [ ] Для Scrum отмечены: итеративность, «достаточная» документация, гибкие изменения, тесная связь с PO/командой.
- [ ] Указаны навыки, выходящие на первый план в каждом контексте.
- [ ] Есть вывод о переносимых и непереносимых аспектах.

## ПОДСКАЗКА
Суть роли (выявить → проанализировать → специфицировать → проверить) одинакова. Меняются **форма артефактов, момент фиксации и темп**. На собеседовании ценят умение объяснить это различие.
