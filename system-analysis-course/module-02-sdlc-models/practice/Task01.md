# Задача 01 — Модуль 02: ЖЦ ПО и модели разработки

## ЗАДАНИЕ
Сопоставьте фазы жизненного цикла ПО с артефактами, которые на них создаются, и с тем, кто за них отвечает.

## ВХОДНЫЕ ДАННЫЕ
Фазы: Анализ требований → Проектирование → Разработка → Тестирование → Внедрение → Сопровождение.
Артефакты (вперемешку): архитектурная схема, план миграции, ТЗ/SRS, тест-кейсы, код, модель данных, отчёт о приёмке, инструкция пользователя, журнал change requests, спецификация API, протоколы дефектов, критерии приёмки.

## ТРЕБОВАНИЯ
- Привяжите каждый артефакт к фазе, на которой он **появляется** (первично).
- Для каждого укажите основную ответственную роль.
- Отметьте артефакты, которые **создаются на одной фазе, но используются на другой** (например, критерии приёмки).

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: `Фаза | Артефакт | Ответственная роль | Где ещё используется`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Все 12 артефактов распределены.
- [ ] ТЗ/SRS и критерии приёмки — на фазе анализа; код — на разработке; план миграции и инструкция — на внедрении; журнал change requests — на сопровождении.
- [ ] Указаны «сквозные» артефакты (критерии приёмки: создаются при анализе, используются при тестировании).
- [ ] Роли назначены корректно (тест-кейсы — QA, код — разработчик, модель данных/спецификация API — СА).

## ПОДСКАЗКА
Артефакт «живёт» дольше своей фазы рождения. Связи «создан тут — использован там» — это основа трассировки (модуль 10).
