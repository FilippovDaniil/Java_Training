# Задача 03 — Модуль 02: ЖЦ ПО и модели разработки

## ЗАДАНИЕ
Постройте соответствие уровней V-модели: каждому уровню требований/проектирования сопоставьте уровень тестирования и покажите, какой артефакт связывает левую и правую ветки.

## ВХОДНЫЕ ДАННЫЕ
Уровни левой ветки: бизнес-требования, системные требования, архитектура, детальный дизайн.
Уровни правой ветки: модульное (unit), интеграционное, системное, приёмочное (UAT) тестирование.

## ТРЕБОВАНИЯ
- Сопоставьте каждый уровень левой ветки с зеркальным уровнем тестирования.
- Для каждой пары укажите, **какой артефакт** служит основанием для тестов этого уровня.
- Объясните главную идею V-модели: почему тесты планируются вместе с требованиями.
- Приведите пример: бизнес-требование для MedConnect и соответствующий приёмочный тест.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: `Левая ветка | Правая ветка | Связующий артефакт`. Ниже — объяснение идеи и пример пары «требование → приёмочный тест».

## КРИТЕРИИ ПРИЁМКИ
- [ ] Бизнес-требования ↔ приёмочное; системные ↔ системное; архитектура ↔ интеграционное; детальный дизайн ↔ модульное.
- [ ] Для каждой пары назван связующий артефакт.
- [ ] Объяснено, что тесты проектируются на этапе требований (раннее обнаружение дефектов).
- [ ] Пример пары конкретен и проверяем (требование → как его проверит приёмочный тест).

## ПОДСКАЗКА
Главная мысль V-модели: каждое требование должно иметь способ проверки уже в момент написания. Если для требования нельзя придумать тест — это сигнал, что требование непроверяемо.
