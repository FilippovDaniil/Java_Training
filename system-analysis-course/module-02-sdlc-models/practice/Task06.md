# Задача 06 — Модуль 02: ЖЦ ПО и модели разработки

## ЗАДАНИЕ
Опишите путь (flow) одной задачи системного аналитика от появления идеи до релиза в продуктовой Scrum-команде FoodExpress. Покажите все этапы, состояния и участников.

## ВХОДНЫЕ ДАННЫЕ
Идея от бизнеса: «Добавить возможность оставить чаевые курьеру при оформлении заказа». Команда: PO, СА, архитектор, разработчики, QA. Двухнедельные спринты.

## ТРЕБОВАНИЯ
- Опишите этапы от идеи до релиза: какие состояния проходит задача и что на каждом делает СА.
- Укажите, где задача может «вернуться назад» и почему.
- Покажите, на каком этапе появляются: user story, критерии приёмки, уточнения, проверка соответствия.
- Отметьте точки взаимодействия с другими ролями.

## ФОРМАТ РЕЗУЛЬТАТА
Линейная схема состояний (Идея → … → Релиз) с описанием действий СА на каждом. Можно таблицей: `Состояние | Действие СА | Артефакт | Участники`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Поток покрывает: идея → проработка (refinement) → планирование → разработка → тестирование → демо/приёмка → релиз.
- [ ] Показано, где СА создаёт user story и критерии приёмки.
- [ ] Указаны точки возврата (например, история не прошла DoR → назад в проработку).
- [ ] Отмечены взаимодействия с PO (приоритет), QA (критерии), архитектором (если нужно техрешение).
- [ ] Поток реалистичен для двухнедельного спринта.

## ПОДСКАЗКА
Это «жизнь» задачи внутри Scrum-процесса из теории. Свяжите этапы с событиями Scrum (refinement, planning, review) из задачи 04.
