# Задача 02 — Модуль 03: Продукт и контекст

## ЗАДАНИЕ
Постройте карту стейкхолдеров проекта **FoodExpress** по матрице «власть/интерес» и определите стратегию работы с каждым квадрантом.

## ВХОДНЫЕ ДАННЫЕ
Стейкхолдеры FoodExpress (агрегатор доставки еды):
- генеральный директор / спонсор проекта;
- клиенты (заказывают еду);
- рестораны-партнёры;
- курьеры;
- служба поддержки;
- отдел маркетинга;
- внешний платёжный провайдер;
- Роспотребнадзор / регулятор (требования к работе с заказами и данными);
- команда разработки;
- инвесторы.

## ТРЕБОВАНИЯ
- Разместите каждого стейкхолдера в одном из 4 квадрантов (власть × интерес).
- Для каждого квадранта укажите стратегию работы.
- Для 3 стейкхолдеров поясните, почему они именно в этом квадранте.
- Отметьте стейкхолдеров, которых легко забыть, но чьи требования критичны.

## ФОРМАТ РЕЗУЛЬТАТА
Матрица 2×2 с распределением стейкхолдеров по квадрантам + стратегия на квадрант + пояснения к 3 размещениям.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Все 10 стейкхолдеров размещены.
- [ ] Спонсор/директор — высокая власть + высокий интерес (вовлекать плотно).
- [ ] Регулятор — высокая власть, обычно низкий интерес (держать удовлетворённым).
- [ ] Клиенты/курьеры/поддержка — высокий интерес, ниже власть (информировать, собирать обратную связь).
- [ ] Для каждого квадранта указана стратегия.
- [ ] Отмечены «легко забываемые, но критичные» (регулятор, поддержка, платёжный провайдер).

## ПОДСКАЗКА
Власть — способность влиять на проект/решения; интерес — насколько его затрагивает результат. Регулятор редко интересуется деталями, но его власть «заблокировать» максимальна — отсюда квадрант «удовлетворять».
