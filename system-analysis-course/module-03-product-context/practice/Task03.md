# Задача 03 — Модуль 03: Продукт и контекст

## ЗАДАНИЕ
Проведите детальный анализ ключевых стейкхолдеров FoodExpress: их интересы, влияние, отношение, стратегию вовлечения и ожидаемые от них требования.

## ВХОДНЫЕ ДАННЫЕ
Возьмите 5 стейкхолдеров из задачи 02 (например: рестораны-партнёры, курьеры, служба поддержки, платёжный провайдер, регулятор). Контекст — запуск нового агрегатора доставки.

## ТРЕБОВАНИЯ
- Для каждого стейкхолдера заполните: роль, интересы (что хочет), влияние (выс/сред/низ), отношение (сторонник/нейтрален/против), стратегия вовлечения, какие требования от него ожидаются.
- Выявите потенциальный конфликт интересов между двумя стейкхолдерами.
- Укажите, какие требования вы рискуете пропустить, если не поговорите с каждым.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица анализа (по одной строке на стейкхолдера, столбцы по требованиям выше) + описание одного конфликта интересов + список рисков пропуска требований.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Проанализированы 5 стейкхолдеров по всем полям.
- [ ] Интересы конкретны (не «хочет, чтобы было хорошо»).
- [ ] Для каждого указаны ожидаемые требования (рестораны — управление меню/заказами; курьеры — маршруты/выплаты; поддержка — инструменты разбора обращений; провайдер — протокол интеграции; регулятор — работа с ПДн/чеками).
- [ ] Описан конкретный конфликт интересов (например, комиссия для ресторанов vs цена для клиентов).
- [ ] Перечислены риски пропуска требований по каждому.

## ПОДСКАЗКА
Анализ стейкхолдеров — это превентивный поиск требований и конфликтов. Каждый стейкхолдер несёт свой набор требований; пропустишь стейкхолдера — пропустишь требования.
