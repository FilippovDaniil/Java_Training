# Задача 04 — Модуль 03: Продукт и контекст

## ЗАДАНИЕ
Переведите размытые запросы бизнеса в измеримые бизнес-цели по критериям SMART.

## ВХОДНЫЕ ДАННЫЕ
Запросы от бизнеса (как есть):
1. «Хотим, чтобы клиенты не уходили с оформления заказа».
2. «Нужно повысить лояльность клиентов».
3. «Сделайте поддержку быстрее».
4. «Хотим больше заказов от ресторанов-партнёров».
5. «Приложение должно работать стабильнее».

## ТРЕБОВАНИЯ
- Для каждого запроса сформулируйте SMART-цель (все 5 критериев).
- Где не хватает данных для measurable/time-bound — явно укажите, какой вопрос задать бизнесу.
- Для 2 целей предложите метрику, по которой будете отслеживать прогресс.
- Отметьте, какой запрос на самом деле относится к нефункциональным требованиям (связь с модулем 09).

## ФОРМАТ РЕЗУЛЬТАТА
Для каждого запроса: исходная формулировка → SMART-цель → (при необходимости) уточняющий вопрос. Ниже — метрики для 2 целей и пометка про NFR.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Все 5 целей переформулированы по SMART (конкретность, измеримость, достижимость, значимость, срок).
- [ ] Там, где данных нет, сформулированы уточняющие вопросы, а не выдуманы цифры.
- [ ] Для 2 целей указаны метрики отслеживания.
- [ ] Запрос 5 («стабильнее») опознан как нефункциональное требование (надёжность/доступность).
- [ ] Цели реалистичны (achievable), а не «конверсия 100%».

## ПОДСКАЗКА
SMART не значит «выдумать цифру». Если бизнес не дал целевое значение — это повод задать вопрос («на сколько сократить? к какому сроку?»), а не угадывать. «Стабильнее» — это про NFR, его меряют через доступность/частоту сбоев.
