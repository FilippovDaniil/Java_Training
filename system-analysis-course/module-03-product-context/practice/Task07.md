# Задача 07 — Модуль 03: Продукт и контекст

## ЗАДАНИЕ
Сформулируйте продуктовые гипотезы в строгом формате «верим → приведёт к → поймём по метрике» и спроектируйте способ их проверки.

## ВХОДНЫЕ ДАННЫЕ
Идеи команды FoodExpress по улучшению:
1. Добавить оплату в один клик.
2. Показывать прогноз времени доставки на главном экране.
3. Включить рейтинг курьеров.
4. Сделать «избранные рестораны».

## ТРЕБОВАНИЯ
- Для каждой идеи сформулируйте гипотезу в формате: МЫ ВЕРИМ, ЧТО … ПРИВЕДЁТ К … ПОЙМЁМ ПО [метрика + пороговое значение + срок].
- Для каждой укажите способ проверки (A/B-тест, аналитика, опрос) и минимальный объём данных.
- Отметьте, какая метрика из AARRR затрагивается.
- Выделите гипотезу с самым высоким риском (если не подтвердится — зря потраченные усилия) и предложите проверить её дешевле.

## ФОРМАТ РЕЗУЛЬТАТА
Для каждой идеи: гипотеза (3 строки) + способ проверки + затрагиваемая метрика AARRR. Ниже — самая рискованная гипотеза и дешёвый способ её предпроверки.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Все 4 гипотезы в строгом формате с пороговым значением метрики и сроком.
- [ ] Для каждой указан конкретный способ проверки.
- [ ] Каждая привязана к этапу AARRR.
- [ ] Выделена самая рискованная гипотеза с обоснованием.
- [ ] Предложен дешёвый способ предпроверки (не сразу полная реализация).

## ПОДСКАЗКА
Гипотеза без порогового значения метрики — это не гипотеза, а пожелание. Дорогую гипотезу часто можно предпроверить дёшево: опрос, фейк-дор, прототип — до полной разработки.
