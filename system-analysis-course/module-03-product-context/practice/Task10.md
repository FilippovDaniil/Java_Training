# Задача 10 (мини-проект) — Модуль 03: Продукт и контекст

## ЗАДАНИЕ
Соберите полный продуктовый контекст для **FoodExpress**: бизнес-цели, карта стейкхолдеров, метрики, границы MVP и проверяемые гипотезы. Это синтез всей теории модуля и завершение Части 1.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress** — новый агрегатор доставки еды (клиенты, рестораны, курьеры). Вы — СА на старте. Бизнес хочет за полгода проверить, что модель «агрегатор берёт комиссию с заказов» жизнеспособна в одном городе, и затем масштабироваться.

## ТРЕБОВАНИЯ
Подготовьте документ «Продуктовый контекст FoodExpress», включающий:
1. **Бизнес-цели** — 2–3 цели по SMART (с измеримыми значениями или вопросами, если данных нет).
2. **Карта стейкхолдеров** — матрица власть/интерес + стратегия по ключевым.
3. **Метрики** — North Star + по 1 метрике на этап AARRR + дерево метрик (минимум 2 уровня).
4. **Границы MVP** — что входит/не входит в первую версию для проверки модели в одном городе, с обоснованием.
5. **Гипотезы** — 3 ключевые гипотезы в формате «верим → приведёт к → поймём по метрике».

## ФОРМАТ РЕЗУЛЬТАТА
Структурированный документ из 5 разделов. Каждый раздел опирается на соответствующие задачи модуля (02, 04, 05, 06–08). Документ должен быть внутренне согласован: метрики поддерживают цели, MVP проверяет гипотезы.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Бизнес-цели по SMART, связаны с проверкой жизнеспособности модели.
- [ ] Карта стейкхолдеров включает внешних (платёжный провайдер, регулятор) и стратегию по квадрантам.
- [ ] North Star отражает ценность (не метрика тщеславия), есть дерево минимум на 2 уровня.
- [ ] MVP ограничен одним городом и ядром для проверки комиссионной модели; исключения обоснованы.
- [ ] 3 гипотезы в строгом формате с пороговыми значениями метрик.
- [ ] Документ внутренне непротиворечив: цели ↔ метрики ↔ MVP ↔ гипотезы связаны.

## ПОДСКАЗКА
Это «нулевой» документ проекта, на который дальше опираются все требования. Проверьте связность: каждая гипотеза должна проверяться тем, что есть в MVP; каждая метрика — поддерживать какую-то цель. Используйте наработки задач 02–08 этого модуля.
