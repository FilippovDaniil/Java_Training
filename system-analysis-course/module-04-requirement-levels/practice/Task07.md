# Задача 07 — Модуль 04: Уровни и типы требований

## ЗАДАНИЕ
Выявите неявные (подразумеваемые) требования в кейсе MedConnect: то, что заказчик не проговорил, но что обязательно должно быть.

## ВХОДНЫЕ ДАННЫЕ
Заказчик MedConnect описал требование:
> «Врач проводит видеоконсультацию с пациентом и по итогу выписывает электронный рецепт, который пациент видит в приложении».

Заказчик считает, что этого описания достаточно. Но за ним скрыто много неявного.

## ТРЕБОВАНИЯ
- Выявите минимум 8 неявных требований, скрытых за этим заявлением.
- Сгруппируйте их по категориям: безопасность/приватность, обработка ошибок, регуляторика, доступ/права, надёжность, удобство.
- Для каждого поясните, почему оно подразумевается и чем грозит его отсутствие.
- Отметьте, какие из них — функциональные, а какие — нефункциональные/ограничения.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: `Неявное требование | Категория | Тип (FR/NFR/ограничение) | Чем грозит отсутствие`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Выявлено ≥8 неявных требований.
- [ ] Покрыты категории: приватность медданных (152-ФЗ, согласие), права доступа (только лечащий врач), обработка ошибок (обрыв видеосвязи), аутентификация сторон, аудит/логирование действий с рецептом, целостность (рецепт нельзя подделать/потерять).
- [ ] Для каждого пояснено, почему подразумевается и риск отсутствия.
- [ ] Указан тип каждого (FR/NFR/ограничение).
- [ ] Есть хотя бы одно требование про регуляторику и одно про обработку ошибок.

## ПОДСКАЗКА
Спрашивайте по каждому шагу: «что если связь оборвётся?», «кто имеет право это видеть?», «что требует закон?», «как подтвердить, что это именно этот врач/пациент?». Неявные требования — главный источник дорогих дефектов в проде.
