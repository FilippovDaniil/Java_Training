# Задача 08 — Модуль 04: Уровни и типы требований

## ЗАДАНИЕ
Обнаружьте и разрешите противоречие между требованиями двух стейкхолдеров.

## ВХОДНЫЕ ДАННЫЕ
В FoodExpress собраны требования от разных сторон:
- **Маркетинг:** «Регистрация должна быть максимально короткой — только номер телефона, чтобы не терять пользователей на входе».
- **Финансовый отдел / безопасность:** «Перед первой оплатой клиент должен пройти подтверждение личности (email + подтверждённая карта), чтобы снизить мошенничество и chargeback».
- **Юрист:** «До любой обработки данных нужно получить согласие на обработку ПДн».
- **Продукт:** «Пользователь должен иметь возможность сделать первый заказ в течение 2 минут после установки приложения».

## ТРЕБОВАНИЯ
- Найдите противоречия между этими требованиями.
- Для каждого противоречия выявите реальный интерес за позицией.
- Предложите решение, удовлетворяющее интересы (а не выбор «кто победил»).
- Зафиксируйте итоговые согласованные требования.
- Укажите, кто Accountable за финальное решение.

## ФОРМАТ РЕЗУЛЬТАТА
Блоки: `Противоречие → Интересы сторон → Предлагаемое решение`. Ниже — список итоговых согласованных требований и кто Accountable.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Обнаружено противоречие «короткая регистрация» vs «подтверждение личности до оплаты».
- [ ] Выявлены интересы (маркетинг — конверсия входа; безопасность — снижение мошенничества; не позиции).
- [ ] Решение разводит во времени: лёгкий вход → подтверждение только перед первой оплатой (прогрессивная регистрация).
- [ ] Согласие на ПДн зафиксировано как обязательное (нельзя выкинуть).
- [ ] Итоговые требования непротиворечивы и сохраняют цель «первый заказ за 2 минуты» для бесплатных действий.
- [ ] Указан Accountable (обычно PO).

## ПОДСКАЗКА
Противоречие часто мнимое: требования конфликтуют только если применять их в один момент. Разведение по времени (прогрессивная регистрация: подтверждение личности нужно к моменту оплаты, а не к входу) снимает конфликт, сохраняя оба интереса.
