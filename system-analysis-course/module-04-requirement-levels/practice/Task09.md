# Задача 09 — Модуль 04: Уровни и типы требований

## ЗАДАНИЕ
Извлеките требования из фрагмента регламента/НПА: переведите юридический/процессный текст в структурированные требования к системе.

## ВХОДНЫЕ ДАННЫЕ
Фрагмент внутреннего регламента MedConnect (на основе требований к работе с медданными):
> «Доступ к медицинским данным пациента имеют только лечащий врач и сам пациент. Любое обращение к медицинской карте фиксируется в журнале с указанием, кто, когда и к каким данным обращался. Согласие пациента на обработку персональных данных оформляется до первой записи на приём и может быть отозвано. Электронный рецепт хранится не менее 5 лет. При отзыве согласия данные пациента обезличиваются в течение 30 дней, кроме данных, которые обязаны храниться по закону».

## ТРЕБОВАНИЯ
- Извлеките отдельные атомарные требования из текста.
- Для каждого определите тип (FR / NFR / бизнес-правило / ограничение).
- Сформулируйте проверяемый критерий приёмки для каждого.
- Отметьте требования, которые порождают конфликт или особую сложность (например, обезличивание vs обязательное хранение).
- Укажите, какие уточняющие вопросы задать юристу.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: `Требование (атомарное) | Тип | Критерий приёмки`. Ниже — отмеченные сложности и список вопросов юристу.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Из текста извлечено ≥6 атомарных требований.
- [ ] Корректно определены типы (доступ только врач/пациент — бизнес-правило/безопасность; журнал обращений — FR + аудит; согласие до приёма — ограничение/FR; хранение 5 лет — ограничение; обезличивание за 30 дней — FR с условием).
- [ ] Для каждого — проверяемый критерий приёмки.
- [ ] Отмечен конфликт «обезличивание при отзыве» vs «обязательное хранение по закону».
- [ ] Сформулированы уточняющие вопросы юристу (какие данные обязаны храниться; что считать обезличиванием).

## ПОДСКАЗКА
Регламент — богатый источник требований, но текст «слипшийся». Разбивайте по предложениям и условиям. Особое внимание — местам с «кроме», «не менее», «в течение»: это пороги и исключения, порождающие отдельные требования и часто конфликты.
