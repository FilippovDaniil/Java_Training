# Задача 01 — Модуль 05: Техники сбора требований

## ЗАДАНИЕ
Для каждой ситуации подберите подходящую технику (или комбинацию) выявления требований и обоснуйте выбор.

## ВХОДНЫЕ ДАННЫЕ
Ситуации:
1. Нужно понять, как операторы поддержки FoodExpress реально разбирают жалобы (они говорят одно, а делают, возможно, другое).
2. Маркетинг, финансы и безопасность не могут договориться о процессе регистрации.
3. Нужно собрать ожидания у 5000 текущих пользователей приложения.
4. Есть подробный регламент работы с возвратами — надо извлечь из него требования.
5. Заказчик не может объяснить, как должен выглядеть экран заказа, — «покажите, тогда скажу».
6. Заменяем старую систему учёта ресторанов на новую.
7. Нужно глубоко разобраться в работе шеф-повара ресторана-партнёра.

## ТРЕБОВАНИЯ
- Для каждой ситуации укажите технику (можно комбинацию) и обоснование.
- Отметьте, какой техники было бы недостаточно в одиночку и чем дополнить.
- Для ситуации 1 объясните, почему интервью может ввести в заблуждение.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Все 7 ситуаций закрыты.
- [ ] Ситуация 1 → наблюдение (говорят ≠ делают); 2 → воркшоп; 3 → анкета/опрос; 4 → анализ документов; 5 → прототип; 6 → анализ существующей системы + интервью; 7 → интервью/наблюдение.
- [ ] Для каждой — обоснование выбора.
- [ ] Отмечено, где одной техники мало (например, анкета 3 → плюс интервью для глубины).
- [ ] Объяснён эффект «говорят одно, делают другое» для ситуации 1.

## ПОДСКАЗКА
Техника выбирается под цель и контекст: глубина (интервью/наблюдение), охват (анкета), согласование (воркшоп), объективная база (документы), снятие неоднозначности (прототип). Если люди могут описывать процесс неточно — наблюдение надёжнее слов.
