# Задача 02 — Модуль 05: Техники сбора требований

## ЗАДАНИЕ
Составьте гайд интервью с заказчиком FoodExpress по новой фиче. Гайд — это структурированный план вопросов, а не «поболтаем».

## ВХОДНЫЕ ДАННЫЕ
Фича: «Подписка FoodExpress Plus» — клиент платит фиксированную сумму в месяц и получает бесплатную доставку. Заказчик — директор по продукту. У вас 45 минут.

## ТРЕБОВАНИЯ
- Сформулируйте цель интервью (1–2 предложения).
- Постройте гайд по структуре: открытие → основная часть (по воронке) → закрытие.
- Минимум 12 вопросов, большинство — открытые; включите зондирующие вопросы про исключения и граничные случаи.
- Отметьте, какие вопросы выявляют бизнес-цель, какие — пользовательские задачи, какие — ограничения.
- Заложите проверку на «решение вместо потребности».

## ФОРМАТ РЕЗУЛЬТАТА
Гайд: цель + блоки (открытие/основная/закрытие) с вопросами. Пометьте у каждого вопроса, что он выявляет.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Указана цель интервью.
- [ ] Структура: открытие, основная часть (воронка от широкого к узкому), закрытие.
- [ ] ≥12 вопросов, преобладают открытые; нет наводящих.
- [ ] Есть зондирующие вопросы про исключения («что если клиент оформил подписку и сразу отменил?»).
- [ ] Вопросы покрывают бизнес-цель, пользовательские задачи и ограничения.
- [ ] Есть вопрос, вытаскивающий потребность, если заказчик начнёт диктовать решение.

## ПОДСКАЗКА
Воронка: сначала «какую проблему решает подписка и для кого», потом детали («какая выгода, какие ограничения, что с возвратом денег»). Закрытие — резюме и следующие шаги. Хороший гайд готовится заранее, а не импровизируется.
