# Задача 03 — Модуль 05: Техники сбора требований

## ЗАДАНИЕ
Переделайте плохие вопросы интервью в хорошие: закрытые и наводящие — в открытые/нейтральные, где это уместно.

## ВХОДНЫЕ ДАННЫЕ
Вопросы из черновика интервью:
1. «Вам ведь нужна возможность оплаты по подписке, верно?»
2. «Вы пользуетесь отчётами?»
3. «Вам не кажется, что текущий процесс возврата слишком медленный?»
4. «Согласны, что push-уведомления лучше, чем email?»
5. «У вас есть проблемы с системой?»
6. «Сколько у вас сотрудников?»
7. «Вы бы хотели, чтобы всё работало быстрее?»

## ТРЕБОВАНИЯ
- Для каждого определите проблему (наводящий / слишком закрытый / бесполезно общий).
- Перепишите в открытый или нейтральный вопрос, дающий содержательный ответ.
- Отметьте вопросы, которые на самом деле уместно оставить закрытыми (уточнение факта).
- Объясните, чем опасны наводящие вопросы для качества требований.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: `№ | Исходный | Проблема | Переписанный`. Ниже — вывод про опасность наводящих вопросов.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Все 7 разобраны.
- [ ] Наводящие (1, 3, 4, 7) опознаны и переписаны нейтрально.
- [ ] Бесполезно общий (5) переформулирован конкретно («опишите случаи, когда система мешала работе»).
- [ ] Вопрос 6 (факт — число сотрудников) обоснованно оставлен закрытым.
- [ ] Объяснено: наводящие вопросы дают эхо гипотез интервьюера, а не реальную потребность.

## ПОДСКАЗКА
Наводящий вопрос содержит «правильный» ответ внутри себя («вам же удобнее…»). Открытый начинается с «как / расскажите / опишите / что происходит, когда». Закрытые вопросы хороши для проверки фактов, а не для исследования потребностей.
