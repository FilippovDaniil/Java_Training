# Задача 04 — Модуль 05: Техники сбора требований

## ЗАДАНИЕ
Спланируйте воркшоп для согласования спорного бизнес-процесса между несколькими отделами.

## ВХОДНЫЕ ДАННЫЕ
В FoodExpress отделы спорят о процессе **отмены заказа и возврата денег**:
- поддержка хочет отменять заказ в один клик;
- финансы требуют проверки и подтверждения перед возвратом;
- рестораны не хотят терять компенсацию за уже начатую готовку;
- продукт хочет минимум трения для клиента.
Нужно за один воркшоп (2 часа) выработать согласованный процесс.

## ТРЕБОВАНИЯ
- Определите цель воркшопа и ожидаемый результат (артефакт на выходе).
- Перечислите участников и почему именно они.
- Составьте повестку с таймингом (блоки по времени).
- Предложите технику фасилитации, чтобы один участник не доминировал.
- Опишите, как фиксируете решения и что делаете с неразрешёнными вопросами.

## ФОРМАТ РЕЗУЛЬТАТА
План воркшопа: цель + результат + участники + повестка с таймингом + приёмы фасилитации + способ фиксации.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Чётко указан результат (например, согласованная схема процесса отмены/возврата).
- [ ] Участники покрывают все конфликтующие стороны + фасилитатор (СА).
- [ ] Повестка с таймингом укладывается в 2 часа.
- [ ] Предложен приём против доминирования (round-robin, тайм-боксы, голосование).
- [ ] Описана фиксация решений и парковка нерешённых вопросов (parking lot).
- [ ] Учтены интересы всех 4 сторон в структуре обсуждения.

## ПОДСКАЗКА
Воркшоп ≠ совещание. У него есть цель, артефакт-результат и фасилитация. Чтобы не доминировал один — структурируйте слово (каждому по очереди), визуализируйте процесс на доске, спорные пункты выносите в «парковку», а не застревайте.
