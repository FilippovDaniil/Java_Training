# Задача 05 — Модуль 05: Техники сбора требований

## ЗАДАНИЕ
Из протокола интервью извлеките структурированные требования и список вопросов-уточнений (что осталось неясным).

## ВХОДНЫЕ ДАННЫЕ
Фрагмент протокола интервью с главврачом для MedConnect:
> «Пациент записывается к врачу через приложение, выбирает удобный слот. Врач должен подтвердить запись — иногда он занят и переносит. Перед консультацией пациент должен оплатить, иначе она не состоится. Бывает, пациент опаздывает — ждём минут 10, потом отменяем, деньги возвращаем... ну, кажется, возвращаем. Рецепт врач выписывает не всегда, только если нужно. Да, и консультации бывают повторные — там, по идее, дешевле, но это не точно. Экстренные случаи мы вообще через приложение не ведём».

## ТРЕБОВАНИЯ
- Извлеките атомарные требования (FR, бизнес-правила, ограничения).
- Отдельно выпишите вопросы-уточнения для всего, что в протоколе сказано неуверенно («кажется», «по идее», «не точно»).
- Отметьте границу системы (что вне scope).
- Сформулируйте выявленные альтернативные/исключительные сценарии (перенос, опоздание, отмена).

## ФОРМАТ РЕЗУЛЬТАТА
Список требований (с типом) + список вопросов-уточнений + пометка границы системы + список исключительных сценариев.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Извлечено ≥6 требований с указанием типа.
- [ ] Все неуверенные места превращены в вопросы-уточнения (возврат при опоздании; цена повторной консультации).
- [ ] Экстренные случаи зафиксированы как out of scope.
- [ ] Выявлены исключения: перенос врачом, опоздание пациента (ожидание 10 мин → отмена), неоплата (консультация не состоится).
- [ ] Требования атомарны и проверяемы.

## ПОДСКАЗКА
Слова «кажется», «по идее», «не точно» — это не требования, а сигналы неопределённости: каждое превращается в вопрос для следующей встречи. «Не ведём через приложение» — явная граница scope. Каждое «иногда/бывает» — это альтернативный сценарий.
