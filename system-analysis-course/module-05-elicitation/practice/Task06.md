# Задача 06 — Модуль 05: Техники сбора требований

## ЗАДАНИЕ
Составьте чек-лист анализа документа: как из регламента/нормативного текста систематически извлекать требования, ничего не упуская.

## ВХОДНЫЕ ДАННЫЕ
СА часто получает документы: регламенты, НПА, договоры, ТЗ предыдущих версий, описания текущих процессов. Нужен повторяемый метод, чтобы анализ не зависел от настроения и не пропускал важное.

## ТРЕБОВАНИЯ
- Составьте чек-лист из 8–12 пунктов: на что смотреть в документе при извлечении требований.
- Включите пункты про: условия и пороги («не менее», «в течение»), исключения («кроме», «за исключением»), роли и права, обязательность (must/should), сроки, ответственность, расхождения с практикой.
- Для 3 пунктов приведите пример формулировки в документе и извлечённое из неё требование.
- Добавьте пункт про актуальность документа.

## ФОРМАТ РЕЗУЛЬТАТА
Чек-лист (Markdown `[ ]`), сгруппированный по смыслу. Ниже — 3 примера «фрагмент → требование».

## КРИТЕРИИ ПРИЁМКИ
- [ ] 8–12 пунктов, каждый — проверяемое действие анализа.
- [ ] Есть пункты про пороги/условия, исключения, роли/права, обязательность, сроки.
- [ ] Есть пункт про проверку актуальности документа и расхождение с реальной практикой.
- [ ] Приведены 3 примера «фрагмент документа → извлечённое требование».
- [ ] Чек-лист применим к регламенту из задачи 09 модуля 04 (проверьте мысленно).

## ПОДСКАЗКА
Документ скрывает требования в служебных словах: «не менее» = порог, «кроме» = исключение, «обязан» = must, «в течение» = срок. Систематический проход по этим маркерам не даёт пропустить требование. И всегда проверяйте: документ ещё актуален?
