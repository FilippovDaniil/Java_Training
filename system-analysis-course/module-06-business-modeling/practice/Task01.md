# Задача 01 — Модуль 06: Бизнес-моделирование

## ЗАДАНИЕ
Заполните Business Model Canvas для FoodExpress по описанию бизнеса.

## ВХОДНЫЕ ДАННЫЕ
FoodExpress — агрегатор доставки еды. Клиенты заказывают еду из ресторанов через приложение, курьеры доставляют. Компания зарабатывает на комиссии с ресторанов и плате за доставку с клиентов. Нужны: платформа, сеть курьеров, договорённости с ресторанами, интеграция с платёжным провайдером и картами. Основные расходы: разработка, маркетинг, выплаты курьерам, поддержка.

## ТРЕБОВАНИЯ
- Заполните все 9 блоков BMC.
- В каждом блоке — конкретные пункты (не общие слова).
- Для блоков «сегменты» и «потоки доходов» учтите многосторонность платформы (клиенты, рестораны, курьеры).
- Отметьте 2–3 связи между блоками (например, партнёр → ключевая активность).

## ФОРМАТ РЕЗУЛЬТАТА
BMC из 9 блоков (можно таблицей/списком по блокам). Ниже — 2–3 отмеченные связи между блоками.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Все 9 блоков заполнены конкретикой.
- [ ] Сегменты клиентов учитывают все стороны платформы (едоки, рестораны, курьеры).
- [ ] Потоки доходов: комиссия с ресторанов + плата за доставку.
- [ ] Ключевые партнёры включают рестораны, платёжный провайдер, картографию.
- [ ] Отмечены связи между блоками.
- [ ] Нет пустых блоков и общих формулировок «делать хорошо».

## ПОДСКАЗКА
FoodExpress — двусторонняя (вернее, трёхсторонняя) платформа: рестораны одновременно и сегмент клиентов, и партнёры. Это нормально для маркетплейсов. Каждый блок должен быть конкретным, чтобы из него можно было вывести требования.
