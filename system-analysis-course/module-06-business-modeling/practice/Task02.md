# Задача 02 — Модуль 06: Бизнес-моделирование

## ЗАДАНИЕ
Постройте контекстную диаграмму системы FoodExpress: система-«чёрный ящик», внешние акторы и системы, потоки данных между ними.

## ВХОДНЫЕ ДАННЫЕ
Окружение FoodExpress: клиенты (заказывают, оплачивают, отслеживают), курьеры (получают задания, шлют геопозицию), рестораны (получают заказы, обновляют меню), платёжный провайдер (списания/возвраты), SMS-шлюз (уведомления), картографический сервис (маршруты), система аналитики (получает события).

## ТРЕБОВАНИЯ
- Изобразите систему одним блоком (без внутренностей).
- Разместите всех внешних акторов и внешние системы.
- Для каждой связи укажите поток (что передаётся) и направление.
- Разделите акторов (люди/роли) и внешние системы.
- Отметьте, какие потоки входящие, какие исходящие.

## ФОРМАТ РЕЗУЛЬТАТА
Контекстная диаграмма (псевдографикой/описанием в нотации «актор —поток→ система»). Ниже — таблица потоков: `Источник | Поток | Приёмник | Направление`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Система — один блок без декомпозиции.
- [ ] Присутствуют все акторы (клиент, курьер, ресторан) и внешние системы (провайдер, SMS, карты, аналитика).
- [ ] Для каждой связи указан поток и направление.
- [ ] Акторы и внешние системы визуально/логически различены.
- [ ] Нет «внутренностей» системы на контекстном уровне (это была бы DFD уровня 1).
- [ ] Потоки конкретны (не просто «данные»).

## ПОДСКАЗКА
Контекстная диаграмма = граница системы. Один блок в центре, всё остальное снаружи, стрелки — обмен данными. Если начали рисовать «модуль заказов», «модуль оплаты» внутри — это уже не контекст, а декомпозиция (DFD/архитектура).
