# Задача 04 — Модуль 06: Бизнес-моделирование

## ЗАДАНИЕ
Постройте 2 персоны для класса пользователей «Клиент FoodExpress» — разные по контексту и потребностям.

## ВХОДНЫЕ ДАННЫЕ
Класс «Клиент» неоднороден: есть занятые офисные работники, семьи с детьми, студенты, пожилые люди. Возьмите 2 разных подтипа и постройте по ним персоны (можно опираться на правдоподобные допущения, отметив их).

## ТРЕБОВАНИЯ
- Для каждой персоны заполните: имя/портрет, цели, боли, поведение/привычки, устройство, типичный сценарий использования.
- Сделайте персоны действительно разными (разные цели и боли).
- Для каждой выведите 1–2 требования к продукту, которые следуют из её потребностей.
- Отметьте, какие допущения стоило бы проверить интервью/данными.

## ФОРМАТ РЕЗУЛЬТАТА
Две карточки персон (по структуре выше) + для каждой 1–2 вытекающих требования + список допущений к проверке.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Две персоны заполнены по всем полям.
- [ ] Персоны различны по целям и болям (не клоны).
- [ ] Сценарий использования конкретен и правдоподобен.
- [ ] Из каждой персоны выведены требования к продукту.
- [ ] Отмечены допущения, требующие проверки (персона ≠ фантазия).

## ПОДСКАЗКА
Персона — «лицо» класса пользователей для эмпатии. Хорошие персоны разные: занятому важна скорость и повтор, семье — большие заказы и оплата по частям, пожилому — простота интерфейса. Из боли персоны прямо следует требование. Помечайте, что выдумано, а что подтверждено.
