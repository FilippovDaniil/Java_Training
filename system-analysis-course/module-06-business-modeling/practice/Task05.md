# Задача 05 — Модуль 06: Бизнес-моделирование

## ЗАДАНИЕ
Свяжите сегменты клиентов из BMC с классами пользователей и персонами. Покажите, что эти модели согласованы между собой.

## ВХОДНЫЕ ДАННЫЕ
Используйте результаты задач 01 (BMC), 03 (классы пользователей), 04 (персоны) для FoodExpress. Если каких-то наработок нет — задайте их кратко.

## ТРЕБОВАНИЯ
- Постройте соответствие: сегмент BMC → класс(ы) пользователей → персона(ы).
- Найдите рассогласования: сегмент без класса, класс без сегмента, персона без класса.
- Объясните, почему один сегмент BMC может давать несколько классов пользователей.
- Покажите, как это соответствие помогает не пропустить требования.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица соответствия: `Сегмент BMC | Класс(ы) пользователей | Персона(ы)`. Ниже — список рассогласований и вывод о пользе сквозной связи.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Все сегменты BMC сопоставлены с классами пользователей.
- [ ] Выявлены рассогласования (если есть) или подтверждена полнота.
- [ ] Объяснено, почему сегмент «рестораны» может давать несколько классов (менеджер, повар).
- [ ] Показано, что несвязанный класс/сегмент = риск пропустить требования.
- [ ] Соответствие непротиворечиво (нет «висящих» персон без класса).

## ПОДСКАЗКА
Сквозная связь BMC ↔ классы ↔ персоны — это проверка полноты. Если в BMC есть сегмент «курьеры», а среди классов их нет — вы рискуете пропустить целую группу требований. Один сегмент часто раскладывается на несколько классов с разными ролями.
