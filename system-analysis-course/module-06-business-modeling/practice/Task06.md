# Задача 06 — Модуль 06: Бизнес-моделирование

## ЗАДАНИЕ
Определите внешние системы MedConnect и их роль в контексте: что каждая делает, какие данные передаёт, какие требования к интеграции порождает.

## ВХОДНЫЕ ДАННЫЕ
MedConnect взаимодействует с внешним окружением: система электронных рецептов (государственная), платёжный провайдер, SMS/push-шлюз, система видеосвязи (внешний провайдер видеозвонков), лаборатории (для направлений на анализы — в будущем), система идентификации (ЕСИА/портал госуслуг — для подтверждения личности).

## ТРЕБОВАНИЯ
- Для каждой внешней системы опишите: её роль, какие данные туда уходят/оттуда приходят, синхронное или асинхронное взаимодействие (предположительно), критичность для основного сценария.
- Постройте контекстную диаграмму MedConnect с этими системами и акторами (пациент, врач).
- Отметьте, какие интеграции содержат чувствительные данные (медицинские/ПДн) и требуют особой защиты.
- Выделите внешнюю систему, недоступность которой блокирует ключевой сценарий.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица внешних систем (`Система | Роль | Данные | Sync/Async | Критичность`) + контекстная диаграмма MedConnect + пометки о чувствительных данных.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Описаны все внешние системы по всем полям.
- [ ] Контекстная диаграмма включает акторов (пациент, врач) и внешние системы.
- [ ] Отмечены интеграции с медданными/ПДн, требующие защиты (рецепты, идентификация).
- [ ] Выделена блокирующая система (например, видеосвязь для консультации, провайдер оплаты до приёма).
- [ ] Указан характер взаимодействия (sync/async) с обоснованием для 2–3 систем.

## ПОДСКАЗКА
Каждая внешняя система = набор требований к интеграции и потенциальная точка отказа. Особое внимание — где ходят медицинские данные (152-ФЗ). Спросите: «без какой внешней системы основной сценарий невозможен?» — это критичная зависимость (тема устойчивости из модулей 25–26).
