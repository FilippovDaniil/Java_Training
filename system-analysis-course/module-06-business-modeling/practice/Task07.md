# Задача 07 — Модуль 06: Бизнес-моделирование

## ЗАДАНИЕ
Из заполненного Business Model Canvas выведите ключевые фичи и требования к системе. Покажите, как бизнес-модель порождает требования.

## ВХОДНЫЕ ДАННЫЕ
Используйте BMC FoodExpress из задачи 01 (или кратко воспроизведите ключевые блоки). Задача — превратить блоки канвы в конкретные требования/фичи.

## ТРЕБОВАНИЯ
- Для блоков «ценностное предложение», «каналы», «потоки доходов», «ключевые партнёры», «отношения с клиентами» выведите минимум по 1 фиче/требованию.
- Для каждой фичи укажите, из какого блока BMC она следует.
- Выделите бизнес-правила, которые порождаются блоком «потоки доходов» (комиссии, плата за доставку).
- Выделите интеграции, которые порождаются блоком «ключевые партнёры».
- Отметьте фичу, без которой ценностное предложение не работает.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: `Блок BMC | Выведенная фича/требование | Тип (FR/бизнес-правило/интеграция)`. Ниже — ключевая фича для ценностного предложения.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Минимум по 1 фиче/требованию из 5 указанных блоков.
- [ ] Каждая фича прослеживается к конкретному блоку.
- [ ] Из «потоков доходов» выведены бизнес-правила расчёта комиссии/доставки.
- [ ] Из «ключевых партнёров» выведены интеграции (оплата, карты).
- [ ] Выделена фича, критичная для ценностного предложения (быстрый поиск/заказ/доставка).
- [ ] Типы требований проставлены корректно.

## ПОДСКАЗКА
BMC — не украшение, а источник требований. «Комиссия с ресторанов» в потоках доходов → требование расчёта и удержания комиссии (бизнес-правило). «Платёжный провайдер» в партнёрах → требование интеграции оплаты. Так канва превращается в каталог требований.
