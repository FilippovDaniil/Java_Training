# Задача 08 — Модуль 06: Бизнес-моделирование

## ЗАДАНИЕ
Сравните две бизнес-модели монетизации для FoodExpress и покажите, как выбор модели меняет требования к системе.

## ВХОДНЫЕ ДАННЫЕ
Два варианта монетизации:
- **Модель A — комиссия с заказа:** FoodExpress берёт процент с каждого заказа ресторана + плату за доставку с клиента.
- **Модель B — подписка:** рестораны платят фиксированную абонентскую плату за размещение, клиенты — подписку за бесплатную доставку.

## ТРЕБОВАНИЯ
- Сравните модели по: предсказуемость дохода, барьер входа для ресторанов/клиентов, сложность расчётов, влияние на поведение пользователей.
- Для каждой модели определите, какие требования и бизнес-правила она порождает (расчёт комиссии vs учёт подписок и их продление).
- Покажите, как меняется модель данных (что нужно хранить в каждом случае).
- Дайте рекомендацию для старта (один город, проверка гипотезы) с обоснованием.

## ФОРМАТ РЕЗУЛЬТАТА
Сравнительная таблица по критериям + для каждой модели список порождаемых требований/правил + влияние на данные + рекомендация.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Сравнение по ≥4 критериям.
- [ ] Для модели A: правила расчёта комиссии и платы за доставку; для B: учёт подписок, биллинг, продление/отмена.
- [ ] Показано различие в данных (комиссия per-order vs состояние подписки и её период).
- [ ] Дана обоснованная рекомендация для старта.
- [ ] Отмечено влияние модели на поведение пользователей (барьер входа, мотивация заказывать).

## ПОДСКАЗКА
Бизнес-модель напрямую диктует требования и данные. Комиссия — расчёт на каждый заказ; подписка — состояние «активна/истекла», биллинг-цикл, продление. Разные модели = разные сущности в БД и разные бизнес-правила. Для проверки гипотезы на старте обычно проще одна модель.
