# Задача 10 (мини-проект) — Модуль 06: Бизнес-моделирование

## ЗАДАНИЕ
Соберите пакет бизнес-моделирования для MedConnect: BMC + контекстная диаграмма + классы пользователей + 2 персоны, согласованные между собой. Это синтез теории модуля.

## ВХОДНЫЕ ДАННЫЕ
**MedConnect** — платформа телемедицины: онлайн-запись, видеоконсультации, электронные рецепты. Зарабатывает на комиссии с консультаций. Партнёры: клиники/врачи, платёжный провайдер, провайдер видеосвязи, госсистема рецептов. Пользователи: пациенты, врачи, администраторы клиник, поддержка.

## ТРЕБОВАНИЯ
Подготовьте пакет, включающий:
1. **BMC** — все 9 блоков, конкретно.
2. **Контекстная диаграмма** — система-«чёрный ящик», акторы и внешние системы, потоки.
3. **Классы пользователей** — минимум 4, с характеристиками и правами.
4. **2 персоны** — для пациента и для врача (структура: цели, боли, поведение, сценарий).
5. **Сквозная согласованность** — сегменты BMC ↔ классы ↔ персоны не противоречат друг другу; контекстная диаграмма покрывает партнёров из BMC.
6. **Выводы** — 3–5 ключевых фич/требований, вытекающих из пакета.

## ФОРМАТ РЕЗУЛЬТАТА
Документ из 6 разделов. Модели должны ссылаться друг на друга (партнёр из BMC → внешняя система на диаграмме → интеграция в выводах).

## КРИТЕРИИ ПРИЁМКИ
- [ ] BMC заполнен по 9 блокам, конкретно, с учётом многосторонности (пациенты, врачи, клиники).
- [ ] Контекстная диаграмма: один блок системы, все акторы и внешние системы (оплата, видео, рецепты), двусторонние потоки.
- [ ] ≥4 класса пользователей с правами; учтена регуляторная роль (доступ к медданным).
- [ ] 2 персоны (пациент + врач) с целями, болями, сценарием.
- [ ] Сквозная согласованность подтверждена: партнёры BMC присутствуют на диаграмме; сегменты ↔ классы ↔ персоны связаны.
- [ ] Выведены 3–5 фич/требований из пакета, включая интеграции и работу с медданными.
- [ ] Учтена приватность/152-ФЗ хотя бы в одном выводе.

## ПОДСКАЗКА
Это связный «портрет» продукта и его контекста, на который опираются use cases (07), модель данных (16) и интеграции (часть 6). Главная ценность — согласованность: одна и та же реальность видна в BMC, на контекстной диаграмме и в персонах без противоречий. Используйте наработки задач 01–08 этого модуля.
