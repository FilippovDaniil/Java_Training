# Задача 01 — Модуль 07: Use Cases

## ЗАДАНИЕ
Выделите акторов и use cases из текстового описания системы. Отделите цели пользователей от внутренних функций.

## ВХОДНЫЕ ДАННЫЕ
Описание FoodExpress:
> Клиенты ищут рестораны, оформляют и оплачивают заказы, отслеживают доставку, оставляют отзывы. Менеджеры ресторанов управляют меню, принимают и подтверждают заказы. Курьеры получают задания, отмечают статусы доставки. Операторы поддержки разбирают жалобы и оформляют возвраты. Система списывает деньги через платёжный провайдер, отправляет уведомления через SMS-шлюз, строит маршруты через картографический сервис. Раз в сутки система формирует отчёт по выручке для администратора.

## ТРЕБОВАНИЯ
- Выделите всех акторов, разделив первичных и вторичных (включая внешние системы и время/планировщик).
- Выделите use cases уровня sea-level (цели пользователей), не функции.
- Отбракуйте кандидатов, которые на самом деле являются внутренними функциями, а не use cases.
- Сгруппируйте use cases по акторам.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица акторов (`Актор | Тип: первичный/вторичный/система/время`) + список use cases, сгруппированный по акторам + список отбракованных «не-use-case» с причиной.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Выделены акторы: Клиент, Менеджер ресторана, Курьер, Оператор поддержки, Администратор (первичные); Платёжный провайдер, SMS-шлюз, Картографический сервис (вторичные/системы); Планировщик/Время (для суточного отчёта).
- [ ] Use cases — цели пользователей (Оформить заказ, Управлять меню, Подтвердить заказ и т.д.), не функции.
- [ ] Отбракованы внутренние функции (списать деньги, отправить SMS — это вторичные шаги, не самостоятельные цели актора-пользователя).
- [ ] Use cases сгруппированы по акторам.
- [ ] Суточный отчёт привязан к актору «Время/планировщик».

## ПОДСКАЗКА
Use case = значимая цель актора, дающая ему ценность. «Списать деньги» ценно не клиенту, а как часть «Оплатить заказ». Внешние системы — обычно вторичные акторы. Регулярные задачи по расписанию инициирует актор «Время».
