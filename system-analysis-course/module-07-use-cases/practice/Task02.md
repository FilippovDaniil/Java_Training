# Задача 02 — Модуль 07: Use Cases

## ЗАДАНИЕ
Постройте use case diagram для FoodExpress на основе акторов и use cases из задачи 01.

## ВХОДНЫЕ ДАННЫЕ
Используйте акторов и use cases, выделенные в задаче 01 (минимум: Клиент, Менеджер ресторана, Курьер, Платёжный провайдер; use cases: Оформить заказ, Оплатить заказ, Отследить доставку, Управлять меню, Подтвердить заказ, Применить промокод).

## ТРЕБОВАНИЯ
- Изобразите границу системы, акторов снаружи, use cases внутри.
- Проведите ассоциации актор—use case.
- Покажите минимум одно отношение include и одно extend.
- Разместите вторичного актора (платёжный провайдер) у соответствующего use case.
- Соблюдайте нотацию (актор снаружи границы, use case — овал внутри).

## ФОРМАТ РЕЗУЛЬТАТА
Use case diagram в псевдографике или текстовой нотации (можно PlantUML-синтаксисом). Ниже — пояснение, где include, где extend и почему.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Есть граница системы; акторы снаружи, use cases внутри.
- [ ] Ассоциации проведены корректно (каждый актор связан со своими use cases).
- [ ] «Оформить заказ» include «Оплатить заказ» (обязательно).
- [ ] «Оформить заказ» extend «Применить промокод» (опционально).
- [ ] Платёжный провайдер — вторичный актор у «Оплатить заказ».
- [ ] Нотация соблюдена (нет use cases снаружи границы, акторов внутри).

## ПОДСКАЗКА
Если используете PlantUML: `actor`, `usecase`, `rectangle` для границы, `-->` для ассоциаций, `.> : include/extend`. Главное — корректная семантика: include для обязательного шага, extend для условного.
