# Задача 03 — Модуль 07: Use Cases

## ЗАДАНИЕ
Напишите текстовое описание use case «Оформить заказ» для FoodExpress — основной (успешный) поток с полной структурой.

## ВХОДНЫЕ ДАННЫЕ
Контекст: клиент авторизован, выбрал ресторан. Цель — оформить и оплатить заказ с доставкой. В оплате участвует внешний платёжный провайдер.

## ТРЕБОВАНИЯ
- Заполните поля: название, первичный актор, вторичные акторы, цель, предусловия, постусловия, триггер.
- Опишите основной поток нумерованными шагами (happy path) — от старта до достижения цели.
- Каждый шаг — одно понятное действие (актор делает / система отвечает).
- Покажите шаг с include (Оплатить заказ).
- Пока БЕЗ альтернатив и исключений (они в задаче 04).

## ФОРМАТ РЕЗУЛЬТАТА
Структурированное описание use case по полям + нумерованный основной поток.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Заполнены все поля (название, акторы, цель, пред/постусловия, триггер).
- [ ] Предусловия и постусловия проверяемы (клиент авторизован → заказ создан и оплачен).
- [ ] Основной поток — последовательность чётких шагов до достижения цели.
- [ ] Чередуются действия актора и ответы системы.
- [ ] Показан include «Оплатить заказ».
- [ ] Уровень — sea-level (одна цель за один присест), не слишком крупно/мелко.

## ПОДСКАЗКА
Хороший шаг основного потока формулируется как «актор делает X» или «система делает Y». Постусловие — это проверяемое состояние мира после успеха. include-шаг ссылается на отдельный use case «Оплатить заказ».
