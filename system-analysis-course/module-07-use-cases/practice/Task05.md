# Задача 05 — Модуль 07: Use Cases

## ЗАДАНИЕ
Сформулируйте корректные предусловия и постусловия для набора use cases. Покажите, что они проверяемы и связывают сценарии между собой.

## ВХОДНЫЕ ДАННЫЕ
Use cases FoodExpress:
1. «Оформить заказ».
2. «Отменить заказ».
3. «Назначить курьера на заказ».
4. «Подтвердить доставку».

## ТРЕБОВАНИЯ
- Для каждого use case сформулируйте предусловия (что истинно ДО) и постусловия (что истинно ПОСЛЕ успеха).
- Покажите связь через состояния: постусловие одного use case = предусловие другого (например, «заказ создан» → можно «назначить курьера»).
- Сделайте все условия проверяемыми (можно ответить да/нет).
- Отметьте, как пред/постусловия связаны с моделью состояний заказа (связь с модулем 13).

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: `Use case | Предусловия | Постусловия`. Ниже — цепочка состояний, показывающая связь postcondition → precondition между use cases.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Для всех 4 use cases заданы пред- и постусловия.
- [ ] Все условия проверяемы (конкретное состояние, а не «всё хорошо»).
- [ ] Показана связь: постусловие «Оформить заказ» (заказ создан/оплачен) = предусловие «Назначить курьера».
- [ ] «Отменить заказ» имеет предусловие про допустимый статус (нельзя отменить уже доставленный).
- [ ] Отмечена связь с моделью состояний заказа (создан → оплачен → назначен → доставлен / отменён).

## ПОДСКАЗКА
Пред/постусловия — это «контракт» use case. Они естественно образуют цепочку через состояния заказа: что одним сценарием достигнуто (postcondition), то другим используется как стартовое (precondition). Это прямой мост к модели состояний (модуль 13).
