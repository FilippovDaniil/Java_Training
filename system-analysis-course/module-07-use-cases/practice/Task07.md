# Задача 07 — Модуль 07: Use Cases

## ЗАДАНИЕ
Корректно примените отношения include и extend в модели use cases. Определите, где какое отношение и почему.

## ВХОДНЫЕ ДАННЫЕ
Use cases MedConnect и связи между ними (нужно классифицировать):
1. «Записаться на приём» всегда требует «Авторизоваться».
2. «Записаться на приём» иногда сопровождается «Выбрать предпочтительного врача».
3. «Провести консультацию» всегда требует «Подтвердить личность пациента».
4. «Провести консультацию» может завершиться «Выписать рецепт» (не всегда).
5. «Оплатить приём» всегда включает «Сформировать чек».
6. «Записаться на приём» может включать «Применить страховой полис» (если есть полис).

## ТРЕБОВАНИЯ
- Для каждой связи определите: include или extend.
- Обоснуйте выбор (обязательно/всегда → include; опционально/при условии → extend).
- Укажите направление отношения (от какого use case к какому).
- Нарисуйте фрагмент use case diagram с этими отношениями.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: `Связь | include/extend | Обоснование | Направление`. Ниже — фрагмент диаграммы (псевдографика/PlantUML).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Связи 1, 3, 5 — include (обязательны/всегда).
- [ ] Связи 2, 4, 6 — extend (опциональны/при условии).
- [ ] Для каждой дано обоснование.
- [ ] Направление указано верно (include: от базового к включаемому; extend: от расширяющего к базовому).
- [ ] Фрагмент диаграммы соответствует таблице.

## ПОДСКАЗКА
Тест: «это происходит ВСЕГДА при выполнении базового сценария?» Да → include. «Это происходит ИНОГДА, при определённом условии?» → extend. Слова «всегда требует» = include, «иногда / может / если» = extend.
