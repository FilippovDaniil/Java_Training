# Задача 08 — Модуль 07: Use Cases

## ЗАДАНИЕ
Нормализуйте набор use cases по уровням: приведите к целевому уровню sea-level, подняв слишком мелкие и опустив слишком крупные.

## ВХОДНЫЕ ДАННЫЕ
Смешанный список «use cases» FoodExpress разных уровней:
1. «Управлять логистикой доставки города» (очень крупно).
2. «Ввести адрес в поле».
3. «Оформить заказ».
4. «Проверить формат email».
5. «Развивать партнёрскую сеть ресторанов».
6. «Отменить заказ».
7. «Сохранить токен сессии».
8. «Отследить доставку».

## ТРЕБОВАНИЯ
- Для каждого определите уровень: cloud (стратегический), sea-level (цель пользователя), fish (подфункция).
- Слишком крупные (cloud) — покажите, на какие sea-level use cases они разбиваются.
- Слишком мелкие (fish) — покажите, частью какого sea-level use case они являются.
- Оставьте список корректных sea-level use cases.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: `№ | Уровень | Действие (разбить/поднять/оставить) | Куда относится`. Ниже — итоговый список чистых sea-level use cases.

## КРИТЕРИИ ПРИЁМКИ
- [ ] 1, 5 опознаны как cloud → разбиты на sea-level цели.
- [ ] 2, 4, 7 опознаны как fish → отнесены к родительским sea-level use cases.
- [ ] 3, 6, 8 подтверждены как sea-level.
- [ ] Для cloud показана декомпозиция, для fish — родитель.
- [ ] Итоговый список содержит только sea-level use cases.

## ПОДСКАЗКА
Sea-level = одна значимая цель пользователя за один присест, обычно завершается за один «сеанс работы». Cloud — это бизнес-процесс/стратегия (разбить). Fish — техническая подфункция внутри сценария (поднять в родителя). «Развивать сеть» — стратегия, «проверить email» — подфункция.
