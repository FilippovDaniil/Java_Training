# Задача 09 — Модуль 07: Use Cases

## ЗАДАНИЕ
Постройте трассировку: свяжите бизнес/пользовательские требования с use cases, а use cases — с функциональными требованиями и тест-кейсами.

## ВХОДНЫЕ ДАННЫЕ
Фрагмент требований FoodExpress:
- BR-1: снизить долю брошенных корзин на оформлении.
- UR-1: клиент может оформить заказ за минимум шагов.
- UR-2: клиент может оплатить сохранённой картой.
- UR-3: клиент может применить промокод.
Use cases: UC-1 «Оформить заказ», UC-2 «Оплатить заказ», UC-3 «Применить промокод».

## ТРЕБОВАНИЯ
- Постройте матрицу трассировки: требование ↔ use case.
- Для UC-1 выведите 2–3 функциональных требования (FR) и для каждого — по тест-кейсу (название).
- Покажите, что каждый use case прослеживается вверх до бизнес-требования.
- Найдите «висящий» элемент, если он есть (use case без требования или требование без use case).

## ФОРМАТ РЕЗУЛЬТАТА
Матрица `Требование | Use case`. Таблица `UC-1 → FR → тест-кейс`. Ниже — проверка трассировки вверх и поиск «висящих».

## КРИТЕРИИ ПРИЁМКИ
- [ ] Матрица связывает UR/BR с use cases (UR-2 → UC-2, UR-3 → UC-3, UR-1 → UC-1).
- [ ] Из UC-1 выведены ≥2 FR с тест-кейсами.
- [ ] Каждый use case прослеживается до BR-1.
- [ ] Проверено наличие/отсутствие «висящих» элементов.
- [ ] Трассировка двунаправленная (вверх к цели, вниз к тестам).

## ПОДСКАЗКА
Трассировка — это «нить» от бизнес-цели через use case к функции и тесту. Она доказывает, что каждая функция работает на цель и проверена. «Висящий» use case без бизнес-требования — кандидат на удаление; требование без use case — пропущенный сценарий.
