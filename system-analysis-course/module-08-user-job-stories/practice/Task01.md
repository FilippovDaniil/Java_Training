# Задача 01 — Модуль 08: User / Job Stories

## ЗАДАНИЕ
Перепишите «сырые» требования в user stories по формату «Как… я хочу… чтобы…», обязательно с ценностью.

## ВХОДНЫЕ ДАННЫЕ
Сырые требования FoodExpress:
1. «Нужна история заказов».
2. «Сделать фильтр ресторанов по кухне».
3. «Push-уведомления о статусе заказа».
4. «Возможность оставить отзыв».
5. «Сохранение нескольких адресов доставки».
6. «Экспорт чека в PDF».

## ТРЕБОВАНИЯ
- Перепишите каждое в user story с тремя частями (роль, действие, ценность).
- Подберите корректную роль (не всегда «клиент» — может быть ресторан, курьер).
- Часть «чтобы…» должна объяснять реальную ценность, а не повторять действие.
- Для 2 историй предложите альтернативную формулировку с другой ролью.

## ФОРМАТ РЕЗУЛЬТАТА
Список user stories (по одной на требование). Для 2 — альтернативная роль.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Все 6 переписаны в формат «Как… я хочу… чтобы…».
- [ ] Часть «чтобы…» несёт ценность, а не дублирует действие («чтобы видеть историю» — плохо; «чтобы быстро повторить заказ» — хорошо).
- [ ] Роли подобраны осмысленно (где-то клиент, где-то ресторан/курьер).
- [ ] Для 2 историй дана альтернативная роль.
- [ ] Истории атомарны (одна история — одна возможность).

## ПОДСКАЗКА
Тест на ценность: если «чтобы…» лишь повторяет «я хочу…» другими словами — ценность не найдена. Спросите «а зачем пользователю это действие?». «Экспорт чека» → «чтобы приложить к авансовому отчёту».
