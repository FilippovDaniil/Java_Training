# Задача 02 — Модуль 08: User / Job Stories

## ЗАДАНИЕ
Проверьте набор user stories по INVEST и исправьте те, что нарушают критерии.

## ВХОДНЫЕ ДАННЫЕ
Истории из бэклога FoodExpress:
1. «Как клиент, я хочу полностью оформить, оплатить, отследить и оценить заказ, чтобы пользоваться сервисом» (одна история).
2. «Как разработчик, я хочу настроить базу данных, чтобы хранить данные».
3. «Как клиент, я хочу красивый интерфейс».
4. «Как клиент, я хочу применить промокод, чтобы получить скидку».
5. «Как клиент, я хочу оплатить заказ способом, который мы потом придумаем».
6. «Как ресторан, я хочу видеть новые заказы, чтобы вовремя их готовить».

## ТРЕБОВАНИЯ
- Для каждой истории проверьте все 6 критериев INVEST.
- Укажите, какие критерии нарушены и почему.
- Исправьте проблемные истории (переписать/разбить).
- Отметьте, какие истории уже хороши.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: `№ | Нарушенные критерии INVEST | Исправление`. Отдельно — список «уже хороших».

## КРИТЕРИИ ПРИЁМКИ
- [ ] История 1 нарушает Small/Independent (эпик) → разбита на отдельные истории.
- [ ] История 2 нарушает Valuable (нет пользовательской ценности, техзадача).
- [ ] История 3 нарушает Testable (нет проверяемых критериев) → переформулирована измеримо.
- [ ] История 5 нарушает Estimable (неопределённость «потом придумаем») → уточнить/вынести в spike.
- [ ] Истории 4 и 6 признаны корректными по INVEST.
- [ ] Для всех нарушений дано конкретное исправление.

## ПОДСКАЗКА
Самые частые нарушения: Small (эпик вместо истории) и Valuable (техзадача без пользовательской ценности). «Красивый интерфейс» не Testable — нужен критерий. «Потом придумаем» не Estimable — это неопределённость, её выносят в исследование (spike).
