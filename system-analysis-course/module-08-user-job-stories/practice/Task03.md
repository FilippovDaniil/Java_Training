# Задача 03 — Модуль 08: User / Job Stories

## ЗАДАНИЕ
Напишите критерии приёмки в формате Gherkin (Given/When/Then) для user story, покрыв успешный и альтернативные/ошибочные сценарии.

## ВХОДНЫЕ ДАННЫЕ
User story FoodExpress:
> Как клиент, я хочу применить промокод при оформлении заказа, чтобы получить скидку.

Известные правила: промокод имеет срок действия; бывает на минимальную сумму заказа; не суммируется со скидкой пенсионера; может быть уже использован клиентом.

## ТРЕБОВАНИЯ
- Напишите минимум 5 сценариев в Gherkin (Дано/Когда/Тогда).
- Покройте: успешное применение, истёкший промокод, несуществующий код, заказ ниже минимальной суммы, конфликт со скидкой пенсионера.
- Критерии должны описывать поведение (что проверить), а не реализацию.
- Каждый сценарий — атомарный и проверяемый.

## ФОРМАТ РЕЗУЛЬТАТА
Набор Gherkin-сценариев (по одному на случай) под заголовком истории.

## КРИТЕРИИ ПРИЁМКИ
- [ ] ≥5 сценариев в корректном Gherkin (Дано/Когда/Тогда).
- [ ] Покрыты успех и ≥4 негативных/граничных случая.
- [ ] Сценарии описывают поведение, а не UI/реализацию.
- [ ] Каждый сценарий атомарный (одна проверка).
- [ ] Учтён конфликт со скидкой пенсионера (бизнес-правило).
- [ ] Учтена минимальная сумма заказа.

## ПОДСКАЗКА
Gherkin: «Дано» — контекст/предусловие, «Когда» — действие, «Тогда» — ожидаемый результат. Один сценарий = один путь. Негативные сценарии (истёк, не существует, ниже минимума) важнее для качества, чем happy path.
