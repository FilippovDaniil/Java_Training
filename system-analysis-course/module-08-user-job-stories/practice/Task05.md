# Задача 05 — Модуль 08: User / Job Stories

## ЗАДАНИЕ
Декомпозируйте эпик на отдельные истории, применяя паттерны разбиения. Сохраните ценность в каждом куске (вертикальные срезы).

## ВХОДНЫЕ ДАННЫЕ
Эпик FoodExpress:
> «Как клиент, я хочу оплачивать заказы, чтобы завершать покупки».
Известно: оплата возможна картой, бонусами, при получении; есть промокоды; нужна оплата как на вебе, так и в мобильном приложении; возможны возвраты.

## ТРЕБОВАНИЯ
- Разбейте эпик минимум на 5 историй, используя разные паттерны декомпозиции (по бизнес-правилам, по вариациям данных, по платформам, по операциям).
- Для каждой укажите применённый паттерн.
- Убедитесь, что каждая история — вертикальный срез (даёт ценность сама по себе), а не слой (БД/UI отдельно).
- Предложите порядок реализации (что в MVP, что потом) с обоснованием.

## ФОРМАТ РЕЗУЛЬТАТА
Список историй (с паттерном для каждой) + порядок реализации с обоснованием.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Эпик разбит на ≥5 историй.
- [ ] Использованы разные паттерны (карта/бонусы — бизнес-правила; веб/мобайл — платформы; оплата/возврат — операции).
- [ ] Каждая история — вертикальный срез с самостоятельной ценностью.
- [ ] Нет историй-слоёв («сделать таблицу платежей»).
- [ ] Предложен и обоснован порядок (MVP — оплата картой в мобайле, потом расширения).

## ПОДСКАЗКА
Хорошая декомпозиция режет «поперёк» (каждый кусок — работающая ценность), а не «вдоль слоёв». «Оплата картой в мобильном приложении» — вертикальный срез; «таблица платежей в БД» — слой без ценности. Начинают с самого важного канала/способа.
