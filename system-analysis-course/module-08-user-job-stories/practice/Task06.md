# Задача 06 — Модуль 08: User / Job Stories

## ЗАДАНИЕ
Постройте story map для FoodExpress: горизонтальный «хребет» действий пользователя и вертикальная разбивка по релизам.

## ВХОДНЫЕ ДАННЫЕ
Путь клиента FoodExpress: найти ресторан → собрать заказ → оформить и оплатить → отслеживать доставку → оценить. Есть множество идей разной важности (поиск, фильтры, рекомендации, корзина, промокоды, оплата картой/бонусами, статус, карта курьера, чат, отзывы, повтор заказа).

## ТРЕБОВАНИЯ
- Постройте backbone (горизонтально) — последовательность ключевых активностей пользователя.
- Под каждой активностью разместите истории по приоритету (вертикально).
- Выделите горизонтальный срез MVP (тонкая сквозная линия ценности).
- Выделите релизы 2 и 3.
- Объясните, почему MVP — это горизонтальный срез, а не «всё про оплату сразу».

## ФОРМАТ РЕЗУЛЬТАТА
Story map (таблица/псевдографика): столбцы — активности backbone, строки — релизы. Ниже — пояснение про горизонтальный срез MVP.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Backbone отражает полный путь пользователя (≥4 активности по порядку).
- [ ] Под каждой активностью — истории, отсортированные по приоритету.
- [ ] MVP — это сквозной горизонтальный срез (по одной базовой истории на каждую активность), позволяющий пройти весь путь.
- [ ] Выделены релизы 2 и 3 с более продвинутыми историями.
- [ ] Объяснено, почему вертикальная нарезка («сначала вся оплата») хуже горизонтальной для MVP.

## ПОДСКАЗКА
Story map читается слева направо как путь пользователя, сверху вниз — по приоритету. MVP — это верхняя тонкая полоса через все активности: пользователь должен пройти весь путь, пусть и в простейшем виде. «Вся оплата сразу, но без возможности оформить заказ» — бесполезно.
