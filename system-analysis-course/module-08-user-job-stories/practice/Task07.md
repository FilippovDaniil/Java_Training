# Задача 07 — Модуль 08: User / Job Stories

## ЗАДАНИЕ
Оцените истории относительным размером (story points) и выявите слишком крупные, требующие декомпозиции.

## ВХОДНЫЕ ДАННЫЕ
Истории FoodExpress для оценки (шкала Фибоначчи: 1, 2, 3, 5, 8, 13, 21):
1. «Клиент видит список ресторанов».
2. «Клиент фильтрует рестораны по кухне и рейтингу».
3. «Клиент оплачивает заказ картой, бонусами или при получении, с промокодами и возвратами».
4. «Клиент получает push при смене статуса заказа».
5. «Клиент отслеживает курьера на карте в реальном времени с прогнозом прибытия».

## ТРЕБОВАНИЯ
- Дайте относительную оценку каждой истории (story points), приняв за эталон самую простую.
- Обоснуйте оценки сравнением между собой (не в часах).
- Определите истории, размер которых сигнализирует о необходимости декомпозиции (обычно ≥13).
- Для одной слишком крупной покажите разбивку на оцениваемые части.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: `История | Оценка (SP) | Обоснование (относительно эталона)`. Ниже — какие декомпозировать и пример разбивки.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Все истории оценены по Фибоначчи относительно эталона.
- [ ] Оценки обоснованы сравнением, а не переведены из часов.
- [ ] История 3 (множество способов + промокоды + возвраты) опознана как слишком крупная → декомпозиция.
- [ ] История 5 (реалтайм-трекинг + прогноз) отмечена как крупная/рискованная.
- [ ] Показана разбивка одной крупной истории на части меньшего размера.

## ПОДСКАЗКА
Story points — относительная мера сложности/объёма, а не часы. Берёте простую историю за эталон (например, 2) и сравниваете остальные с ней. Высокая оценка (13, 21) — сигнал «это эпик, разбей». Реалтайм и множество правил почти всегда крупные.
