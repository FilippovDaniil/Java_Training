# Задача 10 (мини-проект) — Модуль 08: User / Job Stories

## ЗАДАНИЕ
Составьте story map и бэклог первого релиза MedConnect с историями и критериями приёмки. Синтез теории модуля.

## ВХОДНЫЕ ДАННЫЕ
**MedConnect** — телемедицина. Главная гипотеза: пациенты готовы получать первичную консультацию онлайн. Путь пациента: найти врача → записаться → оплатить → пройти видеоконсультацию → получить рецепт/итоги. (Можно опереться на MVP из мини-проекта модуля 03.)

## ТРЕБОВАНИЯ
Подготовьте пакет, включающий:
1. **Story map** — backbone пути пациента + истории по релизам.
2. **MVP-релиз** — горизонтальный срез, проверяющий главную гипотезу.
3. **Истории первого релиза** (≥6) в формате «Как… я хочу… чтобы…», проверенные по INVEST.
4. **Критерии приёмки** (Gherkin) минимум для 3 ключевых историй, с негативными сценариями.
5. **Оценка и приоритет** для историй релиза + зависимости.
6. **Учёт регуляторики** — где в историях/критериях отражены согласие на ПДн и доступ к медданным.

## ФОРМАТ РЕЗУЛЬТАТА
Документ: story map + список историй MVP (с оценкой/приоритетом) + Gherkin-критерии для 3 историй + пометки по регуляторике. Внутренне согласованный.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Story map с backbone пути пациента и разбивкой по релизам.
- [ ] MVP — горизонтальный срез, реально проверяющий гипотезу (пациент проходит весь путь).
- [ ] ≥6 историй в корректном формате, прошедших INVEST.
- [ ] Gherkin-критерии для ≥3 историй с успешными и негативными сценариями.
- [ ] Указаны оценки, приоритеты и зависимости.
- [ ] Регуляторика отражена (согласие на ПДн до записи, доступ к рецепту/медданным).
- [ ] Пакет согласован: истории MVP покрывают backbone и проверяют гипотезу.

## ПОДСКАЗКА
Это рабочий бэклог, с которого начинается разработка. Главные проверки: MVP — сквозной срез (а не «всё про оплату»), истории проходят INVEST, критерии приёмки покрывают негативные случаи, регуляторика заложена с самого начала, а не «потом». Используйте задачи 03, 05, 06, 09 этого модуля.
