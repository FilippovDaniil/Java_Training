# Задача 04 — Модуль 09: Нефункциональные требования

## ЗАДАНИЕ
Рассчитайте, что означают разные уровни доступности на практике, и подберите обоснованный уровень для разных сервисов FoodExpress.

## ВХОДНЫЕ ДАННЫЕ
Сервисы FoodExpress:
- сервис заказов (без него нельзя заказать — деньги бизнеса);
- сервис отзывов (приятно, но не критично);
- сервис оплаты (критичен, завязан на деньги);
- сервис рекомендаций (улучшает опыт, некритичен).
Уровни доступности: 99%, 99.9%, 99.95%, 99.99%.

## ТРЕБОВАНИЯ
- Для каждого уровня доступности укажите допустимый простой в месяц (в минутах/часах).
- Подберите обоснованный целевой уровень (SLO) для каждого сервиса.
- Объясните, почему не для всех сервисов нужен максимум.
- Опишите, как уровень доступности влияет на стоимость и архитектуру (резервирование).
- Различите понятия SLA, SLO, SLI на одном примере.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица доступности (`Уровень | Простой в месяц`) + таблица сервисов (`Сервис | Целевой SLO | Обоснование`) + пояснение SLA/SLO/SLI на примере сервиса оплаты.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Простой в месяц рассчитан корректно для всех уровней (99% ≈ 7.3 ч, 99.9% ≈ 43.8 мин, 99.99% ≈ 4.4 мин).
- [ ] Сервисы заказов и оплаты получили более высокий SLO, чем отзывы/рекомендации.
- [ ] Обосновано, почему максимум не нужен везде (стоимость девяток).
- [ ] Описано влияние на архитектуру/стоимость (резервирование, дублирование).
- [ ] Корректно разведены SLA (внешнее обязательство), SLO (внутренняя цель), SLI (показатель).

## ПОДСКАЗКА
Каждая «девятка» кратно дороже предыдущей. Критичные для денег сервисы (заказы, оплата) требуют высокой доступности; отзывы могут полежать. SLA — обещание клиенту с санкциями, SLO — внутренняя цель, SLI — то, что реально меряем (например, доля успешных запросов).
