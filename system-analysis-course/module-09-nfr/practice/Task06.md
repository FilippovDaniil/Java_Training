# Задача 06 — Модуль 09: Нефункциональные требования

## ЗАДАНИЕ
Выявите конфликты (trade-offs) между нефункциональными требованиями и предложите осознанный баланс под бизнес-приоритеты.

## ВХОДНЫЕ ДАННЫЕ
Набор желаемых NFR FoodExpress:
1. Максимальная безопасность (много проверок при входе и оплате).
2. Минимальное трение для клиента (вход и заказ в пару тапов).
3. Данные всегда актуальны (строгая согласованность).
4. Максимальная скорость отклика (агрессивное кэширование).
5. Доступность 99.99%.
6. Минимальная стоимость инфраструктуры.

## ТРЕБОВАНИЯ
- Найдите минимум 3 пары конфликтующих NFR из списка.
- Для каждого конфликта объясните суть противоречия.
- Предложите осознанный баланс с привязкой к бизнес-приоритету (что важнее для FoodExpress и почему).
- Покажите, где можно «развести» конфликт (например, разный уровень для разных операций).

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: `Конфликт (NFR ↔ NFR) | Суть противоречия | Предлагаемый баланс | Обоснование`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Найдено ≥3 реальных конфликта (безопасность↔удобство, согласованность↔скорость, доступность↔стоимость).
- [ ] Для каждого объяснена суть.
- [ ] Баланс привязан к бизнес-приоритетам FoodExpress, а не абстрактен.
- [ ] Показан приём «разведения» (например, строгая безопасность только на оплате, лёгкий вход для просмотра).
- [ ] Признано, что нельзя максимизировать всё одновременно.

## ПОДСКАЗКА
NFR конкурируют: кэш (скорость) против актуальности (согласованность); проверки (безопасность) против простоты (удобство); девятки (доступность) против бюджета. Часто конфликт «разводят» по контексту: критичные операции — строго, некритичные — легко. Это связь с CAP (модуль 19) и архитектурой (26).
