# Задача 07 — Модуль 10: Приоритизация и управление требованиями

## ЗАДАНИЕ
Сравните методы приоритизации (MoSCoW, RICE, Kano, WSJF) и подберите подходящий под разные контексты.

## ВХОДНЫЕ ДАННЫЕ
Контексты:
1. Стартап на старте: нужно быстро решить, что в MVP, с минимумом данных.
2. Зрелый продукт с аналитикой: есть данные о пользователях и метриках, нужно ранжировать большой бэклог.
3. Команда хочет понять, какие фичи дают «вау», а какие просто обязательны.
4. Поток разнородных задач, нужно выбрать последовательность с учётом стоимости задержки.

## ТРЕБОВАНИЯ
- Сопоставьте каждому контексту подходящий метод (или комбинацию) и обоснуйте.
- Для каждого метода укажите его сильные и слабые стороны.
- Объясните, почему RICE плохо работает без данных (контекст 1).
- Предложите, как методы можно комбинировать (например, MoSCoW + RICE).

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: `Контекст | Метод | Обоснование`. Отдельно — таблица сильных/слабых сторон методов + идея комбинирования.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Контекст 1 → MoSCoW (быстро, мало данных); 2 → RICE (есть данные); 3 → Kano; 4 → WSJF.
- [ ] Для каждого метода названы сильные и слабые стороны.
- [ ] Объяснено, почему RICE требует данных (иначе мнимая точность).
- [ ] Предложена комбинация методов с пользой.
- [ ] Обоснования привязаны к контексту, а не общие.

## ПОДСКАЗКА
Нет «лучшего» метода — есть подходящий под контекст. MoSCoW — быстро и без данных. RICE — когда есть метрики. Kano — про тип удовлетворённости. WSJF — про последовательность с учётом задержки. Часто комбинируют: Kano выявляет базовые (→ Must в MoSCoW), RICE ранжирует остальное.
