# Задача 10 (мини-проект) — Модуль 10: Приоритизация и управление требованиями

## ЗАДАНИЕ
Подготовьте пакет приоритизации и управления требованиями для релиза MedConnect: приоритизация бэклога + матрица трассировки + процесс управления изменениями. Синтез теории модуля.

## ВХОДНЫЕ ДАННЫЕ
**MedConnect**, первый релиз. Бизнес-цели: BR-1 проверить готовность пациентов к онлайн-консультациям; BR-2 соответствие 152-ФЗ. Бэклог (фичи): регистрация, согласие на ПДн, поиск врача, запись, оплата, видеоконсультация, электронный рецепт, отзывы, история консультаций, напоминания о приёме. Команда работает спринтами.

## ТРЕБОВАНИЯ
Подготовьте пакет, включающий:
1. **Приоритизацию бэклога** методом MoSCoW (с учётом регуляторики как Must) + обоснование.
2. **RICE или Kano** для ранжирования спорной части бэклога (на выбор, обоснуйте выбор метода).
3. **Матрицу трассировки**: бизнес-цель → требование → история → тест (хотя бы для топ-5 фич).
4. **Процесс управления изменениями**: как обрабатывается change request (импакт-анализ через трассировку).
5. **Baseline и версионирование**: что фиксируется для релиза 1.0.
6. **Выявленный gold-plating** (если есть) с обоснованием сокращения.

## ФОРМАТ РЕЗУЛЬТАТА
Документ из 6 разделов. Приоритизация и трассировка — таблицами. Процесс изменений — кратким описанием шагов. Внутренне согласованный.

## КРИТЕРИИ ПРИЁМКИ
- [ ] MoSCoW: согласие на ПДн и ядро (регистрация, поиск, запись, оплата, видео) — Must; Must не раздут.
- [ ] Применён RICE или Kano к спорной части с обоснованием выбора метода.
- [ ] Матрица трассировки связывает цели → требования → истории → тесты для топ-5.
- [ ] Описан процесс обработки change request с импакт-анализом по трассировке.
- [ ] Определён baseline релиза 1.0 и подход к версионированию.
- [ ] Выявлен gold-plating (или обоснованно отсутствует).
- [ ] Пакет согласован: приоритеты, трассировка и регуляторика не противоречат друг другу.

## ПОДСКАЗКА
Это управленческий слой над требованиями: что делаем первым, как связано, как меняем. Главные проверки middle+: регуляторика всегда Must, приоритеты обоснованы (не «по громкости»), трассировка позволяет импакт-анализ, baseline зафиксирован. Собирайте из задач 01–09 этого модуля.
