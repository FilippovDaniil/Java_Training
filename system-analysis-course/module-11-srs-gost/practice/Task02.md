# Задача 02 — Модуль 11: Спецификация (SRS, ТЗ ГОСТ)

## ЗАДАНИЕ
Заполните раздел ТЗ по ГОСТ 34.602 «Назначение и цели создания системы» и фрагмент «Требования к функциям» для FoodExpress.

## ВХОДНЫЕ ДАННЫЕ
FoodExpress — система доставки еды. Бизнес-цели: запустить агрегатор в одном городе, проверить комиссионную модель, обеспечить оформление/оплату/доставку заказов. Основные функции: поиск ресторанов, оформление заказа, оплата, отслеживание доставки, управление меню рестораном.

## ТРЕБОВАНИЯ
- Заполните раздел 2 ГОСТ «Назначение и цели создания системы»: назначение + измеримые цели и критерии их достижения.
- Заполните фрагмент раздела 4.2 «Требования к функциям»: минимум 4 функции в формализованном виде (что система должна выполнять).
- Цели — измеримые (с критериями достижения).
- Функции — атомарные и проверяемые.
- Соблюдайте стиль ГОСТ (формальный, «система должна…»).

## ФОРМАТ РЕЗУЛЬТАТА
Раздел 2 (назначение + цели с критериями) + раздел 4.2 (нумерованный список функций в формате ГОСТ).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Назначение системы сформулировано чётко.
- [ ] Цели измеримы и имеют критерии достижения (не «улучшить сервис»).
- [ ] В разделе 4.2 ≥4 функции в формализованном виде «система должна…».
- [ ] Функции атомарны и проверяемы.
- [ ] Соблюдён формальный стиль ГОСТ.
- [ ] В раздел функций не попали NFR (они в 4.1).

## ПОДСКАЗКА
Раздел «Цели» ГОСТ требует не «зачем вообще», а измеримые цели с критериями достижения (например, «обеспечить оформление заказа за ≤3 шага», «выйти на N заказов/день»). Раздел 4.2 — только функции (что делает); NFR (скорость, надёжность) идут в 4.1.
