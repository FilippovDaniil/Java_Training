# Задача 05 — Модуль 11: Спецификация (SRS, ТЗ ГОСТ)

## ЗАДАНИЕ
Напишите раздел ТЗ «Требования к видам обеспечения» (раздел 4.3 ГОСТ 34.602) для FoodExpress.

## ВХОДНЫЕ ДАННЫЕ
FoodExpress — система доставки еды. Известно: данные о заказах/ресторанах/пользователях; интерфейсы на русском и английском; интеграции с провайдером оплаты и картами; разворачивается в контейнерах; за эксплуатацию отвечает DevOps-команда; нужны регламенты работы операторов поддержки.

## ТРЕБОВАНИЯ
- Заполните требования к видам обеспечения: информационному, лингвистическому, программному, техническому, организационному (минимум 5 видов).
- Для каждого вида — конкретные требования (не «должно быть обеспечение»).
- Свяжите виды обеспечения с другими требованиями (например, информационное → модель данных).
- Отметьте, какой вид обеспечения чаще всего забывают.

## ФОРМАТ РЕЗУЛЬТАТА
Раздел 4.3 по видам обеспечения: для каждого вида — список требований. Ниже — пометка о часто забываемом виде.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Заполнено ≥5 видов обеспечения с конкретными требованиями.
- [ ] Информационное (структура данных, классификаторы), лингвистическое (языки, термины), программное (ПО, СУБД, технологии-ограничения), техническое (серверы/контейнеры), организационное (роли, регламенты).
- [ ] Требования конкретны, а не «должно быть обеспечение».
- [ ] Показаны связи с другими разделами (информационное ↔ модель данных).
- [ ] Отмечен часто забываемый вид (например, организационное/методическое).

## ПОДСКАЗКА
«Виды обеспечения» — особенность ГОСТ 34, которой нет в SRS. Это про то, что нужно для работы системы: данные (информационное), языки/термины (лингвистическое), софт (программное), железо (техническое), люди и регламенты (организационное). Организационное и методическое забывают чаще всего.
