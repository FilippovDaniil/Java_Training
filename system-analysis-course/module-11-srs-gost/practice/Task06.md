# Задача 06 — Модуль 11: Спецификация (SRS, ТЗ ГОСТ)

## ЗАДАНИЕ
Составьте глоссарий (раздел терминов) для спецификации MedConnect: ключевые термины предметной области с однозначными определениями.

## ВХОДНЫЕ ДАННЫЕ
В обсуждениях MedConnect термины используются по-разному: «приём», «консультация», «запись», «слот», «рецепт», «пациент», «лечащий врач», «сессия». Возникает путаница: «приём» — это запись, видеозвонок или оплаченная услуга?

## ТРЕБОВАНИЯ
- Дайте однозначное определение минимум 8 терминам.
- Устраните выявленную неоднозначность (разведите «приём», «консультация», «запись», «слот»).
- Отметьте синонимы и запрещённые к смешению термины.
- Покажите связь глоссария с моделью данных (термины → будущие сущности).
- Объясните, зачем глоссарий в спецификации.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица глоссария: `Термин | Определение | Синонимы/не путать с`. Ниже — связь с будущими сущностями данных + зачем нужен глоссарий.

## КРИТЕРИИ ПРИЁМКИ
- [ ] ≥8 терминов с однозначными определениями.
- [ ] Разведены «приём», «консультация», «запись», «слот» (каждый — своё).
- [ ] Указаны синонимы и «не путать с».
- [ ] Показана связь терминов с сущностями данных (модуль 16).
- [ ] Объяснено: глоссарий = единый язык, предотвращает разное понимание (связь с ubiquitous language, модуль 24).
- [ ] Определения не противоречат друг другу.

## ПОДСКАЗКА
Глоссарий устраняет главный источник дефектов — разное понимание терминов. «Запись» — намерение прийти в слот; «консультация» — сам процесс общения; «приём» — возможно, оплаченная завершённая услуга. Определите каждый отдельно. Это основа единого языка (модуль 24) и модели данных (модуль 16).
