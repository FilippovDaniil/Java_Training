# Задача 10 (мини-проект) — Модуль 11: Спецификация (SRS, ТЗ ГОСТ)

## ЗАДАНИЕ
Подготовьте фрагмент ТЗ по ГОСТ 34.602 для FoodExpress: 3–4 содержательных раздела, согласованных между собой. Синтез теории модуля и Части 2.

## ВХОДНЫЕ ДАННЫЕ
FoodExpress — система доставки еды для запуска в одном городе. Используйте наработки Части 2: требования (модули 04, 09), приоритизацию (10), use cases/истории (07, 08). Заказчик требует ТЗ по ГОСТ.

## ТРЕБОВАНИЯ
Подготовьте фрагмент ТЗ, включающий минимум 4 раздела ГОСТ 34.602:
1. **Раздел 2 «Назначение и цели создания системы»** — назначение + измеримые цели.
2. **Раздел 4.1 «Требования к системе в целом»** — ключевые NFR (надёжность, безопасность, производительность) в формальном виде.
3. **Раздел 4.2 «Требования к функциям»** — основные функции (≥5), формализованно.
4. **Раздел 6 «Порядок контроля и приёмки»** — виды испытаний и критерии приёмки.
Плюс: глоссарий ключевых терминов и раздел «Ограничения и допущения».

## ФОРМАТ РЕЗУЛЬТАТА
Структурированный фрагмент ТЗ по разделам ГОСТ. Формальный стиль. Внутренне согласованный (функции из 4.2 проверяются критериями приёмки из р.6; NFR измеримы).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Минимум 4 раздела ГОСТ заполнены содержательно (не «рыба»).
- [ ] Цели измеримы; NFR в 4.1 измеримы (перцентили, доступность, безопасность/152-ФЗ).
- [ ] Функции в 4.2 атомарны, проверяемы; FR и NFR не перемешаны (4.1 vs 4.2).
- [ ] Раздел приёмки связывает критерии с измеримыми требованиями.
- [ ] Есть глоссарий и раздел ограничений/допущений.
- [ ] Документ внутренне непротиворечив; стиль формальный (ГОСТ).
- [ ] Учтена регуляторика (152-ФЗ, хранение ПДн в РФ).

## ПОДСКАЗКА
Это завершающий артефакт Части 2 — спецификация, собирающая все требования. Главные проверки middle+: разделы не перепутаны (NFR в 4.1, функции в 4.2), всё измеримо и проверяемо, приёмка опирается на критерии, регуляторика учтена. Собирайте из наработок модулей 04, 07–10 и задач 02, 05, 07 этого модуля.
