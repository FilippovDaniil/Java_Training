# Задача 01 — Модуль 12: BPMN 2.0

## ЗАДАНИЕ
Прочитайте готовую BPMN-схему (описанную текстом) и опишите процесс словами: что происходит, где развилки, что означает каждый элемент.

## ВХОДНЫЕ ДАННЫЕ
BPMN-схема процесса возврата товара (текстовое описание элементов):
- Start event (message): «получена заявка на возврат».
- User task: «оператор проверяет заявку».
- Exclusive gateway (XOR): «заявка обоснована?».
  - ветка «да» → Service task: «инициировать возврат средств» → Intermediate event (message): «уведомить клиента» → End event: «возврат завершён».
  - ветка «нет» → Service task: «отправить отказ с причиной» → End event: «возврат отклонён».
- На задаче «оператор проверяет заявку» висит boundary timer event (2 дня): если не обработано → Service task «эскалировать руководителю».

## ТРЕБОВАНИЯ
- Опишите процесс простым языком (как для заказчика).
- Объясните роль каждого типа элемента (start/message event, user/service task, XOR-шлюз, timer).
- Укажите, сколько у процесса возможных исходов (end events) и какие.
- Объясните, что делает граничное timer-событие.

## ФОРМАТ РЕЗУЛЬТАТА
Текстовое описание процесса + таблица `Элемент | Тип | Роль в процессе` + перечень исходов.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Процесс описан понятным языком.
- [ ] Объяснён каждый тип элемента (message start, user vs service task, XOR, timer).
- [ ] Указаны исходы: «возврат завершён», «возврат отклонён» (+ ветка эскалации).
- [ ] Объяснён граничный timer (эскалация при простое >2 дней).
- [ ] Различены user task (оператор) и service task (система).

## ПОДСКАЗКА
Чтение BPMN — базовый навык. Идите по потоку от старта. XOR — «или-или» по условию. Граничный timer на задаче ловит превышение срока, не прерывая остальное. Считайте исходы по end-событиям.
