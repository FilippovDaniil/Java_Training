# Задача 02 — Модуль 12: BPMN 2.0

## ЗАДАНИЕ
Смоделируйте линейный (без ветвлений) процесс оформления заказа FoodExpress в BPMN — базовый happy path.

## ВХОДНЫЕ ДАННЫЕ
Процесс (успешный сценарий): клиент собрал корзину → подтвердил адрес → выбрал оплату → система провела оплату → система создала заказ → система отправила заказ ресторану → клиент получил подтверждение.

## ТРЕБОВАНИЯ
- Постройте BPMN: один старт, последовательность задач, один конец.
- Для каждой задачи укажите тип (user task / service task).
- Используйте корректные типы событий (start, end; при необходимости message).
- Соблюдайте семантику BPMN (события — круги, задачи — скруглённые прямоугольники, поток — сплошная стрелка).

## ФОРМАТ РЕЗУЛЬТАТА
BPMN-схема (рисунок из bpmn.io/draw.io ИЛИ текстовая нотация: перечень элементов по порядку с типами и связями).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Есть ровно один start и один end event.
- [ ] Все шаги представлены задачами в правильной последовательности.
- [ ] У каждой задачи указан тип (user/service).
- [ ] «Провести оплату», «создать заказ», «отправить ресторану» — service task; «собрать корзину», «подтвердить адрес» — user task.
- [ ] Поток управления сплошной, элементы названы глагол+объект.
- [ ] Нет ветвлений (это happy path; ветки добавятся в задаче 03).

## ПОДСКАЗКА
Начните с простого линейного потока. User task — то, что делает человек через систему; service task — что система делает сама. Называйте задачи действием («Провести оплату»), а не существительным («Оплата»).
