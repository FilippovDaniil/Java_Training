# Задача 03 — Модуль 12: BPMN 2.0

## ЗАДАНИЕ
Добавьте к процессу оформления заказа шлюзы для ветвлений: успех/отказ оплаты, наличие товара, применение промокода.

## ВХОДНЫЕ ДАННЫЕ
Возьмите линейный процесс из задачи 02. Развилки:
- клиент может применить промокод (опционально) → пересчёт суммы;
- оплата может пройти успешно или быть отклонённой;
- при отклонении — предложить другой способ (до 2 попыток), затем отменить заказ;
- товар может закончиться при подтверждении рестораном.

## ТРЕБОВАНИЯ
- Добавьте корректные шлюзы (XOR для оплаты успех/отказ; что подойдёт для опционального промокода).
- Каждое ветвление корректно слейте (что развели — то свели).
- Смоделируйте повтор оплаты (до 2 попыток) — покажите цикл/возврат.
- Добавьте альтернативные концы (заказ оформлен / заказ отменён).
- Используйте правильные типы шлюзов и обоснуйте выбор.

## ФОРМАТ РЕЗУЛЬТАТА
BPMN-схема (рисунок или текстовая нотация) с шлюзами и ветками. Ниже — пояснение выбора типа каждого шлюза.

## КРИТЕРИИ ПРИЁМКИ
- [ ] XOR-шлюз для «оплата успешна?» с обеими ветками.
- [ ] Ветвления корректно сведены (нет висящих веток).
- [ ] Реализован повтор оплаты (до 2 попыток) с последующей отменой.
- [ ] Несколько корректных end-событий (оформлен / отменён).
- [ ] Тип каждого шлюза обоснован (XOR для взаимоисключающих исходов).
- [ ] Опциональный промокод смоделирован корректно (XOR с веткой «без промокода»).

## ПОДСКАЗКА
Оплата успех/отказ — классический XOR (ровно один исход). Повтор — это возврат потока к задаче оплаты с счётчиком попыток (или XOR «попытки исчерпаны?»). Каждый XOR-разрыв нужно свести. Каждая ветка должна закончиться end-событием.
