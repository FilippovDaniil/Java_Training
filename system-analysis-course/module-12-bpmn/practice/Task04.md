# Задача 04 — Модуль 12: BPMN 2.0

## ЗАДАНИЕ
Смоделируйте межролевое взаимодействие в процессе FoodExpress с использованием пулов и дорожек.

## ВХОДНЫЕ ДАННЫЕ
Участники процесса доставки: Клиент, Система FoodExpress (внутри — дорожки «Система» автоматическая и «Оператор поддержки»), Ресторан, Курьер. Сценарий: клиент оформляет заказ → система передаёт ресторану → ресторан подтверждает и готовит → система назначает курьера → курьер забирает и доставляет → клиент подтверждает получение.

## ТРЕБОВАНИЯ
- Разместите участников по пулам (Клиент, FoodExpress, Ресторан, Курьер).
- Внутри пула FoodExpress сделайте дорожки (Система, Оператор).
- Между пулами — потоки сообщений (пунктир); внутри пула — поток управления (сплошная).
- Покажите минимум 4 обмена сообщениями между пулами.
- Корректно разграничьте, что выполняет каждый участник.

## ФОРМАТ РЕЗУЛЬТАТА
BPMN-схема с пулами/дорожками (рисунок или текстовая нотация с указанием пулов, дорожек, типов потоков). Ниже — список межпуловых сообщений.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Все 4 участника в отдельных пулах.
- [ ] Внутри пула FoodExpress — дорожки Система и Оператор.
- [ ] Между пулами — только потоки сообщений (пунктир).
- [ ] Внутри пула — поток управления (сплошная).
- [ ] ≥4 межпуловых сообщения (заказ ресторану, подтверждение, назначение курьеру, статус клиенту).
- [ ] Действия корректно распределены по участникам (готовит ресторан, доставляет курьер).

## ПОДСКАЗКА
Главное правило: между пулами ходят ТОЛЬКО сообщения (пунктирные стрелки), потому что разные участники не управляют потоком друг друга напрямую. Внутри одного пула — сплошной поток управления. Дорожки делят пул на роли.
