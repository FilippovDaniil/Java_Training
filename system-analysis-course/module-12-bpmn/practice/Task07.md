# Задача 07 — Модуль 12: BPMN 2.0

## ЗАДАНИЕ
Декомпозируйте сложный процесс в подпроцессы: упростите перегруженную схему, выделив логические блоки.

## ВХОДНЫЕ ДАННЫЕ
Процесс FoodExpress «Обработка заказа» получился громоздким: проверка корзины, расчёт суммы, применение промокода, проверка минимальной суммы, выбор адреса, выбор оплаты, проверка карты, проведение оплаты, обработка ошибок оплаты, создание заказа, передача ресторану, ожидание подтверждения, назначение курьера, отслеживание, подтверждение доставки, запрос отзыва — всё на одной схеме (20+ элементов).

## ТРЕБОВАНИЯ
- Сгруппируйте шаги в 3–4 логических подпроцесса (например: «Оформление», «Оплата», «Исполнение и доставка»).
- Постройте верхнеуровневую схему из подпроцессов.
- Для одного подпроцесса покажите раскрытую (детальную) схему.
- Объясните, по какому принципу группировали и как выбрать уровень детализации.

## ФОРМАТ РЕЗУЛЬТАТА
Верхнеуровневая BPMN из подпроцессов + детальная схема одного подпроцесса + обоснование группировки.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Выделено 3–4 осмысленных подпроцесса.
- [ ] Верхнеуровневая схема читается за один взгляд.
- [ ] Один подпроцесс раскрыт детально.
- [ ] Группировка логична (по этапам/ответственности), а не случайна.
- [ ] Объяснён принцип выбора уровня детализации (под аудиторию).
- [ ] Подпроцессы связаны корректным потоком на верхнем уровне.

## ПОДСКАЗКА
Декомпозиция спасает читаемость: верхний уровень — «Оформление → Оплата → Доставка», каждый раскрывается отдельно. Группируйте по этапам или ответственности. Уровень детализации — под аудиторию: заказчику верхний уровень, разработке — детальный.
