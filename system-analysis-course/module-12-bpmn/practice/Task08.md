# Задача 08 — Модуль 12: BPMN 2.0

## ЗАДАНИЕ
Смоделируйте обмен сообщениями между пулами в коллаборативной диаграмме: клиент ↔ система ↔ курьер при доставке заказа.

## ВХОДНЫЕ ДАННЫЕ
Сценарий доставки FoodExpress: система назначает курьера → курьер принимает/отклоняет задание → если принял, забирает заказ в ресторане → отмечает «в пути» → клиент видит статус и геопозицию → курьер отмечает «доставлено» → клиент подтверждает получение. Если курьер отклонил — система ищет другого.

## ТРЕБОВАНИЯ
- Постройте коллаборацию из 3 пулов (Клиент, Система, Курьер).
- Покажите все потоки сообщений между пулами (пунктир) с содержанием.
- Внутри пула Курьер — поток управления с развилкой «принять/отклонить».
- Отразите цикл поиска другого курьера при отклонении.
- Покажите, как статус и геопозиция доходят до клиента (сообщения).

## ФОРМАТ РЕЗУЛЬТАТА
Коллаборативная BPMN (рисунок или текстовая нотация с пулами и сообщениями). Таблица сообщений: `От | Сообщение | Кому`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] 3 пула, между ними только сообщения (пунктир).
- [ ] Внутри пула Курьер — поток управления с XOR «принять/отклонить».
- [ ] Цикл поиска другого курьера при отклонении.
- [ ] Сообщения статуса/геопозиции идут от Системы к Клиенту.
- [ ] Подтверждение получения — сообщение от Клиента к Системе.
- [ ] Все сообщения содержательны (не просто «данные»).

## ПОДСКАЗКА
Коллаборация показывает «кто кому что шлёт» (хореография). Каждое взаимодействие между участниками — message flow. Развилка «принять/отклонить» — внутри пула курьера (его поток управления). Отклонение запускает повтор назначения (цикл в пуле системы).
