# Задача 04 — Модуль 13: UML поведенческие

## ЗАДАНИЕ
Сравните activity diagram и BPMN: для набора ситуаций выберите подходящую нотацию и обоснуйте. Постройте сравнительную таблицу.

## ВХОДНЫЕ ДАННЫЕ
Ситуации, для которых нужно выбрать нотацию (activity UML или BPMN):
1. Согласование договора между отделом продаж, юристом и финансами (межролевой процесс с передачей документов).
2. Алгоритм расчёта стоимости доставки внутри сервиса (логика с ветвлениями).
3. Процесс «как есть» обслуживания клиента в колл-центре для согласования с заказчиком-нетехнарём.
4. Внутренняя логика обработки входящего платёжного вебхука (шаги системы).
5. Процесс возврата товара с участием клиента, оператора, склада и платёжной системы, где важны таймауты и сообщения.

## ТРЕБОВАНИЯ
- Для каждой ситуации: выбор (activity / BPMN) + 1–2 предложения обоснования.
- Сравнительная таблица activity vs BPMN минимум по 5 критериям (назначение, аудитория, участники, события, стандарт).
- Сформулируйте правило выбора одной фразой.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Ситуация | Нотация | Обоснование` + сравнительная таблица activity vs BPMN + правило выбора.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Для всех 5 ситуаций выбрана нотация с обоснованием.
- [ ] Межролевые/бизнес-процессы с участниками и сообщениями → BPMN (ситуации 1, 3, 5).
- [ ] Алгоритм/логика системы → activity (ситуации 2, 4).
- [ ] Сравнительная таблица ≥ 5 критериев.
- [ ] Правило выбора сформулировано кратко.

## ПОДСКАЗКА
Ключ: BPMN — для бизнес-процессов с участниками (пулы), сообщениями и богатыми событиями (timer/message/error), понятных бизнесу. Activity — для алгоритма/логики, в т.ч. внутри системы. «Есть пулы и сообщения между сторонами?» → BPMN. «Это шаги одного алгоритма?» → activity.
