# Задача 05 — Модуль 13: UML поведенческие

## ЗАДАНИЕ
Постройте state machine diagram статуса видеоконсультации в MedConnect: жизненный цикл консультации от записи до завершения, с учётом реальных сбоев.

## ВХОДНЫЕ ДАННЫЕ
**MedConnect.** Консультация проходит статусы: Запланирована → Подтверждена → Идёт → Завершена. Реальные ситуации:
- пациент или врач может отменить до начала (с возвратом/без, по правилам);
- врач может перенести (новое время → снова Запланирована);
- пациент не явился в течение 15 минут от начала слота — «Не состоялась» (no-show);
- во время консультации видеосвязь оборвалась — попытка переподключения; если не восстановилась за 10 минут — «Прервана» с частичным возвратом;
- после завершения врач может выписать рецепт (это уже за пределами статуса консультации — отметить как действие при переходе/выходе).

## ТРЕБОВАНИЯ
- Состояния = статусы консультации.
- Переходы по событиям (`подтверждена`, `начало слота`, `обрыв связи`, `таймаут 10 мин`, `перенос`, …).
- Guard и действия где нужно (отмена с возвратом по правилам, частичный возврат при обрыве).
- Финальные исходы: Завершена, Отменена, Не состоялась, Прервана.
- Корректность: нет тупиков/недостижимости; есть путь к финалу.

## ФОРМАТ РЕЗУЛЬТАТА
State machine diagram (рисунок/PlantUML ИЛИ текстовая нотация переходов).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Состояния — статусы консультации; начальное и финальные состояния присутствуют.
- [ ] Все финальные исходы отражены (Завершена/Отменена/Не состоялась/Прервана).
- [ ] No-show смоделирован событием/таймером (15 мин).
- [ ] Обрыв связи: попытка переподключения + таймаут 10 мин → Прервана (частичный возврат).
- [ ] Перенос возвращает в Запланирована (с новым временем).
- [ ] Нет тупиковых/недостижимых состояний; из каждого статуса есть путь к финалу.

## ПОДСКАЗКА
MedConnect — домен с регуляторикой и реальными сбоями: моделируйте не только happy path. Таймауты (no-show 15 мин, обрыв 10 мин) — это события по времени. Эта state machine стыкуется с BPMN-процессом из модуля 12 (задача 10) и с sequence (модуль 33).
