# Задача 06 — Модуль 13: UML поведенческие

## ЗАДАНИЕ
Найдите дефекты в готовой («битой») диаграмме состояний: недостижимые и тупиковые состояния, неполные/неоднозначные переходы. Предложите исправления.

## ВХОДНЫЕ ДАННЫЕ
State machine заявки на возврат средств (текстовая нотация):
- Начальное → `Новая`
- `Новая` --отправлена на проверку--> `На проверке`
- `На проверке` --одобрена--> `Одобрена`
- `На проверке` --отклонена--> `Отклонена`
- `Одобрена` --деньги переведены--> `Возврат выполнен`
- `Отклонена` — (исходящих переходов нет, не финальное)
- `Заблокирована` — (входящих переходов нет; --разблокирована--> `На проверке`)
- `На проверке` --одобрена--> `Эскалация` (второй переход по тому же событию «одобрена», без guard)
- `Возврат выполнен` — финальное

## ТРЕБОВАНИЯ
- Найдите **недостижимое** состояние (нет входящих переходов).
- Найдите **тупиковое** состояние (нет выхода, не финальное).
- Найдите **неоднозначный** переход (два перехода по одному событию без различающих guard).
- Найдите **неполноту** (события, реакция на которые не описана: отмена заявителем, таймаут).
- Предложите исправление каждого дефекта.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Дефект | Где | Тип | Исправление` + при желании исправленная диаграмма.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Найдено недостижимое состояние (`Заблокирована` — нет входящих).
- [ ] Найдено тупиковое состояние (`Отклонена` — нет выхода, не финальное).
- [ ] Найден неоднозначный переход (`На проверке` --одобрена--> в два состояния без guard).
- [ ] Отмечена неполнота (нет реакции на отмену/таймаут).
- [ ] Для каждого дефекта предложено корректное исправление.

## ПОДСКАЗКА
Чек-лист корректности: в каждое состояние (кроме начального) есть вход; из каждого нефинального есть выход; на одно событие из состояния — один детерминированный переход (либо различающие guard); на все возможные события есть реакция; из любого состояния достижим финал. `Отклонена` нужно либо сделать финальным, либо добавить выход (например, «обжаловать»).
