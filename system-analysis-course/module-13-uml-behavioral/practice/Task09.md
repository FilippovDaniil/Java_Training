# Задача 09 — Модуль 13: UML поведенческие

## ЗАДАНИЕ
Преобразуйте словесное описание жизненного цикла сущности в диаграмму состояний. Выделите состояния, события, guard'ы из текста.

## ВХОДНЫЕ ДАННЫЕ
Текстовый регламент (MedConnect, рецепт): «После завершения консультации врач может оформить электронный рецепт. Черновик рецепта врач заполняет и проверяет; затем подписывает усиленной подписью. Подписанный рецепт отправляется в государственную систему рецептов. Если госсистема приняла — рецепт становится действующим, пациент видит его в кабинете. Если госсистема отклонила (ошибка данных) — рецепт возвращается врачу на исправление. Действующий рецепт можно погасить (когда лекарство выдано в аптеке) или он истекает по сроку действия. Врач может аннулировать рецепт до погашения, если назначение ошибочно».

## ТРЕБОВАНИЯ
- Извлеките состояния (статусы рецепта) из текста.
- Извлеките события переходов и guard'ы (принят/отклонён госсистемой, истёк срок, погашен, аннулирован).
- Постройте state machine с начальным и финальными состояниями.
- Отметьте, где переход условный (guard) и где есть действие при переходе.
- Проверьте на тупики/недостижимость.

## ФОРМАТ РЕЗУЛЬТАТА
Перечень «фраза текста → элемент диаграммы» + state machine diagram (рисунок/PlantUML/текстовая нотация).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Состояния извлечены из текста (Черновик, Подписан, Отправлен, Действующий, Погашен/Истёк/Аннулирован и т.п.).
- [ ] События переходов соответствуют тексту (подписал, принят/отклонён ГИС, погашен, истёк, аннулирован).
- [ ] Отклонение госсистемой возвращает рецепт на исправление (цикл).
- [ ] Финальные состояния выделены (Погашен/Истёк/Аннулирован).
- [ ] Нет тупиковых/недостижимых состояний.
- [ ] Показано соответствие «фраза → элемент».

## ПОДСКАЗКА
Маркеры в тексте: «становится / переходит в» → состояние; «если … то» → guard; «можно / врач может» → событие-переход; «возвращается на исправление» → переход обратно (цикл). Существительные-статусы — состояния, глаголы — события. Это домен с регуляторикой: подпись и отправка в ГИС — обязательные шаги.
