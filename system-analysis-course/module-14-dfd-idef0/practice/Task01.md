# Задача 01 — Модуль 14: DFD и IDEF0

## ЗАДАНИЕ
Постройте контекстную диаграмму потоков данных (DFD уровня 0) для системы FoodExpress: вся система как один процесс + внешние сущности + потоки данных между ними.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress.** Система взаимодействует с: клиентом (оформляет заказ, получает статусы/чек), рестораном (получает заказ, шлёт статус готовности), курьером (получает задание на доставку, шлёт статус доставки), платёжной системой (запрос оплаты, ответ об оплате), сервисом уведомлений (отправка SMS/push).

## ТРЕБОВАНИЯ
- Один процесс «FoodExpress» в центре (уровень 0).
- Все внешние сущности вокруг.
- Именованные потоки данных в обе стороны (что именно течёт).
- Только потоки данных (не управление, не последовательность).
- Нет потоков напрямую между внешними сущностями.

## ФОРМАТ РЕЗУЛЬТАТА
Контекстная DFD (рисунок из draw.io ИЛИ текстовая нотация: процесс, список внешних сущностей, список потоков вида `Источник → [данные] → Приёмник`).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Ровно один процесс (вся система) — уровень 0.
- [ ] Все 5 внешних сущностей присутствуют.
- [ ] Потоки именованы данными (не «информация», а «заказ», «статус оплаты»).
- [ ] Потоки идут в обе стороны там, где есть обмен.
- [ ] Нет потоков напрямую между внешними сущностями.
- [ ] Показаны границы системы (что внутри / снаружи).

## ПОДСКАЗКА
Контекстная DFD = граница системы: один «чёрный ящик» + окружение (как контекстная диаграмма из модуля 06). Поток должен называть конкретные данные. Внешние сущности между собой в DFD не соединяются — обмен идёт только через систему.
