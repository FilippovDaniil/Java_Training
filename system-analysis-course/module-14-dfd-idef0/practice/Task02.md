# Задача 02 — Модуль 14: DFD и IDEF0

## ЗАДАНИЕ
Декомпозируйте контекстную DFD FoodExpress (из задачи 01) в DFD уровня 1: разложите систему на основные процессы, добавьте хранилища данных, сохраните баланс потоков.

## ВХОДНЫЕ ДАННЫЕ
Контекстная DFD из задачи 01. Основные функции системы: приём и оформление заказа, обработка оплаты, передача заказа ресторану и отслеживание готовки, организация доставки, отправка уведомлений.

## ТРЕБОВАНИЯ
- Разложите единый процесс на 4–6 пронумерованных процессов (1, 2, 3…).
- Добавьте хранилища данных (например, D1 Заказы, D2 Клиенты, D3 Платежи).
- Сохраните **баланс**: внешние потоки уровня 0 присутствуют и на уровне 1.
- Покажите потоки между процессами и хранилищами.
- Каждый процесс имеет вход и выход.

## ФОРМАТ РЕЗУЛЬТАТА
DFD уровня 1 (рисунок/текстовая нотация: пронумерованные процессы, хранилища D1..Dn, потоки между процессами/хранилищами/внешними сущностями).

## КРИТЕРИИ ПРИЁМКИ
- [ ] 4–6 пронумерованных процессов.
- [ ] Введены хранилища данных (data store).
- [ ] Баланс соблюдён: входные/выходные потоки уровня 0 есть на уровне 1.
- [ ] Каждый процесс имеет хотя бы один вход и один выход (нет чёрных дыр/чудес).
- [ ] Нет потоков «хранилище → хранилище» напрямую.
- [ ] Потоки именованы данными.

## ПОДСКАЗКА
Декомпозиция = раскрыть «чёрный ящик» уровня 0. Правило баланса: что входило/выходило у всей системы, должно входить/выходить и у её процессов. Хранилища появляются только на уровне 1+. Между двумя хранилищами всегда стоит процесс.
