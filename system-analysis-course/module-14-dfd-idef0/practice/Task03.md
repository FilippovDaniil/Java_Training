# Задача 03 — Модуль 14: DFD и IDEF0

## ЗАДАНИЕ
Выделите хранилища данных и потоки для подсистемы обработки заказов: определите, какие данные где хранятся и как движутся между процессами.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, подсистема заказа. Известно: при оформлении сохраняются данные заказа и связь с клиентом; статусы заказа меняются по ходу; история платежей хранится отдельно; каталог ресторанов и меню читается при оформлении; назначения курьеров фиксируются.

## ТРЕБОВАНИЯ
- Перечислите хранилища данных (с кратким составом: что лежит).
- Для каждого хранилища укажите: какой процесс пишет (создаёт/обновляет), какой читает.
- Опишите потоки данных к/от хранилищ (с именами).
- Отметьте, какие хранилища — справочные (только чтение в этом процессе), какие — транзакционные.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Хранилище | Состав (основные данные) | Кто пишет | Кто читает | Тип (справочное/транзакционное)` + перечень потоков.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Выделены хранилища (Заказы, Клиенты, Платежи, Каталог/Меню, Назначения курьеров и т.п.).
- [ ] Для каждого указан состав основных данных.
- [ ] Указаны процессы-писатели и процессы-читатели.
- [ ] Потоки к/от хранилищ именованы.
- [ ] Разделены справочные и транзакционные хранилища.

## ПОДСКАЗКА
Хранилище в DFD — пассивно: само данные не двигает, всё через процессы. «Кто пишет/читает» — это будущие операции над сущностями (модуль 16). Справочник (меню, рестораны) обычно читается; транзакционные (заказы, платежи) — пишутся и читаются. Это прямой задел к ER-модели.
