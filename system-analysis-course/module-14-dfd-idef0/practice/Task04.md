# Задача 04 — Модуль 14: DFD и IDEF0

## ЗАДАНИЕ
Постройте IDEF0-диаграмму одной функции с корректными ICOM-стрелками: верно разнесите вход, управление, выход и механизм.

## ВХОДНЫЕ ДАННЫЕ
**MedConnect**, функция «Выписать электронный рецепт». В контексте функции участвуют: данные консультации и диагноз (что обрабатывается); клинические протоколы, формуляр лекарств, требования 152-ФЗ и регламент ГИС рецептов (правила); готовый подписанный рецепт (результат); врач, система MedConnect, государственная система рецептов (исполнители/средства).

## ТРЕБОВАНИЯ
- Нарисуйте функцию-прямоугольник с глаголом.
- Разнесите элементы по ICOM:
  - **I (слева)** — что преобразуется;
  - **C (сверху)** — правила/управление;
  - **O (справа)** — результат;
  - **M (снизу)** — кто/чем выполняется.
- Поясните, почему 152-ФЗ/протоколы — это Control, а не Input.

## ФОРМАТ РЕЗУЛЬТАТА
IDEF0-функция (рисунок/текстовая нотация) + таблица `Стрелка | ICOM | Обоснование`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Функция названа глаголом.
- [ ] Input (данные консультации, диагноз) — слева.
- [ ] Control (протоколы, формуляр, 152-ФЗ, регламент ГИС) — сверху.
- [ ] Output (подписанный рецепт) — справа.
- [ ] Mechanism (врач, MedConnect, ГИС) — снизу.
- [ ] Объяснено различие Input vs Control.

## ПОДСКАЗКА
Главная ловушка IDEF0: путать Input и Control. Input — то, что преобразуется в выход (данные на обработку). Control — правила, по которым идёт обработка, сами они не «расходуются» (регламент, НПА, протокол). Mechanism — ресурс-исполнитель.
