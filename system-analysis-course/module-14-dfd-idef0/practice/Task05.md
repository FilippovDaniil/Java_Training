# Задача 05 — Модуль 14: DFD и IDEF0

## ЗАДАНИЕ
Найдите ошибки в готовой («битой») DFD: чёрные дыры, чудеса, висящие потоки, недопустимые соединения. Предложите исправления.

## ВХОДНЫЕ ДАННЫЕ
DFD уровня 1 (текстовая нотация) подсистемы заказа:
- `[Клиент] ──заказ──► (1. Принять заказ)` — у процесса 1 нет ни одного выхода.
- `(2. Рассчитать стоимость) ──итог──► [Клиент]` — у процесса 2 нет входов.
- `|D1 Заказы| ──► |D2 Архив|` — поток напрямую между двумя хранилищами.
- `[Клиент] ──► [Ресторан]` — поток напрямую между двумя внешними сущностями.
- `(3. Уведомить) ◄──статус──` — поток без источника (из ниоткуда).
- `(4. Обработать оплату) ──команда «начать»──► (5. Готовить)` — поток управления (команда), а не данные.

## ТРЕБОВАНИЯ
- Найдите и классифицируйте каждую ошибку.
- Для каждой укажите тип (чёрная дыра / чудо / поток между хранилищами / поток между внешними / висящий поток / поток управления).
- Предложите корректное исправление.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Фрагмент | Тип ошибки | Почему неверно | Исправление`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Процесс 1 — чёрная дыра (есть вход, нет выхода).
- [ ] Процесс 2 — чудо (есть выход, нет входа).
- [ ] D1→D2 — недопустимый поток между хранилищами (нужен процесс).
- [ ] Клиент→Ресторан — недопустимый поток между внешними сущностями.
- [ ] Процесс 3 — висящий поток (нет источника).
- [ ] Поток «команда начать» — это управление, а не данные.
- [ ] Для каждой ошибки предложено исправление.

## ПОДСКАЗКА
Чек-лист DFD: у каждого процесса есть и вход, и выход; потоки только данные (не команды); хранилище↔хранилище и внешняя↔внешняя напрямую запрещены (между ними процесс / вне системы); у каждого потока есть источник и приёмник.
