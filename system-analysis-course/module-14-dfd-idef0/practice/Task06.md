# Задача 06 — Модуль 14: DFD и IDEF0

## ЗАДАНИЕ
Для набора ситуаций выберите подходящую нотацию (DFD / IDEF0 / BPMN) и обоснуйте. Постройте сравнительную таблицу трёх нотаций.

## ВХОДНЫЕ ДАННЫЕ
Ситуации:
1. Показать, как данные о заказе движутся через систему и где хранятся (подготовка к проектированию БД).
2. Описать бизнес-процесс согласования заявки с участием 3 отделов и таймаутами — для согласования с заказчиком.
3. Строго описать функцию госуслуги с указанием регламентов (Control) и исполнителей (Mechanism) для ГОСТ-документации.
4. Понять, какие хранилища данных нужны системе, по потокам информации.
5. Смоделировать последовательность шагов оформления заказа во времени с ветвлениями.

## ТРЕБОВАНИЯ
- Для каждой ситуации: выбор нотации + обоснование (1–2 предложения).
- Сравнительная таблица DFD vs IDEF0 vs BPMN минимум по 5 критериям (фокус, узлы, время/порядок, хранилища, управление/правила).
- Сформулируйте правило выбора.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Ситуация | Нотация | Обоснование` + сравнительная таблица + правило выбора.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Для всех 5 ситуаций выбор с обоснованием.
- [ ] Движение/хранение данных (1, 4) → DFD.
- [ ] Функция с регламентами и исполнителями (3) → IDEF0.
- [ ] Процесс во времени с участниками/ветвлениями (2, 5) → BPMN.
- [ ] Сравнительная таблица ≥ 5 критериев.
- [ ] Правило выбора сформулировано.

## ПОДСКАЗКА
Ключ: DFD — про данные (потоки + хранилища, мост к БД); IDEF0 — про функции с управлением (Control) и ресурсами (Mechanism); BPMN — про процесс во времени с участниками. «Где хранятся данные?» → DFD; «по каким правилам и кем выполняется функция?» → IDEF0; «в каком порядке шаги?» → BPMN.
