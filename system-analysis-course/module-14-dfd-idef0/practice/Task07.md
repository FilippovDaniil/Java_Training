# Задача 07 — Модуль 14: DFD и IDEF0

## ЗАДАНИЕ
Выполните декомпозицию IDEF0: разложите контекстную функцию A-0 на диаграмму A0 с подфункциями A1..A3, сохранив баланс стрелок ICOM.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, контекстная функция **A-0 «Обработать заказ»**:
- Input: заявка клиента;
- Control: тарифы, правила скидок, регламент доставки;
- Output: выполненный заказ (доставлен), чек;
- Mechanism: система FoodExpress, ресторан, курьер.

Подфункции для A0: A1 «Оформить и оплатить», A2 «Приготовить», A3 «Доставить».

## ТРЕБОВАНИЯ
- Разложите A-0 на A1, A2, A3 (3–6 функций).
- Проведите стрелки ICOM между подфункциями (выход одной = вход следующей и т.д.).
- Сохраните **баланс**: внешние стрелки A-0 (I/C/O/M) присутствуют на диаграмме A0.
- Покажите, как Control родителя распределяется по детям (тарифы → A1, регламент доставки → A3).

## ФОРМАТ РЕЗУЛЬТАТА
Диаграмма A0 (рисунок/текстовая нотация: функции A1..A3 + стрелки ICOM между ними и наружу).

## КРИТЕРИИ ПРИЁМКИ
- [ ] A0 содержит 3 подфункции (A1, A2, A3), глаголы.
- [ ] Стрелки ICOM связывают подфункции (output→input по цепочке).
- [ ] Баланс соблюдён: внешние I/C/O/M из A-0 есть на A0.
- [ ] Control распределён по подфункциям осмысленно (тарифы→A1, регламент доставки→A3).
- [ ] Mechanism подфункций соответствует (ресторан→A2, курьер→A3).

## ПОДСКАЗКА
Декомпозиция IDEF0 = раскрыть одну функцию в 3–6 подфункций с сохранением граничных стрелок (баланс, как в DFD). Выход A1 (оплаченный заказ) становится входом A2; выход A2 (готовый заказ) — входом A3. Control и Mechanism родителя «спускаются» к подходящим детям.
