# Задача 08 — Модуль 14: DFD и IDEF0

## ЗАДАНИЕ
Свяжите хранилища и потоки данных DFD с будущими сущностями БД: покажите, как DFD служит мостом к ER-модели (модуль 16).

## ВХОДНЫЕ ДАННЫЕ
DFD уровня 1 FoodExpress (из задач 02–03). Хранилища: D1 Заказы, D2 Клиенты, D3 Платежи, D4 Каталог (рестораны+меню), D5 Назначения курьеров. Потоки: «данные заказа» (содержит позиции, адрес, сумму), «данные клиента», «данные платежа», «позиция меню».

## ТРЕБОВАНИЯ
- Сопоставьте каждое хранилище с будущей сущностью (или несколькими).
- Из состава потоков выведите предполагаемые атрибуты сущностей.
- Наметьте связи между сущностями (по тому, как процессы соединяют хранилища).
- Отметьте, где из одного хранилища, вероятно, выделятся несколько сущностей (например, Заказ и Позиция заказа).

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Хранилище DFD | Сущность(и) | Предполагаемые атрибуты | Связи` + краткое пояснение перехода DFD→ER.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Каждое хранилище сопоставлено с сущностью(ями).
- [ ] Атрибуты выведены из состава потоков данных.
- [ ] Намечены связи между сущностями (Заказ–Клиент, Заказ–Позиция, Заказ–Платёж).
- [ ] Показано выделение нескольких сущностей из одного хранилища (Заказ / Позиция заказа).
- [ ] Объяснён принцип перехода DFD → ER.

## ПОДСКАЗКА
DFD — естественный мост к модели данных: хранилище → сущность/таблица, состав потока → атрибуты, процесс, соединяющий хранилища → связь. «Данные заказа» с позициями → две сущности (Заказ 1:N Позиция). Это вход в модуль 16 (ER).
