# Задача 09 — Модуль 14: DFD и IDEF0

## ЗАДАНИЕ
Постройте DFD из словесного описания процесса: извлеките процессы, потоки, хранилища и внешние сущности из текста.

## ВХОДНЫЕ ДАННЫЕ
Текст (MedConnect, запись на приём): «Пациент через приложение выбирает специализацию и врача, система показывает доступные слоты из расписания врачей. Пациент бронирует слот — система создаёт запись и резервирует слот. Для подтверждения пациент оплачивает приём через платёжный провайдер; после успешной оплаты система фиксирует оплату и отправляет пациенту и врачу уведомление о записи. Данные записей и оплат хранятся в системе; расписание врачей ведётся отдельно».

## ТРЕБОВАНИЯ
- Выделите внешние сущности (Пациент, Врач, Платёжный провайдер).
- Выделите процессы (показать слоты, забронировать, оплатить, уведомить).
- Выделите хранилища (Записи, Оплаты, Расписание врачей).
- Проведите именованные потоки данных.
- Проверьте на корректность (нет чёрных дыр/чудес/недопустимых соединений).

## ФОРМАТ РЕЗУЛЬТАТА
Перечень «фраза текста → элемент DFD» + DFD уровня 1 (рисунок/текстовая нотация).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Внешние сущности извлечены из текста.
- [ ] Процессы — преобразования данных (глаголы), пронумерованы.
- [ ] Хранилища выделены (Записи, Оплаты, Расписание).
- [ ] Потоки именованы данными и имеют источник/приёмник.
- [ ] Нет чёрных дыр/чудес и недопустимых соединений.
- [ ] Показано соответствие «фраза → элемент».

## ПОДСКАЗКА
Маркеры в тексте: существительные-данные («слоты», «запись», «оплата») → потоки/хранилища; «система показывает/создаёт/фиксирует» → процессы; «хранятся/ведётся» → хранилища; внешние участники («пациент», «провайдер») → внешние сущности. Сначала границы (контекст), потом процессы и хранилища.
