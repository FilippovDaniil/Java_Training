# Задача 10 (мини-проект) — Модуль 14: DFD и IDEF0

## ЗАДАНИЕ
Постройте полную DFD-модель ключевой подсистемы MedConnect на двух уровнях: контекстную диаграмму (уровень 0) и декомпозицию (уровень 1) с хранилищами, балансом и корректностью. Свяжите хранилища с будущими сущностями данных. Синтез теории модуля.

## ВХОДНЫЕ ДАННЫЕ
**MedConnect**, подсистема «Запись и проведение консультации». Окружение: Пациент, Врач, Платёжный провайдер, Провайдер видеосвязи, Государственная система рецептов, Сервис уведомлений. Функции: запись на слот, оплата, проведение видеоконсультации, выписка рецепта, уведомления. Данные хранятся: записи/консультации, оплаты, расписание врачей, рецепты, профили пользователей.

## ТРЕБОВАНИЯ
Сдайте пакет:
1. **Контекстная DFD (уровень 0)** — система как один процесс + все внешние сущности + именованные потоки; границы системы.
2. **DFD уровня 1** — 5–6 пронумерованных процессов + хранилища (D1..Dn) + потоки между процессами/хранилищами/внешними; баланс с уровнем 0.
3. **Корректность** — нет чёрных дыр/чудес, потоков «хранилище↔хранилище» и «внешняя↔внешняя», висящих потоков; потоки = данные.
4. **Мост к данным** — таблица «хранилище → будущая сущность + основные атрибуты» (вход в модуль 16).

## ФОРМАТ РЕЗУЛЬТАТА
Две DFD (рисунок/текстовая нотация) + таблица соответствия хранилищ сущностям + краткое сопроводительное описание.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Контекстная DFD: один процесс, все внешние сущности, именованные потоки, границы.
- [ ] DFD уровня 1: 5–6 процессов + хранилища, потоки между ними.
- [ ] Баланс между уровнями 0 и 1 соблюдён.
- [ ] Учтены реальные внешние системы (видеосвязь, ГИС рецептов, платежи).
- [ ] Нет ошибок корректности (чёрные дыры/чудеса/недопустимые соединения/висящие потоки).
- [ ] Таблица «хранилище → сущность + атрибуты» построена.
- [ ] Регуляторика отражена (например, хранение рецептов/медданных).

## ПОДСКАЗКА
Синтез: контекст из задачи 01, декомпозиция и хранилища из 02–03, корректность из 05, мост к данным из 08, извлечение из текста из 09. Главные проверки middle+: правильные границы системы, баланс уровней, чистая семантика DFD (данные, не управление) и осознанный переход к ER. Эта DFD стыкуется с BPMN-процессом MedConnect (модуль 12) и станет основой ER-модели (модуль 16).
