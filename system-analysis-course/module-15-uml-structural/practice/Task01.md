# Задача 01 — Модуль 15: UML структурные

## ЗАДАНИЕ
Выделите сущности (классы) и их атрибуты предметной области FoodExpress: постройте первичный список классов концептуальной модели.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress.** Клиенты оформляют заказы в ресторанах. Заказ состоит из позиций (блюдо + количество). У ресторана есть меню из блюд. Заказ доставляет курьер. У клиента может быть несколько адресов доставки. Заказ имеет статус и сумму. Оплата фиксируется по заказу.

## ТРЕБОВАНИЯ
- Выделите классы-существительные домена (Клиент, Заказ, Ресторан, Блюдо, Позиция заказа, Курьер, Адрес, Оплата…).
- Для каждого класса перечислите 3–6 ключевых атрибутов (без типов БД, без id-как-FK).
- Не вносите операции (концептуальный уровень).
- Отметьте кандидатов, которые на самом деле не сущности (а атрибуты/значения).

## ФОРМАТ РЕЗУЛЬТАТА
Список классов с атрибутами + краткое пояснение, почему отброшены «ложные» сущности.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Выделены основные сущности домена (≥ 6 классов).
- [ ] Классы — существительные, не действия.
- [ ] У каждого класса 3–6 осмысленных атрибутов.
- [ ] Атрибуты без типов БД и без внешних ключей (концептуальный уровень).
- [ ] «Позиция заказа» выделена как отдельная сущность (не атрибут заказа).
- [ ] Отброшены ложные сущности с обоснованием.

## ПОДСКАЗКА
Концептуальная модель — словарь домена. Существительные текста → кандидаты в классы; их свойства → атрибуты. «Список позиций» — не атрибут, а связь с сущностью «Позиция заказа». Не добавляйте типы столбцов и FK — это уровень ER (модуль 16).
