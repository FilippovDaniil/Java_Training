# Задача 02 — Модуль 15: UML структурные

## ЗАДАНИЕ
Определите связи и кратности между классами FoodExpress (из задачи 01): постройте class diagram с ассоциациями и multiplicity.

## ВХОДНЫЕ ДАННЫЕ
Классы из задачи 01. Правила домена: у клиента 0+ заказов, у заказа ровно 1 клиент; заказ состоит из 1+ позиций; позиция ссылается на 1 блюдо; ресторан имеет 1+ блюд; заказ доставляет 0..1 курьер (пока не назначен — ноль); у клиента 1+ адресов; у заказа ровно 1 оплата (после оформления).

## ТРЕБОВАНИЯ
- Проведите ассоциации между классами.
- Проставьте кратности на обоих концах каждой связи.
- Подпишите роли/смысл связи там, где неочевидно.
- Проверьте, что кратности стоят на правильных сторонах.

## ФОРМАТ РЕЗУЛЬТАТА
Class diagram (рисунок/PlantUML/текстовая нотация связей вида `Клиент 1 -- 0..* Заказ`).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Все основные связи проведены.
- [ ] Кратности проставлены на обоих концах.
- [ ] Клиент 1 — 0..* Заказ (а не наоборот).
- [ ] Заказ 1 — 1..* Позиция.
- [ ] Курьер 0..1 — Заказ (необязательность назначения отражена).
- [ ] Кратности на корректных сторонах (проверено чтением «с другого конца»).

## ПОДСКАЗКА
Кратность читается с противоположного конца: число у «Заказ» = «сколько заказов у одного клиента». Необязательность (курьер ещё не назначен) → `0..1`. «Минимум один» → `1..*`. Сверяйте каждую связь фразой «один X связан со сколькими Y».
