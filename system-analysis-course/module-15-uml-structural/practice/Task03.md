# Задача 03 — Модуль 15: UML структурные

## ЗАДАНИЕ
Различите агрегацию и композицию на примерах домена: для каждой пары «часть-целое» определите тип и обоснуйте через жизненный цикл части.

## ВХОДНЫЕ ДАННЫЕ
Пары «целое — часть» (FoodExpress / общие):
1. Заказ — Позиция заказа.
2. Ресторан — Блюдо (меню).
3. Корзина — Товар в корзине.
4. Клиент — Адрес доставки.
5. Команда курьеров — Курьер.
6. Чек — Строка чека.

## ТРЕБОВАНИЯ
- Для каждой пары определите: агрегация или композиция.
- Обоснуйте через тест «живёт ли часть без целого / принадлежит ли только одному целому».
- Покажите нотацию (пустой/закрашенный ромб) и кратности.
- Отметьте спорные случаи и как бы вы их разрешили.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Целое — Часть | Тип | Обоснование (жизненный цикл) | Кратности`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Заказ — Позиция: композиция (позиция не живёт без заказа).
- [ ] Ресторан — Блюдо: ближе к агрегации/ассоциации (блюдо как понятие может существовать/переиспользоваться).
- [ ] Команда — Курьер: агрегация (курьер остаётся при роспуске команды).
- [ ] Для каждой пары приведено обоснование через жизненный цикл.
- [ ] Указаны нотация (ромб) и кратности.
- [ ] Спорные случаи отмечены с разрешением.

## ПОДСКАЗКА
Тест: удалили целое — часть имеет смысл сама? Да → агрегация (пустой ромб). Нет → композиция (закрашенный ромб). Композиция: часть принадлежит ровно одному целому и удаляется с ним (Заказ→Позиция, Чек→Строка). Агрегация: часть переживает целое (Команда→Курьер).
