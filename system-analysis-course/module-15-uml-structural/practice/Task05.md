# Задача 05 — Модуль 15: UML структурные

## ЗАДАНИЕ
Постройте object diagram (диаграмму объектов) — снимок системы для конкретного сценария. Проверьте по нему корректность кратностей class diagram.

## ВХОДНЫЕ ДАННЫЕ
Class diagram FoodExpress (из задач 01–02). Сценарий-снимок: клиент Иванов оформил заказ №1024 в ресторане «Суши-бар» на 2 позиции (ролл «Филадельфия» ×2 и суп мисо ×1), сумма 1500 ₽, статус «Оплачен», доставка на адрес «ул. Ленина, 5», курьер пока не назначен, оплата картой проведена.

## ТРЕБОВАНИЯ
- Изобразите экземпляры классов с конкретными значениями атрибутов (`заказ_1024 : Заказ`).
- Проведите связи между конкретными объектами.
- Проверьте, что снимок не нарушает кратности модели (например, заказ имеет ≥1 позицию, ровно 1 клиента).
- Отметьте, как «курьер не назначен» отражается (0..1 → нет связи).

## ФОРМАТ РЕЗУЛЬТАТА
Object diagram (рисунок/текстовая нотация экземпляров и связей) + проверка соответствия кратностям.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Экземпляры классов с конкретными значениями (объект : Класс).
- [ ] Связи между конкретными объектами проведены.
- [ ] Заказ связан ровно с 1 клиентом и ≥1 позицией.
- [ ] Отсутствие курьера корректно отражает кратность 0..1.
- [ ] Снимок не нарушает ни одной кратности class diagram.
- [ ] Показана роль object diagram (проверка модели на примере).

## ПОДСКАЗКА
Object diagram — снимок в момент времени: конкретные экземпляры вместо типов. Это способ проверить class diagram: если реальный сценарий не укладывается в кратности — модель неверна. «Курьер не назначен» при `0..1` — просто отсутствие связи, и это допустимо.
