# Задача 09 — Модуль 15: UML структурные

## ЗАДАНИЕ
Выведите классы предметной области из use case: на основе сценария use case определите сущности, атрибуты и связи концептуальной модели.

## ВХОДНЫЕ ДАННЫЕ
Use case (MedConnect) «Записаться на консультацию», основной поток: «Пациент выбирает специализацию → система показывает врачей и их свободные слоты → пациент выбирает врача и слот → система создаёт запись → пациент оплачивает → система подтверждает запись и резервирует слот → пациент и врач получают уведомление». Сущности проявляются в существительных сценария.

## ТРЕБОВАНИЯ
- Извлеките сущности из существительных use case (Пациент, Врач, Специализация, Слот, Запись, Оплата, Уведомление).
- Определите атрибуты каждой сущности.
- Проведите связи и кратности.
- Покажите соответствие «фраза use case → элемент модели».
- Отметьте сущности, которые проявляются неявно (например, Расписание/Слот).

## ФОРМАТ РЕЗУЛЬТАТА
Перечень «фраза → сущность/связь» + concept class diagram (рисунок/текст) с кратностями.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Сущности извлечены из существительных use case.
- [ ] У каждой сущности есть атрибуты.
- [ ] Связи и кратности проставлены (Врач 1—0..* Слот, Запись—Пациент, Запись—Слот и т.п.).
- [ ] Показано соответствие «фраза → элемент».
- [ ] Выделены неявные сущности (Слот/Расписание).
- [ ] Модель концептуальная (без типов БД/FK).

## ПОДСКАЗКА
Существительные сценария → сущности; глаголы → связи/операции; «система создаёт запись» → сущность «Запись» с владельцами. Use case (модуль 07) — естественный источник концептуальной модели. Неявные сущности (Слот, Расписание) часто прячутся за «свободными слотами».
