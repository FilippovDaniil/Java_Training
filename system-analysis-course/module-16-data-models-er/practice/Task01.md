# Задача 01 — Модуль 16: Модели данных и ER

## ЗАДАНИЕ
Постройте концептуальную ER-модель предметной области FoodExpress: сущности, атрибуты и связи без ключей и типов БД.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress.** Клиенты делают заказы в ресторанах. Заказ состоит из позиций (блюдо + количество). Ресторан имеет меню из блюд по категориям. Заказ доставляет курьер на адрес клиента. По заказу проходит оплата. Клиент оставляет отзыв о заказе.

## ТРЕБОВАНИЯ
- Выделите сущности (Клиент, Заказ, Позиция, Блюдо, Ресторан, Категория, Курьер, Адрес, Оплата, Отзыв).
- Для каждой — основные атрибуты (по смыслу, без типов БД).
- Проведите связи с кратностями (1:1 / 1:N / M:N).
- Не вводите пока PK/FK как столбцы (концептуальный уровень).

## ФОРМАТ РЕЗУЛЬТАТА
Концептуальная ER (рисунок/текстовая нотация: сущности + атрибуты + связи с кратностями).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Выделены основные сущности домена (≥ 8).
- [ ] У каждой сущности есть атрибуты по смыслу.
- [ ] Связи проведены с кратностями.
- [ ] M:N-связи отмечены (например, Заказ—Блюдо через позиции).
- [ ] Уровень концептуальный (без типов БД и FK-столбцов).
- [ ] Кратности на верных сторонах.

## ПОДСКАЗКА
Концептуальная ER близка к концептуальной class diagram (модуль 15). Существительные → сущности, свойства → атрибуты, связи → отношения с кратностями. M:N пока просто отметьте (разрешать будете в задаче 03). Ключи и типы — позже (логический/физический уровень).
