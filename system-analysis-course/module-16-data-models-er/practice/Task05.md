# Задача 05 — Модуль 16: Модели данных и ER

## ЗАДАНИЕ
Преобразуйте концептуальную ER-модель в логическую: добавьте ключи, разрешите M:N, расставьте FK, наметьте обязательность атрибутов.

## ВХОДНЫЕ ДАННЫЕ
Концептуальная ER FoodExpress (из задач 01–03). Перевести в логическую модель по алгоритму: сущности → таблицы, атрибуты → столбцы, PK, связи → FK/промежуточные таблицы.

## ТРЕБОВАНИЯ
- Каждую сущность представьте как таблицу с PK.
- Связи 1:N → FK на стороне «многие»; M:N → промежуточные таблицы.
- Отметьте обязательность (NOT NULL) и уникальность (UNIQUE) ключевых атрибутов.
- Производные атрибуты (сумма заказа) — решите: хранить или вычислять, обоснуйте.
- Многозначные/составные атрибуты — вынесите при необходимости.

## ФОРМАТ РЕЗУЛЬТАТА
Логическая модель: список таблиц вида `Таблица(колонка PK, колонка, FK→…)` + пометки NOT NULL/UNIQUE + решение по производным атрибутам.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Все сущности → таблицы с PK.
- [ ] FK расставлены корректно (1:N — на «многие»).
- [ ] M:N разрешены промежуточными таблицами.
- [ ] Отмечены NOT NULL / UNIQUE для ключевых атрибутов (email UNIQUE).
- [ ] Принято и обосновано решение по производным атрибутам (сумма).
- [ ] Многозначные/составные атрибуты обработаны (телефоны/адрес).

## ПОДСКАЗКА
Логический уровень = структура без привязки к СУБД: PK/FK, обязательность, уникальность, нормализация (модуль 17 углубит). Производная сумма: хранить ради скорости отчётов (с риском рассинхрона) ИЛИ вычислять (всегда верно, но дороже) — обоснуйте под контекст. Типы столбцов и индексы — это уже физический уровень (задача 08).
