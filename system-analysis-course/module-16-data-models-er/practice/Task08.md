# Задача 08 — Модуль 16: Модели данных и ER

## ЗАДАНИЕ
Преобразуйте логическую модель в физическую (наброском): подберите типы данных столбцов и наметьте индексы под основные запросы.

## ВХОДНЫЕ ДАННЫЕ
Логическая модель FoodExpress (из задачи 05). СУБД — PostgreSQL. Основные запросы: заказы клиента по дате; заказы ресторана за период; поиск блюд по названию; статус оплаты по заказу.

## ТРЕБОВАНИЯ
- Для ключевых столбцов подберите типы (id, суммы, даты, статусы, текст).
- Наметьте индексы под перечисленные запросы (FK-столбцы, столбцы фильтрации/сортировки).
- Отметьте, что суммы — не float (деньги), статусы — enum/справочник.
- Кратко: какие индексы дадут эффект, а какие избыточны.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Таблица.столбец | Тип PostgreSQL | Обоснование` + список индексов `Индекс | Под какой запрос`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Типы подобраны (bigint/uuid для id, numeric для денег, timestamptz для дат, varchar/text, enum/ref для статусов).
- [ ] Деньги — numeric/decimal, НЕ float (обоснование точности).
- [ ] Индексы намечены под основные запросы (client_id+дата, restaurant_id+дата, название блюда).
- [ ] FK-столбцы проиндексированы.
- [ ] Отмечены избыточные/нежелательные индексы (на часто меняющихся/низкоселективных).
- [ ] Статусы — enum или справочная таблица, не свободный текст.

## ПОДСКАЗКА
Физический уровень = реализация в конкретной СУБД: типы, индексы, ограничения. Деньги — всегда numeric/decimal (float теряет копейки). Индекс ускоряет чтение по столбцу фильтра/сортировки, но замедляет запись — не индексируйте всё подряд. Подробно индексы/перформанс — модуль 20.
