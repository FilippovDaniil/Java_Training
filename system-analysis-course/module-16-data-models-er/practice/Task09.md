# Задача 09 — Модуль 16: Модели данных и ER

## ЗАДАНИЕ
Постройте ER-модель из требований: извлеките сущности, атрибуты и связи из текстовых требований MedConnect.

## ВХОДНЫЕ ДАННЫЕ
Требования (MedConnect): «Пациент имеет профиль (ФИО, дата рождения, полис, контакты). Врач имеет специализацию (одну или несколько) и ведёт расписание из слотов. Пациент записывается на слот — создаётся запись на консультацию. По консультации может быть выписан рецепт с препаратами (несколько позиций). Консультация привязана к врачу и пациенту. Оплата фиксируется по записи. Все медицинские данные подпадают под 152-ФЗ (хранение и доступ ограничены)».

## ТРЕБОВАНИЯ
- Извлеките сущности (Пациент, Врач, Специализация, Слот, Запись/Консультация, Рецепт, Позиция рецепта, Оплата).
- Определите атрибуты и PK.
- Проведите связи и кратности; разрешите M:N (Врач—Специализация).
- Покажите соответствие «фраза требования → элемент ER».
- Отметьте, какие сущности содержат перс./медданные (152-ФЗ) — для будущих требований к доступу.

## ФОРМАТ РЕЗУЛЬТАТА
Перечень «фраза → элемент» + ER-модель (рисунок/текстовая нотация с PK/FK и кратностями).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Сущности извлечены из требований.
- [ ] Атрибуты и PK определены.
- [ ] Врач—Специализация (M:N) разрешена промежуточной сущностью.
- [ ] Рецепт 1—N Позиция рецепта смоделирована.
- [ ] Показано соответствие «фраза → элемент».
- [ ] Отмечены сущности с перс./медданными (152-ФЗ).

## ПОДСКАЗКА
Существительные требований → сущности; «имеет/ведёт/привязана» → связи; «одну или несколько» → кратность (тут M:N для специализаций). 152-ФЗ важен: пометьте сущности с медданными — это вход для модулей 09 (NFR безопасности) и 34 (доступ). «Несколько позиций» → отдельная сущность 1:N.
