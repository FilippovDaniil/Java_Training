# Задача 07 — Модуль 17: Нормализация

## ЗАДАНИЕ
Составьте словарь данных (data dictionary) для ключевых сущностей FoodExpress: для каждого атрибута — тип, обязательность, ограничения, смысл, источник.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, сущности: Заказ (order_id, дата, статус, сумма, способ_оплаты), Клиент (client_id, email, телефон, дата_регистрации), ПозицияЗаказа (order_id, dish_id, количество, цена_на_момент). Статусы заказа: Создан, Оплачен, Принят, Готовится, В доставке, Доставлен, Отменён.

## ТРЕБОВАНИЯ
- Для каждого атрибута заполните словарь: имя, тип, обязательность, ограничения/допустимые значения, описание/смысл, источник/правило.
- Для статуса — перечислите все допустимые значения.
- Для денег — укажите единицы и точность.
- Для дат — укажите таймзону/формат.
- Отметьте производные атрибуты (сумма) и их формулу.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица словаря данных `Атрибут | Тип | Обязат. | Ограничения/значения | Описание | Источник`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Все перечисленные атрибуты внесены в словарь.
- [ ] Статус — перечислены все допустимые значения (enum/справочник).
- [ ] Деньги — единицы (₽/копейки) и точность (numeric(10,2)).
- [ ] Даты — формат/таймзона.
- [ ] email/телефон — формат и уникальность.
- [ ] Производные атрибуты (сумма) помечены с формулой/источником.

## ПОДСКАЗКА
Словарь данных снимает разночтения. Самые «опасные» атрибуты: статусы (перечислить ВСЕ значения), деньги (единицы + точность, не float), даты (таймзона), коды/справочники. Производные (сумма) — указать формулу. Это артефакт middle+, дополняет глоссарий (модуль 11).
