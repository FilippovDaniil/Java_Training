# Задача 09 — Модуль 17: Нормализация

## ЗАДАНИЕ
Нормализуйте модель из плоского экспорта: дан выгруженный «широкий» CSV — приведите его к 3NF с корректными таблицами, ключами и связями.

## ВХОДНЫЕ ДАННЫЕ
Плоский экспорт (MedConnect, одна строка = одна позиция рецепта):

```
рецепт_id; дата; пациент_id; пациент_фио; пациент_полис; врач_id; врач_фио;
диагноз_код; диагноз_название; препарат_код; препарат_название; дозировка; количество
```

В одном рецепте несколько препаратов (несколько строк с одинаковым `рецепт_id`).

## ТРЕБОВАНИЯ
- Выпишите ФЗ.
- Приведите к 3NF: выделите таблицы (Рецепт, Пациент, Врач, Диагноз, Препарат, ПозицияРецепта).
- Расставьте PK/FK и кратности.
- Учтите 152-ФЗ: отметьте таблицы с медданными.
- Покажите, какие нарушения НФ были в плоском экспорте.

## ФОРМАТ РЕЗУЛЬТАТА
Список ФЗ + нормализованные таблицы (PK/FK) + перечень устранённых нарушений + пометка медданных.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Выписаны ФЗ (пациент_id→ФИО/полис; врач_id→ФИО; диагноз_код→название; препарат_код→название).
- [ ] Выделены таблицы: Рецепт, Пациент, Врач, Диагноз, Препарат, ПозицияРецепта.
- [ ] ПозицияРецепта хранит `(рецепт_id, препарат_код) → дозировка, количество`.
- [ ] Справочники (Диагноз, Препарат) вынесены.
- [ ] PK/FK и кратности расставлены.
- [ ] Отмечены медданные (152-ФЗ): Пациент, Диагноз, Рецепт.

## ПОДСКАЗКА
Плоский экспорт = классический набор нарушений: повторяющиеся данные пациента/врача/диагноза в каждой строке (транзитивные ФЗ), справочники-препараты, многострочный рецепт (1:N к позициям). Разносим по ФЗ. Справочники (диагнозы, препараты) — отдельные таблицы. Медданные помечаем для требований доступа (модули 09, 34).
