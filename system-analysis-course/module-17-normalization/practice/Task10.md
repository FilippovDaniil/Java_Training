# Задача 10 (мини-проект) — Модуль 17: Нормализация

## ЗАДАНИЕ
Постройте полностью нормализованную схему (3NF) ключевой подсистемы MedConnect + словарь данных к ней. Обоснуйте точечную денормализацию под один отчёт. Синтез теории модуля.

## ВХОДНЫЕ ДАННЫЕ
**MedConnect**, подсистема «Консультации и рецепты». Исходные данные — частично из плоского экспорта (задача 09) и из ER-модуля 16: пациенты, врачи (со специализациями), записи/консультации, рецепты с позициями, диагнозы, препараты, оплаты. Есть отчёт «число консультаций и средний чек по врачу за месяц», который читается часто и тяжело джойнится.

## ТРЕБОВАНИЯ
Сдайте пакет:
1. **Нормализованная схема (3NF)** — все таблицы, PK/FK, кратности; ФЗ, на которых основана декомпозиция.
2. **Доказательство НФ** — для 2–3 ключевых таблиц покажите отсутствие частичных/транзитивных зависимостей.
3. **Словарь данных** — для ключевых атрибутов (типы, обязательность, ограничения, смысл, источник; статусы и медданные).
4. **Денормализация под отчёт** — предложите витрину/агрегат для отчёта по врачам, опишите поддержку согласованности и риски.
5. **152-ФЗ** — пометьте таблицы с перс./медданными.

## ФОРМАТ РЕЗУЛЬТАТА
Схема (текст/диаграмма) + список ФЗ + доказательство НФ + словарь данных + решение по денормализации + пометки медданных.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Схема в 3NF: таблицы, PK/FK, кратности, M:N разрешены.
- [ ] Приведены ФЗ, на которых основана декомпозиция.
- [ ] Для 2–3 таблиц доказано отсутствие частичных/транзитивных зависимостей.
- [ ] Словарь данных заполнен для ключевых атрибутов (статусы, деньги, даты, медданные).
- [ ] Денормализация под отчёт обоснована + механизм поддержания + риски.
- [ ] Таблицы с перс./медданными помечены (152-ФЗ).
- [ ] Сопроводительное описание связывает решения.

## ПОДСКАЗКА
Синтез: 1NF–3NF из задач 01–03, BCNF-чутьё из 04, аномалии из 05, денормализация из 06, словарь из 07, ФЗ из 08, нормализация экспорта из 09. Главные проверки middle+: декомпозиция по ФЗ (а не «на глаз»), достижение 3NF, осознанная денормализация под измеренную проблему, полный словарь данных. Схема идёт в SQL-запросы (модуль 18) и перформанс (модуль 20).
