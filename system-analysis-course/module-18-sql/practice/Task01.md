# Задача 01 — Модуль 18: SQL

## ЗАДАНИЕ
Напишите выборку с фильтрацией и сортировкой.

## ВХОДНЫЕ ДАННЫЕ
Схема и данные — из `theory.md` модуля (таблицы `client`, `restaurant`, `order`, `order_item`).

## ТРЕБОВАНИЯ
1. Выведите имена и города клиентов из Москвы, отсортированные по имени.
2. Выведите все доставленные заказы (`status = 'Доставлен'`) с суммой ≥ 500, сортировка по сумме по убыванию.

## ФОРМАТ РЕЗУЛЬТАТА
Два SQL-запроса + полученные результаты.

## ОЖИДАЕМЫЙ РЕЗУЛЬТАТ
Запрос 1 (клиенты Москвы по имени):
```
Иванов   | Москва
Сидорова | Москва
```
Запрос 2 (доставленные ≥ 500, по убыванию суммы):
```
id=103 | 1500
id=100 | 900
id=101 | 600
```

## КРИТЕРИИ ПРИЁМКИ
- [ ] Запрос 1 использует `WHERE city='Москва'` и `ORDER BY name`.
- [ ] Запрос 1 возвращает Иванов и Сидорова (без Петрова/Кузнецова).
- [ ] Запрос 2 использует `WHERE status='Доставлен' AND total>=500`.
- [ ] Запрос 2 отсортирован по `total DESC` (103, 100, 101).
- [ ] Отменённый заказ 102 и заказ 104 (450) не попали.

## ПОДСКАЗКА
`WHERE` фильтрует строки, `ORDER BY name` / `ORDER BY total DESC` сортирует. Условие диапазона — `total >= 500`. Текстовые значения в кавычках.
