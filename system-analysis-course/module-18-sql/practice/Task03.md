# Задача 03 — Модуль 18: SQL

## ЗАДАНИЕ
Посчитайте выручку по ресторанам через GROUP BY + агрегат, отфильтруйте группы через HAVING.

## ВХОДНЫЕ ДАННЫЕ
Схема и данные — из `theory.md` модуля.

## ТРЕБОВАНИЯ
Посчитайте суммарную выручку (`SUM(total)`) по ресторанам **только по доставленным** заказам. Выведите название ресторана и выручку. Оставьте только рестораны с выручкой > 1000. Сортировка по выручке убыв.

## ФОРМАТ РЕЗУЛЬТАТА
SQL-запрос (JOIN + GROUP BY + HAVING) + результат.

## ОЖИДАЕМЫЙ РЕЗУЛЬТАТ
```
Суши-бар | 2700
Пиццерия | 1050
```
(Бургерная — только отменённый заказ, выручка 0, не проходит. Расчёт: Суши-бар = 900+1500+300; Пиццерия = 600+450.)

## КРИТЕРИИ ПРИЁМКИ
- [ ] `WHERE status='Доставлен'` (фильтр строк до группировки).
- [ ] `GROUP BY` по ресторану, `SUM(total)`.
- [ ] `HAVING SUM(total) > 1000` (фильтр по агрегату).
- [ ] Результат: Суши-бар 2700, Пиццерия 1050.
- [ ] Бургерная отсутствует.
- [ ] Сортировка по выручке убыв.

## ПОДСКАЗКА
`WHERE` фильтрует до группировки, `HAVING` — после (по агрегату). Нельзя поставить `SUM(...)>1000` в WHERE. Чтобы вывести название, соедините с `restaurant` и группируйте по нему.
