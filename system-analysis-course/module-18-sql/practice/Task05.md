# Задача 05 — Модуль 18: SQL

## ЗАДАНИЕ
Используйте CTE (`WITH`) для многошагового расчёта: выручка по клиентам и отбор крупных.

## ВХОДНЫЕ ДАННЫЕ
Схема и данные — из `theory.md` модуля.

## ТРЕБОВАНИЯ
Через CTE: (1) отберите доставленные заказы; (2) посчитайте суммарную выручку по клиентам; (3) выведите имена клиентов с суммарной выручкой > 1000, по убыванию.

## ФОРМАТ РЕЗУЛЬТАТА
SQL с `WITH` (1–2 CTE) + результат.

## ОЖИДАЕМЫЙ РЕЗУЛЬТАТ
Выручка по клиентам (доставленные): Иванов = 900+600+300 = 1800; Сидорова = 1500+450 = 1950; Петров = 0 (только отменён).
Фильтр > 1000:
```
Сидорова | 1950
Иванов   | 1800
```

## КРИТЕРИИ ПРИЁМКИ
- [ ] Используется `WITH` (CTE), а не только вложенный подзапрос.
- [ ] CTE отбирает доставленные заказы.
- [ ] Агрегация выручки по клиенту.
- [ ] Фильтр итоговой выручки > 1000.
- [ ] Результат: Сидорова 1950, Иванов 1800 (Петров отсутствует).
- [ ] Сортировка по убыванию.

## ПОДСКАЗКА
```sql
WITH delivered AS (SELECT * FROM "order" WHERE status='Доставлен'),
     rev AS (SELECT client_id, SUM(total) s FROM delivered GROUP BY client_id)
SELECT c.name, rev.s FROM rev JOIN client c ON c.id=rev.client_id
WHERE rev.s > 1000 ORDER BY rev.s DESC;
```
CTE читается сверху вниз — удобнее вложенных подзапросов для многошаговой логики.
