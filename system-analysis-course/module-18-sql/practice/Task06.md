# Задача 06 — Модуль 18: SQL

## ЗАДАНИЕ
Используйте LEFT JOIN для поиска «осиротевших»/отсутствующих записей.

## ВХОДНЫЕ ДАННЫЕ
Схема и данные — из `theory.md` модуля.

## ТРЕБОВАНИЯ
1. Найдите клиентов, которые **ни разу** не делали заказ (нет ни одной записи в `order`).
2. Найдите рестораны, у которых **нет ни одного доставленного** заказа.

## ФОРМАТ РЕЗУЛЬТАТА
Два SQL-запроса (LEFT JOIN + IS NULL) + результаты.

## ОЖИДАЕМЫЙ РЕЗУЛЬТАТ
Запрос 1 (клиенты без заказов):
```
Кузнецов
```
Запрос 2 (рестораны без доставленных заказов):
```
Бургерная
```
(Кузнецов не имеет заказов вообще; Бургерная имеет только отменённый заказ 102.)

## КРИТЕРИИ ПРИЁМКИ
- [ ] Запрос 1: `client LEFT JOIN "order" ... WHERE order.id IS NULL` → Кузнецов.
- [ ] Запрос 2: LEFT JOIN с условием доставленных, отбор ресторанов без них → Бургерная.
- [ ] Не использован INNER JOIN (он бы скрыл отсутствие пары).
- [ ] Петров НЕ попал в запрос 1 (у него есть заказ, хоть и отменённый).
- [ ] Результаты точны.

## ПОДСКАЗКА
LEFT JOIN сохраняет все строки левой таблицы; у кого нет пары — справа NULL. Фильтр `WHERE правая.id IS NULL` = «нет связанных записей». Для запроса 2 условие доставленности перенесите в ON (иначе WHERE отфильтрует NULL раньше) либо отберите через `NOT IN`/`NOT EXISTS`.
