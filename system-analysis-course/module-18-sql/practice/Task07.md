# Задача 07 — Модуль 18: SQL ⭐

## ЗАДАНИЕ
Используйте оконную функцию для нумерации/рейтинга без схлопывания строк.

## ВХОДНЫЕ ДАННЫЕ
Схема и данные — из `theory.md` модуля.

## ТРЕБОВАНИЯ
1. Пронумеруйте заказы каждого клиента по дате создания (`ROW_NUMBER() OVER (PARTITION BY client_id ORDER BY created_at)`).
2. Постройте рейтинг всех заказов по сумме (`RANK() OVER (ORDER BY total DESC)`).

## ФОРМАТ РЕЗУЛЬТАТА
Два SQL-запроса с оконными функциями + результаты.

## ОЖИДАЕМЫЙ РЕЗУЛЬТАТ
Запрос 1 (нумерация заказов клиента по дате):
```
client 1 Иванов:   100 (n=1), 101 (n=2), 105 (n=3)
client 2 Петров:   102 (n=1)
client 3 Сидорова: 103 (n=1), 104 (n=2)
```
Запрос 2 (RANK по сумме убыв):
```
103 | 1500 | rank 1
100 |  900 | rank 2
101 |  600 | rank 3
104 |  450 | rank 4
105 |  300 | rank 5
102 |    0 | rank 6
```

## КРИТЕРИИ ПРИЁМКИ
- [ ] Запрос 1: `ROW_NUMBER() OVER (PARTITION BY client_id ORDER BY created_at)`.
- [ ] Нумерация перезапускается для каждого клиента (1,2,3 у Иванова).
- [ ] Запрос 2: `RANK() OVER (ORDER BY total DESC)`.
- [ ] Ранги верны (103=1 … 102=6).
- [ ] Строки НЕ схлопнуты (выводятся все заказы).

## ПОДСКАЗКА
Оконная функция считает по «окну», сохраняя строки (в отличие от GROUP BY). `PARTITION BY` задаёт разделы (как «группы», но без схлопывания), `ORDER BY` внутри окна — порядок для нумерации/ранга. `ROW_NUMBER` — сквозной номер, `RANK` — ранг с пропусками при равенстве.
