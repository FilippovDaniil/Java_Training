# Задача 08 — Модуль 18: SQL ⭐

## ЗАДАНИЕ
Используйте LAG/LEAD для анализа динамики «период к периоду».

## ВХОДНЫЕ ДАННЫЕ
Схема и данные — из `theory.md` модуля.

## ТРЕБОВАНИЯ
Посчитайте дневную выручку по **доставленным** заказам, затем с помощью `LAG` выведите для каждого дня: дату, выручку, выручку предыдущего дня и прирост (текущая − предыдущая).

## ФОРМАТ РЕЗУЛЬТАТА
SQL (агрегация по дате + `LAG() OVER (ORDER BY дата)`) + результат.

## ОЖИДАЕМЫЙ РЕЗУЛЬТАТ
Дневная выручка (доставленные): 06-01=900, 06-03=600, 06-05=1500, 06-10=450, 06-12=300.
```
дата       | выручка | пред.день | прирост
2026-06-01 |   900   |   NULL    |  NULL
2026-06-03 |   600   |   900     |  -300
2026-06-05 |  1500   |   600     |  +900
2026-06-10 |   450   |  1500     | -1050
2026-06-12 |   300   |   450     |  -150
```

## КРИТЕРИИ ПРИЁМКИ
- [ ] Сначала агрегируется выручка по дате (только доставленные).
- [ ] `LAG(выручка) OVER (ORDER BY дата)` даёт предыдущий день.
- [ ] Первый день имеет NULL в «пред.день» и «прирост».
- [ ] Прирост = текущая − предыдущая, значения верны.
- [ ] Отменённый заказ (06-03, заказ 102) не влияет на выручку дня.

## ПОДСКАЗКА
Сначала CTE с дневной выручкой (`GROUP BY created_at`), потом окно: `LAG(rev) OVER (ORDER BY created_at)`. `LAG` берёт значение из предыдущей строки окна, `LEAD` — из следующей. Прирост — разность; у первой строки предыдущего нет → NULL.
