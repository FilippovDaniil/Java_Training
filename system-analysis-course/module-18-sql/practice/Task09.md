# Задача 09 — Модуль 18: SQL

## ЗАДАНИЕ
Используйте self-join для обхода иерархии (категория → родительская категория).

## ВХОДНЫЕ ДАННЫЕ
Дополнительная таблица категорий меню (создайте отдельно):
```sql
CREATE TABLE category (id INT PRIMARY KEY, name VARCHAR(30), parent_id INT);
INSERT INTO category VALUES
 (1,'Еда',NULL),
 (2,'Японская',1),
 (3,'Итальянская',1),
 (4,'Роллы',2),
 (5,'Пицца',3);
```

## ТРЕБОВАНИЯ
Выведите каждую категорию вместе с названием её родительской категории (self-join). Категории без родителя (корень) можно не выводить (INNER) или вывести с NULL (LEFT) — на выбор, укажите какой.

## ФОРМАТ РЕЗУЛЬТАТА
SQL (self-join `category c JOIN category p ON c.parent_id = p.id`) + результат.

## ОЖИДАЕМЫЙ РЕЗУЛЬТАТ
INNER self-join (корень «Еда» без родителя не выводится):
```
Японская     | Еда
Итальянская  | Еда
Роллы        | Японская
Пицца        | Итальянская
```

## КРИТЕРИИ ПРИЁМКИ
- [ ] Таблица соединена сама с собой (self-join) с двумя алиасами.
- [ ] Условие соединения `child.parent_id = parent.id`.
- [ ] Выведены пары «категория — родитель».
- [ ] Указано поведение для корня (INNER скрывает «Еда»; LEFT покажет с NULL).
- [ ] Результат соответствует ожидаемому.

## ПОДСКАЗКА
Self-join — таблица соединяется с собой через два алиаса: `FROM category c JOIN category p ON c.parent_id = p.id`. Один алиас — «ребёнок», другой — «родитель». Для корня (`parent_id IS NULL`) пары нет → INNER его отбросит, LEFT покажет с NULL-родителем.
