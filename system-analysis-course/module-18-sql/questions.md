## Ключ (эталонные ответы)

1. `SELECT` (столбцы), `FROM` (источник), `WHERE` (фильтр строк), `GROUP BY` (группировка), `HAVING` (фильтр групп), `ORDER BY` (сортировка), `LIMIT` (ограничение).
2. **WHERE** фильтрует отдельные строки до группировки; **HAVING** — группы после агрегации (по значению агрегата).
3. `FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT`. Поэтому агрегат недоступен в WHERE и алиасы SELECT — в WHERE.
4. INNER (только совпадения), LEFT (все левые + совпавшие правые), RIGHT (все правые + совпавшие левые), FULL (все из обеих), SELF (таблица с собой).
5. INNER возвращает только строки, имеющие пару в обеих таблицах; LEFT возвращает все строки левой, а отсутствующие правые заполняет NULL.
6. LEFT JOIN + `WHERE правая.ключ IS NULL` (или `NOT EXISTS`/`NOT IN`): остаются строки без связанных записей.
7. **Self-join** — соединение таблицы с собой через два алиаса; нужен для иерархий (категория↔родитель, сотрудник↔менеджер).
8. Группирует строки по значениям столбцов; агрегаты считаются внутри каждой группы.
9. COUNT, SUM, AVG, MIN, MAX.
10. На этапе WHERE агрегаты ещё не вычислены (группировка идёт позже). Фильтр по агрегату — в HAVING.
11. **Подзапрос** — запрос внутри другого (в WHERE/FROM/SELECT). Может возвращать скаляр, список или таблицу.
12. **Коррелированный** подзапрос ссылается на строки внешнего запроса и вычисляется для каждой из них — потенциально дорого (повторное выполнение).
13. **CTE** (`WITH`) — именованный подзапрос; лучше при многошаговой логике или повторном использовании: читается сверху вниз, не загромождает.
14. Оконная функция считает по «окну» строк, **сохраняя все строки** (добавляет столбец); GROUP BY **схлопывает** строки в группы.
15. `ROW_NUMBER()` присваивает сквозной порядковый номер строкам в пределах окна согласно ORDER BY.
16. При равных значениях **RANK()** оставляет пропуски (1,2,2,4), **DENSE_RANK()** — без пропусков (1,2,2,3).
17. `LAG(x)` возвращает значение x из предыдущей строки окна, `LEAD(x)` — из следующей; используются для динамики «период к периоду».
18. `PARTITION BY` делит данные на разделы (окна считаются независимо в каждом); `ORDER BY` задаёт порядок строк внутри окна.
19. Оконной суммой с упорядочением: `SUM(x) OVER (ORDER BY дата)` — накапливает значение по мере движения по строкам.
20. `NULL` означает «неизвестно»: любое сравнение с ним даёт «неизвестно», поэтому `= NULL` не работает — только `IS NULL`/`IS NOT NULL`.
21. `COUNT(*)` считает все строки (включая с NULL); `COUNT(столбец)` — только строки, где столбец не NULL.
22. Соединение 1:N размножает строки «одной» стороны по числу связанных «многих»; если потом агрегировать по столбцам «одной» стороны, значения могут задвоиться — следить за гранулярностью.
23. Через `HAVING` после `GROUP BY` (например, `HAVING SUM(total) > 1000`).
24. Сортировка по показателю + `LIMIT N` (или оконная `RANK()`/`ROW_NUMBER()` с фильтром по рангу ≤ N).
25. **Подзапрос** — для простого однократного вложения; **CTE** — для многошаговой/повторно используемой логики и читаемости; **оконная функция** — когда нужны рейтинг/нумерация/динамика с сохранением всех строк.
