# Задача 02 — Модуль 19: NoSQL и CAP

## ЗАДАНИЕ
Объясните CAP-выбор для конкретных подсистем: определите, что важнее при сетевом разделении — согласованность (CP) или доступность (AP).

## ВХОДНЫЕ ДАННЫЕ
Подсистемы:
1. Списание средств при оплате заказа.
2. Лента «новинки ресторанов» на главной.
3. Остаток порций блюда (стоп-лист) у ресторана.
4. Счётчик лайков/просмотров блюда.
5. Бронирование слота времени у врача (MedConnect).

## ТРЕБОВАНИЯ
- Для каждой подсистемы выберите CP или AP.
- Объясните, чем жертвуем при разделении в каждом случае.
- Опишите последствие неправильного выбора (например, AP для денег).
- Подчеркните, почему P обязательна (нельзя выбрать «CA»).

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Подсистема | CP/AP | Чем жертвуем | Последствие ошибки выбора`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Оплата → CP (корректность важнее; AP → двойное списание).
- [ ] Лента → AP (отзывчивость; устаревшие данные допустимы).
- [ ] Стоп-лист/остаток → CP (иначе продадут отсутствующее).
- [ ] Счётчик лайков → AP (неточность некритична).
- [ ] Бронь слота → CP (иначе овербукинг).
- [ ] Объяснено, что P неизбежна → выбор только CP/AP.

## ПОДСКАЗКА
P обязательна (сеть рвётся), поэтому реальный выбор — CP или AP при разделении. CP: лучше отказать, чем показать/записать неверное (деньги, остатки, бронь). AP: лучше ответить устаревшим, чем не ответить (лента, счётчики). Цена ошибки: AP для денег = двойные списания; CP для ленты = ненужные отказы.
