# Задача 10 (мини-проект) — Модуль 19: NoSQL и CAP

## ЗАДАНИЕ
Спроектируйте стратегию хранения данных для MedConnect: выберите хранилища под каждый тип данных, обоснуйте CAP/PACELC-профили и модели согласованности, опишите согласование. Синтез теории модуля.

## ВХОДНЫЕ ДАННЫЕ
**MedConnect** (телемедицина, 152-ФЗ). Данные: профили пациентов/врачей (перс. и медданные); расписание и слоты; записи/консультации; оплаты; рецепты; чат/сообщения консультации; рекомендации врачей; полнотекстовый поиск врачей; телеметрия качества видеосвязи. Система геораспределённая. Регуляторика 152-ФЗ влияет на хранение медданных.

## ТРЕБОВАНИЯ
Сдайте стратегию:
1. **Карта данных → хранилища** — для каждого типа: семейство + обоснование моделью доступа.
2. **CAP/PACELC и согласованность** — для критичных подсистем (оплаты, бронь слота, баланс) — strong/CP/EC; для ленты/счётчиков/поиска — eventual/AP/EL.
3. **Согласование между хранилищами** — события/CDC, источник истины; что синхронизируется (запись → поиск/рекомендации).
4. **152-ФЗ** — где хранятся перс./медданные, требования к их размещению/доступу (вход для модулей 26, 34).
5. **Обоснование полиглота** — что выиграли и какой ценой; где можно обойтись одной БД.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица «данные → хранилище → согласованность → CAP/PACELC» + схема синхронизации + раздел 152-ФЗ + вывод о целесообразности полиглота.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Каждый тип данных сопоставлен с хранилищем и обоснован.
- [ ] Бронь слота, оплаты, баланс → strong/CP (последствия рассинхрона названы — овербукинг/двойное списание).
- [ ] Поиск/рекомендации/телеметрия → eventual/AP с обоснованием.
- [ ] PACELC-профили для геораспределённости (Latency vs Consistency в норме).
- [ ] Описана синхронизация (события/CDC) и источники истины.
- [ ] Медданные размещены с учётом 152-ФЗ; отмечены требования к доступу.
- [ ] Полиглот обоснован (выгода vs операционная цена), указано где избыточен.

## ПОДСКАЗКА
Синтез: выбор хранилищ (зад.01,09), CAP-профили (02), document/graph/KV (03–05,07), согласованность (06), PACELC (08). Главные проверки middle+: выбор под модель доступа (а не «по моде»), strong там где деньги/бронь/остатки, осознанная синхронизация с источником истины, учёт 152-ФЗ для медданных. Стыкуется с модулями 25 (распределёнка), 26 (NFR-арх), 34 (безопасность/доступ).
