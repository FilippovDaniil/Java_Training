# Задача 01 — Модуль 20: Производительность данных

## ЗАДАНИЕ
Подберите индексы под набор запросов: для каждого запроса определите нужный индекс и обоснуйте.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, таблица `order(id, client_id, restaurant_id, created_at, status, total)`. Частые запросы:
1. Заказы клиента, сортированные по дате: `WHERE client_id=? ORDER BY created_at DESC`.
2. Заказы ресторана за период: `WHERE restaurant_id=? AND created_at BETWEEN ? AND ?`.
3. Поиск заказа по id (PK).
4. Все заказы в статусе 'Доставлен' (статусов всего 7, доставлено ~80%).

## ТРЕБОВАНИЯ
- Для каждого запроса предложите индекс (или обоснуйте его отсутствие).
- Для составных индексов укажите порядок столбцов и почему.
- Отметьте запрос, где индекс бесполезен/вреден (низкая селективность).
- Кратко: цена добавленных индексов.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Запрос | Индекс | Порядок столбцов | Обоснование`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Запрос 1: составной индекс `(client_id, created_at)`.
- [ ] Запрос 2: составной индекс `(restaurant_id, created_at)`.
- [ ] Запрос 3: PK уже индексирован — отдельный индекс не нужен.
- [ ] Запрос 4: индекс по `status` бесполезен (низкая селективность, ~80% совпадений).
- [ ] Объяснён порядок столбцов в составных индексах (префикс).
- [ ] Названа цена индексов (замедление записи, место).

## ПОДСКАЗКА
Индекс под фильтр+сортировку: столбец фильтра первым, столбец сортировки — следом (составной по префиксу). Низкоселективный столбец (status с 2-3 частыми значениями) индексировать бессмысленно — СУБД всё равно прочитает почти всю таблицу. PK индексирован автоматически.
