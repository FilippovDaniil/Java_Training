# Задача 02 — Модуль 20: Производительность данных

## ЗАДАНИЕ
Сопоставьте уровни изоляции с аномалиями: для набора сценариев определите, какая аномалия возникает и какой уровень её устраняет.

## ВХОДНЫЕ ДАННЫЕ
Сценарии параллельных транзакций:
1. Транзакция A читает баланс, который B ещё не зафиксировала (потом B откатилась).
2. Транзакция A дважды читает строку заказа; между чтениями B изменила сумму — A видит разные значения.
3. Транзакция A дважды выполняет `SELECT COUNT(*) WHERE status='новый'`; между ними B вставила новый заказ — счёт изменился.
4. Финансовый отчёт должен видеть полностью согласованный снимок данных без любых аномалий.

## ТРЕБОВАНИЯ
- Для каждого сценария назовите аномалию (dirty / non-repeatable / phantom).
- Укажите минимальный уровень изоляции, который её устраняет.
- Для сценария 4 — обоснуйте Serializable.
- Отметьте цену повышения изоляции.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Сценарий | Аномалия | Минимальный уровень | Цена`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Сценарий 1 → dirty read → устраняет Read Committed.
- [ ] Сценарий 2 → non-repeatable read → Repeatable Read.
- [ ] Сценарий 3 → phantom read → Serializable (или RR в PostgreSQL).
- [ ] Сценарий 4 → Serializable (полная изоляция).
- [ ] Указана цена изоляции (блокировки, конфликты, снижение параллелизма, риск deadlock).
- [ ] Корректное соответствие аномалия↔уровень.

## ПОДСКАЗКА
Иерархия: Read Uncommitted < Read Committed (нет dirty) < Repeatable Read (нет non-repeatable) < Serializable (нет phantom). Чем выше — меньше аномалий, но больше блокировок и конфликтов. Phantom — про появление новых строк по условию, его убирает Serializable.
