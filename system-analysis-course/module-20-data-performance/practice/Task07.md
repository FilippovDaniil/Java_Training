# Задача 07 — Модуль 20: Производительность данных

## ЗАДАНИЕ
Спроектируйте партиционирование таблицы по времени и сравните его с шардированием.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, таблица `order` за несколько лет, сотни миллионов строк в одной БД. 95% запросов — по последним 30 дням; старые данные нужны для отчётов и редко. Удаление старых данных архивацией.

## ТРЕБОВАНИЯ
- Предложите партиционирование `order` по времени (по месяцам/кварталам).
- Покажите выгоду: partition pruning (запрос к свежим данным читает только свежие партиции).
- Опишите упрощение архивации/удаления старых данных (drop партиции).
- Сравните партиционирование и шардирование (одна БД vs разные узлы; что решает каждое).

## ФОРМАТ РЕЗУЛЬТАТА
Схема партиционирования + выгоды (pruning, архивация) + сравнительная таблица партиционирование vs шардирование.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Партиционирование по диапазону дат (месяц/квартал).
- [ ] Объяснён partition pruning (запрос за 30 дней читает только нужные партиции).
- [ ] Архивация/удаление через drop/detach партиции (без тяжёлого DELETE).
- [ ] Сравнение: партиционирование — части одной таблицы в одной БД; шардирование — данные на разных узлах.
- [ ] Указано, что партиционирование решает управляемость/скорость, шардирование — масштаб записи/объёма.

## ПОДСКАЗКА
Партиционирование — деление одной таблицы в пределах одной БД (по дате — частый кейс). Запрос с фильтром по дате читает только подходящие партиции (pruning). Удаление старого = drop партиции (мгновенно vs медленный DELETE). Шардирование — другое: разнос по узлам ради масштаба. Их можно сочетать.
