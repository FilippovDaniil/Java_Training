# Задача 10 (мини-проект) — Модуль 20: Производительность данных

## ЗАДАНИЕ
Спроектируйте data performance design для FoodExpress под рост нагрузки: индексы, репликация, шардирование/партиционирование, кэш, изоляция критичных операций. Синтез теории модуля.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress** на пике (Чёрная пятница): заказы — сотни млн строк, чтения >> записи, каталог читается массово, оплаты требуют корректности, дашборды тяжёлые, история заказов большая. Требования: быстрый каталог, корректные оплаты/остатки, масштаб чтения, управляемость исторических данных.

## ТРЕБОВАНИЯ
Сдайте пакет проектных решений:
1. **Индексы** — под основные запросы (заказы клиента/ресторана/период, поиск), с обоснованием и ценой.
2. **Изоляция/транзакции** — для оплат и списания остатков (уровень/блокировки), почему.
3. **Репликация** — read replicas под чтение; где лаг недопустим (read-your-writes).
4. **Шардирование/партиционирование** — ключ/стратегия для огромной таблицы заказов; разбор перекоса.
5. **Кэширование** — что кэшируем (каталог), TTL/инвалидация; что НЕ кэшируем (остатки/баланс).
6. **Связь с NFR** — как решения отражаются в NFR производительности (модуль 09).

## ФОРМАТ РЕЗУЛЬТАТА
Документ проектных решений по 6 пунктам + краткое обоснование каждого + перечень компромиссов (чтение/запись, свежесть/скорость).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Индексы под реальные запросы (составные, порядок столбцов), указана цена.
- [ ] Оплаты/остатки — повышенная изоляция или блокировки с обоснованием.
- [ ] Read replicas под чтение; названы чтения, требующие свежести (с мастера).
- [ ] Выбран ключ шардирования/партиционирование с разбором перекоса.
- [ ] Кэш каталога с инвалидацией; критичные данные не из кэша.
- [ ] Решения увязаны с NFR (latency p95, throughput, доступность).
- [ ] Явно перечислены компромиссы.

## ПОДСКАЗКА
Синтез: индексы (зад.01,08), изоляция/ACID (02,06), репликация (03), шардирование/партиционирование (04,07), кэш и согласованность (05,09). Главные проверки middle+: решения под измеренные паттерны (не «всё подряд»), strong-гарантии для денег/остатков, осознанные компромиссы чтение/запись и свежесть/скорость, связь с NFR. Стыкуется с модулями 09 (NFR), 25 (распределёнка), 26 (NFR-арх).
