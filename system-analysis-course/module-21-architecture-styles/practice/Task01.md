# Задача 01 — Модуль 21: Архитектурные стили

## ЗАДАНИЕ
Сравните архитектурные стили по критериям: постройте таблицу trade-offs монолита, модульного монолита, SOA и микросервисов.

## ВХОДНЫЕ ДАННЫЕ
Стили: монолит, модульный монолит, SOA, микросервисы. Критерии сравнения: простота разработки, развёртывание, масштабирование, транзакции/целостность, изоляция отказов, операционная сложность, автономия команд, технологическая свобода.

## ТРЕБОВАНИЯ
- Постройте таблицу «стиль × критерий» (≥ 8 критериев).
- В каждой ячейке — краткая характеристика (плюс/минус/нюанс).
- Выделите, где каждый стиль силён и где слаб.
- Сделайте вывод: какой стиль — разумный дефолт для старта и почему.

## ФОРМАТ РЕЗУЛЬТАТА
Сравнительная таблица + краткие выводы по сильным/слабым сторонам каждого стиля.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Таблица покрывает 4 стиля и ≥ 8 критериев.
- [ ] Монолит: прост, ACID в одной БД, но сложно масштабировать по частям.
- [ ] Микросервисы: независимость/масштаб ценой операционной сложности и распределённых сбоев.
- [ ] Модульный монолит показан как промежуточный (границы без сетевой цены).
- [ ] Указаны сильные/слабые стороны каждого.
- [ ] Вывод: (модульный) монолит — разумный дефолт для старта.

## ПОДСКАЗКА
Это не рейтинг «лучше-хуже», а компромиссы. Ось: простота ↔ гибкость/масштаб, с ростом операционной сложности вправо. Микросервисы выигрывают в масштабе/автономии команд, проигрывают в сложности эксплуатации и отсутствии общих транзакций.
