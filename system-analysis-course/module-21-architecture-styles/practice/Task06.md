# Задача 06 — Модуль 21: Архитектурные стили

## ЗАДАНИЕ
Оцените операционную стоимость микросервисов: перечислите, что нужно «вокруг» сервисов, и сопоставьте с выгодами для конкретного контекста.

## ВХОДНЫЕ ДАННЫЕ
Команда хочет «перейти на микросервисы», потому что «это современно». Контекст: 6 разработчиков, один продукт, умеренная нагрузка, нет выделенных DevOps/SRE.

## ТРЕБОВАНИЯ
- Перечислите операционные требования микросервисов (CI/CD на сервис, observability, оркестрация, обработка распределённых сбоев, контракты, тестирование интеграций).
- Сопоставьте с возможностями команды (6 человек, без SRE).
- Дайте рекомендацию (вероятно, модульный монолит) с обоснованием.
- Укажите, при каких изменениях контекста микросервисы станут оправданы.

## ФОРМАТ РЕЗУЛЬТАТА
Чек-лист операционных затрат + сопоставление с командой + рекомендация + условия оправданности.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Перечислены операционные затраты (CI/CD, мониторинг/трейсинг/логи, оркестрация/K8s, обработка сбоев, версионирование контрактов).
- [ ] Сопоставлено с командой 6 человек без SRE (затраты превышают выгоду).
- [ ] Рекомендация: модульный монолит (границы без операционной цены).
- [ ] Названы условия, когда микросервисы оправданы (рост команды/нагрузки/потребность независимого масштаба).
- [ ] Отвергнут мотив «потому что современно».

## ПОДСКАЗКА
Микросервисы — не бесплатны: каждый сервис нужно собирать, деплоить, мониторить, трейсить, оркестрировать, обрабатывать его сбои и версионировать контракты. Для маленькой команды это отнимает время у продукта. «Современно» — не критерий. Дефолт — модульный монолит; микросервисы под реальную потребность.
