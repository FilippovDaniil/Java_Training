# Задача 01 — Модуль 22: Клиент-серверная архитектура

## ЗАДАНИЕ
Разнесите компоненты системы по слоям (presentation / business / data access) и проверьте корректность зависимостей.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, компоненты: страница оформления заказа, контроллер заказа (приём HTTP), сервис расчёта стоимости и применения скидок, репозиторий заказов (SQL), валидатор корзины (бизнес-правила), DTO ответа клиенту, маппер entity↔DTO, компонент рендеринга меню.

## ТРЕБОВАНИЯ
- Отнесите каждый компонент к слою (presentation / business / data access).
- Покажите направление зависимостей (сверху вниз).
- Отметьте компоненты, которые легко «уехать не в свой слой».
- Сформулируйте, что каждый слой НЕ должен делать.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Компонент | Слой | Обоснование` + схема зависимостей + правила «чего слой не делает».

## КРИТЕРИИ ПРИЁМКИ
- [ ] Presentation: страница оформления, рендеринг меню, DTO ответа, контроллер (на границе presentation/app).
- [ ] Business: сервис расчёта стоимости/скидок, валидатор корзины.
- [ ] Data access: репозиторий заказов, маппер entity↔DTO (на границе).
- [ ] Зависимости идут сверху вниз (presentation → business → data).
- [ ] Указано, что presentation не содержит бизнес-правил, data access — тоже.
- [ ] Отмечены «пограничные» компоненты (контроллер, маппер).

## ПОДСКАЗКА
Слой = ответственность: presentation (UI/ввод-вывод), business (правила, расчёты), data access (хранилище). Расчёт скидок — бизнес-правило, не место для UI или репозитория. Зависимости только вниз: UI не лезет в SQL мимо бизнес-слоя.
