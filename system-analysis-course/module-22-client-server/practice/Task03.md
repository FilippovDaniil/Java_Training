# Задача 03 — Модуль 22: Клиент-серверная архитектура

## ЗАДАНИЕ
Спроектируйте 3-tier архитектуру для веб-приложения FoodExpress: разнесите компоненты по звеньям и слоям, объясните разницу.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress** — веб-приложение. Компоненты: браузер пользователя, сервер приложений (бизнес-логика, API), сервер БД (PostgreSQL). Нужно показать и физическое размещение (звенья), и логические слои.

## ТРЕБОВАНИЯ
- Распределите систему по 3 звеньям (tier): что на каждом физически.
- Покажите логические слои (presentation/business/data) и где они «живут» относительно звеньев.
- Явно объясните разницу tier vs layer на этом примере.
- Отметьте выгоды 3-tier (масштабирование, безопасность — БД не наружу).

## ФОРМАТ РЕЗУЛЬТАТА
Схема 3-tier (звенья) + наложение слоёв + пояснение tier vs layer + выгоды.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Tier 1 — клиент (браузер), tier 2 — сервер приложений, tier 3 — сервер БД.
- [ ] Показаны слои presentation/business/data и их расположение по звеньям.
- [ ] Явно объяснено: tier = физическое размещение, layer = логическая организация.
- [ ] Названы выгоды 3-tier (независимое масштабирование звеньев, БД скрыта за app-сервером).
- [ ] Подчёркнуто, что БД не доступна клиенту напрямую (в отличие от 2-tier).

## ПОДСКАЗКА
3-tier: браузер (tier 1) → сервер приложений (tier 2) → сервер БД (tier 3). Слои (presentation/business/data) — логические; в 3-tier presentation частично на клиенте, business на app-сервере, data — между app и БД. Главное: tier — где физически, layer — как организован код. БД за app-сервером = безопасность и масштабируемость.
