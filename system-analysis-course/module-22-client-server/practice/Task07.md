# Задача 07 — Модуль 22: Клиент-серверная архитектура

## ЗАДАНИЕ
Спроектируйте N-tier для масштабирования: добавьте к 3-tier дополнительные звенья под нагрузку и объясните роль каждого.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress** под ростом: всплески трафика, тяжёлые чтения, нужна устойчивость. К базовому 3-tier (клиент / сервер приложений / БД) нужно добавить звенья для масштаба и надёжности.

## ТРЕБОВАНИЯ
- Дополните архитектуру звеньями: балансировщик нагрузки, кэш, очередь сообщений, read replicas БД, CDN для статики.
- Для каждого звена — роль и какую проблему решает.
- Покажите путь запроса через звенья (чтение и запись).
- Свяжите с предыдущими модулями (кэш/реплики — модуль 20; очередь — модуль 32).

## ФОРМАТ РЕЗУЛЬТАТА
Схема N-tier (звенья + связи) + таблица `Звено | Роль | Какую проблему решает` + путь запроса.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Добавлен балансировщик перед серверами приложений (масштаб/доступность).
- [ ] Добавлен кэш (разгрузка БД на чтении).
- [ ] Добавлены read replicas (масштаб чтения).
- [ ] Добавлена очередь для асинхронных задач (уведомления и т.п.).
- [ ] CDN для статики (разгрузка, близость к пользователю).
- [ ] Показан путь запроса через звенья; связь с модулями 20/32.

## ПОДСКАЗКА
N-tier = 3-tier + инфраструктурные звенья под нефункциональные требования: балансировщик (горизонтальный масштаб app-серверов), кэш и реплики (масштаб чтения — модуль 20), очередь (асинхронность — модуль 32), CDN (статика ближе к пользователю). Каждое звено решает конкретную проблему масштаба/надёжности.
