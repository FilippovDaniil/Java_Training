# Задача 10 (мини-проект) — Модуль 22: Клиент-серверная архитектура

## ЗАДАНИЕ
Спроектируйте компонентную клиент-серверную архитектуру FoodExpress: звенья, слои, клиенты (с BFF/Gateway), потоки запросов и инфраструктурные звенья под нагрузку. Синтез теории модуля.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**: клиенты — веб (SSR-публичные страницы + SPA-кабинет), мобильное приложение, партнёрские интеграции. Backend — несколько сервисов (Каталог, Заказы, Платежи, Доставка). Нагрузка неравномерна, есть тяжёлые чтения и асинхронные задачи (уведомления). Требуется безопасность (БД не наружу) и масштаб чтения.

## ТРЕБОВАНИЯ
Сдайте архитектуру:
1. **Звенья (tier)** — физическое размещение (клиенты, gateway, серверы приложений/сервисы, БД, кэш, реплики, очередь, CDN).
2. **Слои (layer)** — логическая организация (presentation/business/data) и где живут.
3. **Клиенты и входная точка** — API Gateway (сквозные задачи) + BFF под веб/мобайл; обоснование.
4. **Потоки запросов** — путь чтения (с кэшем/репликами) и записи (через сервисы, async-уведомления).
5. **Толстый/тонкий и SSR/SPA** — какие части веба SSR, какие SPA; где логика.
6. **Безопасность и слоистость** — БД скрыта, критичная валидация на сервере, корректные зависимости слоёв.
7. **Связь с NFR** — масштабируемость/доступность (модули 20, 26).

## ФОРМАТ РЕЗУЛЬТАТА
Компонентная схема (звенья + слои + клиенты + Gateway/BFF + инфраструктура) + описание потоков + обоснование ключевых решений.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Чётко разделены звенья (tier) и слои (layer) с пояснением разницы.
- [ ] Входная точка: API Gateway (auth/routing/limits) + BFF под клиентов.
- [ ] Описаны потоки чтения (кэш/реплики) и записи (сервисы, async-уведомления через очередь).
- [ ] Указано, что SSR/SPA где применяется и как смещается логика.
- [ ] БД скрыта за серверами приложений; критичная валидация на сервере.
- [ ] Слоистость соблюдена (зависимости сверху вниз).
- [ ] Решения увязаны с NFR (масштаб чтения, доступность, безопасность).

## ПОДСКАЗКА
Синтез: слои (зад.01,06), клиент (02,08), 3/N-tier (03,07), BFF (04), Gateway (05), SSR/SPA (09). Главные проверки middle+: не путать tier и layer, единая входная точка + BFF под клиентов, БД не наружу, критичная логика на сервере, инфраструктурные звенья под измеренные NFR. Стыкуется с сетями (23), DDD/слоями (24), NFR-арх (26), REST/контрактами (29).
