# Задача 01 — Модуль 23: Сети

## ЗАДАНИЕ
Сопоставьте протоколы и технологии со слоями модели OSI/TCP-IP.

## ВХОДНЫЕ ДАННЫЕ
Технологии/протоколы: HTTP, TCP, IP, TLS, Ethernet, DNS, UDP, gRPC, MAC-адрес, HTTPS.

## ТРЕБОВАНИЯ
- Сопоставьте каждый элемент с уровнем OSI (и при возможности с уровнем TCP/IP).
- Для TCP и UDP объясните разницу (гарантии, когда что).
- Объясните, где «находится» HTTPS относительно HTTP и TLS.
- Отметьте, какие уровни наиболее важны аналитику и почему.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Элемент | Уровень OSI | Уровень TCP/IP | Комментарий` + пояснение TCP vs UDP.

## КРИТЕРИИ ПРИЁМКИ
- [ ] HTTP, DNS, gRPC → прикладной (L7).
- [ ] TLS → представления (поверх TCP); HTTPS = HTTP+TLS.
- [ ] TCP, UDP → транспортный (L4).
- [ ] IP → сетевой (L3); Ethernet/MAC → канальный (L2).
- [ ] Объяснена разница TCP (надёжный, порядок) vs UDP (быстрый, без гарантий).
- [ ] Отмечено, что аналитику важнее верхние уровни (прикладной/транспортный).

## ПОДСКАЗКА
Прикладной (L7): протоколы приложений (HTTP/DNS/gRPC). Транспортный (L4): TCP (надёжный, для HTTP/БД) и UDP (быстрый, для видео/DNS-запросов). Сетевой (L3): IP. Канальный (L2): Ethernet/MAC. TLS — поверх TCP (шифрование), HTTPS = HTTP по TLS.
