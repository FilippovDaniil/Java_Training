# Задача 05 — Модуль 23: Сети

## ЗАДАНИЕ
Опишите, как работает HTTPS/TLS — рукопожатие словами и какие гарантии оно даёт.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress** переводит API на HTTPS. Нужно объяснить команде (нетехнической части) и зафиксировать в требованиях: что даёт TLS, как устроено рукопожатие, что такое сертификат.

## ТРЕБОВАНИЯ
- Опишите TLS-рукопожатие пошагово простыми словами.
- Назовите три гарантии TLS (шифрование, целостность, аутентификация сервера).
- Объясните роль сертификата и центра сертификации (CA).
- Объясните, почему используется и асимметричное, и симметричное шифрование.
- Отметьте, что такое mTLS и где он нужен.

## ФОРМАТ РЕЗУЛЬТАТА
Пошаговое описание рукопожатия + три гарантии + роль сертификата/CA + пояснение асимметрия→симметрия + mTLS.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Рукопожатие описано по шагам (hello → сертификат → проверка → согласование ключа → шифрованный обмен).
- [ ] Названы три гарантии: конфиденциальность, целостность, аутентификация сервера.
- [ ] Объяснена роль сертификата и CA (подтверждение подлинности домена).
- [ ] Объяснено: асимметрия — для обмена ключом, симметрия — для данных (быстрее).
- [ ] Описан mTLS (обе стороны с сертификатами) и где нужен (межсервисное).
- [ ] Отмечено последствие невалидного/истёкшего сертификата.

## ПОДСКАЗКА
TLS даёт: шифрование (не прочитать), целостность (не подменить), аутентификацию сервера (сертификат от доверенного CA). Рукопожатие: клиент/сервер договариваются о шифрах, сервер шлёт сертификат, клиент проверяет его у CA, стороны согласуют симметричный сеансовый ключ (через асимметрию), дальше — быстрый симметричный обмен. mTLS — взаимная проверка сертификатов (модуль 34).
