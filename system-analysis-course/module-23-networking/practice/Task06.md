# Задача 06 — Модуль 23: Сети

## ЗАДАНИЕ
Опишите DNS-резолюцию шаг за шагом и разберите влияние TTL при переключении.

## ВХОДНЫЕ ДАННЫЕ
Пользователь открывает `api.foodexpress.ru`. Также есть задача: миграция сервиса на новый IP без даунтайма.

## ТРЕБОВАНИЯ
- Опишите шаги резолюции имени `api.foodexpress.ru` в IP (кэш → резолвер → иерархия → ответ).
- Объясните роль кэша и TTL.
- Назовите типы записей (A, AAAA, CNAME, MX).
- Разберите, как TTL влияет на переключение IP (почему старый IP «живёт» какое-то время) и как готовиться к миграции (снизить TTL заранее).

## ФОРМАТ РЕЗУЛЬТАТА
Пошаговая схема резолюции + роль TTL + таблица типов записей + стратегия миграции IP.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Шаги: проверка кэша → DNS-резолвер → root → TLD → авторитетный → IP.
- [ ] Объяснена роль кэша и TTL (как долго ответ считается актуальным).
- [ ] Перечислены типы записей (A/AAAA/CNAME/MX) с назначением.
- [ ] Разобрано, что после смены записи старый IP кэширован до истечения TTL.
- [ ] Стратегия миграции: заранее снизить TTL, потом менять запись.
- [ ] Отмечено, что DNS не переключается мгновенно.

## ПОДСКАЗКА
DNS переводит имя → IP через иерархию (root → TLD → авторитетный сервер), с кэшированием по TTL. При миграции IP: старые клиенты используют закэшированный IP до окончания TTL — поэтому TTL снижают заранее, и только потом меняют A-запись. Типы: A (IPv4), AAAA (IPv6), CNAME (псевдоним), MX (почта).
