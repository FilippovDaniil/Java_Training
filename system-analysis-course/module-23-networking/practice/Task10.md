# Задача 10 (мини-проект) — Модуль 23: Сети

## ЗАДАНИЕ
Опишите сетевое взаимодействие компонентов FoodExpress «от ввода адреса до ответа»: протоколы, статусы, TLS, DNS, балансировка, кэширование. Синтез теории модуля.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**: пользователь открывает мобильное приложение и оформляет заказ. Инфраструктура: DNS, CDN для статики, балансировщик, несколько серверов приложений (stateless), API Gateway, БД с репликами. Требования: HTTPS везде, доступность, разумная нагрузка.

## ТРЕБОВАНИЯ
Опишите сквозной путь и решения:
1. **DNS** — резолюция домена API, роль TTL.
2. **TLS** — установление HTTPS, что гарантирует.
3. **Балансировка** — стратегия и уровень (L4/L7), health checks, stateless.
4. **HTTP-обмен** — пример запроса оформления заказа (метод, заголовки, idempotency) и ответа (статус).
5. **Статусы** — какие коды на каких исходах (создание, ошибки, конфликт).
6. **Кэширование** — что кэшируется (статика через CDN, каталог), что нет (статус заказа).
7. **Диагностируемость** — что логировать/смотреть при инциденте (связь с наблюдаемостью, модуль 26).

## ФОРМАТ РЕЗУЛЬТАТА
Сквозное описание пути запроса (диаграмма/текст) + решения по 7 пунктам + пример HTTP-обмена.

## КРИТЕРИИ ПРИЁМКИ
- [ ] DNS-резолюция и роль TTL описаны.
- [ ] HTTPS/TLS: гарантии и где применяется (везде).
- [ ] Балансировка: стратегия + уровень + health checks + stateless-серверы.
- [ ] Пример HTTP-обмена оформления заказа с корректными методом/заголовками/idempotency.
- [ ] Корректные статусы под исходы (201/4xx/409/5xx).
- [ ] Кэширование: статика/каталог кэшируется, статус заказа — нет.
- [ ] Отмечено, что логировать/мониторить для диагностики.

## ПОДСКАЗКА
Синтез: OSI/протоколы (зад.01), HTTP-обмен (02), статусы (03), идемпотентность (04), TLS (05), DNS (06), балансировка (07), кэш (08), диагностика (09). Главные проверки middle+: корректный сквозной путь (DNS→TLS→балансировщик→gateway→сервис→БД), правильные статусы и идемпотентность, HTTPS везде, осознанное кэширование, диагностируемость. Стыкуется с клиент-сервером (22), REST (29), безопасностью (34), NFR-арх (26).
