# Задача 03 — Модуль 24: DDD

## ЗАДАНИЕ
Определите агрегаты и их границы: сгруппируйте сущности в агрегаты, выделите корни, зафиксируйте инварианты.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, контекст «Заказы». Сущности/объекты: Заказ, Позиция заказа, Адрес доставки, Оплата, Клиент, Блюдо. Инварианты: у заказа минимум 1 позиция; сумма заказа = сумме позиций; статус заказа меняется по правилам; позиция не существует без заказа.

## ТРЕБОВАНИЯ
- Сгруппируйте в агрегаты; для каждого укажите корень (aggregate root).
- Определите, что внутри агрегата, а что — отдельный агрегат, на который ссылаются по id.
- Зафиксируйте инварианты, которые держит корень.
- Объясните правило «одна транзакция = один агрегат» и ссылки между агрегатами по id.

## ФОРМАТ РЕЗУЛЬТАТА
Список агрегатов (корень + содержимое + инварианты) + пояснение границ и ссылок между агрегатами.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Агрегат «Заказ»: корень Order, внутри — Позиции (и Адрес как VO/часть).
- [ ] Клиент и Блюдо — отдельные агрегаты, ссылка по id (не внутрь Заказа).
- [ ] Оплата — отдельный агрегат/контекст (Биллинг), ссылка по id.
- [ ] Инварианты (≥1 позиция, сумма=сумме позиций) держит корень Order.
- [ ] Изменение позиций — только через корень.
- [ ] Объяснено «одна транзакция = один агрегат» и ссылки по id между агрегатами.

## ПОДСКАЗКА
Агрегат — кластер, меняемый как целое через корень. Заказ+Позиции — один агрегат (инвариант «сумма=сумме позиций» держится внутри). Клиент, Блюдо, Оплата — отдельные агрегаты: ссылаемся по id, не вкладываем. Граница агрегата = граница транзакции; между агрегатами — eventual consistency/saga (модуль 25).
