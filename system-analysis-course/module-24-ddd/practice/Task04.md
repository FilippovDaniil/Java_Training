# Задача 04 — Модуль 24: DDD

## ЗАДАНИЕ
Классифицируйте объекты как entity или value object и обоснуйте через идентичность/изменяемость.

## ВХОДНЫЕ ДАННЫЕ
Объекты домена FoodExpress / MedConnect:
1. Заказ.
2. Деньги (сумма + валюта).
3. Адрес доставки.
4. Клиент.
5. Координаты (широта, долгота).
6. Период времени (с–по).
7. Позиция заказа.
8. Email.

## ТРЕБОВАНИЯ
- Для каждого объекта определите: entity или value object.
- Обоснуйте через критерии: есть ли идентичность, важна ли изменяемость, как сравнивается.
- Отметьте «пограничные» случаи (Позиция заказа, Email) и контекст-зависимость.
- Сформулируйте правило различения.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Объект | Entity/VO | Обоснование (идентичность/равенство)`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Заказ, Клиент → entity (идентичность во времени, сравнение по id).
- [ ] Деньги, Координаты, Период, Адрес → value object (по значениям, неизменяемы).
- [ ] Email → обычно VO (значение), но может быть атрибутом.
- [ ] Позиция заказа → пограничный (часто entity внутри агрегата Заказ).
- [ ] Для каждого приведено обоснование.
- [ ] Сформулировано правило (идентичность → entity; только значения → VO).

## ПОДСКАЗКА
Entity — есть идентичность, живущая во времени (Заказ №1024 тот же при изменении полей), сравнение по id. Value Object — определяется значениями, неизменяем, сравнивается по значениям (два одинаковых адреса равны). Деньги/Координаты/Период — классические VO. Контекст влияет: иногда одно и то же может быть entity или VO в разных контекстах.
