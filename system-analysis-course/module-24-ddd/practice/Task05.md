# Задача 05 — Модуль 24: DDD

## ЗАДАНИЕ
Составьте глоссарий единого языка (ubiquitous language) для bounded context MedConnect и покажите его проникновение в код/БД/API.

## ВХОДНЫЕ ДАННЫЕ
**MedConnect**, контекст «Консультации». Понятия и текущая путаница: бизнес говорит «приём»/«консультация»/«сеанс» вперемешку; «запись» и «бронь» используются как синонимы и нет; «врач» и «специалист»; статусы консультации называются по-разному в обсуждениях.

## ТРЕБОВАНИЯ
- Зафиксируйте единый термин для каждого понятия (выбрать один из синонимов).
- Составьте глоссарий: термин → определение в контексте.
- Покажите, как термин отражается единообразно: класс/метод, поле БД, эндпоинт API.
- Объясните, почему недопустимы синонимы внутри одного контекста.

## ФОРМАТ РЕЗУЛЬТАТА
Глоссарий `Термин | Определение | В коде | В БД | В API` + правило единственности термина.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Для каждого понятия выбран один канонический термин (устранены синонимы).
- [ ] Глоссарий с определениями в рамках контекста.
- [ ] Показано единое проникновение термина в код/БД/API.
- [ ] Объяснено, почему синонимы внутри контекста — источник дефектов.
- [ ] Отмечено, что в другом контексте тот же термин может значить иное (это нормально).
- [ ] Связь с глоссарием спецификации (модуль 11).

## ПОДСКАЗКА
Единый язык = один термин на понятие, одинаково в речи бизнеса, коде, БД и API. Выбрать: «консультация» (а не сеанс/приём вперемешку), определить «запись» vs «бронь» однозначно. Термин «консультация» → класс Consultation, таблица consultation, эндпоинт /consultations. Синонимы внутри контекста = путаница; глоссарий фиксирует выбор (модуль 11).
