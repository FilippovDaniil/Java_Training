# Задача 06 — Модуль 24: DDD

## ЗАДАНИЕ
Классифицируйте поддомены FoodExpress на core / supporting / generic и определите стратегию для каждого.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, области функциональности: алгоритм матчинга заказов и оптимизации маршрутов доставки (главное преимущество), каталог и меню ресторанов, расчёт стоимости/скидок, аутентификация и авторизация, отправка email/SMS-уведомлений, платёжный шлюз, аналитические дашборды.

## ТРЕБОВАНИЯ
- Отнесите каждую область к core / supporting / generic.
- Для каждого типа определите стратегию (строить лучшими силами / проще своими / купить готовое).
- Обоснуйте, что является core (конкурентное преимущество).
- Укажите типичную ошибку: вложения в generic вместо core.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Область | Поддомен (core/supporting/generic) | Стратегия | Обоснование`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Матчинг/оптимизация маршрутов → core (строить самим, лучшие силы).
- [ ] Каталог/меню, расчёт стоимости → supporting (свои силы, проще).
- [ ] Аутентификация, уведомления, платёжный шлюз → generic (готовые решения).
- [ ] Для каждого определена стратегия.
- [ ] Обосновано, почему core даёт преимущество.
- [ ] Отмечена ошибка «вкладываться в generic вместо core».

## ПОДСКАЗКА
Core — конкурентное преимущество (для FoodExpress — логистика/матчинг): строить самим, тщательно. Supporting — нужно, но не уникально (каталог): своими силами, без перфекционизма. Generic — стандартное (авторизация, email, платёжный шлюз): брать готовое/купить, не изобретать. Ошибка — тратить лучших инженеров на generic.
