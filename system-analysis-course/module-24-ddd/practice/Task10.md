# Задача 10 (мини-проект) — Модуль 24: DDD

## ЗАДАНИЕ
Проведите стратегический DDD-анализ MedConnect: bounded context'ы + context map + единый язык + поддомены + агрегаты ключевого контекста. Синтез теории модуля.

## ВХОДНЫЕ ДАННЫЕ
**MedConnect** (телемедицина): пациенты записываются к врачам, проходят видеоконсультации, получают рецепты; есть оплаты, расписания, поддержка, уведомления; интеграция с государственной системой рецептов (legacy) и платёжным провайдером; 152-ФЗ для медданных. Конкурентное преимущество — качество и удобство видеоконсультаций и матчинг «пациент–врач».

## ТРЕБОВАНИЯ
Сдайте стратегический анализ:
1. **Bounded context'ы** — выделить и описать модель/язык каждого.
2. **Context map** — отношения между контекстами; ACL для ГИС рецептов и платёжного провайдера.
3. **Поддомены** — core / supporting / generic + стратегия (что строить, что купить).
4. **Единый язык** — глоссарий ключевых терминов одного контекста (устранить синонимы).
5. **Агрегаты** — для контекста «Консультации»: корни, границы, инварианты.
6. **Связь с архитектурой** — контексты → сервисы (database per service), транзакции между ними (saga); 152-ФЗ для медданных.

## ФОРМАТ РЕЗУЛЬТАТА
Документ: контексты + context map (с ACL) + классификация поддоменов + глоссарий + агрегаты + связь с сервисами/saga + замечания по 152-ФЗ.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Bounded context'ы выделены и описаны (модель/язык).
- [ ] Context map с типами отношений; ACL для ГИС и платёжного провайдера.
- [ ] Поддомены классифицированы (core = видеоконсультации/матчинг; generic = авторизация/уведомления) со стратегией.
- [ ] Глоссарий единого языка для одного контекста (синонимы устранены).
- [ ] Агрегаты «Консультации» с корнями, границами и инвариантами.
- [ ] Контексты → сервисы (database per service), межсервисные транзакции → saga.
- [ ] Медданные/152-ФЗ отмечены (контексты с перс./медданными).

## ПОДСКАЗКА
Синтез: контексты (зад.01), context map (02), агрегаты (03), entity/VO (04), язык (05), поддомены (06), ACL (07), связь с сервисами (09). Главные проверки middle+: контексты по домену с единым языком, корректная context map с ACL для legacy, верная классификация core/generic (видеоконсультации = core), агрегаты как границы согласованности, переход к сервисам с saga, учёт 152-ФЗ. Стыкуется с модулями 21 (стили), 25 (распределёнка), 27 (интеграции), 34 (доступ).
