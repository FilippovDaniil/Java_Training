# Задача 02 — Модуль 25: Распределённые системы

## ЗАДАНИЕ
Спроектируйте saga для распределённой транзакции заказа: локальные транзакции + компенсации при сбое.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, оформление заказа через сервисы: Заказы (создать заказ), Платежи (списать оплату), Каталог/Стоп-лист (зарезервировать блюда), Доставка (назначить курьера). Нет общей транзакции. Любой шаг может упасть.

## ТРЕБОВАНИЯ
- Опишите шаги saga (последовательность локальных транзакций).
- Для каждого шага определите компенсирующее действие.
- Покажите сценарий сбоя (например, нет курьера) и откат через компенсации.
- Подчеркните: компенсация — бизнес-действие (вернуть деньги), не rollback БД.
- Отметьте, что нужна идемпотентность шагов (повторы).

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Шаг | Локальная транзакция | Компенсация` + сценарий сбоя с откатом.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Шаги saga перечислены по порядку.
- [ ] Для каждого шага указана компенсация (отменить заказ, вернуть оплату, снять резерв, отменить доставку).
- [ ] Разобран сценарий сбоя (нет курьера → компенсации в обратном порядке).
- [ ] Подчёркнуто, что компенсация — бизнес-действие, не откат БД.
- [ ] Отмечена необходимость идемпотентности шагов и компенсаций.
- [ ] Учтено, что часть компенсаций «грязные» (деньги уже списаны → возврат).

## ПОДСКАЗКА
Saga = цепочка локальных транзакций; при сбое — компенсации в обратном порядке. Шаг «списать оплату» компенсируется «вернуть оплату» (бизнес-действие, деньги уже ушли — это возврат, не rollback). Резерв блюд → снять резерв; заказ → отменить. Шаги и компенсации должны быть идемпотентны (возможны повторы).
