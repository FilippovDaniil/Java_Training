# Задача 03 — Модуль 25: Распределённые системы

## ЗАДАНИЕ
Выберите между оркестрацией и хореографией для saga и обоснуйте под сценарии.

## ВХОДНЫЕ ДАННЫЕ
Сценарии:
1. Оформление заказа FoodExpress: 4 шага с ветвлениями (если оплата не прошла — иной путь; если нет курьера — компенсации), нужен явный контроль и отслеживание.
2. Уведомления: при событии «заказ доставлен» несколько сервисов независимо реагируют (отправить чек, начислить бонусы, попросить отзыв) — простая реакция без общего потока.

## ТРЕБОВАНИЯ
- Для каждого сценария выберите оркестрацию или хореографию.
- Обоснуйте через: сложность потока, ветвления, отслеживаемость, связанность.
- Опишите плюсы и минусы каждого подхода.
- Отметьте риск каждого (оркестратор — связность; хореография — трудно проследить поток).

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Сценарий | Подход | Обоснование` + плюсы/минусы оркестрации vs хореографии.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Сложный заказ с ветвлениями → оркестрация (явный контроль, отслеживаемость).
- [ ] Реакция нескольких сервисов на событие → хореография (слабая связанность).
- [ ] Описаны плюсы/минусы обоих подходов.
- [ ] Отмечен риск оркестратора (центральная точка сложности/связности).
- [ ] Отмечен риск хореографии (трудно проследить общий поток, «событийный спагетти»).
- [ ] Выбор обоснован сложностью потока.

## ПОДСКАЗКА
Оркестрация — центральный «дирижёр» командует шагами: хороша для сложных процессов с ветвлениями и компенсациями (видно весь поток). Хореография — сервисы реагируют на события друг друга: хороша для простых независимых реакций (слабая связанность), но общий поток размыт. Сложный заказ → оркестрация; «на событие реагируют многие» → хореография.
