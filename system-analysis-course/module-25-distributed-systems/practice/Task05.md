# Задача 05 — Модуль 25: Распределённые системы

## ЗАДАНИЕ
Спроектируйте Circuit Breaker: состояния, пороги и поведение при отказе зависимости.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, сервис Заказы вызывает внешний платёжный сервис. Платёжный сервис иногда деградирует (отвечает медленно/ошибками). Без защиты — заказы зависают, потоки исчерпываются, каскадный отказ.

## ТРЕБОВАНИЯ
- Опишите три состояния circuit breaker (Closed / Open / Half-Open) и переходы.
- Задайте пороги (например, % ошибок за окно → Open; таймер → Half-Open; успех → Closed).
- Опишите поведение в каждом состоянии (что происходит с запросами).
- Предложите fallback при Open (что вернуть клиенту/как деградировать).
- Объясните, как CB предотвращает каскадный отказ.

## ФОРМАТ РЕЗУЛЬТАТА
Диаграмма состояний CB + пороги переходов + поведение по состояниям + fallback.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Три состояния: Closed (норма), Open (не зовём), Half-Open (пробный запрос).
- [ ] Переходы: много ошибок → Open; по таймеру → Half-Open; успех → Closed; ошибка в Half-Open → Open.
- [ ] Заданы пороги (% ошибок/окно, таймаут открытия).
- [ ] В Open запросы сразу получают ошибку/fallback, не идут в мёртвый сервис.
- [ ] Предложен fallback (очередь на потом, понятная ошибка, деградация).
- [ ] Объяснено предотвращение каскадного отказа.

## ПОДСКАЗКА
Circuit Breaker как электрический предохранитель: Closed — пропускает; при превышении порога ошибок → Open (запросы сразу падают/fallback, сервис не добивается); через таймаут → Half-Open (один пробный запрос); успех → Closed, ошибка → снова Open. Защищает от каскадных отказов (не держит потоки на мёртвом сервисе). Сочетается с таймаутом и retry.
