# Задача 06 — Модуль 25: Распределённые системы

## ЗАДАНИЕ
Спроектируйте Retry с backoff и защитой от дублей: безопасные повторы при временных сбоях.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, сервис Заказы вызывает сервис Доставка для назначения курьера. Иногда Доставка временно недоступна (короткие сбои сети). Нужно повторять, но: не создавать дубли назначений и не добить сервис лавиной повторов.

## ТРЕБОВАНИЯ
- Спроектируйте стратегию retry: число попыток, экспоненциальный backoff, jitter.
- Покажите, почему retry обязателен В ПАРЕ с идемпотентностью (иначе дубли).
- Объясните проблему retry storm и роль backoff+jitter.
- Определите, какие ошибки повторять (временные/5xx/таймаут), а какие — нет (4xx/бизнес-ошибки).
- Свяжите с circuit breaker (когда прекратить повторы).

## ФОРМАТ РЕЗУЛЬТАТА
Стратегия retry (попытки/backoff/jitter) + какие ошибки повторять + связка с идемпотентностью и CB.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Заданы число попыток и экспоненциальный backoff с jitter.
- [ ] Подчёркнуто: retry безопасен только при идемпотентности операции (иначе дубли).
- [ ] Объяснён retry storm и роль backoff+jitter в его предотвращении.
- [ ] Разделены повторяемые ошибки (таймаут, 5xx, сетевые) и неповторяемые (4xx, бизнес-ошибки).
- [ ] Указана связь с circuit breaker (прекратить повторы при стойком отказе).
- [ ] Idempotency key используется для назначения курьера.

## ПОДСКАЗКА
Retry повторяет при временных сбоях, но: (1) только идемпотентные операции (или с idempotency key — иначе дубли); (2) с экспоненциальным backoff + jitter (иначе retry storm добьёт сервис); (3) только повторяемые ошибки (таймаут/5xx/сеть, НЕ 4xx/бизнес); (4) ограниченное число попыток, дальше — circuit breaker/DLQ. Retry + idempotency — неразлучны.
