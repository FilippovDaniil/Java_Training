# Задача 10 (мини-проект) — Модуль 25: Распределённые системы

## ЗАДАНИЕ
Спроектируйте устойчивость сквозного сценария оформления заказа FoodExpress: saga + outbox + идемпотентность + circuit breaker + retry + гарантии доставки + UX. Синтез теории модуля.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, оформление заказа в распределённой архитектуре: сервисы Заказы, Платежи (внешний провайдер), Каталог/Стоп-лист, Доставка, Уведомления. Брокер сообщений (at-least-once). Требования: не двойных списаний, не потерянных событий, устойчивость к сбоям внешнего платёжного провайдера, честный UX при асинхронной обработке.

## ТРЕБОВАНИЯ
Сдайте проект устойчивости:
1. **Saga** — шаги оформления + компенсации; оркестрация или хореография (обосновать).
2. **Идемпотентность** — idempotency key для оплаты и ключевых шагов; защита от дублей.
3. **Outbox** — надёжная публикация событий (OrderCreated и др.) без dual write.
4. **Гарантии доставки** — at-least-once + дедупликация на потребителях.
5. **Устойчивость** — timeout/retry+backoff/circuit breaker/fallback на внешних вызовах (платёжный провайдер).
6. **UX при eventual consistency** — промежуточные состояния, read-your-writes.
7. **Диагностика** — что логировать/трейсить для распределённого сценария (связь с модулем 26).

## ФОРМАТ РЕЗУЛЬТАТА
Схема саги (sequence/BPMN или текст) + решения по 7 пунктам + перечень компенсаций + перечень примененных паттернов устойчивости.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Saga: шаги + компенсации; выбран и обоснован способ координации.
- [ ] Idempotency key для оплаты/шагов; защита от двойного списания.
- [ ] Outbox для надёжной публикации событий (решена проблема dual write).
- [ ] At-least-once + дедупликация на потребителях.
- [ ] Timeout/retry+backoff/circuit breaker/fallback на внешнем платёжном вызове.
- [ ] UX: промежуточные состояния и read-your-writes при асинхронности.
- [ ] Отмечено, что логировать/трейсить (корреляция, наблюдаемость — модуль 26).

## ПОДСКАЗКА
Синтез всего модуля: идемпотентность (зад.01), saga+компенсации (02), оркестрация/хореография (03), outbox (04), circuit breaker (05), retry+backoff (06), гарантии+дедуп (07), UX (08), точки сбоя (09). Главные проверки middle+: распределённая операция как saga (не ACID), безопасные повторы (идемпотентность+retry), надёжные события (outbox+at-least-once+дедуп), устойчивость к внешним сбоям (timeout/CB/fallback), честный UX. Стыкуется с DDD (24), async (32), sequence (33), NFR-арх (26).
