# Задача 01 — Модуль 26: Нефункциональная архитектура

## ЗАДАНИЕ
Рассчитайте доступность системы из компонентов: для последовательного и параллельного соединения.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, запрос оформления заказа проходит через компоненты (синхронно, каждый обязателен):
- API Gateway — 99.9%
- Сервис Заказы — 99.9%
- Сервис Платежи — 99.5%
- БД — 99.95%
Отдельно: рассматривается резервирование сервиса Платежи двумя экземплярами (по 99.5% каждый).

## ТРЕБОВАНИЯ
- Рассчитайте итоговую доступность последовательной цепочки (перемножение).
- Рассчитайте доступность пары резервированных экземпляров Платежей (`1−(1−A)²`).
- Покажите, как резервирование Платежей меняет итог цепочки.
- Объясните вывод: цепочка зависимостей снижает доступность.

## ФОРМАТ РЕЗУЛЬТАТА
Расчёты (с формулами и числами) + вывод о влиянии длины цепочки и резервирования.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Последовательная доступность = 0.999 × 0.999 × 0.995 × 0.9995 ≈ 0.9925 (~99.25%).
- [ ] Резерв Платежей: 1 − (1−0.995)² = 1 − 0.000025 = 0.999975.
- [ ] Показано улучшение цепочки при замене 99.5% на резервированные ~99.9975%.
- [ ] Сформулирован вывод: умножение доступностей → итог ниже каждого звена.
- [ ] Объяснено, что резервирование (параллель) повышает доступность.
- [ ] Формулы приведены явно.

## ПОДСКАЗКА
Последовательно (все обязательны) — доступности перемножаются: итог ниже самого слабого. Параллельно (резерв) — считается через вероятность одновременного отказа: `1−(1−A)ⁿ`. Слабое звено (Платежи 99.5%) тянет цепочку вниз; резервирование его поднимает. Вывод: меньше синхронных зависимостей + резерв критичных.
