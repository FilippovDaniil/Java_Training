# Задача 08 — Модуль 26: Нефункциональная архитектура

## ЗАДАНИЕ
Найдите и устраните точки отказа (SPOF) в архитектуре.

## ВХОДНЫЕ ДАННЫЕ
Архитектура FoodExpress с проблемами:
- один экземпляр API Gateway;
- единственный сервер БД (без реплик);
- один балансировщик нагрузки;
- все сервисы в одной зоне доступности (AZ);
- сессии хранятся на одном Redis-узле;
- единственный экземпляр сервиса Платежи.

## ТРЕБОВАНИЯ
- Найдите все SPOF (single point of failure).
- Для каждого предложите устранение (резервирование, мультизона, кластер).
- Расставьте приоритеты (что критичнее устранить первым).
- Отметьте, что после резервирования нужны health checks и failover.
- Свяжите с расчётом доступности (задача 01) — как резерв повышает SLA.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `SPOF | Риск | Устранение | Приоритет` + замечание про health checks/failover.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Найдены все SPOF (Gateway, БД, балансировщик, одна AZ, Redis, Платежи).
- [ ] Для каждого предложено резервирование (несколько экземпляров/кластер/мультизона).
- [ ] БД → реплики/кластер с failover; Gateway/балансировщик → несколько экземпляров.
- [ ] Одна AZ → размещение в нескольких зонах.
- [ ] Расставлены приоритеты устранения.
- [ ] Отмечена роль health checks и автоматического failover.

## ПОДСКАЗКА
SPOF — компонент, отказ которого роняет всю систему. Любой «единственный» (один Gateway, одна БД, один балансировщик, одна зона, один Redis, один сервис) — SPOF. Устранение: резервирование (≥2 экземпляра), кластер с failover, мультизона. После — health checks для автоматического переключения. Резерв повышает доступность (задача 01: параллель `1−(1−A)ⁿ`).
