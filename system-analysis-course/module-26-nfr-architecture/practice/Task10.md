# Задача 10 (мини-проект) — Модуль 26: Нефункциональная архитектура

## ЗАДАНИЕ
Спроектируйте NFR-архитектуру FoodExpress: масштабируемость, доступность (SLA/SLO/SLI + расчёт), отказоустойчивость (SPOF/degradation), наблюдаемость, capacity planning, error budget. Синтез теории модуля и связь с NFR (модуль 09).

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress** на пике (Чёрная пятница, ×5 нагрузка). Бизнес: критичны оформление и оплата заказа (downtime = потеря денег), некритичны рекомендации/баннеры. Требования: высокая доступность ключевого пути, масштаб под пик, быстрая диагностика инцидентов, обоснованные SLA.

## ТРЕБОВАНИЯ
Сдайте NFR-архитектуру:
1. **Масштабируемость** — stateless-сервисы, горизонтальный масштаб, автоскейлинг, capacity под пик.
2. **Доступность** — SLI/SLO/SLA ключевого пути; расчёт доступности из компонентов; обоснование «девяток».
3. **Отказоустойчивость** — устранение SPOF (резерв/мультизона), graceful degradation некритичного, failover.
4. **Наблюдаемость** — логи/метрики/трейсы, correlation ID, золотые сигналы → SLI.
5. **Error budget** — политика «релизить/стабилизировать».
6. **Связь с NFR (модуль 09)** — измеримые требования ↔ архитектурные решения.

## ФОРМАТ РЕЗУЛЬТАТА
Документ NFR-архитектуры по 6 пунктам + расчёт доступности ключевого пути + перечень устранённых SPOF + набор SLI/SLO.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Stateless + горизонтальный масштаб + capacity под пик (×5).
- [ ] SLI/SLO/SLA ключевого пути + расчёт доступности из компонентов.
- [ ] SPOF найдены и устранены (резерв/мультизона); failover + health checks.
- [ ] Graceful degradation некритичных зависимостей.
- [ ] Наблюдаемость: три столпа + correlation ID + золотые сигналы → SLI.
- [ ] Error budget с политикой релиз/стабилизация.
- [ ] Решения увязаны с измеримыми NFR (модуль 09).

## ПОДСКАЗКА
Синтез: расчёт SLA (зад.01), stateless (02), SLI/SLO (03), degradation (04), наблюдаемость (05), capacity (06), error budget (07), SPOF (08), health checks (09). Главные проверки middle+: измеримые SLO под бизнес-критичность (не «100%»), расчёт доступности цепочки, устранённые SPOF + degradation, наблюдаемость для распределёнки, error budget как политика. Стыкуется с NFR (09), данными (20), распределёнкой (25), эксплуатацией (37).
