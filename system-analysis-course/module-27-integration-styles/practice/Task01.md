# Задача 01 — Модуль 27: Способы интеграции (EIP)

## ЗАДАНИЕ
Подберите способ интеграции под 5 кейсов и обоснуйте через связанность и требования.

## ВХОДНЫЕ ДАННЫЕ
Кейсы (FoodExpress / MedConnect):
1. Ночная выгрузка отчётов в систему бухгалтерии (пакетно, раз в сутки).
2. Проверка баланса бонусов при оформлении заказа (нужен немедленный ответ).
3. Уведомление сервисов о событии «заказ создан» (несколько подписчиков, развязка).
4. Интеграция со старой системой партнёра, умеющей только обмен файлами по FTP.
5. Синхронизация каталога между сервисом Каталог и поисковым индексом.

## ТРЕБОВАНИЯ
- Для каждого кейса выберите способ (file / RPC / messaging; общую БД — не предлагать).
- Обоснуйте через: нужен ли ответ сейчас, развязка, надёжность, возможности партнёра.
- Отметьте связанность каждого варианта.
- Укажите, где async предпочтительнее sync и наоборот.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Кейс | Способ | Обоснование | Связанность`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Отчёты в бухгалтерию → file transfer (пакетно, не нужен realtime).
- [ ] Проверка баланса → sync RPC (немедленный ответ).
- [ ] Событие «заказ создан» → messaging (несколько подписчиков, развязка).
- [ ] Партнёр с FTP → file transfer (по возможностям партнёра).
- [ ] Синхронизация каталога с поиском → messaging/события (CDC).
- [ ] Для каждого указана связанность; общая БД не предложена.

## ПОДСКАЗКА
Выбор по требованиям: нужен немедленный ответ → sync RPC; развязка/несколько подписчиков/устойчивость → messaging; пакетный обмен или ограничения партнёра → file. Общая БД — антипаттерн, не предлагать. Async снижает временную связанность; sync проще, когда ответ нужен сейчас.
