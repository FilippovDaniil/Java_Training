# Задача 02 — Модуль 27: Способы интеграции (EIP)

## ЗАДАНИЕ
Спроектируйте файловый обмен: формат, расписание, обработку ошибок и идемпотентность загрузки.

## ВХОДНЫЕ ДАННЫЕ
**MedConnect** обменивается с государственной системой реестром оказанных консультаций: ежедневная выгрузка файла (CSV/XML) на FTP, ответный файл с результатами обработки. Возможны проблемы: файл не пришёл, пришёл битый, пришёл дважды, опоздал.

## ТРЕБОВАНИЯ
- Определите формат файла и его структуру (с заголовком/контрольной суммой/числом записей).
- Задайте расписание и окно обмена.
- Спроектируйте обработку ошибок: нет файла, битый файл, частичная обработка.
- Обеспечьте идемпотентность загрузки (повторный/дублированный файл не задваивает данные).
- Учтите 152-ФЗ (медданные в файле: шифрование/защищённый канал).

## ФОРМАТ РЕЗУЛЬТАТА
Описание формата + расписание + сценарии ошибок и реакции + механизм идемпотентности + замечание о безопасности.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Формат файла с метаданными (число записей, контрольная сумма/хеш).
- [ ] Заданы расписание и окно обмена.
- [ ] Обработка ошибок: отсутствие файла, битый, частичная обработка → реакция (алерт/повтор/карантин).
- [ ] Идемпотентность: повторный файл (по имени/хешу/идентификатору пакета) не задваивает данные.
- [ ] Подтверждение обработки (ответный файл/квитанция).
- [ ] 152-ФЗ: защищённый канал/шифрование медданных.

## ПОДСКАЗКА
Файловый обмен надёжен, если продуман: метаданные (число записей + контрольная сумма для проверки целостности), расписание/окно, реакции на сбои (нет файла → алерт; битый → карантин; частичная обработка → откат/докачка). Идемпотентность: распознавать повтор по идентификатору/хешу пакета. Медданные — только по защищённому каналу (152-ФЗ).
