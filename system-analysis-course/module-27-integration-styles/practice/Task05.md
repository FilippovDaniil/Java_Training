# Задача 05 — Модуль 27: Способы интеграции (EIP)

## ЗАДАНИЕ
Спроектируйте message router для маршрутизации заказов: направьте сообщения нужным обработчикам по содержимому.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, единый поток входящих заказов нужно направлять по-разному: заказы еды → на кухню ресторана; заказы товаров (магазин) → на склад; заказы с алкоголем → на проверку возраста; срочные заказы → в приоритетную очередь.

## ТРЕБОВАНИЯ
- Спроектируйте content-based router: правила маршрутизации по содержимому сообщения.
- Определите критерии маршрутизации (тип товара, флаг срочности, наличие алкоголя).
- Покажите каналы-назначения (кухня, склад, проверка, приоритет).
- Учтите случай «не подошло ни одно правило» (default/DLQ).
- Отметьте, что router не содержит бизнес-логики обработки, только маршрутизацию.

## ФОРМАТ РЕЗУЛЬТАТА
Схема content-based router (правила → каналы) + обработка default/нераспознанного.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Правила маршрутизации по содержимому (тип товара/срочность/алкоголь).
- [ ] Каждое правило ведёт в свой канал (кухня/склад/проверка/приоритет).
- [ ] Обработан случай «ни одно правило не подошло» (default-канал/DLQ).
- [ ] Router только маршрутизирует, не обрабатывает (нет бизнес-логики).
- [ ] Возможна комбинация правил (срочный + еда).
- [ ] Каналы независимы (получатели не знают о router).

## ПОДСКАЗКА
Content-Based Router (EIP) направляет сообщение по его содержимому: смотрит поля (тип, флаги) → выбирает канал-назначение. Правила: еда→кухня, товар→склад, алкоголь→проверка возраста, срочный→приоритет. Нераспознанное → default/DLQ. Router не обрабатывает заказ — только решает, куда отправить (разделение маршрутизации и обработки).
