# Задача 01 — Модуль 28: Форматы данных и схемы

## ЗАДАНИЕ
Спроектируйте JSON-представление сущности «Заказ» FoodExpress с корректными типами и структурой.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, сущность Заказ: id, клиент (id и имя), ресторан, список позиций (блюдо, количество, цена), сумма, статус, дата создания, адрес доставки. Деньги — рубли с копейками.

## ТРЕБОВАНИЯ
- Спроектируйте JSON-объект заказа со всеми полями.
- Выберите корректные типы: деньги (не float — строка или копейки-целое), дата (ISO 8601 с таймзоной), статус (enum-строка).
- Позиции — массив объектов (не поля-через-индекс).
- Адрес — вложенный объект или строка (обоснуйте).
- Отметьте обязательные и опциональные поля.

## ФОРМАТ РЕЗУЛЬТАТА
JSON-объект заказа + таблица `Поле | Тип | Обязательность | Комментарий по типу`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] JSON валиден, со всеми полями заказа.
- [ ] Деньги — не float (строка "300.00" или целое в копейках), обоснование.
- [ ] Дата — ISO 8601 с таймзоной.
- [ ] Статус — строка из перечня (enum).
- [ ] Позиции — массив объектов (не item_1, item_2).
- [ ] Обязательные/опциональные поля размечены.

## ПОДСКАЗКА
Корректные типы критичны: деньги во float теряют копейки → строка "300.00" или целое в копейках (30000); даты — ISO 8601 с таймзоной (2026-06-25T10:00:00Z); статус — enum-строка. Список → массив объектов, не поля-через-индекс (антипаттерн). Вложенность под цельный объект. Это вход для JSON Schema (задача 02).
