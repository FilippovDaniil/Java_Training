# Задача 02 — Модуль 28: Форматы данных и схемы

## ЗАДАНИЕ
Напишите JSON Schema для заказа (из задачи 01) и провалидируйте корректные/некорректные примеры.

## ВХОДНЫЕ ДАННЫЕ
JSON-объект заказа из задачи 01. Нужно формализовать контракт схемой и проверить валидацию.

## ТРЕБОВАНИЯ
- Напишите JSON Schema: типы полей, `required`, ограничения (`minimum`, `enum`, `minItems`).
- Опишите вложенные объекты (клиент, адрес) и массив позиций.
- Приведите 1 валидный пример и 2 невалидных (нарушают схему) с указанием, что именно нарушено.
- Проверьте схему в валидаторе (или объясните результат валидации).

## ФОРМАТ РЕЗУЛЬТАТА
JSON Schema + валидный пример + 2 невалидных примера с пояснением нарушений.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Схема описывает типы всех полей и `required`.
- [ ] Заданы ограничения: `minimum` для суммы (≥0), `enum` для статуса, `minItems:1` для позиций.
- [ ] Описаны вложенные объекты и массив позиций.
- [ ] Валидный пример проходит схему.
- [ ] Невалидный 1: нарушает тип/required (например, нет обязательного поля).
- [ ] Невалидный 2: нарушает ограничение (статус вне enum / пустой массив позиций / отрицательная сумма).

## ПОДСКАЗКА
JSON Schema = контракт: `type`, `required`, `properties`, ограничения (`minimum`, `enum`, `minItems`). Для вложенных — `type:object` с `properties`; для массива — `type:array` + `items` + `minItems`. Невалидные примеры подбирайте так, чтобы они нарушали конкретные правила (нет required-поля; статус не из enum). Проверка — на jsonschemavalidator.net.
