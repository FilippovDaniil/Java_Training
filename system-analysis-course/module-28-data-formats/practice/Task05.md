# Задача 05 — Модуль 28: Форматы данных и схемы

## ЗАДАНИЕ
Спроектируйте XML + XSD для интеграции с госсистемой и объясните особенности XML.

## ВХОДНЫЕ ДАННЫЕ
**MedConnect** интегрируется с государственной системой рецептов, которая принимает только XML с валидацией по XSD. Нужно описать рецепт: id, дата, пациент (полис), врач, позиции (препарат, дозировка, количество).

## ТРЕБОВАНИЯ
- Спроектируйте XML-документ рецепта.
- Опишите ключевые элементы XSD (типы, обязательность, повторяющиеся элементы для позиций).
- Покажите особенности XML: элементы vs атрибуты, namespaces, типизация через XSD.
- Сравните с JSON-вариантом: что в XML сложнее/строже.
- Учтите 152-ФЗ (медданные: защищённый канал).

## ФОРМАТ РЕЗУЛЬТАТА
XML-документ рецепта + фрагмент XSD (ключевые элементы) + сравнение XML vs JSON для этого случая.

## КРИТЕРИИ ПРИЁМКИ
- [ ] XML-документ рецепта со вложенными позициями (повторяющиеся элементы).
- [ ] XSD: типы, обязательность, повторяемость (`maxOccurs`), ограничения.
- [ ] Показаны особенности XML (элементы/атрибуты, namespace, строгая типизация через XSD).
- [ ] Сравнение с JSON (XML многословнее/строже, есть namespaces/XSD).
- [ ] Учтён 152-ФЗ (защищённый канал передачи медданных).
- [ ] XML валиден относительно описанной XSD.

## ПОДСКАЗКА
XML строже и многословнее JSON: элементы vs атрибуты, namespaces, валидация по XSD (типы, `minOccurs`/`maxOccurs` для повторяющихся позиций). Госсистемы часто требуют именно XML+XSD. Позиции рецепта → повторяющиеся элементы. Медданные → защищённый канал (152-ФЗ). Внутри MedConnect можно держать свою модель, а к ГИС переводить через translator/ACL (модули 27/24).
