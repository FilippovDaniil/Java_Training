# Задача 06 — Модуль 28: Форматы данных и схемы

## ЗАДАНИЕ
Найдите ошибки в JSON-контракте и исправьте их.

## ВХОДНЫЕ ДАННЫЕ
JSON-контракт ответа API (FoodExpress) с проблемами:
```json
{
  "order_id": "1024",
  "total": 599.99,
  "createdAt": "25.06.2026 10:00",
  "status": 2,
  "item_1": "Ролл",
  "item_2": "Суп",
  "isPaid": "true",
  "client": "Иванов; i@mail.ru; +7900..."
}
```

## ТРЕБОВАНИЯ
- Найдите проблемы: тип id, деньги float, формат даты, статус как число без словаря, поля-через-индекс, булево строкой, склеенные данные клиента.
- Для каждой объясните, чем плоха, и предложите исправление.
- Приведите исправленный JSON.
- Свяжите с принципами (типы, массивы вместо индексов, атомарность — модуль 17).

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Проблема | Чем плохо | Исправление` + исправленный JSON.

## КРИТЕРИИ ПРИЁМКИ
- [ ] `order_id` как строка вместо числа (или несогласованность с типом) — отмечено.
- [ ] `total` float → строка/копейки (точность денег).
- [ ] `createdAt` нестандартный формат → ISO 8601 с таймзоной.
- [ ] `status: 2` (магическое число) → строковый enum.
- [ ] `item_1/item_2` → массив `items`.
- [ ] `isPaid: "true"` (строка) → булево `true`.
- [ ] Склеенный `client` → объект `{name, email, phone}` (атомарность).
- [ ] Приведён исправленный JSON.

## ПОДСКАЗКА
Типовые ошибки контракта: деньги float; даты в произвольном формате; статус магическим числом; поля-через-индекс вместо массива; булево строкой; несколько значений в одном поле (нарушение атомарности, модуль 17). Исправления: корректные типы, ISO-даты, enum-статусы, массивы, вложенный объект клиента. Хороший контракт = предсказуемые типы + схема (задача 02).
