# Задача 01 — Модуль 29: REST API

## ЗАДАНИЕ
Спроектируйте ресурсную модель и URI для FoodExpress: существительные, множественное число, без глаголов, вложенность ≤ 2.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, сущности и связи: клиенты, заказы (у клиента), позиции заказа, рестораны, блюда (у ресторана), оплаты, отзывы.

## ТРЕБОВАНИЯ
- Определите URI для основных ресурсов (коллекции и отдельные ресурсы).
- Используйте существительные во множественном числе, без глаголов.
- Покажите вложенные ресурсы (заказы клиента, блюда ресторана) с вложенностью ≤ 2 уровней.
- Отметьте, где НЕ стоит вкладывать (а ссылаться на ресурс верхнего уровня).

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Ресурс | URI коллекции | URI элемента | Комментарий` + примеры вложенных ресурсов.

## КРИТЕРИИ ПРИЁМКИ
- [ ] URI — существительные во множественном числе (`/orders`, `/clients`).
- [ ] Нет глаголов в URI.
- [ ] Вложенность показана (`/clients/{id}/orders`, `/restaurants/{id}/dishes`) и не глубже 2 уровней.
- [ ] Отдельный ресурс — по id (`/orders/{orderId}`).
- [ ] Отмечено, где лучше плоский URI вместо глубокой вложенности.
- [ ] Модель покрывает основные сущности.

## ПОДСКАЗКА
Ресурс = существительное во множественном числе. Действия — методами, не в URI. Вложенность отражает принадлежность (`/clients/{id}/orders`), но ≤ 2 уровней: глубже (`/clients/{id}/orders/{id}/items/{id}/options`) — неудобно, лучше `/order-items/{id}`. Конвенция проекта: множественное число, без глаголов, ≤ 2 уровня.
