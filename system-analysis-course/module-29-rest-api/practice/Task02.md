# Задача 02 — Модуль 29: REST API

## ЗАДАНИЕ
Назначьте HTTP-методы и статусы CRUD-операциям над ресурсами.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, операции над заказами и блюдами:
1. Создать заказ.
2. Получить заказ по id.
3. Получить список заказов клиента.
4. Полностью заменить данные блюда.
5. Частично изменить блюдо (цена).
6. Удалить блюдо.
7. Запросить несуществующий заказ.
8. Создать заказ с нарушением бизнес-правила (пустая корзина).

## ТРЕБОВАНИЯ
- Для каждой операции укажите метод, URI и статус успеха.
- Для ошибок (7, 8) укажите корректный статус.
- Отметьте заголовки (Location при создании).
- Обоснуйте различения 201/200/204 и 404/422.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Операция | Метод | URI | Статус успеха | Статус ошибки`.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Создать → `POST /orders` → 201 + Location.
- [ ] Получить по id → `GET /orders/{id}` → 200.
- [ ] Список заказов клиента → `GET /clients/{id}/orders` → 200.
- [ ] Полная замена блюда → `PUT /dishes/{id}` → 200/204.
- [ ] Частичное изменение → `PATCH /dishes/{id}` → 200/204; удаление → `DELETE` → 204.
- [ ] Несуществующий заказ → 404; пустая корзина (бизнес-правило) → 422.

## ПОДСКАЗКА
Метод по семантике: POST=создать (201+Location), GET=читать (200), PUT=полная замена, PATCH=частичное, DELETE=удалить (204). Ошибки: нет ресурса → 404; валиден, но нарушает бизнес-правило → 422 (не 400). 201 для создания, 204 для «успех без тела».
