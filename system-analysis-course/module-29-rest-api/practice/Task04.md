# Задача 04 — Модуль 29: REST API

## ЗАДАНИЕ
Выберите тип пагинации (offset vs cursor) для разных коллекций и обоснуйте.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, коллекции с пагинацией:
1. Лента новых заказов в реальном времени (постоянно добавляются, бесконечная прокрутка).
2. Админка: таблица заказов с переходом на произвольную страницу и общим числом страниц.
3. Каталог блюд ресторана (стабильный, небольшой).
4. История заказов клиента за всё время (большой объём).

## ТРЕБОВАНИЯ
- Для каждой коллекции выберите offset/limit или cursor.
- Обоснуйте через: размер данных, изменчивость, нужны ли номера страниц/прыжки, стабильность.
- Покажите формат параметров запроса для каждого варианта.
- Объясните проблемы offset на больших/изменяющихся данных.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Коллекция | Пагинация | Параметры | Обоснование` + разбор проблем offset.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Лента реального времени → cursor (стабильность при вставках, бесконечная прокрутка).
- [ ] Админка с номерами страниц → offset/limit (нужны прыжки и общее число).
- [ ] Стабильный небольшой каталог → offset (просто, данные малы).
- [ ] Большая история → cursor (производительность на больших данных).
- [ ] Показаны параметры (`?offset=&limit=` / `?cursor=&limit=`).
- [ ] Объяснены проблемы offset (медленно на больших, сдвиг при вставках).

## ПОДСКАЗКА
Offset/limit — прост, позволяет прыжки на страницу и подсчёт страниц, но на больших данных медленный (БД пропускает N строк) и нестабилен при вставках (дубли/пропуски). Cursor (keyset) — стабилен и быстр для лент/больших объёмов, но нельзя прыгнуть на произвольную страницу. Лента/большие данные → cursor; админка с номерами → offset.
