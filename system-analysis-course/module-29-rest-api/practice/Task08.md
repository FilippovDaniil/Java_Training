# Задача 08 — Модуль 29: REST API

## ЗАДАНИЕ
Спроектируйте фильтрацию, сортировку и проекцию полей для коллекции заказов.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, `GET /orders`. Нужны: фильтр по статусу, ресторану, диапазону суммы и дат; сортировка по дате/сумме (по возр./убыв.); выбор только нужных полей (для мобильного — компактно).

## ТРЕБОВАНИЯ
- Спроектируйте query-параметры фильтрации (status, restaurantId, minTotal, dateFrom/dateTo).
- Спроектируйте сортировку (`sort=-createdAt`, поддержка нескольких полей).
- Спроектируйте проекцию полей (`fields=id,total,status`).
- Покажите комбинированный пример запроса.
- Отметьте связь с пагинацией (задача 04) и безопасностью (не давать фильтровать/сортировать по произвольному полю).

## ФОРМАТ РЕЗУЛЬТАТА
Спецификация query-параметров (фильтр/сортировка/проекция) + комбинированный пример + замечание о безопасности.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Фильтры через query-параметры (status, restaurantId, minTotal, dateFrom/dateTo).
- [ ] Сортировка с направлением (`sort=-createdAt`), поддержка нескольких ключей.
- [ ] Проекция полей (`fields=...`) для компактных ответов.
- [ ] Приведён комбинированный пример с пагинацией.
- [ ] Отмечено ограничение допустимых полей фильтра/сортировки (безопасность/перформанс).
- [ ] Параметры согласованы и предсказуемы.

## ПОДСКАЗКА
Фильтрация/сортировка/проекция — через query-параметры: `?status=delivered&minTotal=500&sort=-createdAt&fields=id,total&limit=20`. Направление сортировки — префикс `-` (убыв.). Проекция (`fields`) экономит трафик (мобайл). Безопасность/перформанс: разрешать фильтр/сортировку только по индексированным/допустимым полям (не по произвольным). Сочетается с пагинацией (задача 04).
