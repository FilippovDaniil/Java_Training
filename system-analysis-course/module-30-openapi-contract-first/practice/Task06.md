# Задача 06 — Модуль 30: Contract-first / OpenAPI

## ЗАДАНИЕ
Найдите ошибки в готовой OpenAPI-спецификации и исправьте их.

## ВХОДНЫЕ ДАННЫЕ
Фрагмент OpenAPI с проблемами:
```yaml
openapi: 3.0.3
info:
  title: FoodExpress API
  # отсутствует version
paths:
  /createOrder:                      # глагол в пути
    post:
      responses:
        '200':                       # создание должно быть 201
          description: ok
          # нет схемы ответа
        # нет описания ошибок (4xx)
  /orders/{id}:
    get:
      # параметр пути {id} не объявлен в parameters
      responses:
        '200':
          content:
            application/json:
              schema:
                type: object
                properties:
                  total: { type: float }   # нет такого типа в OpenAPI
components:
  schemas:
    Order:
      # схема Order объявлена, но нигде не используется ($ref отсутствует)
      type: object
```

## ТРЕБОВАНИЯ
- Найдите все ошибки (нет version, глагол в пути, неверный статус, нет схемы/ошибок, необъявленный path-параметр, неверный тип, неиспользуемая схема).
- Объясните каждую.
- Предложите исправление.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица `Ошибка | Чем плоха | Исправление` + при желании исправленный фрагмент.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Отсутствует `info.version` → добавить.
- [ ] `/createOrder` (глагол) → `/orders` + POST.
- [ ] Создание возвращает 200 → должно быть 201 (+ Location).
- [ ] Нет схемы ответа и нет ошибок 4xx → добавить.
- [ ] Path-параметр `{id}` не объявлен в `parameters` → объявить.
- [ ] `type: float` невалиден → `type: number` (или string для денег).
- [ ] Неиспользуемая схема Order → подключить через `$ref`.

## ПОДСКАЗКА
Типовые ошибки спеки: нет обязательного `info.version`; глагол в пути; неверные статусы (200 вместо 201); отсутствие схем ответов и ошибок; необъявленные path-параметры (нужен блок `parameters` с `in: path, required: true`); неверные типы (`float` нет — есть `number` с `format`, но деньги лучше string); «мёртвые» схемы без `$ref`. Валидация в Swagger Editor подсветит часть.
