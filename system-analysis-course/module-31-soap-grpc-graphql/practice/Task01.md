# Задача 01 — Модуль 31: SOAP / gRPC / GraphQL

## ЗАДАНИЕ
Постройте сравнительную таблицу четырёх стилей API: REST, SOAP, gRPC, GraphQL.

## ВХОДНЫЕ ДАННЫЕ
Обобщённый кейс: вы готовите для команды памятку «какой стиль API когда выбирать».

## ТРЕБОВАНИЯ
- Сравните 4 стиля минимум по критериям: формат данных, транспорт, контракт, стиль (ресурсы/RPC/граф), стриминг, кэширование, работа в браузере, over/underfetching, типичная ниша.
- По каждому стилю — одна короткая фраза «где силён».
- Без воды: таблица + 4 строки вывода.

## ФОРМАТ РЕЗУЛЬТАТА
Markdown-таблица (строки — критерии или стили) + 4 строки «где силён».

## КРИТЕРИИ ПРИЁМКИ
- [ ] Все 4 стиля присутствуют (REST/SOAP/gRPC/GraphQL).
- [ ] Минимум 7 критериев сравнения.
- [ ] Указаны форматы: JSON / XML / Protobuf / JSON.
- [ ] Указаны контракты: OpenAPI / WSDL / `.proto` / Schema(SDL).
- [ ] Корректно отражён стриминг (gRPC — да 4 типа; GraphQL — subscriptions; REST/SOAP — ограниченно/нет).
- [ ] Для каждого стиля сформулирована ниша.

## ПОДСКАЗКА
Опора — таблица из §4 теории. Ключевые различающие оси: формат (текст/бинарный), контракт, ресурсы vs RPC vs граф, стриминг, кэш, браузер. Ниши: REST — публичный CRUD; SOAP — enterprise/legacy/гос; gRPC — межсервис/low-latency; GraphQL — гибкие клиенты/BFF.
