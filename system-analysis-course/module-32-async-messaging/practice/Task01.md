# Задача 01 — Модуль 32: Асинхронная интеграция

## ЗАДАНИЕ
Выберите модель доставки — очередь (point-to-point) или топик (pub/sub) — под каждый сценарий и обоснуйте.

## ВХОДНЫЕ ДАННЫЕ
Сценарии интеграции:
1. Задание на генерацию PDF-чека — обработать должен ровно один воркер из пула.
2. Событие «заказ оплачен» нужно складу, сервису уведомлений и аналитике независимо.
3. Команда «списать средства» — выполняется один раз одним обработчиком.
4. Событие «статус заказа изменился» — на него подписаны мобильное приложение и партнёрский сервис.

## ТРЕБОВАНИЯ
- Для каждого сценария: очередь или топик + 1 фраза почему.
- Сформулируйте критерий различения (один обработчик vs многие подписчики).
- Отметьте, что будет не так при неверном выборе (например, событие в очередь → получит только один потребитель).

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: Сценарий | Очередь/Топик | Обоснование. + 1–2 строки про критерий и риск ошибки.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Сценарии 1 и 3 → очередь (работу делает один).
- [ ] Сценарии 2 и 4 → топик (копию получает каждый подписчик).
- [ ] Сформулирован критерий: «один обработчик» → очередь, «многим» → топик.
- [ ] Указан риск неверного выбора (событие в очередь получит лишь один из потребителей).

## ПОДСКАЗКА
Опора — §2 теории. Очередь = распределение работы (задача/команда, один исполнитель); топик = широковещание события (многим подписчикам). Маркеры «команда/задание» → очередь; «событие, факт произошёл» → топик.
