# Задача 03 — Модуль 32: Асинхронная интеграция

## ЗАДАНИЕ
Выберите брокер — Kafka или RabbitMQ — под две задачи и обоснуйте.

## ВХОДНЫЕ ДАННЫЕ
1. Поток событий действий пользователей (клики, просмотры) для аналитики в реальном времени; объём очень высокий; нужна возможность перечитать историю за сутки.
2. Очередь фоновых задач (генерация чеков, отправка писем) с гибкой маршрутизацией по типу задачи; после обработки сообщение не нужно хранить.

## ТРЕБОВАНИЯ
- Для каждой задачи: брокер + 2–3 решающих критерия (модель, хранение/реплей, throughput, маршрутизация).
- Сравните по сути: лог (Kafka) vs очереди+exchange (RabbitMQ).
- Отметьте, что выбор — про модель данных/нагрузки, а не «модность».

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: Задача | Брокер | Решающие критерии. + 2 строки сравнения моделей.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Задача 1 → Kafka (высокий throughput, хранение/реплей, event streaming).
- [ ] Задача 2 → RabbitMQ (очереди задач, гибкая маршрутизация, удаление после ack).
- [ ] Названы различия моделей: распределённый лог vs exchange→queue.
- [ ] Упомянуты retention/реплей (Kafka) и routing (RabbitMQ).
- [ ] Выбор обоснован свойствами, не вкусом.

## ПОДСКАЗКА
Опора — §3 теории. Маркеры Kafka: «поток событий», «высокий объём», «перечитать историю». Маркеры RabbitMQ: «очередь задач», «маршрутизация по типу», «не хранить после обработки».
