# Задача 03 — Модуль 33: Sequence diagram

## ЗАДАНИЕ
Добавьте в сценарий оплаты фреймы `alt` и `opt` (успех/ошибка + опциональный шаг).

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, сценарий из задачи 01. Добавьте логику:
- `alt`: оплата успешна (резерв + 201) ИЛИ оплата отклонена (402, заказ не оформлен).
- `opt`: если у клиента есть промокод — применить скидку перед оплатой.

## ТРЕБОВАНИЯ
- Используйте фрейм `alt` с двумя ветками (успех/ошибка) и условиями.
- Используйте фрейм `opt` для опционального шага (промокод).
- Корректные ответы клиенту в каждой ветке (201 / 402).
- Сохраните корректность sync-вызовов и return внутри фреймов.

## ФОРМАТ РЕЗУЛЬТАТА
Sequence diagram (PlantUML или текстовая нотация) с фреймами `alt` (успех/ошибка) и `opt` (промокод).

## КРИТЕРИИ ПРИЁМКИ
- [ ] `alt` с двумя взаимоисключающими ветками и условиями.
- [ ] Ветка успеха: резерв + 201; ветка ошибки: 402, без резерва.
- [ ] `opt` для применения промокода (условный шаг).
- [ ] Ответы клиенту различаются по ветке.
- [ ] Сообщения и return внутри фреймов корректны.

## ПОДСКАЗКА
Опора — §3 теории. `alt оплата успешна … else отклонена … end`. `opt есть промокод … end`. `opt` — это `alt` без else (один условный блок). Это перевод «альтернативных потоков/исключений» use case на sequence.
