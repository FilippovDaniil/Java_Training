# Задача 04 — Модуль 33: Sequence diagram

## ЗАДАНИЕ
Примените фреймы `loop` и `par` в сценарии обработки заказа.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, оформление заказа из нескольких позиций:
- `loop`: для каждой позиции заказа проверить наличие на складе.
- `par`: после подтверждения параллельно отправить уведомление клиенту и задание ресторану.

## ТРЕБОВАНИЯ
- Фрейм `loop` по позициям заказа (повторяющаяся проверка наличия).
- Фрейм `par` с двумя параллельными блоками (уведомление + задание ресторану).
- Корректные участники и сообщения внутри фреймов.
- Кратко: почему эти шаги моделируются как цикл и как параллель.

## ФОРМАТ РЕЗУЛЬТАТА
Sequence diagram (PlantUML или текстовая нотация) с `loop` и `par` + 2 строки обоснования.

## КРИТЕРИИ ПРИЁМКИ
- [ ] `loop` по позициям заказа (повтор проверки наличия).
- [ ] `par` с двумя параллельными блоками (уведомление, задание ресторану).
- [ ] Внутри фреймов — корректные сообщения участникам.
- [ ] Обосновано: проверка позиций — повтор (loop); независимые действия после подтверждения — параллель (par).
- [ ] Диаграмма читаема (фреймы не пересекаются некорректно).

## ПОДСКАЗКА
Опора — §3 теории. `loop по позициям … end` — повтор для каждого элемента. `par … and … end` — одновременные блоки. par уместен для **независимых** действий (уведомление и задание ресторану друг от друга не зависят).
