# Задача 05 — Модуль 33: Sequence diagram

## ЗАДАНИЕ
Постройте sequence diagram распределённой саги с компенсациями.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, распределённая транзакция заказа (модуль 25): шаги — списать оплату → зарезервировать товар на складе → назначить курьера. Если шаг падает — выполнить компенсации уже выполненных шагов в обратном порядке (оркестрация).

## ТРЕБОВАНИЯ
- Участники: Оркестратор саги, Платёж, Склад, Доставка.
- Прямые шаги (sync-вызовы) + ветка сбоя через `alt`.
- Компенсации при сбое (например: не нашёлся курьер → освободить товар → вернуть оплату).
- Показать обратный порядок компенсаций.
- Кратко: чем оркестрация отличается от хореографии (1 строка).

## ФОРМАТ РЕЗУЛЬТАТА
Sequence diagram (PlantUML или текстовая нотация) с шагами саги, `alt` на сбой и компенсациями + 1 строка про оркестрацию.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Оркестратор управляет шагами (вызывает Платёж, Склад, Доставка).
- [ ] Прямые шаги в правильном порядке.
- [ ] `alt`/ветка сбоя на одном из шагов.
- [ ] Компенсации в обратном порядке (освободить товар, вернуть оплату).
- [ ] Отмечено отличие оркестрации (центральный координатор) от хореографии (события).

## ПОДСКАЗКА
Опора — §3, §6 теории и модуль 25. Оркестратор последовательно вызывает сервисы; при сбое шага N — компенсирует шаги N-1…1 в обратном порядке. Это визуализация саги-оркестрации на sequence.
