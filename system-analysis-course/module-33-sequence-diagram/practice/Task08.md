# Задача 08 — Модуль 33: Sequence diagram

## ЗАДАНИЕ
Смоделируйте взаимодействие с внешней системой, включая тайм-аут и ретрай.

## ВХОДНЫЕ ДАННЫЕ
**MedConnect**: сервис записи проверяет полис пациента через внешний государственный шлюз (медленный, иногда не отвечает). Нужно: тайм-аут на ожидание, повтор (ретрай) с ограничением, обработка окончательного отказа (модуль 25).

## ТРЕБОВАНИЯ
- Участники: Сервис записи, Внешний шлюз (граница системы).
- Sync-вызов проверки полиса с возможным тайм-аутом.
- `loop` для ретраев с ограничением числа попыток.
- `alt`: ответ получен (продолжить) / попытки исчерпаны (graceful degradation или ошибка).
- Отметьте, что внешняя система — отдельный участник с ненадёжностью.

## ФОРМАТ РЕЗУЛЬТАТА
Sequence diagram (PlantUML или текстовая нотация) с тайм-аутом, `loop` ретраев и `alt` исхода + 1 строка про обработку отказа.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Внешний шлюз — отдельный участник (граница системы).
- [ ] Sync-вызов с тайм-аутом на ожидание ответа.
- [ ] `loop` ретраев с лимитом попыток.
- [ ] `alt`: успех / исчерпание попыток (деградация или ошибка пользователю).
- [ ] Отмечена ненадёжность внешней системы (нельзя полагаться на мгновенный ответ).

## ПОДСКАЗКА
Опора — §3, §7 теории и модуль 25. Тайм-аут = ветка «нет ответа за T»; ретрай = `loop` с лимитом; исчерпание = `alt` в ошибку/деградацию. Внешние вызовы всегда моделируем с учётом сбоя — это признак зрелого дизайна.
