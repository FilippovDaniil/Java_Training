# Задача 09 — Модуль 33: Sequence diagram

## ЗАДАНИЕ
Выберите нотацию (sequence / activity / BPMN) под каждую задачу моделирования и обоснуйте.

## ВХОДНЫЕ ДАННЫЕ
Ситуации:
1. Показать бизнес-процесс «оформление заказа» бизнес-заказчику с ролями (клиент, ресторан, курьер).
2. Спроектировать точную последовательность вызовов между сервисами при оплате (кто кому шлёт API-запросы).
3. Описать алгоритм расчёта стоимости доставки с ветвлениями и параллельными шагами, без акцента «кто исполняет».

## ТРЕБОВАНИЯ
- Для каждой ситуации: нотация + 1–2 решающих критерия.
- Объясните различие фокусов: взаимодействие участников (sequence) vs поток действий (activity) vs бизнес-процесс с ролями (BPMN).
- Отметьте, почему неверный выбор плох (например, sequence для бизнес-аудитории — слишком технично).

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: Ситуация | Нотация | Критерий. + 2 строки про различие фокусов.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Ситуация 1 → BPMN (бизнес-процесс, роли/дорожки, бизнес-аудитория).
- [ ] Ситуация 2 → Sequence (взаимодействие сервисов, API-сообщения).
- [ ] Ситуация 3 → Activity (алгоритм/поток действий с ветвлениями/параллелизмом).
- [ ] Объяснены различия фокусов трёх нотаций.
- [ ] Отмечен риск неверного выбора.

## ПОДСКАЗКА
Опора — §7 теории (+ модули 12, 13). Маркеры: «бизнес-процесс/роли/для бизнеса» → BPMN; «кто кому шлёт сообщения/API» → sequence; «алгоритм/логика без акцента на исполнителя» → activity.
