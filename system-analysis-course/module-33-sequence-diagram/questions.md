---

## Ключ (эталонные ответы)

1. Обмен **сообщениями между участниками во времени** в рамках конкретного сценария: кто кому что шлёт и в каком порядке (динамика, не структура).
2. Время — сверху вниз; участники — слева направо (каждый со своей вертикальной lifeline).
3. **Lifeline** — вертикальная линия участника (объекта/сервиса/актора), показывающая его существование в течение сценария.
4. **Актор** — внешний инициатор (пользователь/внешняя система, человечек); **участник** — внутренний объект/сервис системы (прямоугольник).
5. **Sync** — сплошная закрашенная стрелка, вызывающий ждёт return; **async** — тонкая открытая стрелка, вызывающий не ждёт (событие/публикация/постановка в очередь).
6. **Return** — возврат результата на синхронный вызов; обозначается пунктирной стрелкой обратно к вызывающему.
7. **Активация (activation bar)** — узкий прямоугольник на lifeline, период, когда участник активно обрабатывает вызов.
8. **Self-call** — сообщение участника самому себе (вызов собственного метода/внутреннего шага).
9. `alt` (ветвление), `opt` (опционально), `loop` (цикл), `par` (параллельно), `ref` (ссылка на диаграмму), `break` (прерывание).
10. Когда есть **альтернативы по условию** (несколько взаимоисключающих веток), например успех/ошибка оплаты (if/else).
11. **`opt`** — один блок, выполняется только при условии (if без else); **`alt`** — несколько взаимоисключающих веток (if/else).
12. Повтор блока сообщений (по коллекции или условию) — например, обработать каждую позицию заказа.
13. Параллельное выполнение нескольких блоков одновременно (например, одновременные вызовы к двум сервисам).
14. Ссылка на другую sequence-диаграмму — декомпозиция/переиспользование, чтобы не перегружать текущую.
15. Через ветку ошибки во фрейме `alt` (else) или `break`; показать ответ с ошибкой/исключение и реакцию.
16. Актор UC → актор/первый участник; основной поток → последовательность sync-сообщений; альтернативные потоки → `alt`/`opt`; исключения → ветка ошибки/`break`; внешние системы → отдельные участники.
17. Async-сообщением (открытая стрелка) к участнику-топику/очереди без немедленного return; подписчики получают событие отдельными async-сообщениями.
18. **Sequence** — взаимодействие участников во времени (кто кому шлёт); **activity** — поток действий/алгоритм без акцента на «кто».
19. **Sequence** — техническое взаимодействие компонентов; **BPMN** — бизнес-процесс с ролями (дорожки) для бизнес-аудитории.
20. Когда нужно показать **обмен сообщениями между компонентами/сервисами** (API, интеграции), а не сквозной бизнес-процесс для бизнеса.
21. Внешняя система — отдельный участник; её сбой — ветка ошибки (`alt`), плюс тайм-аут/ретрай (модуль 25) как повтор вызова/альтернативная ветка.
22. Игнорирует ошибки и альтернативы → нереалистично; реальная система должна обрабатывать отказы, поэтому нужны `alt`/`opt`/`break`.
23. Декомпозировать через `ref` на отдельные диаграммы; оставить на одной только ключевые участники/сообщения.
24. Тайм-аут — ветка `alt` «нет ответа за T»; ретрай — `loop` повтора вызова с ограничением попыток (и переход в ошибку/DLQ при исчерпании).
25. Только happy path; путаница sync/async; нет return на sync; перегруженность; внешний вызов без сбоя/тайм-аута; подмена state/structure/BPMN диаграмм.
