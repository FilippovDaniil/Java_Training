# Задача 06 — Модуль 34: Безопасность интеграций

## ЗАДАНИЕ
Найдите уязвимости API по OWASP API Security Top 10 и предложите меры.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, API с проблемами:
1. `GET /orders/{id}` отдаёт заказ любому аутентифицированному пользователю по id.
2. `GET /clients/{id}` возвращает полный объект клиента, включая хэш пароля и внутренние флаги.
3. Нет ограничения частоты запросов; эндпоинт списка без пагинации.
4. `POST /admin/restaurants` доступен любому пользователю с валидным токеном.
5. Включён отладочный эндпоинт `/debug` с дампом конфигурации.

## ТРЕБОВАНИЯ
- Сопоставьте каждую проблему с классом угрозы OWASP API Top 10.
- Для каждой — мера защиты.
- Особо выделите BOLA (№1) и excessive data exposure.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: Проблема | Угроза (OWASP) | Мера.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Проблема 1 → BOLA; мера: проверка владения объектом.
- [ ] Проблема 2 → Excessive Data Exposure; мера: DTO/проекция, не отдавать лишнее.
- [ ] Проблема 3 → Unrestricted Resource Consumption; мера: rate limiting + пагинация.
- [ ] Проблема 4 → Broken Function Level Authorization; мера: проверка прав на админ-функции.
- [ ] Проблема 5 → Security Misconfiguration; мера: отключить отладочные эндпоинты.

## ПОДСКАЗКА
Опора — §6 теории. BOLA — доступ к чужому объекту по id; excessive exposure — лишние поля; нет лимитов — DoS; админ-функция всем — function level authorization; debug-эндпоинт — misconfiguration.
