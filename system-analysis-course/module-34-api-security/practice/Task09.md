# Задача 09 — Модуль 34: Безопасность интеграций

## ЗАДАНИЕ
Сформулируйте требования к приватности и согласиям (152-ФЗ/GDPR) для интеграций с персональными данными.

## ВХОДНЫЕ ДАННЫЕ
**MedConnect** передаёт персональные и медицинские данные между внутренними сервисами и во внешние системы (страховой шлюз, СМС-провайдер, аналитика). Требуется соблюсти 152-ФЗ (РФ) / GDPR-подобные принципы.

## ТРЕБОВАНИЯ
- Сформулируйте требования: согласие на обработку, минимизация данных (передавать только необходимое), цель обработки, срок хранения.
- Опишите защиту ПДн в интеграциях: шифрование канала (TLS), маскирование/псевдонимизация, ограничение полей во внешние системы.
- Отметьте права субъекта (доступ, удаление/«право на забвение»).
- Свяжите с предыдущими задачами (BOLA — доступ к чужим ПДн; excessive exposure; секреты).
- Зафиксируйте, что аналитик выносит в требования/контракт.

## ФОРМАТ РЕЗУЛЬТАТА
Список требований по приватности (согласие, минимизация, цель, срок, права субъекта) + меры защиты в интеграциях + связь с угрозами API.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Согласие на обработку ПДн (цель, объём).
- [ ] Минимизация: во внешние системы — только необходимые поля (не всё).
- [ ] Срок хранения и цель обработки зафиксированы.
- [ ] Меры: TLS, маскирование/псевдонимизация, ограничение полей.
- [ ] Права субъекта (доступ, удаление) учтены.
- [ ] Связь с BOLA/excessive exposure/секретами обозначена.

## ПОДСКАЗКА
Опора — §6, §7 теории и модуль 09 (NFR безопасности/приватности). Ключевые принципы: согласие, минимизация, цель/срок, права субъекта. В интеграциях — не передавать лишние ПДн (минимизация = защита от excessive exposure), шифровать канал, маскировать. Аналитик фиксирует это как требования.
