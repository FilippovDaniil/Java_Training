# Задача 10 (мини-проект) — Модуль 34: Безопасность интеграций

## ЗАДАНИЕ
Спроектируйте модель безопасности API: аутентификация, авторизация, роли, угрозы и меры. Синтез теории модуля.

## ВХОДНЫЕ ДАННЫЕ
**MedConnect** (телемедицина), API: запись на приём, оплата, медкарта, отчёты. Клиенты: мобильное приложение пациента (публичный), веб-кабинет врача (серверный), внутренние сервисы (межсервис), внешний страховой шлюз. Данные чувствительны (ПДн/медицина, 152-ФЗ).

## ТРЕБОВАНИЯ
Сдайте модель безопасности:
1. **Аутентификация** — механизм (OAuth2/OIDC), flow по каждому типу клиента; mTLS для межсервиса.
2. **Токены** — структура JWT (claims), валидация, TTL/refresh, отзыв.
3. **Авторизация** — RBAC-модель (роли пациент/врач/админ/сервис × действия) + объектная проверка владения (BOLA).
4. **Угрозы** — топ-5 по OWASP API Top 10 для этого API + меры (обязательно BOLA, excessive exposure, function level auth, rate limiting).
5. **Секреты** — хранение, ротация, минимальный scope.
6. **Приватность** — 152-ФЗ: согласие, минимизация, защита ПДн в интеграциях.
7. **Отражение в контракте** — что фиксируется в OpenAPI `securitySchemes`/`security` (модуль 30).
Короткий вывод: как меры покрывают угрозы.

## ФОРМАТ РЕЗУЛЬТАТА
Структурированная модель безопасности (AuthN, токены, AuthZ/RBAC+BOLA, угрозы→меры, секреты, приватность, контракт) + матрица «угроза → мера» + 2–3 строки вывода.

## КРИТЕРИИ ПРИЁМКИ
- [ ] AuthN: OAuth2 flow по типам клиента (мобайл+PKCE, серверный web — Authorization Code, межсервис — Client Credentials/mTLS).
- [ ] JWT: claims, валидация (подпись/exp/iss/aud), короткий TTL + refresh.
- [ ] RBAC-матрица (роли × действия) + проверка владения объектом (анти-BOLA).
- [ ] Топ-5 угроз OWASP с мерами (BOLA, excessive exposure, function level auth, resource consumption, misconfiguration).
- [ ] Секреты: secret-хранилище, ротация, минимальный scope.
- [ ] Приватность 152-ФЗ: согласие, минимизация, защита ПДн.
- [ ] Отражение в OpenAPI (securitySchemes/security).
- [ ] Матрица «угроза → мера» и вывод о покрытии.

## ПОДСКАЗКА
Синтез всего модуля: AuthN/AuthZ (зад.01), flows (02), JWT (03), RBAC (04), mTLS (05), OWASP (06), BOLA (07), секреты (08), приватность (09). Главные проверки middle+: разделение AuthN/AuthZ, защита от BOLA (владение объектом), корректные токены/flows, минимизация данных и секретов, отражение в контракте. Это завершает Часть 6 — дальше тестирование (модуль 35), в т.ч. проверка этих правил безопасности.
