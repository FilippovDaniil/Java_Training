# Задача 01 — Модуль 35: Тестирование для аналитика

## ЗАДАНИЕ
Выведите набор тест-кейсов из use case (основной поток + альтернативы + исключения).

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, use case «Оплатить заказ»: клиент оплачивает созданный заказ картой через платёжный шлюз. Альтернативы/исключения: недостаточно средств; истёкшая карта; таймаут шлюза; заказ уже оплачен.

## ТРЕБОВАНИЯ
- Выведите минимум 5 тест-кейсов: happy path + альтернативы + исключения.
- Для каждого — краткое название и ожидаемый результат (статус заказа, ответ).
- Покажите соответствие «поток use case → тест-кейс».
- Включите хотя бы 2 негативных сценария.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: ID | Сценарий | Поток UC (основной/альт/исключение) | Ожидаемый результат.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Есть happy path (успешная оплата → статус PAID).
- [ ] ≥2 негативных сценария (недостаточно средств, истёкшая карта).
- [ ] Покрыты исключения (таймаут шлюза, повторная оплата).
- [ ] У каждого тест-кейса конкретный ожидаемый результат.
- [ ] Видно соответствие потокам use case.

## ПОДСКАЗКА
Опора — §3 теории. Один UC → набор тестов: основной поток (happy) + каждая альтернатива/исключение = отдельный тест-кейс. Это перевод модуля 07 (use case с потоками) в проверки.
