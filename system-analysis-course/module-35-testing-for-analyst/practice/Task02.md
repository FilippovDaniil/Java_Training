# Задача 02 — Модуль 35: Тестирование для аналитика

## ЗАДАНИЕ
Примените классы эквивалентности и граничные значения к правилам валидации.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, правила оформления заказа:
- Количество порций блюда: целое от 1 до 20.
- Сумма заказа для оформления: от 300 ₽ (минимальный заказ) до 15 000 ₽.
- Промокод: строка ровно 8 символов.

## ТРЕБОВАНИЯ
- Для каждого правила выделите классы эквивалентности (валидные/невалидные).
- Определите граничные значения для проверки.
- Сведите в таблицу: правило → классы → граничные точки → ожидаемый результат (принять/отклонить).
- Объясните, почему достаточно по одному значению из класса + границы.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица тест-дизайна: Правило | Классы эквивалентности | Граничные значения | Ожидание. + 1 строка про экономию тестов.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Классы эквивалентности для каждого правила (валид/невалид).
- [ ] Граничные значения: для 1..20 → 0,1,20,21; для суммы → 299,300,15000,15001; для промокода → 7,8,9 символов.
- [ ] Ожидаемый результат для каждой точки (принять/отклонить).
- [ ] Объяснено сокращение числа тестов через классы.
- [ ] Учтены оба края диапазона.

## ПОДСКАЗКА
Опора — §5 теории. Класс = группа с одинаковым поведением (проверяем 1 значение). Границы = края (min−1, min, max, max+1) — там off-by-one баги. Промокод «ровно 8» → проверять 7/8/9.
