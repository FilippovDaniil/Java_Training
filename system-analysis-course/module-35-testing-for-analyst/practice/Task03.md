# Задача 03 — Модуль 35: Тестирование для аналитика

## ЗАДАНИЕ
Напишите полноценный тест-кейс по стандартной структуре.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, проверяемое требование: «Клиент не может оформить заказ на сумму меньше минимальной (300 ₽); система отклоняет с понятной ошибкой, заказ не создаётся».

## ТРЕБОВАНИЯ
- Оформите тест-кейс со всеми полями: ID/название, предусловия, тест-данные, шаги (пронумерованные), ожидаемый результат, приоритет.
- Ожидаемый результат — конкретный и проверяемый (статус ответа, состояние заказа, сообщение).
- Тест-данные — конкретные значения (сумма у границы, например 299 ₽).
- Сделайте кейс воспроизводимым.

## ФОРМАТ РЕЗУЛЬТАТА
Тест-кейс таблицей/структурой с полями: ID, название, предусловия, тест-данные, шаги, ожидаемый результат, приоритет.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Присутствуют все поля структуры тест-кейса.
- [ ] Предусловия заданы (состояние корзины/клиента).
- [ ] Конкретные тест-данные (сумма у границы, напр. 299 ₽).
- [ ] Пронумерованные шаги.
- [ ] Ожидаемый результат конкретен (например: ошибка 400/422, заказ не создан, понятное сообщение).
- [ ] Указан приоритет.

## ПОДСКАЗКА
Опора — §4 теории. Шаблон: предусловия → тест-данные → шаги → ожидаемый результат. Избегай «работает правильно» — пиши проверяемо («HTTP 422, заказ не создан, сообщение про минимальную сумму»). Возьми граничное значение из задачи 02.
