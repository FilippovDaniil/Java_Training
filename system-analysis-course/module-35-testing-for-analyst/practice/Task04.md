# Задача 04 — Модуль 35: Тестирование для аналитика

## ЗАДАНИЕ
Разграничьте use case и test case, покажите связь между ними.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, use case «Отменить заказ» (модуль 07): клиент отменяет оплаченный заказ до передачи в доставку; средства возвращаются. Альтернатива: заказ уже в доставке — отмена невозможна.

## ТРЕБОВАНИЯ
- Объясните различие: что описывает use case, что — test case.
- Покажите, почему из одного use case рождается несколько тест-кейсов.
- Приведите 3–4 тест-кейса для этого UC (с ожидаемыми результатами).
- Сведите в таблицу соответствие «элемент UC → тест-кейс(ы)».

## ФОРМАТ РЕЗУЛЬТАТА
Краткое различие UC vs TC + список 3–4 тест-кейсов + таблица соответствия.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Чётко сформулировано различие (UC — поведение/сценарий; TC — конкретная проверка).
- [ ] Объяснено «один UC → много TC».
- [ ] 3–4 тест-кейса с ожидаемыми результатами (успешная отмена, отмена в доставке, возврат средств, чужой заказ).
- [ ] Таблица соответствия UC → TC.
- [ ] Учтены основной поток и альтернатива.

## ПОДСКАЗКА
Опора — §3 теории. UC = «что система делает» (потоки), TC = «как проверить» (данные+ожидание). Основной поток + альтернатива + граница/негатив → отдельные тест-кейсы.
