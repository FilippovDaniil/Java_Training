# Задача 05 — Модуль 35: Тестирование для аналитика

## ЗАДАНИЕ
Спроектируйте позитивные и негативные сценарии для функции.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, функция «Применить промокод к заказу»: промокод даёт скидку, если валиден, активен, не истёк, не использован этим клиентом и заказ проходит по условиям (мин. сумма).

## ТРЕБОВАНИЯ
- Спроектируйте 1–2 позитивных сценария (валидный промокод применяется).
- Спроектируйте минимум 5 негативных сценариев (невалидный/истёкший/уже использованный/неактивный/сумма ниже порога).
- Для каждого — ожидаемый результат (скидка применена / отклонено с причиной).
- Отметьте, почему негативные сценарии важнее для выявления дефектов.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: Сценарий | Тип (позитив/негатив) | Входные данные | Ожидаемый результат.

## КРИТЕРИИ ПРИЁМКИ
- [ ] ≥1 позитивный сценарий (скидка применяется).
- [ ] ≥5 негативных сценариев (разные причины отказа).
- [ ] У каждого конкретный ожидаемый результат с причиной отказа.
- [ ] Покрыты ключевые условия (истёк, использован, неактивен, мин. сумма, несуществующий).
- [ ] Объяснена ценность негативных сценариев.

## ПОДСКАЗКА
Опора — §6 теории. Позитив = happy path; негатив = каждое нарушение условия = отдельный сценарий с понятной причиной отказа. Негативные сценарии обычно дают больше найденных дефектов.
