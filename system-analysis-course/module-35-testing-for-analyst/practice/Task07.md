# Задача 07 — Модуль 35: Тестирование для аналитика

## ЗАДАНИЕ
Найдите пробелы в тест-покрытии требований и предложите недостающие тест-кейсы.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, требования к функции «Доставка заказа» и существующие тест-кейсы:
- Требования: R1 заказ доставляется в выбранный интервал; R2 клиент видит статус курьера в реальном времени; R3 при недоставке — повторная попытка/возврат; R4 нельзя доставить отменённый заказ; R5 уведомление при смене статуса.
- Существующие тесты: TC1 успешная доставка в срок (R1); TC2 отображение статуса (R2).

## ТРЕБОВАНИЯ
- Постройте матрицу трассировки требование → тест (что покрыто, что нет).
- Выявите непокрытые требования.
- Предложите недостающие тест-кейсы (с ожидаемым результатом), включая негативные.
- Отметьте, почему трассировка нужна для оценки покрытия.

## ФОРМАТ РЕЗУЛЬТАТА
Матрица трассировки (требование → тест-кейсы, покрыто/нет) + список недостающих тест-кейсов.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Матрица показывает покрытие R1–R5 существующими тестами.
- [ ] Выявлены непокрытые: R3 (недоставка), R4 (отменённый заказ), R5 (уведомление).
- [ ] Предложены тест-кейсы на пробелы с ожидаемыми результатами.
- [ ] Есть негативный сценарий (R4 — доставка отменённого заказа запрещена).
- [ ] Объяснена роль трассировки в оценке покрытия.

## ПОДСКАЗКА
Опора — §1 «Нет трассировки» и модуль 10. Сопоставь каждое требование с тест-кейсами; пустые строки = пробелы. R3/R4/R5 не покрыты — допиши тесты, включая негатив (R4).
