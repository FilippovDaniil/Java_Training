# Задача 08 — Модуль 35: Тестирование для аналитика

## ЗАДАНИЕ
Преобразуйте критерии приёмки (Gherkin) в приёмочные тесты.

## ВХОДНЫЕ ДАННЫЕ
**MedConnect**, user story «Запись на приём» с критериями приёмки (Gherkin, модуль 08):
```
Сценарий: успешная запись в свободный слот
  Дано пациент авторизован и слот свободен
  Когда пациент бронирует слот
  Тогда слот становится занятым и пациент получает подтверждение

Сценарий: слот занят
  Дано слот уже забронирован
  Когда пациент пытается забронировать его
  Тогда система отказывает и предлагает другие слоты
```

## ТРЕБОВАНИЯ
- Преобразуйте каждый Gherkin-сценарий в приёмочный тест-кейс (предусловия, шаги, ожидаемый результат).
- Добавьте недостающие сценарии (негативные/граничные), которых нет в критериях, но нужны (например: бронь без авторизации, двойная бронь).
- Покажите прямую связь Given/When/Then → предусловия/шаги/ожидание.
- Отметьте роль аналитика в UAT.

## ФОРМАТ РЕЗУЛЬТАТА
Приёмочные тест-кейсы из Gherkin + дополнительные сценарии + пояснение связи Gherkin↔тест-кейс.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Каждый Gherkin-сценарий преобразован в тест-кейс.
- [ ] Given → предусловия, When → шаги, Then → ожидаемый результат.
- [ ] Добавлены недостающие сценарии (негативные/граничные).
- [ ] Ожидаемые результаты конкретны.
- [ ] Отмечена связь приёмочных тестов с UAT и критериями приёмки.

## ПОДСКАЗКА
Опора — §2, §3 теории и модуль 08. Gherkin прямо ложится в тест-кейс: Given=предусловия, When=шаги, Then=ожидание. Критерии приёмки задают приёмочные тесты; аналитик дополняет негативами.
