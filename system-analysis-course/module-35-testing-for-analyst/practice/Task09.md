# Задача 09 — Модуль 35: Тестирование для аналитика

## ЗАДАНИЕ
Подготовьте тест-данные для набора сценариев одной функции.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, функция «Оформление заказа» тестируется по сценариям: успешный заказ, заказ ниже минимальной суммы, заказ с промокодом, заказ неактивного ресторана, заказ с недоступным блюдом.

## ТРЕБОВАНИЯ
- Для каждого сценария определите необходимые тест-данные (клиент, ресторан, блюда, суммы, промокод, статусы).
- Укажите предусловия-состояния (что должно быть в системе до теста).
- Покажите, что данные конкретны и обеспечивают воспроизводимость.
- Отметьте данные на границах (мин. сумма) и для негативных сценариев.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: Сценарий | Тест-данные | Предусловия-состояния | Ожидаемый результат.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Для каждого сценария заданы конкретные тест-данные.
- [ ] Указаны предусловия-состояния (статус ресторана, наличие блюда и т.п.).
- [ ] Есть граничные данные (сумма у минимума).
- [ ] Данные для негативных сценариев (неактивный ресторан, недоступное блюдо).
- [ ] Данные обеспечивают воспроизводимость.

## ПОДСКАЗКА
Опора — §4, §5 теории. Тест-данные = конкретные значения + состояние системы (ресторан активен/нет, блюдо доступно/нет). Без них тест невоспроизводим. Включи границы и негативные данные.
