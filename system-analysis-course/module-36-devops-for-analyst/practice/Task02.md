# Задача 02 — Модуль 36: DevOps для аналитика

## ЗАДАНИЕ
Опишите этапы CI/CD пайплайна и место проверок качества в нём.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, команда настраивает пайплайн от коммита до прода. Нужно показать, где «срабатывают» тесты и критерии приёмки (модуль 35).

## ТРЕБОВАНИЯ
- Перечислите этапы пайплайна по порядку (commit → … → prod).
- Укажите, какие проверки на каждом этапе (build, unit, integration/e2e, анализ кода, approve).
- Объясните, что происходит при провале этапа.
- Отметьте роль аналитика (где UAT/approve, критерии приёмки).
- Разграничьте CI и CD.

## ФОРМАТ РЕЗУЛЬТАТА
Последовательность этапов с проверками + поведение при провале + разграничение CI/CD.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Этапы по порядку: commit→build→тесты→анализ→артефакт→deploy test→integration→stage→approve→prod.
- [ ] На каждом этапе указана проверка.
- [ ] Объяснено: провал этапа останавливает продвижение.
- [ ] Разграничены CI (сборка+тесты на коммит) и CD (поставка/выкат).
- [ ] Отмечена точка участия аналитика (UAT/approve/критерии).

## ПОДСКАЗКА
Опора — §2 теории. CI = автоматическая сборка и тесты на каждый коммит; CD = автоматическая поставка/выкат. Провал любого этапа = фича не едет дальше. Критерии приёмки и тесты (модуль 35) «живут» в пайплайне.
