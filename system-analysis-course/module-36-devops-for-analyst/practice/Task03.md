# Задача 03 — Модуль 36: DevOps для аналитика

## ЗАДАНИЕ
Объясните, что контейнеризация и оркестрация дают понимать аналитику (без написания Dockerfile).

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress** разворачивается в контейнерах под управлением Kubernetes (несколько реплик, health checks). Аналитику нужно понять, как это влияет на требования и поставку.

## ТРЕБОВАНИЯ
- Объясните, какую проблему решает контейнер («работает у меня»).
- Опишите, что даёт оркестратор (масштаб, перезапуск, выкат, балансировка).
- Свяжите с NFR (доступность, масштабируемость, health checks — модуль 26).
- Покажите, как это упрощает откат (вернуть предыдущий образ).
- Разграничьте образ и контейнер, реплику.

## ФОРМАТ РЕЗУЛЬТАТА
Объяснение «зачем контейнеры/оркестрация аналитику» + связь с NFR + 1–2 строки про откат + краткий глоссарий (образ/контейнер/реплика).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Объяснена воспроизводимость сред (контейнер).
- [ ] Описаны функции оркестратора (масштаб/перезапуск/выкат/балансировка).
- [ ] Связь с NFR: доступность, масштабируемость, health checks (liveness/readiness).
- [ ] Показан быстрый откат через предыдущий образ.
- [ ] Разграничены образ/контейнер/реплика.

## ПОДСКАЗКА
Опора — §3 теории и модуль 26. Аналитику не нужен Dockerfile, но нужно понимать: контейнер = воспроизводимость; оркестратор = масштаб + самовосстановление + откат; это прямо влияет на NFR доступности/масштабируемости.
