# Задача 04 — Модуль 36: DevOps для аналитика

## ЗАДАНИЕ
Спроектируйте постепенный выкат фичи через feature flag.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, фича «рекомендации блюд на основе истории заказов». Риск: новая логика может ошибаться. Нужно выкатить постепенно, с возможностью мгновенно выключить.

## ТРЕБОВАНИЯ
- Опишите план выката через флаг: деплой с флагом OFF → включение для доли (5%→50%→100%).
- Объясните разделение деплоя и релиза на этом примере.
- Опишите сценарий отката (флаг OFF мгновенно).
- Отметьте таргетинг (на сегмент/процент) и как измерять успех.
- Упомяните гигиену флага (удалить после полного выката).

## ФОРМАТ РЕЗУЛЬТАТА
План выката (этапы с долями) + объяснение деплой≠релиз + сценарий отката + 1 строка про гигиену флага.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Деплой с флагом OFF, затем постепенное включение (5%→50%→100%).
- [ ] Объяснено разделение деплоя и релиза.
- [ ] Сценарий мгновенного отката флагом (без отката деплоя).
- [ ] Таргетинг по доле/сегменту и метрика успеха.
- [ ] Отмечена необходимость удалить флаг после стабилизации.

## ПОДСКАЗКА
Опора — §5 теории. Флаг разводит деплой (код в проде, OFF) и релиз (включение). Постепенно расширяем долю, следим за метриками; проблема → OFF мгновенно. После 100% — убрать флаг (не плодить зомби).
