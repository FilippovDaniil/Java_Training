# Задача 05 — Модуль 36: DevOps для аналитика

## ЗАДАНИЕ
Выберите релизную стратегию (canary vs blue-green) под две ситуации и обоснуйте.

## ВХОДНЫЕ ДАННЫЕ
1. Критичный платёжный сервис: нужен мгновенный откат при любой проблеме; есть ресурсы держать две полные среды.
2. Новая версия ленты рекомендаций: хотим обнаружить проблемы на реальном трафике рано, выкатывая сначала на малую долю пользователей и наблюдая метрики.

## ТРЕБОВАНИЯ
- Для каждой ситуации: стратегия + 2 решающих критерия.
- Сравните blue-green и canary по откату, ресурсам, раннему обнаружению.
- Отметьте, где уместно сочетание с feature flags.
- Покажите, чем плох неверный выбор.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: Ситуация | Стратегия | Критерии. + сравнение blue-green vs canary + 1 строка про сочетание с флагами.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Ситуация 1 → blue-green (мгновенный откат, две среды).
- [ ] Ситуация 2 → canary (раннее обнаружение на малой доле).
- [ ] Сравнение по откату/ресурсам/раннему обнаружению.
- [ ] Отмечено сочетание canary + feature flags.
- [ ] Обоснование привязано к свойствам стратегий.

## ПОДСКАЗКА
Опора — §6 теории. Blue-green = две полные среды + мгновенное переключение (быстрый откат, но дорого по ресурсам). Canary = выкат на 5%→… с наблюдением (раннее обнаружение, дешевле по ресурсам).
