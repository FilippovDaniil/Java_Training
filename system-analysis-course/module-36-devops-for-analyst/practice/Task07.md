# Задача 07 — Модуль 36: DevOps для аналитика

## ЗАДАНИЕ
Определите, что отличается в конфигурации по средам, и отделите конфигурацию от кода/секретов.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, один артефакт разворачивается на test, stage, prod. Различаются: URL БД, эндпоинт платёжного шлюза (тест/боевой), уровень логирования, лимиты, ключи/секреты, значения feature flags.

## ТРЕБОВАНИЯ
- Составьте таблицу «что отличается по средам» (параметр × среда).
- Разделите: код (одинаков), конфигурация (env/configmap), секреты (Secret).
- Объясните принцип «build once, deploy many» на этом примере.
- Отметьте, что секреты не идут в код/гит/ConfigMap (модуль 34).
- Покажите, где живут значения feature flags.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица конфигурации по средам + классификация (код/конфиг/секрет) + 2 строки про build once, deploy many и секреты.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Таблица: параметры × (test/stage/prod) с отличиями.
- [ ] Разделение код / конфигурация / секреты.
- [ ] Принцип build once, deploy many объяснён.
- [ ] Секреты — в Secret/хранилище, не в коде/гите/ConfigMap.
- [ ] Учтены значения feature flags по средам.

## ПОДСКАЗКА
Опора — §4 теории и модуль 34. Код один → поведение по средам задаёт конфигурация (env). Платёжный шлюз: тестовый на test/stage, боевой на prod. Секреты отдельно и защищённо. 12-factor: конфиг в окружении.
