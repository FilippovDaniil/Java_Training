# Задача 03 — Модуль 37: Эксплуатация и сопровождение

## ЗАДАНИЕ
Составьте план миграции данных из legacy-системы.

## ВХОДНЫЕ ДАННЫЕ
**MedConnect** заменяет старую систему учёта пациентов. Legacy-данные: плоский экспорт, дубли пациентов, разные форматы телефонов/дат, пустые обязательные поля, ~200 тыс. записей. Целевая модель — нормализованная (модуль 17).

## ТРЕБОВАНИЯ
- Распишите этапы миграции: профилирование → маппинг → очистка → перенос (ETL) → верификация → запуск.
- Опишите конкретные проблемы качества данных и как их обработать (дубли, форматы, пропуски).
- Составьте фрагмент маппинга «поле источника → поле цели + правило трансформации».
- Выберите стратегию запуска (big-bang vs параллельно) с обоснованием (мед-данные критичны).
- Опишите верификацию (как убедиться, что перенесено корректно).

## ФОРМАТ РЕЗУЛЬТАТА
План миграции по этапам + таблица маппинга (фрагмент) + обработка проблем качества + выбор стратегии запуска + способ верификации.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Все этапы: профилирование, маппинг, очистка, ETL, верификация, запуск.
- [ ] Конкретные проблемы качества и их обработка (дубли/форматы/пропуски).
- [ ] Фрагмент маппинга с правилами трансформации.
- [ ] Выбрана стратегия запуска с обоснованием (для мед-данных — параллельная/осторожная).
- [ ] Описана верификация (количество, контрольные суммы, выборка).

## ПОДСКАЗКА
Опора — §3 теории. Legacy = грязные данные → профилирование и очистка обязательны. Маппинг = источник→цель+правило. Для критичных мед-данных безопаснее параллельный запуск со сверкой. Верификация: счётчики + выборка.
