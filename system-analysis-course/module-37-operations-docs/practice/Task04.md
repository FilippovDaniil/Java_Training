# Задача 04 — Модуль 37: Эксплуатация и сопровождение

## ЗАДАНИЕ
Спроектируйте структуру руководства пользователя.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, мобильное/веб-приложение клиента. Нужно руководство пользователя для конечных клиентов (не технических).

## ТРЕБОВАНИЯ
- Составьте оглавление руководства (разделы по основным сценариям).
- Покройте ключевые задачи пользователя: регистрация, поиск ресторана, заказ, оплата, отслеживание, отмена, промокоды, поддержка.
- Для 1–2 разделов покажите формат описания (шаги + что видит пользователь).
- Учтите нетехническую аудиторию (язык, скриншоты, FAQ).
- Отметьте, чем это отличается от руководства администратора.

## ФОРМАТ РЕЗУЛЬТАТА
Оглавление руководства пользователя + пример оформления 1–2 разделов + 1 строка про отличие от рук-ва админа.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Оглавление покрывает основные пользовательские сценарии.
- [ ] Разделы организованы по задачам пользователя (не по экранам кода).
- [ ] Показан формат описания (пошагово, доступным языком).
- [ ] Учтена нетехническая аудитория (FAQ/скриншоты).
- [ ] Отмечено отличие от руководства администратора.

## ПОДСКАЗКА
Опора — §1 теории. Руководство пользователя = «как сделать задачу» доступным языком, по сценариям (модуль 07). Не путать с админским (установка/настройка). Добавь FAQ и раздел «куда обращаться» (поддержка).
