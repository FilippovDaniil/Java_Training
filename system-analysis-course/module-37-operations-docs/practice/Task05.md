# Задача 05 — Модуль 37: Эксплуатация и сопровождение

## ЗАДАНИЕ
Распределите инциденты по линиям поддержки L1/L2/L3 и опишите эскалацию.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, обращения в поддержку:
1. «Как изменить адрес доставки в профиле?»
2. «Не приходит SMS с кодом» (для части пользователей).
3. «При оплате заказа возникает ошибка 500» (баг в коде).
4. «Забыл пароль».
5. «Списали деньги дважды» (требует разбора транзакций и, возможно, правки логики).

## ТРЕБОВАНИЯ
- Отнесите каждое обращение к L1/L2/L3 с обоснованием.
- Опишите путь эскалации (когда L1→L2→L3).
- Отметьте, какие обращения L1 закрывает по инструкции, а какие требуют разработки.
- Покажите, как повторяющиеся инциденты становятся change requests.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: Обращение | Линия | Обоснование | Действие. + описание эскалации + связь с change requests.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Обращения 1 и 4 → L1 (типовое, по инструкции).
- [ ] Обращение 2 → L2 (диагностика, нетиповое).
- [ ] Обращения 3 и 5 → L3 (дефект кода/правка логики).
- [ ] Описана эскалация снизу вверх.
- [ ] Показано, как повторяющиеся инциденты → change requests/требования.

## ПОДСКАЗКА
Опора — §4, §5 теории. L1 = типовое по инструкции; L2 = диагностика нетипового; L3 = код/баг. Дважды списанные деньги и ошибка 500 → L3 (и, вероятно, CR). Повторы → новые требования.
