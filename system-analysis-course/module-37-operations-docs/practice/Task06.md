# Задача 06 — Модуль 37: Эксплуатация и сопровождение

## ЗАДАНИЕ
Оформите change request с импакт-анализом.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, после запуска бизнес просит добавить «чаевые курьеру» при оформлении заказа: новое поле, влияние на оплату, расчёт, отчётность и выплаты курьерам.

## ТРЕБОВАНИЯ
- Оформите change request: описание, инициатор, обоснование/ценность.
- Проведите импакт-анализ: что затронет (требования, UI, API, данные/схема, интеграция с оплатой, отчётность, NFR, тесты).
- Оцените риски и зависимости.
- Дайте рекомендацию (принять/отложить/отклонить) с приоритетом.
- Отметьте, какие артефакты нужно обновить (трассировка, модуль 10).

## ФОРМАТ РЕЗУЛЬТАТА
Карточка change request + таблица импакт-анализа (область → влияние) + риски + рекомендация.

## КРИТЕРИИ ПРИЁМКИ
- [ ] CR содержит описание, инициатора, обоснование.
- [ ] Импакт-анализ охватывает требования/UI/API/данные/оплату/отчётность/тесты.
- [ ] Указаны риски и зависимости.
- [ ] Дана рекомендация с приоритетом.
- [ ] Отмечены артефакты к обновлению (трассировка).

## ПОДСКАЗКА
Опора — §5 теории и модуль 10. CR не реализуют «вслепую»: сначала импакт-анализ (что затронет, стоимость, риски). «Чаевые» тянут оплату, расчёты, отчёты, выплаты — всё это в анализ.
