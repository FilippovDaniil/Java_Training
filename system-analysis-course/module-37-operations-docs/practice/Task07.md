# Задача 07 — Модуль 37: Эксплуатация и сопровождение

## ЗАДАНИЕ
Определите метрики поддержки и SLA (реакция/решение по приоритетам).

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, служба поддержки обрабатывает обращения разной критичности: от «не работает оплата на проде» до «вопрос по интерфейсу».

## ТРЕБОВАНИЯ
- Определите уровни приоритетов инцидентов (Critical/High/Medium/Low) с критериями.
- Задайте SLA: время реакции и время решения по каждому приоритету.
- Предложите метрики поддержки (например: время реакции, время решения, доля в SLA, число эскалаций, повторные обращения).
- Объясните, как метрики и SLA связаны с доступностью (модуль 26) и удовлетворённостью.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица приоритетов с критериями + SLA (реакция/решение) + список метрик поддержки + связь с NFR.

## КРИТЕРИИ ПРИЁМКИ
- [ ] 4 приоритета с понятными критериями отнесения.
- [ ] SLA: время реакции и решения для каждого приоритета.
- [ ] ≥4 метрики поддержки.
- [ ] Critical (прод недоступен) имеет жёсткие сроки.
- [ ] Связь с доступностью/удовлетворённостью обозначена.

## ПОДСКАЗКА
Опора — §6 теории. Приоритет ↔ срочность: Critical (прод лежит) — минуты на реакцию, часы на решение; Low — дни. Метрики: время реакции/решения, % в SLA, эскалации, повторы. Связь с SLA доступности (модуль 26).
