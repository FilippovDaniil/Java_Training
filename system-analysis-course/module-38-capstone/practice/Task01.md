# Задача 01 — Capstone: Бизнес-контекст

## ЗАДАНИЕ
Сформируйте бизнес-контекст IoT-платформы «Умный дом»: стейкхолдеры, цели, метрики, границы MVP.

## ВХОДНЫЕ ДАННЫЕ
**Умный дом / IoT** (домен из theory.md, §1). Стартап выводит платформу управления умным домом: устройства, сценарии автоматизации, телеметрия. Запрос размытый: «хотим удобную платформу, чтобы люди пользовались автоматизацией».

## ТРЕБОВАНИЯ
- Карта стейкхолдеров (power/interest grid), включая «нечеловеческих» акторов (устройства, автоматизация) и регулятора (ПДн).
- Бизнес-цели по SMART (минимум 2) из размытого запроса.
- Продуктовые метрики: North Star + 2–3 поддерживающие (дерево метрик).
- Границы MVP: что входит / не входит + обоснование.
- 2–3 продуктовые гипотезы (гипотеза → проверка → метрика).

## ФОРМАТ РЕЗУЛЬТАТА
Бизнес-контекст: grid стейкхолдеров + SMART-цели + метрики (North Star + дерево) + MVP (вошло/не вошло) + гипотезы.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Power/interest grid с ≥6 стейкхолдерами (вкл. устройства/автоматизацию, регулятора).
- [ ] ≥2 цели по SMART (измеримые, со сроком).
- [ ] North Star + поддерживающие метрики (дерево).
- [ ] Границы MVP с обоснованием (вошло/не вошло).
- [ ] 2–3 гипотезы с проверкой и метрикой.

## ПОДСКАЗКА
Опора — модули 01, 03. Начинаем НЕ с требований, а с «зачем». North Star для умного дома — что-то про активную ценность (напр. число активных сценариев на дом). MVP = ключевые сценарии (управление + базовая автоматизация + критичные датчики), остальное отложить. Это фундамент для всех следующих задач.
