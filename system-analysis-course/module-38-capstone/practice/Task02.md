# Задача 02 — Capstone: Каталог требований (функц. + NFR)

## ЗАДАНИЕ
Соберите и структурируйте каталог требований платформы: функциональные + нефункциональные, с приоритизацией и трассировкой к целям.

## ВХОДНЫЕ ДАННЫЕ
Бизнес-контекст из задачи 01 (цели, MVP). Домен «Умный дом / IoT».

## ТРЕБОВАНИЯ
- Иерархия требований: бизнес-цель → пользовательские → функциональные (минимум 8 функциональных по MVP).
- NFR (минимум 6) по категориям ISO 25010, **измеримые** (latency срабатывания, доступность, поток телеметрии, безопасность и т.д.).
- Приоритизация функциональных (MoSCoW или RICE).
- Фрагмент трассировки: цель → требование (показать связь).
- Отметьте источники и хотя бы 2 неявных/подразумеваемых требования.

## ФОРМАТ РЕЗУЛЬТАТА
Каталог: иерархия требований + таблица NFR (категория, формулировка, метрика) + приоритизация + фрагмент трассировки.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Иерархия бизнес → пользовательские → функциональные (≥8 функц.).
- [ ] ≥6 измеримых NFR по категориям ISO 25010.
- [ ] Приоритизация (MoSCoW/RICE) с обоснованием.
- [ ] Трассировка цель → требование (фрагмент).
- [ ] Указаны источники + ≥2 неявных требования.
- [ ] Требования согласованы с MVP из задачи 01.

## ПОДСКАЗКА
Опора — модули 04, 05, 09, 10, 11. NFR для IoT обязательно измеримы: «срабатывание критичного сценария < 1 с (p95)», «доступность 99.9%», «обработка N событий телеметрии/сек». Неявные: восстановление после потери связи, безопасность замков. Требования = вход для процессов, данных, архитектуры (след. задачи).
