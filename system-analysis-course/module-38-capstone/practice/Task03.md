# Задача 03 — Capstone: Бизнес-процессы (BPMN)

## ЗАДАНИЕ
Смоделируйте ключевые процессы платформы в BPMN, включая критичный сценарий автоматизации.

## ВХОДНЫЕ ДАННЫЕ
Требования из задачи 02. Ключевые сценарии: (а) настройка сценария автоматизации пользователем; (б) срабатывание критичного сценария «протечка → перекрыть воду + уведомить».

## ТРЕБОВАНИЯ
- BPMN процесса настройки сценария (пользователь + система; шлюзы валидации).
- BPMN критичного сценария «протечка»: событие датчика → проверка → параллельные действия (команда клапану + уведомление) → обработка ошибок (клапан не ответил) + таймер.
- Корректные элементы: пулы/дорожки, события (start/intermediate/end), шлюзы (XOR/AND), обработка ошибок.
- Отметьте, где процесс пересекается с устройствами/внешними системами.

## ФОРМАТ РЕЗУЛЬТАТА
2 BPMN-диаграммы (PlantUML/текстовая нотация) + краткое пояснение элементов.

## КРИТЕРИИ ПРИЁМКИ
- [ ] BPMN настройки сценария с валидацией (шлюзы).
- [ ] BPMN критичного сценария с событием-стартом от датчика.
- [ ] Параллельный шлюз (AND): клапан + уведомление одновременно.
- [ ] Обработка ошибки (клапан не ответил) + таймер/эскалация.
- [ ] Пулы/дорожки (пользователь/система/устройство) корректны.
- [ ] Процессы согласованы с требованиями (задача 02).

## ПОДСКАЗКА
Опора — модуль 12. Критичный сценарий — не happy path: клапан может не ответить → повтор/эскалация (модуль 25 — устойчивость, разовьём в задаче 08). AND-шлюз для параллельных действий. Эти процессы лягут в sequence-диаграммы (задача 07).
