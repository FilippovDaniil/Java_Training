# Задача 08 — Capstone: Устойчивость + NFR + безопасность

## ЗАДАНИЕ
Спроектируйте нефункциональную архитектуру: распределённую устойчивость, NFR-реализацию и модель безопасности.

## ВХОДНЫЕ ДАННЫЕ
Архитектура и интеграции (задачи 06–07), NFR (задача 02). Критичность: замки/камеры, критичные сценарии (протечка/дым), неустойчивая связь устройств, ПДн (присутствие людей).

## ТРЕБОВАНИЯ
- **Устойчивость:** примените паттерны (idempotency для команд/событий, retry+backoff, circuit breaker для внешних, saga при необходимости, graceful degradation при потере облака → edge).
- **NFR-реализация:** SLA/SLO для критичных сценариев (расчёт доступности), масштабирование под поток телеметрии (stateless, партиции), наблюдаемость (что логировать/мерить/трейсить).
- **Безопасность:** authn (OAuth2/OIDC, MFA для критичных действий), authz (роли владелец/член/гость + проверка владения — анти-BOLA), mTLS устройств, топ-5 угроз (OWASP/IoT) + меры.
- **Приватность:** ПДн (присутствие, видео), 152-ФЗ: согласие, минимизация, доступ гостей.

## ФОРМАТ РЕЗУЛЬТАТА
Три блока (устойчивость / NFR / безопасность+приватность) с конкретными паттернами, расчётом SLA и матрицей «угроза → мера».

## КРИТЕРИИ ПРИЁМКИ
- [ ] Паттерны устойчивости применены к конкретным точкам (команды, телеметрия, внешние, потеря облака).
- [ ] Graceful degradation: критичные сценарии на edge при потере облака.
- [ ] SLA/SLO для критичных сценариев + расчёт доступности.
- [ ] Масштабирование телеметрии (stateless, партиции) + наблюдаемость.
- [ ] Безопасность: authn/authz + анти-BOLA + mTLS; топ-5 угроз с мерами.
- [ ] Приватность ПДн (152-ФЗ): согласие, минимизация, гости.
- [ ] Согласованность с архитектурой (задачи 06–07).

## ПОДСКАЗКА
Опора — модули 25, 26, 34. IoT-специфика: критичные сценарии должны работать при потере облака (edge), команды идемпотентны (дубли), доступ к замкам — строгая authz + анти-BOLA (нельзя управлять чужим домом). Угрозы IoT: захват устройства, перехват команд, утечка присутствия. Это покрывает NFR из задачи 02.
