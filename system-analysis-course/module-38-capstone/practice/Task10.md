# Задача 10 (мини-проект) — Capstone: Сборка пакета + защита решения

## ЗАДАНИЕ
Соберите итоговый аналитический пакет по всему кейсу и подготовьте защиту решения (уровень middle+).

## ВХОДНЫЕ ДАННЫЕ
Артефакты задач 01–09 по платформе «Умный дом / IoT». Цель — единый связный пакет и его обоснование.

## ТРЕБОВАНИЯ
Сдайте аналитический пакет (синтез по чек-листу theory §4):
1. **Резюме решения** (1 абзац): что за система, ключевые решения, для каких целей.
2. **Сводка по слоям** (ссылки/выжимки задач 01–09): контекст, требования, процессы, данные, модели, архитектура, интеграции/API, NFR/безопасность, качество/поставка.
3. **Сквозная трассировка** (фрагмент): бизнес-цель → требование → модель/сущность → API → тест — на одном ключевом сценарии (напр. критичный «протечка» или «создание сценария»).
4. **Защита решения:**
   - обоснование 3 ключевых решений (стиль архитектуры, хранение телеметрии, стратегия выкатов) с trade-offs;
   - топ-5 рисков/ограничений + допущения;
   - как решение закрывает бизнес-цели и критичные NFR;
   - где данных не хватает (честная неопределённость).
5. **Самооценка по чек-листу** middle+ (что покрыто, что осталось).

## ФОРМАТ РЕЗУЛЬТАТА
Аналитический пакет: резюме + сводка по слоям + сквозная трассировка одного сценария + защита решения (обоснования/риски/соответствие целям/неопределённость) + самооценка по чек-листу.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Резюме решения (система, ключевые решения, цели).
- [ ] Сводка покрывает все слои (задачи 01–09), артефакты согласованы.
- [ ] Сквозная трассировка цель → требование → модель → API → тест на одном сценарии (без разрывов).
- [ ] Обоснование 3 ключевых решений с trade-offs (как ADR).
- [ ] Топ-5 рисков/ограничений + допущения.
- [ ] Показано соответствие целям и критичным NFR.
- [ ] Честно указана неопределённость (где не хватает данных).
- [ ] Самооценка по чек-листу middle+.
- [ ] Единый язык терминов через весь пакет (ubiquitous language).

## ПОДСКАЗКА
Синтез всего курса. Главное middle+: не объём диаграмм, а **связность и обоснованность**. Покажи трассировку на одном сценарии — это доказывает, что слои не противоречат друг другу. Защита = «почему так, что стоит, где риски, где неизвестно». Это и есть результат capstone — пакет, который не стыдно защитить на собеседовании.
