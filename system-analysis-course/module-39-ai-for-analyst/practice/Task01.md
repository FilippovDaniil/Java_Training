# Задача 01 — Модуль 39: ИИ для системного аналитика

## ЗАДАНИЕ
Составьте эффективный промпт (роль + контекст + задача + формат) для конкретной задачи СА.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**. Нужно получить от ИИ черновик критериев приёмки для требования «Клиент применяет промокод к заказу» (скидка при валидном коде, ограничения по сумме/сроку).

## ТРЕБОВАНИЯ
- Напишите промпт со всеми элементами: роль, контекст (домен + требование + ограничения), задача, формат ответа, при необходимости few-shot.
- Добавьте ограничение «отметить допущения, не выдумывать поля/статусы».
- Покажите слабый промпт-антипример и объясните, чем он хуже.
- Кратко: что именно делает ваш промпт лучше (по элементам).

## ФОРМАТ РЕЗУЛЬТАТА
Готовый промпт (структурированный) + антипример слабого промпта + разбор отличий.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Промпт содержит роль, контекст, задачу, формат.
- [ ] Есть ограничение «отметить допущения / не выдумывать».
- [ ] Указан желаемый формат (Gherkin/таблица).
- [ ] Приведён слабый антипример с разбором, чем хуже.
- [ ] Объяснено, как элементы повышают качество ответа.

## ПОДСКАЗКА
Опора — §2 теории. Структура: [Роль] ты СА middle+ → [Контекст] домен+требование+ограничения → [Задача] критерии приёмки → [Формат] Gherkin позитив/негатив → [Ограничения] отметь допущения. Слабый: «напиши критерии приёмки» (нет контекста/формата → общий ответ).
