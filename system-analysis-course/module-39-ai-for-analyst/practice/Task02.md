# Задача 02 — Модуль 39: ИИ для системного аналитика

## ЗАДАНИЕ
Сгенерируйте SQL по бизнес-описанию через ИИ и спроектируйте его проверку.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, схема (упрощённо): `orders(id, client_id, restaurant_id, total, created_at, status)`, `restaurants(id, name)`. Бизнес-вопрос: «выручка по ресторанам за прошлый месяц по доставленным заказам, топ-10».

## ТРЕБОВАНИЯ
- Сформулируйте промпт для генерации SQL (с указанием схемы и СУБД-нейтральности или конкретной СУБД).
- Приведите ожидаемый SQL-черновик (JOIN + агрегат + фильтр по статусу/дате + сортировка + LIMIT).
- Спроектируйте верификацию: как проверить корректность (на схеме, на тестовых данных, что именно сверять).
- Перечислите типовые ошибки, которые ИИ может допустить (неверный статус, граница дат, не те джойны).

## ФОРМАТ РЕЗУЛЬТАТА
Промпт + SQL-черновик + план верификации + список вероятных ошибок ИИ.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Промпт содержит схему и точную формулировку запроса.
- [ ] SQL-черновик: JOIN, агрегат (SUM/GROUP BY), фильтр статуса+даты, ORDER BY, LIMIT 10.
- [ ] План верификации (прогнать на схеме/данных, сверить результат).
- [ ] Перечислены вероятные ошибки ИИ (статус, границы дат, джойны, NULL).
- [ ] Подчёркнуто: SQL принимать только после прогона.

## ПОДСКАЗКА
Опора — §3, §7 теории и модуль 18. Дай ИИ схему — иначе он выдумает поля. SQL всегда прогонять: частые ошибки модели — неверный фильтр статуса, неточная граница «прошлого месяца», лишние/неверные джойны, потеря строк из-за NULL.
