# Задача 03 — Модуль 39: ИИ для системного аналитика

## ЗАДАНИЕ
Структурируйте «сырые» требования через ИИ и верифицируйте результат.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, сырые заметки после интервью (хаотичный текст):
«…клиент хочет видеть статус заказа… ещё нужно чтобы можно было отменить пока не готовят… оплата картой и может быть СБП… важно быстро, никто не будет ждать… промокоды для новых… ну и чтобы курьера на карте видно было…»

## ТРЕБОВАНИЯ
- Промпт, просящий структурировать заметки в каталог требований (с разделением функц./NFR, отметкой неоднозначностей и вопросов-уточнений).
- Приведите ожидаемый структурированный черновик (функц. требования + NFR + открытые вопросы).
- Верификация: сверьте черновик с исходными заметками — что ИИ мог потерять, добавить от себя, неверно интерпретировать.
- Отметьте «быстро» как неизмеримое → требует уточнения (не дать ИИ выдумать число).

## ФОРМАТ РЕЗУЛЬТАТА
Промпт + структурированный каталог (функц./NFR/вопросы) + результат сверки с источником.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Промпт просит структурировать + отметить неоднозначности/вопросы.
- [ ] Черновик разделяет функц. требования и NFR.
- [ ] Выделены открытые вопросы (СБП — точно? промокоды только новым?).
- [ ] «Быстро» помечено как неизмеримое (уточнить, не выдумывать число).
- [ ] Сверка с источником: что потеряно/добавлено/искажено.

## ПОДСКАЗКА
Опора — §4, §5 теории и модуль 05. ИИ хорошо причёсывает хаос в структуру, но может потерять пункт или выдумать деталь. Обязательно сверить с исходником. «Быстро» — это не NFR, пока нет метрики; ИИ не должен подставлять число сам.
