# Задача 04 — Модуль 39: ИИ для системного аналитика

## ЗАДАНИЕ
Получите от ИИ черновик user stories с критериями приёмки и отревьюйте его по INVEST/Gherkin.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, эпик «Избранное»: клиент сохраняет любимые рестораны/блюда и быстро заказывает повторно.

## ТРЕБОВАНИЯ
- Промпт на декомпозицию эпика в user stories + критерии приёмки (Gherkin).
- Приведите черновик (3–4 истории с критериями).
- Ревью по **INVEST**: проверьте каждую историю, найдите проблемы (слишком крупная, без ценности, непроверяемая) и исправьте.
- Проверьте критерии приёмки на проверяемость; добавьте недостающий негативный сценарий.
- Отметьте, что в черновике пришлось править (ответственность аналитика).

## ФОРМАТ РЕЗУЛЬТАТА
Промпт + черновик историй с критериями + ревью по INVEST (проблемы→правки) + добавленный негатив.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Промпт просит US + критерии приёмки в Gherkin.
- [ ] Черновик: 3–4 истории формата роль-цель-ценность.
- [ ] Ревью по INVEST с конкретными проблемами и правками.
- [ ] Критерии приёмки проверены на проверяемость.
- [ ] Добавлен ≥1 негативный сценарий, упущенный ИИ.
- [ ] Отмечено, что именно доработано аналитиком.

## ПОДСКАЗКА
Опора — §3, §4 теории и модуль 08. ИИ даёт болванку историй быстро, но часто: истории слишком крупные (нарушает INVEST), критерии без негатива. Аналитик доводит: разбить, добавить ценность, дописать негатив.
