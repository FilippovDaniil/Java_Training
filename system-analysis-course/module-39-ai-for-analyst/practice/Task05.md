# Задача 05 — Модуль 39: ИИ для системного аналитика

## ЗАДАНИЕ
Получите от ИИ диаграмму (BPMN/PlantUML) по текстовому сценарию и проверьте её.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, сценарий: «Клиент оформляет заказ → система проверяет минимальную сумму → оплата через шлюз → при успехе резерв на складе и подтверждение, при отказе — сообщение об ошибке».

## ТРЕБОВАНИЯ
- Промпт на генерацию диаграммы в текстовой нотации (PlantUML BPMN/sequence) по сценарию.
- Приведите ожидаемый черновик диаграммы.
- Верификация: проверьте нотацию (корректность синтаксиса/элементов) и логику (есть ли ветка ошибки, не потерян ли шаг).
- Найдите типовые промахи ИИ: только happy path, неверные шлюзы/события, пропуск проверки суммы.
- Исправьте найденное.

## ФОРМАТ РЕЗУЛЬТАТА
Промпт + черновик диаграммы (PlantUML/текст) + результат проверки нотации и логики + исправления.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Промпт задаёт нотацию и сценарий.
- [ ] Черновик диаграммы в корректной текстовой нотации.
- [ ] Проверена нотация (синтаксис/элементы).
- [ ] Проверена логика: ветка ошибки оплаты, проверка суммы, резерв.
- [ ] Найдены и исправлены промахи (happy-path, неверные шлюзы).
- [ ] Подчёркнуто: диаграмму сверять с требованием, а не принимать на веру.

## ПОДСКАЗКА
Опора — §3, §7 теории и модули 12, 33. ИИ часто рисует только happy path и путает sync/async или шлюзы. Сверь диаграмму со сценарием: все ли шаги, есть ли ошибка оплаты, верны ли элементы нотации.
