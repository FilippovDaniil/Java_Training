# Задача 07 — Модуль 39: ИИ для системного аналитика

## ЗАДАНИЕ
Сгенерируйте тест-кейсы по критериям приёмки через ИИ и проверьте покрытие.

## ВХОДНЫЕ ДАННЫЕ
**FoodExpress**, критерии приёмки требования «Применение промокода»:
- валидный активный промокод → скидка применена;
- истёкший промокод → отказ;
- сумма заказа ниже порога промокода → отказ.

## ТРЕБОВАНИЯ
- Промпт на генерацию тест-кейсов по критериям приёмки (структура: предусловия/шаги/ожидание).
- Приведите черновик тест-кейсов от ИИ.
- Проверка покрытия: трассировка критерий → тест-кейс; найдите непокрытое.
- Добавьте недостающие негативные/граничные кейсы, которые ИИ упустил (несуществующий код, уже использованный, граница суммы).
- Отметьте, почему нельзя доверять «полноте» ИИ без проверки трассировки.

## ФОРМАТ РЕЗУЛЬТАТА
Промпт + черновик тест-кейсов + матрица трассировки критерий→тест + добавленные кейсы.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Промпт просит тест-кейсы по критериям с полной структурой.
- [ ] Черновик покрывает заданные критерии.
- [ ] Матрица трассировки критерий → тест-кейс.
- [ ] Добавлены упущенные кейсы (несуществующий/использованный код, граница суммы).
- [ ] Конкретные ожидаемые результаты.
- [ ] Отмечено: проверять покрытие, не верить «полноте» ИИ.

## ПОДСКАЗКА
Опора — §3, §7 теории и модуль 35. ИИ сгенерит кейсы по явным критериям, но упустит неявные негативы/границы. Трассировка критерий→тест показывает пробелы; аналитик дополняет. «ИИ покрыл всё» — не доверять без проверки.
