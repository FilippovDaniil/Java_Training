# Задача 08 — Модуль 39: ИИ для системного аналитика

## ЗАДАНИЕ
Распознайте галлюцинацию/ошибку в ответе ИИ и исправьте промпт.

## ВХОДНЫЕ ДАННЫЕ
Аналитик спросил ИИ про требования, и получил ответ (с дефектами):
- «Согласно ГОСТ Р 12345-2020 "О промокодах", скидка не может превышать 40%.» (норма выдумана)
- SQL ссылается на колонку `orders.discount_amount` (в схеме её нет).
- «REST требует, чтобы все эндпоинты возвращали 200 OK.» (неверное утверждение)
- «Для надёжности достаточно at-most-once доставки.» (сомнительно для важных событий)

## ТРЕБОВАНИЯ
- Для каждого пункта: это галлюцинация/ошибка? как распознали? как проверить?
- Классифицируйте типы (выдуманный факт/НПА, несуществующее поле, неверное «правило», сомнительная рекомендация).
- Предложите исправленный промпт, снижающий такие ошибки (контекст, «отметь допущения», запрет выдумывать НПА/поля).
- Сформулируйте правило: что от ИИ всегда проверять.

## ФОРМАТ РЕЗУЛЬТАТА
Разбор каждого пункта (галлюцинация? как проверить?) + классификация + исправленный промпт + правило верификации.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Каждый пункт оценён как галлюцинация/ошибка с признаком распознавания.
- [ ] Выдуманный ГОСТ — проверять в реестре стандартов.
- [ ] Несуществующее поле — сверять со схемой.
- [ ] Неверное «правило REST/доставки» — сверять с теорией (модули 29, 25/32).
- [ ] Исправленный промпт (контекст + запрет выдумывать + отметить допущения).
- [ ] Правило: факты/цифры/НПА/имена полей — всегда верифицировать.

## ПОДСКАЗКА
Опора — §5, §7 теории. Маркеры галлюцинаций: точные номера ГОСТ/законов, конкретные поля, безапелляционные «правила». Проверка — первоисточник/схема/теория курса. Промпт-фикс: дать схему, запретить выдумывать НПА/поля, просить отмечать неуверенность.
