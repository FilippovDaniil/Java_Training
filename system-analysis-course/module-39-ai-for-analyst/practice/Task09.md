# Задача 09 — Модуль 39: ИИ для системного аналитика

## ЗАДАНИЕ
Определите, что нельзя отдавать в ИИ, и подготовьте данные через обезличивание.

## ВХОДНЫЕ ДАННЫЕ
**MedConnect**, аналитику нужно попросить ИИ помочь с SQL/анализом, но исходные данные содержат: ФИО пациентов, телефоны, диагнозы, реальный ключ платёжного API, фрагмент боевого дампа БД, внутренний документ под NDA.

## ТРЕБОВАНИЯ
- Классифицируйте, что из перечисленного **нельзя** отправлять в публичный ИИ и почему (152-ФЗ/тайна/секреты).
- Опишите обезличивание: как подготовить данные для безопасного промпта (маскирование, синтетика, абстрактная схема).
- Предложите безопасную альтернативу для чувствительного (корпоративный/локальный контур, политика провайдера).
- Свяжите с безопасностью (модуль 34): данные не покидают контур.
- Приведите пример: «было (с ПДн) → стало (обезличено)» для одного случая.

## ФОРМАТ РЕЗУЛЬТАТА
Классификация (можно/нельзя + причина) + правила обезличивания + безопасная альтернатива + пример «было→стало».

## КРИТЕРИИ ПРИЁМКИ
- [ ] ФИО/телефоны/диагнозы — нельзя (ПДн, 152-ФЗ); ключ API/секреты — нельзя; дамп/NDA — нельзя.
- [ ] Правила обезличивания (маскирование/синтетика/абстрактная схема).
- [ ] Безопасная альтернатива для чувствительного (локальный/корп. контур).
- [ ] Связь с модулем 34 (данные не покидают контур).
- [ ] Пример «было (ПДн) → стало (обезличено)».

## ПОДСКАЗКА
Опора — §6 теории и модуль 34. Правило: не отправляй в публичный ИИ то, что не дал бы постороннему. Для SQL достаточно **схемы без реальных строк** + синтетические данные. Секреты/ПДн/NDA — никогда; для них — защищённый контур.
