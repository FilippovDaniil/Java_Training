# Задача 04 — Модуль 01: Что такое System Design

## ЗАДАНИЕ
Для заданной системы перечислите оси дизайна, которые будут определять ключевые решения, и проранжируйте их по важности.

## ВХОДНЫЕ ДАННЫЕ
Система: мессенджер 1-на-1 и групповые чаты. Контекст: 200 млн активных пользователей, доставка сообщений «как можно быстрее», история переписки хранится бессрочно, важна доставка каждого сообщения (не терять), пользователи по всему миру.

## ТРЕБОВАНИЯ
- Перечислите активные оси дизайна (из §4 теории) для этой системы.
- Для каждой оси сформулируйте конкретный вопрос, на который придётся ответить в дизайне.
- Проранжируйте 3 самые важные оси для этого домена и обоснуйте ранжирование.
- Назовите хотя бы один компромисс между двумя осями.

## ФОРМАТ РЕЗУЛЬТАТА
Список осей с вопросами + топ-3 с обоснованием + один named trade-off.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Перечислены ≥ 5 осей (нагрузка, латентность, доступность, надёжность/доставка, данные, география).
- [ ] Для каждой оси — конкретный вопрос (не общий).
- [ ] Топ-3 включает латентность (быстрая доставка) и надёжность доставки (не терять сообщения), с обоснованием.
- [ ] Назван минимум один компромисс (например, гарантия доставки/порядок ↔ латентность; глобальная доступность ↔ согласованность порядка сообщений).

## ПОДСКАЗКА
Начните с фраз требований: «как можно быстрее» → латентность; «не терять» → надёжность доставки; «бессрочно» → данные/хранение; «по всему миру» → география/мультирегион. Оси, прямо названные в требованиях, обычно и есть топовые.
