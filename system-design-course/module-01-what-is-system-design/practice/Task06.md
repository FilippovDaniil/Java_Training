# Задача 06 — Модуль 01: Что такое System Design

## ЗАДАНИЕ
Составьте чек-лист «что делает дизайн хорошим» и примените его как рубрику к конкретному кейсу.

## ВХОДНЫЕ ДАННЫЕ
Кейс: коллега защищает дизайн системы загрузки и просмотра фото. Он принёс схему из 6 прямоугольников (клиент, API, сервис фото, БД, хранилище, CDN) со стрелками. На вопросы «сколько загрузок в секунду?», «какой объём за год?», «почему именно CDN, а не отдавать из хранилища напрямую?» внятного ответа нет. Альтернатив не рассматривалось.

## ТРЕБОВАНИЯ
- Составьте чек-лист из 6–8 критериев хорошего дизайна (на основе §6 теории).
- Примените чек-лист к кейсу: для каждого критерия — выполнен / не выполнен / частично, с пояснением.
- Сформулируйте 3 вопроса, которые вернут дизайн «на землю».

## ФОРМАТ РЕЗУЛЬТАТА
Чек-лист (критерий + что проверяем) + таблица оценки кейса + 3 наводящих вопроса.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Чек-лист включает: покрытие FR, удовлетворение NFR, обоснование числами, наличие trade-offs, выявленные узкие места, соразмерность, эволюционируемость.
- [ ] Кейс оценён по каждому критерию; отмечено отсутствие estimation и trade-offs как главный пробел.
- [ ] Наводящие вопросы бьют в числа (RPS/объём) и в обоснование выбора (CDN vs прямая отдача).
- [ ] Вывод: схема есть, дизайна (обоснованных решений) — нет.

## ПОДСКАЗКА
Хороший дизайн — не самый сложный, а обоснованный и соразмерный. Схема из прямоугольников — это начало (HLD-картинка), но без чисел и trade-offs она не отвечает на главный вопрос дизайна: «почему именно так?».
