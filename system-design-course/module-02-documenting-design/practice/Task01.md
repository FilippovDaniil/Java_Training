# Задача 01 — Модуль 02: Документирование дизайна

## ЗАДАНИЕ
Постройте диаграмму C4 уровня **Context** для заданной системы: система как чёрный ящик, её пользователи и внешние системы.

## ВХОДНЫЕ ДАННЫЕ
Система: интернет-магазин. Известно: покупатели заходят через веб и мобильное приложение; для оплаты используется внешний платёжный шлюз; уведомления о заказе шлются через внешний email/SMS-провайдер; каталог товаров частично импортируется из системы поставщика; сотрудники склада работают с заказами через админ-панель.

## ТРЕБОВАНИЯ
- Изобразите систему как один элемент (чёрный ящик).
- Покажите всех акторов (типы пользователей) и внешние системы.
- Подпишите связи (что передаётся / по какому каналу на уровне смысла, без деталей протокола).
- Не опускайтесь до контейнеров (БД, сервисы) — это следующий уровень.

## ФОРМАТ РЕЗУЛЬТАТА
Текстовая C4-Context диаграмма (PlantUML/Mermaid/псевдо-нотация) + список акторов и внешних систем.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Система показана как единый чёрный ящик.
- [ ] Акторы: покупатель, сотрудник склада (и при желании администратор).
- [ ] Внешние системы: платёжный шлюз, email/SMS-провайдер, система поставщика.
- [ ] Связи подписаны по смыслу («оформляет заказ», «проводит платёж», «шлёт уведомление», «импорт каталога»).
- [ ] Нет внутренних контейнеров/БД/сервисов (не тот уровень).

## ПОДСКАЗКА
Context отвечает на вопрос «с кем и с чем система взаимодействует». Всё, что внутри (сервисы, БД), — это уже Container. На этом уровне важна граница системы и её окружение, а не внутреннее устройство.
