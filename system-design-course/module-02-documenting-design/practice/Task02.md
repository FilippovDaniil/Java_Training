# Задача 02 — Модуль 02: Документирование дизайна

## ЗАДАНИЕ
Постройте диаграмму C4 уровня **Container** для той же системы: внутренние контейнеры (приложения, сервисы, хранилища, очереди) и протоколы между ними.

## ВХОДНЫЕ ДАННЫЕ
Тот же интернет-магазин (см. Task01). Дополнительно известно: есть SPA на React (веб), мобильное приложение, серверный API, БД заказов и каталога, кэш для каталога, очередь для отправки уведомлений.

## ТРЕБОВАНИЯ
- Покажите контейнеры внутри системы (SPA, мобильное приложение, API, БД, кэш, очередь, воркер уведомлений).
- Для каждой связи укажите **протокол/технологию** (HTTPS/REST, SQL, Redis-протокол, сообщения в очередь).
- Сохраните на схеме внешние системы из Context (платёжный шлюз, email/SMS-провайдер) с их связями.
- Не опускайтесь до классов и схем таблиц (это уровни Component/Code).

## ФОРМАТ РЕЗУЛЬТАТА
Текстовая C4-Container диаграмма + таблица «связь → протокол».

## КРИТЕРИИ ПРИЁМКИ
- [ ] Показаны контейнеры: SPA, мобильное приложение, API-сервис, БД, кэш, очередь, воркер уведомлений.
- [ ] У каждой связи указан протокол/технология (REST/JSON, SQL, Redis, сообщения).
- [ ] Внешние системы и их связи (платёж, email/SMS) присутствуют.
- [ ] «Container» трактуется как деплоимая единица, не как Docker-контейнер.
- [ ] Нет деталей уровня Component/Code (классы, таблицы).

## ПОДСКАЗКА
Container = то, что отдельно запускается/деплоится. На этом уровне впервые появляются протоколы — это важно для понимания связности и узких мест. SPA и мобильное приложение — тоже контейнеры (клиентские).
