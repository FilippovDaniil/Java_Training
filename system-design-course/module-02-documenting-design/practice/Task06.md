# Задача 06 — Модуль 02: Документирование дизайна

## ЗАДАНИЕ
Найдите ошибки в C4-диаграмме: смешение уровней, отсутствие протоколов, неверная трактовка элементов.

## ВХОДНЫЕ ДАННЫЕ
Диаграмма, помеченная автором как «C4 Container»:
> На одной схеме изображены: пользователь; «Web App»; «API Service»; класс `OrderController`; класс `OrderRepository`; «PostgreSQL»; «Платёжный шлюз»; Docker-контейнер `nginx`. Связи между элементами нарисованы стрелками **без подписей**. «PostgreSQL» подписана как «таблица orders».

## ТРЕБОВАНИЯ
- Перечислите все ошибки диаграммы.
- Для каждой объясните, какое правило C4 нарушено.
- Предложите исправленную версию (что убрать/добавить/переподписать), оставаясь на уровне Container.

## ФОРМАТ РЕЗУЛЬТАТА
Список ошибок с указанием нарушенного правила + описание исправленной диаграммы.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Отмечено смешение уровней: классы `OrderController`/`OrderRepository` — это уровень Code/Component, не Container.
- [ ] Отмечено отсутствие подписей протоколов на связях (нужны REST/JSON, SQL и т.д.).
- [ ] Отмечена путаница: «PostgreSQL как таблица orders» — БД это контейнер, таблица — деталь уровня ниже.
- [ ] Отмечено, что Docker-контейнер nginx смешивает понятие инфраструктуры с C4-container (можно показать как контейнер reverse-proxy, но не как «docker»).
- [ ] Исправленная версия: убрать классы, подписать протоколы, БД как контейнер без таблиц, единый уровень детализации.

## ПОДСКАЗКА
Главное правило: один уровень = одна диаграмма. Классы на Container-диаграмме — самая частая ошибка. Связи без протоколов на уровне Container обесценивают схему — именно протоколы показывают характер взаимодействия и потенциальные узкие места.
