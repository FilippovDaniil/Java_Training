# Задача 07 — Модуль 02: Документирование дизайна

## ЗАДАНИЕ
Набросайте структуру дизайн-документа по arc42 для конкретного кейса: какие разделы взять и что в них поместить.

## ВХОДНЫЕ ДАННЫЕ
Кейс: внутренний сервис аутентификации (SSO) для продуктов компании. Команда — 4 человека, документ нужен для онбординга и для аудита безопасности. Система средней сложности: несколько контейнеров, интеграция с внешними IdP, требования к безопасности и доступности.

## ТРЕБОВАНИЯ
- Выберите из 12 разделов arc42 те, что реально нужны этому кейсу (не все).
- Для каждого выбранного раздела укажите, что конкретно туда войдёт (1–2 строки).
- Обоснуйте, почему отбросили остальные разделы.
- Отметьте, где будут жить ADR и где — C4-диаграммы.

## ФОРМАТ РЕЗУЛЬТАТА
Список выбранных разделов arc42 с наполнением + обоснование отброшенных + размещение ADR/C4.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Выбраны как минимум: 1 (цели/качества), 3 (контекст — C4 Context), 5 (building blocks — C4 Container), 8 (сквозные концепции — безопасность), 9 (решения — ADR), 10 (качества — доступность/безопасность).
- [ ] Для безопасностного аудита явно усилены разделы 8 и 10.
- [ ] Обосновано отбрасывание лишних разделов (например, 12 глоссарий — опционально для маленькой команды).
- [ ] Указано, что ADR живут в разделе 9 (ссылками), C4 — в разделах 3 и 5.
- [ ] Подчёркнуто: arc42 — оглавление, берём нужное под цель документа.

## ПОДСКАЗКА
arc42 не требует заполнять всё. Цель документа (онбординг + аудит безопасности) подсказывает акценты: контекст и состав для онбординга, сквозная безопасность и качества — для аудита. Лишние разделы только разбавляют документ.
