# Задача 01 — Модуль 03: Фреймворк design-интервью

## ЗАДАНИЕ
Сформулируйте уточняющие вопросы к размытой постановке «Спроектируйте Twitter», сгруппировав их по типам.

## ВХОДНЫЕ ДАННЫЕ
Постановка от интервьюера: «Спроектируйте Twitter». Больше ничего не сказано — это намеренно.

## ТРЕБОВАНИЯ
- Сформулируйте 8–12 уточняющих вопросов.
- Сгруппируйте их: функциональные (scope фич), нефункциональные (масштаб/latency/доступность/согласованность), ограничения/контекст.
- Для каждого вопроса покажите, **как ответ повлияет на дизайн** (1 строка).
- Предложите разумный набор допущений на случай, если интервьюер скажет «решай сам».

## ФОРМАТ РЕЗУЛЬТАТА
Сгруппированный список вопросов + влияние на дизайн + набор дефолтных допущений.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Есть функциональные вопросы (какие фичи в фокусе: твит, лента, подписки, лайки, поиск, медиа?).
- [ ] Есть NFR-вопросы (DAU, read/write-ratio, latency ленты, доступность, согласованность ленты).
- [ ] Для каждого вопроса показано влияние на дизайн (например, read/write-ratio → стратегия fan-out).
- [ ] Предложены дефолтные допущения (например: фокус на твит+лента+подписки, 200 млн DAU, read-heavy, лента с допустимой задержкой в секунды).
- [ ] Не начинается проектирование — только сужение задачи.

## ПОДСКАЗКА
Цель шага 1 — превратить «спроектируйте Twitter» в выполнимую за 45 минут задачу. Хорошие вопросы сразу отсекают лишнее («лайки и DM вне scope?») и вытаскивают числа (DAU, read/write). Каждый вопрос должен иметь дизайн-последствие — иначе он бессмысленный.
