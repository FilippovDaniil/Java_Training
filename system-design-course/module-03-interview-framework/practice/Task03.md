# Задача 03 — Модуль 03: Фреймворк design-интервью

## ЗАДАНИЕ
Распределите 45 минут интервью по этапам фреймворка (тайм-бокс) и опишите риски срыва тайминга.

## ВХОДНЫЕ ДАННЫЕ
Интервью длится 45 минут (из них ~3–5 минут в конце — вопросы кандидата). Задача средней сложности: спроектировать сервис коротких ссылок.

## ТРЕБОВАНИЯ
- Постройте тайм-бокс на 8 шагов фреймворка с минутами.
- Отметьте, какие шаги нельзя сжимать (фундамент) и какие можно при нехватке времени.
- Опишите 2 типичных сценария срыва тайминга и как их избежать.
- Покажите, как озвучить план в начале интервью.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица «шаг → минуты → можно ли сжимать» + 2 сценария срыва + вступительная реплика с планом.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Сумма по шагам укладывается в ~40 минут (плюс резерв на вопросы).
- [ ] High-Level Design получает наибольшее время (ядро, ~10–12 мин).
- [ ] Помечено: требования + estimation сжимать нельзя; data model/estimation можно ужать при спешке, но не выбрасывать.
- [ ] Описаны сценарии срыва (зависание на требованиях; ранний уход в LLD) и противодействие.
- [ ] Дана вступительная реплика, проговаривающая план этапов.

## ПОДСКАЗКА
Тайм-бокс — это страховка от двух крайностей: «утонул в требованиях» и «проскочил требования». Объявленный вслух план («пройду требования → оценку → API → данные → схему → углубление → узкие места») и сам структурирует ответ, и помогает интервьюеру вас вести.
