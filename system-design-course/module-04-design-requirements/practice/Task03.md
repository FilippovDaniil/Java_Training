# Задача 03 — Модуль 04: Требования для дизайна

## ЗАДАНИЕ
Определите scope дизайна: разделите функциональность на «в границах» и «вне границ» для первой итерации.

## ВХОДНЫЕ ДАННЫЕ
Система: платформа онлайн-курсов. Полный набор возможностей: регистрация/профили, каталог курсов, видеоуроки, тесты и задания, прогресс обучения, сертификаты, оплата подписки, форум/обсуждения, рекомендации курсов, аналитика для преподавателей, мобильное приложение, live-вебинары.

## ТРЕБОВАНИЯ
- Выберите 3–4 возможности «в границах» первой итерации дизайна и обоснуйте.
- Перечислите «вне границ» с краткой причиной отсрочки.
- Проверьте, что выбранное ядро самодостаточно (можно спроектировать связно).
- Сформулируйте, как вы озвучите scope (контракт с интервьюером/командой).

## ФОРМАТ РЕЗУЛЬТАТА
Списки «в границах» (с обоснованием) и «вне границ» (с причиной) + реплика-формулировка scope.

## КРИТЕРИИ ПРИЁМКИ
- [ ] В границах — 3–4 связные ядровые возможности (например: каталог, просмотр видеоуроков, прогресс обучения, оплата).
- [ ] Вне границ — остальное (форум, рекомендации, вебинары, аналитика) с причиной отсрочки.
- [ ] Обосновано, почему ядро самодостаточно и архитектурно показательно (видео = нагрузка/CDN, оплата = надёжность).
- [ ] Дана явная формулировка scope для согласования.
- [ ] Не делается попытки взять всё сразу.

## ПОДСКАЗКА
Хороший scope — связное ядро, которое можно спроектировать целиком и которое демонстрирует ключевые архитектурные вызовы. Видеоуроки тянут за собой хранение/CDN, оплата — надёжность; это богаче, чем форум. Out-of-scope — не «не умеем», а «осознанно отложили».
