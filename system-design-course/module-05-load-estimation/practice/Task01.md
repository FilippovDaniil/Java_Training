# Задача 01 — Модуль 05: Оценка нагрузки

## ЗАДАНИЕ
Рассчитайте average RPS чтения и записи из заданного DAU, пройдя цепочку оценки.

## ВХОДНЫЕ ДАННЫЕ
Сервис микроблогов (Twitter-подобный). Допущения для расчёта: DAU = 200 млн; каждый пользователь в среднем читает ленту 20 раз в день; публикует 0.5 поста в день (в среднем).

## ТРЕБОВАНИЯ
- Посчитайте число чтений в день и число записей в день.
- Переведите в average read RPS и average write RPS (используя 1 сутки ≈ 10^5 c).
- Вычислите read/write-ratio.
- Сделайте вывод: read-heavy или write-heavy, и что это значит для дизайна.

## ФОРМАТ РЕЗУЛЬТАТА
Пошаговый расчёт с единицами + read/write-ratio + вывод для дизайна.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Чтений/день = 2×10^8 × 20 = 4×10^9; записей/день = 2×10^8 × 0.5 = 10^8.
- [ ] Average read RPS = 4×10^9 / 10^5 = 4×10^4 ≈ 40 000 RPS.
- [ ] Average write RPS = 10^8 / 10^5 = 10^3 = 1 000 RPS.
- [ ] Read/write-ratio = 40 000 : 1 000 = 40:1 (read-heavy).
- [ ] Вывод: выраженно read-heavy → акцент на кэш/реплики чтения; счёт ведётся в степенях десятки, числа округлены.

## ПОДСКАЗКА
Используйте 1 сутки ≈ 10^5 c — деление становится вычитанием степеней. 4×10^9 / 10^5 = 4×10^4. Не гонитесь за точностью: порядок величины и прозрачность важнее. Закончите выводом — ради него и считали.
