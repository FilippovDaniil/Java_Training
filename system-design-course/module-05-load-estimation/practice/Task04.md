# Задача 04 — Модуль 05: Оценка нагрузки

## ЗАДАНИЕ
Оцените внутреннюю нагрузку с учётом fan-out: покажите, как один внешний запрос усиливается во множество внутренних операций.

## ВХОДНЫЕ ДАННЫЕ
Соцсеть с моделью fan-out-on-write (пост раскладывается по лентам подписчиков при публикации). Допущения: публикуется 10^7 постов/день; средний автор имеет 500 подписчиков; но есть «звёзды» — 1000 аккаунтов с 10 млн подписчиков каждый, они дают 10^5 постов/день.

## ТРЕБОВАНИЯ
- Оцените число операций записи в ленты от обычных авторов (fan-out × посты).
- Оцените число операций записи от «звёзд».
- Сравните вклад обычных авторов и «звёзд» в общую нагрузку записи.
- Сделайте вывод: где узкое место и почему наивный fan-out-on-write ломается на «звёздах».

## ФОРМАТ РЕЗУЛЬТАТА
Расчёт fan-out по двум группам + сравнение вклада + вывод об узком месте.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Обычные авторы: ~ (10^7 − 10^5) постов × 500 ≈ 5×10^9 операций записи/день.
- [ ] «Звёзды»: 10^5 постов × 10^7 подписчиков = 10^12 операций записи/день.
- [ ] Показано, что «звёзды» дают на порядки больше нагрузки, несмотря на меньшее число постов.
- [ ] Вывод: наивный fan-out-on-write для «звёзд» нежизнеспособен (10^12/день) → нужен гибрид (on-read для «звёзд»).
- [ ] Различие внешнего (посты) и внутреннего (записи в ленты) RPS явно показано.

## ПОДСКАЗКА
Fan-out — усилитель нагрузки. Ключевой инсайт: маленькая группа «звёзд» с огромным fan-out доминирует над всей остальной нагрузкой. 10^5 постов × 10^7 подписчиков = 10^12 — это и есть причина, почему реальные ленты используют гибрид (fan-out-on-write для обычных, fan-out-on-read для «звёзд»).
