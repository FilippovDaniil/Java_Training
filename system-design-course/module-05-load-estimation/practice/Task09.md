# Задача 09 — Модуль 05: Оценка нагрузки

## ЗАДАНИЕ
Оцените необходимое число серверов под заданную нагрузку, исходя из пропускной способности одного сервера.

## ВХОДНЫЕ ДАННЫЕ
Сервис API. Peak нагрузка = 60 000 RPS. По нагрузочным тестам один инстанс приложения уверенно держит ~5 000 RPS при целевой latency. Требование: запас на отказ части инстансов и на рост, целевая утилизация инстанса ≤ 70%.

## ТРЕБОВАНИЯ
- Посчитайте минимальное число инстансов «впритык» (peak / на инстанс).
- Скорректируйте под целевую утилизацию ≤ 70%.
- Добавьте запас на отказоустойчивость (N+ к, чтобы переживать отказ нескольких инстансов).
- Сделайте вывод об итоговом числе инстансов и почему «впритык» — плохо.

## ФОРМАТ РЕЗУЛЬТАТА
Пошаговый расчёт (впритык → с утилизацией → с запасом) + итоговое число + вывод.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Впритык: 60 000 / 5 000 = 12 инстансов.
- [ ] С утилизацией ≤ 70%: 12 / 0.7 ≈ 18 инстансов.
- [ ] Добавлен запас на отказ (например, +2–3 инстанса / резерв зоны) → ~20+ инстансов.
- [ ] Вывод: «впритык» опасно — нет места под всплески, отказы и деплой; целятся в утилизацию с запасом.
- [ ] Расчёт прозрачен, единицы (RPS, инстансы) согласованы.

## ПОДСКАЗКА
Базовая формула: число инстансов = нагрузка / пропускная-на-инстанс, затем делим на целевую утилизацию (запас под всплески) и добавляем избыточность под отказы. Работать на 100% утилизации нельзя: не останется ресурса на пик, отказ инстанса или выкатку новой версии.
