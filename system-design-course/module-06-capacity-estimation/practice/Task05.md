# Задача 05 — Модуль 06: Оценка ёмкости

## ЗАДАНИЕ
Упорядочьте операции по задержке и примените «числа, которые надо знать» для оценки производительности сценария.

## ВХОДНЫЕ ДАННЫЕ
Операции для упорядочивания (вперемешку): чтение из RAM, round-trip между континентами, SSD random read, обращение к L1-кэшу, чтение 1 МБ с HDD, round-trip в пределах дата-центра, disk seek HDD.
Сценарий для оценки: запрос обслуживается либо (А) из кэша в RAM в том же ДЦ, либо (Б) с обращением к БД на HDD в другом дата-центре на другом континенте.

## ТРЕБОВАНИЯ
- Упорядочьте операции от самой быстрой к самой медленной с порядком задержки.
- Оцените суммарную задержку сценария А (кэш в RAM, локально) и Б (HDD на другом континенте).
- Посчитайте, во сколько раз примерно А быстрее Б (по порядку).
- Сделайте вывод для дизайна: что это говорит о кэшировании и географии.

## ФОРМАТ РЕЗУЛЬТАТА
Упорядоченный список операций с порядками + оценка А и Б + соотношение + вывод.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Порядок: L1 (~1 нс) < RAM (~100 нс) < SSD read (~0.1 мс) < round-trip в ДЦ (~0.5 мс) < disk seek/HDD 1MB (~10 мс) < межконтинентальный round-trip (~150 мс).
- [ ] Сценарий А ≈ десятки–сотни нс (RAM) — единицы мкс.
- [ ] Сценарий Б ≈ ~150 мс (сеть) + ~10 мс (HDD) ≈ ~160 мс.
- [ ] А быстрее Б примерно на 5–6 порядков (нс/мкс против сотен мс).
- [ ] Вывод: кэш в RAM и близость к пользователю (региональность/CDN) дают разницу в тысячи–миллионы раз; география и диск — главные источники latency.

## ПОДСКАЗКА
Запомните иерархию порядков: RAM ≪ SSD ≪ HDD ≪ сеть в ДЦ ≪ межконтинент. Главный инсайт: межконтинентальный round-trip (~150 мс) на ~5–6 порядков медленнее RAM (~100 нс). Отсюда два правила дизайна: кэшируй в памяти и держи данные близко к пользователю.
