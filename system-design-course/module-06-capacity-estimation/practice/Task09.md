# Задача 09 — Модуль 06: Оценка ёмкости

## ЗАДАНИЕ
Определите число кэш-серверов под горячий набор данных с учётом отказоустойчивости и запаса.

## ВХОДНЫЕ ДАННЫЕ
Распределённый кэш для ленты. Горячий набор данных оценён в 600 GB. Кэш-сервер: 128 GB RAM, под данные доступно ~100 GB. Требования: целевая утилизация памяти ≤ 75%, нужна репликация кэша (каждый ключ в 2 копиях для переживания отказа ноды).

## ТРЕБОВАНИЯ
- Посчитайте число серверов «впритык» под 600 GB.
- Учтите целевую утилизацию ≤ 75%.
- Учтите репликацию кэша ×2.
- Сделайте вывод об итоговом числе нод и почему репликацию кэша важно закладывать.

## ФОРМАТ РЕЗУЛЬТАТА
Пошаговый расчёт (впритык → утилизация → ×replication) + итоговое число + вывод.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Впритык: 600 GB / 100 GB = 6 серверов.
- [ ] С утилизацией ≤ 75%: 6 / 0.75 = 8 серверов.
- [ ] С репликацией кэша ×2: 8 × 2 = 16 серверов.
- [ ] Вывод: ~16 нод; репликация кэша нужна, чтобы отказ ноды не вызывал лавину промахов и удара по БД (cache stampede).
- [ ] Расчёт прозрачен, единицы согласованы.

## ПОДСКАЗКА
Цепочка та же, что для серверов приложения: данные / на ноду → /утилизация → ×replication. Особенность кэша: репликация защищает не только от потери данных, но и от внезапного провала hit rate при отказе ноды — иначе вся снятая нагрузка обрушится на БД. Закладывайте копии заранее.
