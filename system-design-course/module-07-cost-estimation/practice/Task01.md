# Задача 01 — Модуль 07: Cost Estimation

## ЗАДАНИЕ
Оцените месячную стоимость compute под заданную нагрузку, отталкиваясь от числа инстансов.

## ВХОДНЫЕ ДАННЫЕ
Сервис API. Из load/capacity-оценки: нужно ~20 инстансов приложения (с запасом). Ориентир цены: один инстанс среднего размера ≈ $100/мес (округлённо, как порядок). Дополнительно: балансировщик ≈ $30/мес, базовый мониторинг/логи ≈ $200/мес суммарно.

## ТРЕБОВАНИЯ
- Посчитайте стоимость compute (инстансы) в месяц.
- Добавьте сопутствующие расходы (балансировщик, мониторинг).
- Покажите, как стоимость масштабируется при удвоении нагрузки.
- Сделайте вывод: какая статья здесь доминирует и где есть запас на оптимизацию.

## ФОРМАТ РЕЗУЛЬТАТА
Расчёт $/мес по статьям + поведение при ×2 нагрузке + вывод.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Compute = 20 × $100 = $2 000/мес.
- [ ] Итого с сопутствующим ≈ $2 000 + $30 + $200 ≈ $2 230/мес.
- [ ] При ×2 нагрузке инстансы ~×2 → compute ~$4 000 (compute растёт с RPS почти линейно).
- [ ] Вывод: доминирует compute (инстансы); оптимизация — автоскейлинг, более плотная утилизация, spot-инстансы для части нагрузки.
- [ ] Цены трактуются как порядки/ориентиры, не точные тарифы.

## ПОДСКАЗКА
Compute обычно линейно растёт с нагрузкой (больше RPS → больше инстансов). Сопутствующие расходы (LB, мониторинг) часто фиксированы или растут медленнее. Считайте в порядках: точная цена инстанса не важна, важно, что compute здесь — главная статья.
