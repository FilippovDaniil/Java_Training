# Задача 03 — Модуль 07: Cost Estimation

## ЗАДАНИЕ
Оцените стоимость egress-трафика и покажите, во сколько раз CDN экономит деньги в медиа-системе.

## ВХОДНЫЕ ДАННЫЕ
Сервис фото. Из capacity-оценки: egress ≈ 32 GB/c при отдаче напрямую из origin. Ориентиры цены: egress из дата-центра в интернет ≈ $0.08 за GB; CDN-трафик ≈ $0.02 за GB; CDN снимает с origin ~90% повторных запросов (высокий cache hit на edge).

## ТРЕБОВАНИЯ
- Переведите egress 32 GB/c в объём за месяц.
- Посчитайте стоимость egress при прямой отдаче из origin.
- Посчитайте стоимость с CDN (90% трафика через CDN по $0.02, 10% через origin по $0.08).
- Сделайте вывод об экономии и других выгодах CDN (latency, разгрузка origin).

## ФОРМАТ РЕЗУЛЬТАТА
Перевод GB/c → GB/мес + стоимость без CDN + стоимость с CDN + экономия + вывод.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Объём/мес ≈ 32 GB/c × 2.6×10^6 c ≈ ~8.3×10^7 GB ≈ ~83 PB/мес (порядок).
- [ ] Без CDN: ~8.3×10^7 GB × $0.08 ≈ ~$6.6М/мес (огромная сумма — нереалистично для origin).
- [ ] С CDN: ~90% × $0.02 + ~10% × $0.08 ≈ средневзвешенно ~$0.026/GB → существенно дешевле.
- [ ] Экономия — кратная; плюс CDN снижает latency и разгружает origin (меньше compute).
- [ ] Вывод: для медиа egress — доминирующая статья, CDN обязателен и по деньгам, и по latency, и по разгрузке.

## ПОДСКАЗКА
Egress — самая коварная статья: гигантский исходящий трафик медиа по $0.08/GB даёт неподъёмный счёт. CDN дешевле за единицу И снимает большую часть трафика с origin (меньше egress origin + меньше compute). Это случай, когда оптимизация стоимости одновременно улучшает latency.
