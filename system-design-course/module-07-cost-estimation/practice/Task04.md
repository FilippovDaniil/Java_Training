# Задача 04 — Модуль 07: Cost Estimation

## ЗАДАНИЕ
Посчитайте $/запрос для системы и сопоставьте с экономикой бизнеса ($/пользователя vs выручка).

## ВХОДНЫЕ ДАННЫЕ
Сервис с суммарным счётом инфраструктуры ≈ $20 000/мес. Нагрузка: 100 млн запросов в день. Пользователи: MAU = 5 млн. Бизнес зарабатывает в среднем ARPU ≈ $0.50 на пользователя в месяц (реклама).

## ТРЕБОВАНИЯ
- Посчитайте число запросов в месяц.
- Посчитайте $/запрос.
- Посчитайте $/пользователя (инфраструктурная себестоимость на MAU).
- Сравните $/пользователя с ARPU и сделайте вывод о жизнеспособности модели.

## ФОРМАТ РЕЗУЛЬТАТА
Расчёт $/запрос и $/пользователя + сравнение с ARPU + вывод о юнит-экономике.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Запросов/мес = 10^8 × 30 = 3×10^9.
- [ ] $/запрос = $20 000 / 3×10^9 ≈ $6.7×10^-6 (≈ доли цента за тысячу запросов).
- [ ] $/пользователя = $20 000 / 5×10^6 = $0.004/мес.
- [ ] Сравнение: $0.004 инфраструктурной себестоимости ≪ $0.50 ARPU → модель сходится с большим запасом.
- [ ] Вывод: инфраструктура — малая доля выручки; если бы $/пользователя приближался к ARPU, модель была бы под угрозой.

## ПОДСКАЗКА
$/запрос и $/пользователя переводят инфраструктуру на язык бизнеса. Ключевая проверка: $/пользователя должен быть существенно меньше ARPU, иначе бизнес теряет деньги с каждого пользователя независимо от масштаба. Здесь запас огромный — инфраструктура «дешёвая» относительно выручки.
