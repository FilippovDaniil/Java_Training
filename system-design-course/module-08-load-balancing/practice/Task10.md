# Задача 10 — Модуль 08 (мини-проект): Слой балансировки системы

## ЗАДАНИЕ
Спроектируйте полный слой балансировки для системы — от точки входа до сервисов — с обоснованием каждого решения. Это синтез всего модуля.

## ВХОДНЫЕ ДАННЫЕ
Система на выбор (зафиксируйте одну): (а) read-heavy API соцсети; (б) интернет-магазин с пиками (распродажи); (в) видеоплатформа с пользователями в нескольких регионах. Пиковую нагрузку возьмите из своих расчётов модулей 05–07 (или задайте явно как допущение).

## ТРЕБОВАНИЯ
Дизайн должен покрывать:
1. **Топология** — глобальный уровень (нужен ли GSLB/anycast) + локальный LB + пул сервисов; диаграмма.
2. **L4 vs L7** — на каком уровне и почему; что маршрутизируете на L7 (если он есть).
3. **Алгоритм** — выбор под профиль запросов с обоснованием.
4. **Health checks** — тип, параметры, liveness/readiness, connection draining.
5. **HA самого LB** — схема, исключающая SPOF на точке входа.
6. **Stateless vs sticky** — где хранится состояние сессии.
7. **Ёмкость пула** — число backend'ов с запасом N+1 (расчёт).

## ФОРМАТ РЕЗУЛЬТАТА
Структурированный дизайн-документ из 7 разделов + диаграмма (текст/ASCII/Mermaid).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Зафиксирована одна система; нагрузка задана (своя оценка или допущение).
- [ ] Диаграмма показывает путь запроса от клиента через все уровни балансировки до сервиса.
- [ ] Уровень (L4/L7), алгоритм, health-checks, HA-LB, stateless/sticky — каждый выбран с обоснованием под кейс, а не по умолчанию.
- [ ] Точка входа не является SPOF (HA-пара/active-active или anycast).
- [ ] Ёмкость пула посчитана с запасом N+1; числа прозрачны.
- [ ] Для multi-region-варианта (в) разделены глобальный и локальный уровни и описан регион-failover.

## ПОДСКАЗКА
Это первый собранный строительный блок: дальше за LB появятся кэш (модуль 09), CDN (модуль 10), БД и очереди. Хороший слой балансировки заканчивается не «поставили nginx», а ответами на: на каком уровне, по какому алгоритму, как проверяем здоровье, как переживаем отказ сервера / зоны / самого LB и где живёт состояние. Все решения должны опираться на числа из Части 2.
