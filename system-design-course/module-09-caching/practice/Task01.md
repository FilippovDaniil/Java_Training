# Задача 01 — Модуль 09: Кэширование

## ЗАДАНИЕ
Выберите паттерн записи (write-through / write-back / write-around) под три сценария и обоснуйте.

## ВХОДНЫЕ ДАННЫЕ
- (А) профиль пользователя: читается часто, меняется редко, устаревшие данные недопустимы;
- (Б) счётчик просмотров: пишется очень часто, точность не критична, допустима потеря единичных инкрементов;
- (В) лог событий: пишется массово, читается редко и не сразу.

## ТРЕБОВАНИЯ
- Для каждого сценария выберите паттерн записи и паттерн чтения (cache-aside/read-through).
- Обоснуйте через trade-off (свежесть, скорость записи, засорение кэша, риск потери).
- Назовите для каждого, какой паттерн был бы ошибкой.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: сценарий → паттерн записи → паттерн чтения → обоснование → анти-выбор.

## КРИТЕРИИ ПРИЁМКИ
- [ ] (А) → write-through (свежесть критична, читается часто) + cache-aside на чтение.
- [ ] (Б) → write-back допустим (скорость, потеря единичных инкрементов терпима); либо in-memory счётчик с периодическим сбросом.
- [ ] (В) → write-around (пишется массово, читается редко → не засорять кэш редко-читаемым).
- [ ] Для каждого назван анти-выбор (напр. write-back для профиля — риск потери критичных данных).
- [ ] Выбор увязан с профилем (свежесть/частота/допустимость потери), а не «по умолчанию».

## ПОДСКАЗКА
Три вопроса решают выбор: важна ли свежесть (→ write-through), допустима ли потеря (→ можно write-back), читаются ли данные часто (редко → write-around, чтобы не засорять кэш). Cache-aside — дефолт на чтение.
