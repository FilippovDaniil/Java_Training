# Задача 01 — Модуль 10: CDN и хранение медиа

## ЗАДАНИЕ
Выберите режим CDN (pull vs push) под разные сценарии и обоснуйте.

## ВХОДНЫЕ ДАННЫЕ
- (А) пользовательские аватарки и картинки постов: огромный каталог, спрос непредсказуем, много новых файлов ежедневно;
- (Б) релизные бинарники приложения: известный набор крупных файлов, выкладываются по графику, скачиваются массово сразу после релиза;
- (В) VOD-видео популярного сериала: крупные файлы, заранее известны, ожидается пик в день премьеры.

## ТРЕБОВАНИЯ
- Для каждого сценария выберите pull или push и обоснуйте.
- Укажите, что произойдёт при первом запросе в каждом режиме.
- Назовите, какой режим был бы здесь неудачным выбором и почему.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: сценарий → режим → поведение первого запроса → обоснование → анти-выбор.

## КРИТЕРИИ ПРИЁМКИ
- [ ] (А) → pull (большой непредсказуемый каталог, само-наполнение по спросу, не хранить лишнее).
- [ ] (Б) и (В) → push (заранее известны, крупные, важно избежать холодного первого запроса при пике).
- [ ] Описано поведение первого запроса: pull — промах → fetch из origin (медленно); push — уже на edge (быстро).
- [ ] Для каждого назван анти-выбор (напр. pull для премьеры VOD → первые зрители ловят холодный fetch).
- [ ] Выбор увязан с предсказуемостью спроса и размером файлов, а не «по умолчанию».

## ПОДСКАЗКА
Pull — дефолт: прост, само-наполняется по спросу. Push — когда контент заранее известен, крупный и нельзя допустить «холодного» первого запроса (пиковые премьеры, релизы). Многие CDN умеют оба режима одновременно.
