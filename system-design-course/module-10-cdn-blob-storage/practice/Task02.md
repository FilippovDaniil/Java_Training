# Задача 02 — Модуль 10: CDN и хранение медиа

## ЗАДАНИЕ
Задайте стратегию cache-control / TTL для разных типов контента на CDN.

## ВХОДНЫЕ ДАННЫЕ
Сайт раздаёт через CDN: (1) версионированные ассеты `app.<hash>.js`/`style.<hash>.css`; (2) HTML главной страницы (меняется несколько раз в день); (3) JSON персонализированной ленты пользователя; (4) пользовательские картинки (immutable после загрузки).

## ТРЕБОВАНИЯ
- Для каждого типа задайте: кэшировать ли на CDN, TTL, `public`/`private`/`no-store`, способ инвалидации.
- Объясните, почему персонализированный JSON нельзя класть в shared edge-кэш.
- Покажите, как versioned URL снимает проблему инвалидации для (1) и (4).

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: контент → кэшировать? → TTL → директива → инвалидация + пояснения.

## КРИТЕРИИ ПРИЁМКИ
- [ ] (1) версионированные ассеты → очень долгий TTL, `public`, инвалидация не нужна (новый hash = новый URL).
- [ ] (2) HTML главной → короткий/умеренный TTL + purge при правке (или `s-maxage`).
- [ ] (3) персональный JSON → `private`/`no-store`, НЕ в shared edge (иначе чужие увидят чужое).
- [ ] (4) картинки immutable → долгий TTL, `public`; имя/URL уникален → инвалидация не нужна.
- [ ] Объяснено, что versioned URL = долгий TTL без purge (контент меняется только вместе с URL).

## ПОДСКАЗКА
Главный приём: immutable + versioned URL → ставь максимальный TTL и забудь про purge. Персональное и приватное помечай `private`/`no-store` — shared CDN-кэш общий для всех. Динамику с правками — короткий TTL + точечный purge.
