# Задача 05 — Модуль 10: CDN и хранение медиа

## ЗАДАНИЕ
Выберите тип хранилища (object/blob vs block vs file) под несколько вариантов использования и обоснуйте.

## ВХОДНЫЕ ДАННЫЕ
- (А) загруженные пользователями видео и картинки;
- (Б) диск под реляционную БД с частыми записями и низкой latency;
- (В) общая папка, к которой несколько worker-машин одновременно обращаются как к файловой системе;
- (Г) ночные бэкапы и архив логов за прошлые годы.

## ТРЕБОВАНИЯ
- Для каждого варианта выберите тип хранилища и обоснуйте характеристиками.
- Для (А) и (Г) уточните класс хранения (hot/cold/archive), если уместно.
- Назовите, какой выбор был бы ошибкой (напр. blob под БД) и почему.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: вариант → тип хранилища → (класс) → обоснование → анти-выбор.

## КРИТЕРИИ ПРИЁМКИ
- [ ] (А) → object/blob (медиа, immutable, раздача через CDN); hot-класс для активного.
- [ ] (Б) → block (диск под БД: низкая latency, частые правки блоков); blob здесь невозможен.
- [ ] (В) → file (POSIX, shared-доступ нескольких машин, иерархия).
- [ ] (Г) → object/blob, класс cold/archive (редкий доступ, дёшево).
- [ ] Назван анти-выбор с причиной (blob под БД — нет случайной записи/низкой latency; block под медиа+CDN — не отдать по HTTP, дорого).
- [ ] Выбор увязан со свойствами (immutable/random-write, shared, частота доступа).

## ПОДСКАЗКА
Три вопроса: правят ли данные по кусочкам и нужна ли низкая latency (→ block); нужен ли shared POSIX-доступ нескольких машин (→ file); это большие immutable-объекты по HTTP (→ blob). Частота доступа выбирает класс хранения внутри blob.
