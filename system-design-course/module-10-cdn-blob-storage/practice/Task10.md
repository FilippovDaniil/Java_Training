# Задача 10 — Модуль 10 (мини-проект): Хранение и доставка медиа

## ЗАДАНИЕ
Спроектируйте полную подсистему хранения и доставки медиа для системы — синтез всего модуля и завершение под-батча Части 3 (a).

## ВХОДНЫЕ ДАННЫЕ
Система на выбор (зафиксируйте одну): (а) сервис обмена фото (Instagram-подобный); (б) видеоплатформа (YouTube-подобная, упрощённо); (в) облачное хранилище файлов (Dropbox-подобное). Объёмы и трафик возьмите из расчётов модулей 05–07 (или задайте явно как допущение).

## ТРЕБОВАНИЯ
Дизайн должен покрывать:
1. **Хранение** — где лежат файлы (blob) и что хранится в БД (метаданные/ссылки); почему не в реляционной БД.
2. **Загрузка** — presigned direct upload, multipart для крупных файлов.
3. **Обработка** — асинхронный конвейер (transcode/resize/preview) через очередь, если нужен.
4. **Доставка** — CDN: pull/push, cache-control/TTL, инвалидация (versioned URL/purge).
5. **Tiering** — классы хранения под профиль доступа (hot/cold/archive).
6. **Стоимость** — оценка egress и экономии CDN, storage по классам (связь с модулем 07).
7. **Схема** — диаграмма потока: upload → blob → обработка → CDN → пользователь.

## ФОРМАТ РЕЗУЛЬТАТА
Структурированный дизайн-документ из 7 разделов + диаграмма с прозрачными расчётами.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Зафиксирована одна система; объёмы/трафик заданы (своя оценка или допущение).
- [ ] Файлы → blob, метаданные → БД; явно сказано, почему не в реляционной БД.
- [ ] Загрузка — presigned direct upload; для крупных файлов multipart.
- [ ] Обработка (если нужна) — асинхронная через очередь; раздача — через CDN.
- [ ] CDN: выбран режим, заданы TTL/инвалидация (versioned URL для статики/immutable).
- [ ] Tiering под профиль доступа; посчитаны egress-экономия CDN и storage по классам.
- [ ] Диаграмма отражает полный поток; решения опираются на числа Части 2.

## ПОДСКАЗКА
Это синтез модуля и финал под-батча (a): балансировка (08) + кэш (09) + CDN/blob (10) — три блока, приближающие данные и режущие нагрузку и стоимость. Канон: файл в blob, метаданные в БД, загрузка presigned, обработка асинхронно, раздача через CDN, инвалидация versioned URL, холодное в архив. Хороший дизайн заканчивается числами: egress-экономия, стоимость хранения, latency для удалённого пользователя. Дальше — Часть 3 (b): базы данных, репликация, шардинг.
