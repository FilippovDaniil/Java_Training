# Задача 03 — Модуль 11: Базы данных в дизайне

## ЗАДАНИЕ
Выберите NoSQL-семейство под профиль доступа для трёх подсистем и обоснуйте.

## ВХОДНЫЕ ДАННЫЕ
Принято решение использовать NoSQL для трёх подсистем:
- (А) лента активности: запись очень частая, чтение по «пользователь + диапазон времени», объём колоссальный;
- (Б) корзина покупок: читается/пишется целиком по id пользователя, структура — вложенный объект;
- (В) система рекомендаций: «похожие пользователи», «купившие это также купили», обходы связей.

## ТРЕБОВАНИЯ
- Для каждой подсистемы выберите семейство (key-value / document / wide-column / graph) и обоснуйте.
- Для (А) укажите, каким должен быть ключ/партиция под паттерн «пользователь + время».
- Назовите анти-выбор для каждой.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: подсистема → семейство → обоснование → (ключ для А) → анти-выбор.

## КРИТЕРИИ ПРИЁМКИ
- [ ] (А) → wide-column (Cassandra): write-heavy + ключ (user_id, time) под range-чтение по времени.
- [ ] (Б) → key-value или document: читается/пишется целиком по id (агрегат), вложенная структура → document.
- [ ] (В) → graph: обход связей и рекомендации.
- [ ] Для (А) предложен ключ с партицией по пользователю и кластеризацией по времени (range по времени внутри партиции).
- [ ] Для каждой назван анти-выбор (напр. graph под ленту — не та модель; wide-column под обход связей — дорого).

## ПОДСКАЗКА
Семейство выбирают по паре «модель данных + паттерн доступа». «Пользователь + диапазон времени» — каноничный wide-column (партиция = пользователь, сортировка = время). Целостный агрегат по id — key-value/document. Обходы связей — только graph.
