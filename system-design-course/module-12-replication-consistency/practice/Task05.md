# Задача 05 — Модуль 12: Репликация и согласованность

## ЗАДАНИЕ
Подберите параметры кворума (N/W/R) в leaderless-репликации под разные приоритеты и проверьте условие согласованности.

## ВХОДНЫЕ ДАННЫЕ
Leaderless-хранилище, N = 3 реплики. Три профиля:
- (А) нужна сильная гарантия «чтение видит последнюю запись» при переживании отказа одного узла;
- (Б) приоритет — максимально быстрая запись, чтения редкие и могут быть чуть устаревшими;
- (В) приоритет — максимально быстрые чтения, запись редкая.

## ТРЕБОВАНИЯ
- Для каждого профиля подберите W и R и проверьте условие `W+R>N`.
- Объясните, что гарантирует/не гарантирует выбор (видит ли чтение последнюю запись).
- Укажите, какой профиль переживает отказ одного узла на запись/чтение.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: профиль → W → R → W+R>N? → что гарантирует + заметка про отказ узла.

## КРИТЕРИИ ПРИЁМКИ
- [ ] (А) → W=2, R=2 (W+R=4>3 ✓): чтение пересекается с записью → видит свежее; переживает отказ 1 узла и на запись, и на чтение.
- [ ] (Б) → W=1, R=… : быстрая запись; если W+R≤N (напр. W=1,R=1) — eventual, чтение может не увидеть свежее.
- [ ] (В) → R=1, W=3: быстрые чтения; запись медленная и уязвима к отказу узла.
- [ ] Для каждого проверено условие `W+R>N` и сделан корректный вывод про свежесть чтения.
- [ ] Отмечено, какой профиль устойчив к отказу одного узла (W=2,R=2 — да; W=3 — запись падает при отказе узла).

## ПОДСКАЗКА
`W+R>N` гарантирует, что прочитанное множество пересекается с писавшим → есть узел с последней записью. W=2,R=2 при N=3 — классический баланс: и свежесть, и устойчивость к отказу одного узла. Чем выше W, тем уязвимее запись к отказам.
