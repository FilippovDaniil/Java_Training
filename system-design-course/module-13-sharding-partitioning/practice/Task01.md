# Задача 01 — Модуль 13: Шардирование и партиционирование

## ЗАДАНИЕ
Выберите стратегию шардирования (range / hash / directory / consistent hashing) под кейсы и обоснуйте.

## ВХОДНЫЕ ДАННЫЕ
- (А) аналитика по диапазонам времени: частые запросы «события за период», нужны range-выборки;
- (Б) профили пользователей: доступ по user_id, нужно равномерное распределение, число узлов будет расти;
- (В) мультиарендная SaaS: данные тенантов сильно разного размера, нужно гибко переселять крупных арендаторов на отдельные узлы.

## ТРЕБОВАНИЯ
- Для каждого кейса выберите стратегию и обоснуйте.
- Назовите главный риск выбранной стратегии для этого кейса.
- Укажите неудачную альтернативу и почему.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: кейс → стратегия → обоснование → главный риск → анти-выбор.

## КРИТЕРИИ ПРИЁМКИ
- [ ] (А) → range-based (дешёвые range-запросы по времени); риск — hotspot на «свежем» диапазоне; анти-выбор — hash (убивает range-запросы).
- [ ] (Б) → hash/consistent hashing (равномерность); т.к. число узлов растёт → consistent hashing (минимум перемещения); анти-выбор — range (hotspot) или `hash % N` (resharding ломает).
- [ ] (В) → directory/lookup (гибкое размещение крупных тенантов); риск — lookup как узкое место/SPOF; анти-выбор — hash (нельзя выборочно переселить «гиганта»).
- [ ] Для каждого назван риск и анти-выбор.
- [ ] Выбор увязан с паттерном запросов и ростом, а не «по умолчанию».

## ПОДСКАЗКА
Range — когда нужны диапазонные запросы (ценой hotspot). Hash/consistent — когда нужна равномерность (ценой range-запросов); растущее число узлов требует consistent hashing. Directory — когда нужно гибко и адресно размещать (ценой lookup-слоя).
