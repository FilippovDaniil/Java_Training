# Задача 02 — Модуль 13: Шардирование и партиционирование

## ЗАДАНИЕ
Выберите shard key для системы и обоснуйте по трём критериям; забракуйте плохих кандидатов.

## ВХОДНЫЕ ДАННЫЕ
Мессенджер. Основные паттерны: (1) загрузить сообщения конкретного чата по времени; (2) отправить сообщение в чат; (3) изредка — статистика по стране. Кандидаты на shard key: `country`, `created_at`, `message_id` (случайный), `chat_id`.

## ТРЕБОВАНИЯ
- Оцените каждого кандидата по трём критериям: кардинальность, равномерность, совпадение с паттернами запросов.
- Выберите лучший shard key и обоснуйте.
- Объясните, почему отвергнуты остальные (с конкретной проблемой).
- Укажите, как при выбранном ключе обслуживается редкий запрос «статистика по стране».

## ФОРМАТ РЕЗУЛЬТАТА
Таблица оценки кандидатов + выбор + объяснение отказов + про cross-shard статистику.

## КРИТЕРИИ ПРИЁМКИ
- [ ] `country` отвергнут: низкая кардинальность + страны-гиганты → hot shard.
- [ ] `created_at` отвергнут: монотонный → вся свежая запись в один шард (hotspot).
- [ ] `message_id` (случайный) равномерен, но НЕ совпадает с запросом «сообщения чата» → scatter-gather.
- [ ] Выбран `chat_id`: высокая кардинальность, совпадает с главными запросами (1)(2) — сообщения чата в одном шарде.
- [ ] Отмечен риск «звёздного» чата (очень крупный чат → hot shard) как остаточная проблема.
- [ ] «Статистика по стране» — редкий cross-shard запрос (scatter-gather/офлайн-аналитика), не оптимизируется под shard key.

## ПОДСКАЗКА
Лучший ключ совпадает с тем, как вы читаете чаще всего. Для мессенджера это `chat_id` (сообщения одного чата — в одном шарде). Монотонные (`created_at`) и низкокардинальные (`country`) ключи дают hotspot. Редкие аналитические запросы обслуживают отдельно, не ломая основной ключ.
