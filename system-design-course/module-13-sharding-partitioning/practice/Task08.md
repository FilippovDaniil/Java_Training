# Задача 08 — Модуль 13: Шардирование и партиционирование

## ЗАДАНИЕ
Рассчитайте число шардов под объём и нагрузку записи, с учётом репликации и запаса на рост.

## ВХОДНЫЕ ДАННЫЕ
- Общий объём данных: 40 ТБ сейчас, прогноз рост ×3 за 2 года.
- Запись: 120 000 RPS.
- Один узел-шард устойчиво держит: ~2 ТБ данных и ~15 000 RPS записи (целевая утилизация, не предел).
- Требование доступности: каждый шард реплицирован (replication factor = 3).

## ТРЕБОВАНИЯ
- Посчитайте число шардов по объёму (сейчас и с учётом роста ×3).
- Посчитайте число шардов по записи.
- Возьмите максимум из двух ограничений (что доминирует?).
- Учтите репликацию: сколько всего узлов (шарды × replication factor).
- Сделайте вывод и заложите запас/pre-sharding под рост.

## ФОРМАТ РЕЗУЛЬТАТА
Прозрачный расчёт по шагам (объём / запись / max / ×RF) + вывод про запас.

## КРИТЕРИИ ПРИЁМКИ
- [ ] По объёму: сейчас 40/2 = 20 шардов; с ростом ×3 → 120 ТБ / 2 = 60 шардов.
- [ ] По записи: 120 000 / 15 000 = 8 шардов.
- [ ] Доминирует объём (60 > 8) → ориентируемся на ~60 шардов под горизонт роста.
- [ ] С репликацией: 60 шардов × RF 3 = 180 узлов (как порядок).
- [ ] Вывод: закладывать запас и pre-sharding (логических шардов с запасом), чтобы рост не требовал болезненного resharding; числа — порядки.

## ПОДСКАЗКА
Число шардов = max(ограничение по объёму, ограничение по записи), с запасом на рост. Часто доминирует объём. Не забудьте умножить на replication factor для общего числа узлов. Pre-sharding (с запасом логических шардов) избавляет от частого resharding.
