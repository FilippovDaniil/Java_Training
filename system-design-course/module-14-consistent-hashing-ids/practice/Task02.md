# Задача 02 — Модуль 14: Consistent hashing и уникальные ID

## ЗАДАНИЕ
Опишите (и схематично изобразите) hash ring: размещение узлов и ключей, определение владельца, добавление узла.

## ВХОДНЫЕ ДАННЫЕ
Кольцо хеша. Узлы A, B, C размещены на кольце. Несколько ключей k1…k5 с известными (условными) позициями. Затем добавляется узел D между B и C.

## ТРЕБОВАНИЯ
- Объясните правило принадлежности ключа узлу (по часовой стрелке до первого узла).
- Нарисуйте кольцо (ASCII/текст) с узлами и ключами; укажите владельца каждого ключа.
- Покажите, какие ключи переедут при добавлении D и почему остальные останутся.
- Объясните, что происходит при удалении узла (куда уходят его ключи).

## ФОРМАТ РЕЗУЛЬТАТА
Правило принадлежности + схема кольца + список переездов при добавлении D + поведение при удалении.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Сформулировано правило: ключ принадлежит первому узлу по часовой стрелке от его позиции.
- [ ] Схема кольца с узлами и ключами; для каждого ключа указан владелец.
- [ ] При добавлении D переезжают только ключи на участке между предыдущим узлом и D; остальные не трогаются.
- [ ] При удалении узла его ключи уходят следующему по часовой стрелке.
- [ ] Видно ключевое свойство: локальность изменений (трогается только сосед), а не всё кольцо.

## ПОДСКАЗКА
Кольцо — это замкнутый диапазон хеша. И узлы, и ключи на нём. Владелец ключа — ближайший узел «вперёд» (по часовой). Поэтому новый узел отбирает ключи только у одного соседа, а не у всех.
