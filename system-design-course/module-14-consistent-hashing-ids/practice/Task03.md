# Задача 03 — Модуль 14: Consistent hashing и уникальные ID

## ЗАДАНИЕ
Объясните проблему неравномерности в базовом consistent hashing и спроектируйте решение через virtual nodes.

## ВХОДНЫЕ ДАННЫЕ
Кластер из 3 физических узлов на кольце. Наблюдается перекос: один узел получает заметно больше ключей и нагрузки. Дополнительно: один из узлов вдвое мощнее остальных и должен брать больше данных.

## ТРЕБОВАНИЯ
- Объясните, почему при немногих узлах участки кольца получаются неравными → перекос.
- Опишите, как virtual nodes выравнивают распределение.
- Покажите, как vnodes позволяют учесть разную мощность узлов (веса).
- Объясните бонус vnodes при отказе узла (как распределяется его нагрузка).

## ФОРМАТ РЕЗУЛЬТАТА
Причина перекоса → как vnodes выравнивают → веса под мощность → поведение при отказе.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Причина: мало узлов → мало точек → участки разной длины → неравномерная нагрузка.
- [ ] vnodes: каждый физический узел = много точек на кольце → усреднение участков → равномерность.
- [ ] Веса: более мощному узлу дают больше vnodes → ему достаётся больше ключей пропорционально.
- [ ] При отказе узла его vnodes (а значит, нагрузка) распределяются по многим соседям, а не валятся на одного.
- [ ] Решение направлено и на равномерность, и на гетерогенность.

## ПОДСКАЗКА
Чем больше точек на кольце, тем равномернее участки (закон больших чисел). vnodes = «размножить» каждый узел в десятки точек. Хочешь дать узлу больше данных — дай ему больше vnodes. Бонус: отказ размазывается, а не бьёт по одному соседу.
