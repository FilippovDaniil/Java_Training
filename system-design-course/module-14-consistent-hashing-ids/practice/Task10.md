# Задача 10 — Модуль 14 (мини-проект): Распределение данных и генерация ID

## ЗАДАНИЕ
Спроектируйте схему распределения данных (consistent hashing) и генерации уникальных ID для системы. Синтез модуля.

## ВХОДНЫЕ ДАННЫЕ
Система на выбор (зафиксируйте одну): (а) распределённый KV-кэш; (б) шардированная БД сообщений; (в) система событий с миллионами записей/с. Нагрузку/объём и число узлов возьмите из расчётов модулей 05–06, 13 (или задайте явно как допущение).

## ТРЕБОВАНИЯ
Дизайн должен покрывать:
1. **Распределение ключей** — consistent hashing + virtual nodes; правило принадлежности.
2. **Эластичность** — что переезжает при добавлении/выводе узла (доля).
3. **Равномерность и веса** — как vnodes выравнивают и учитывают разную мощность.
4. **Генерация ID** — выбор схемы (Snowflake/ULID/UUID/...) под требования (сортируемость, координация, размер).
5. **Раскладка ID** — если Snowflake-подобный: биты под нагрузку и число узлов.
6. **Clock skew** — как защищаетесь (если ID сортируемый по времени).
7. **Связи** — как это стыкуется с кэшем (09), шардированием (13), репликацией (12).

## ФОРМАТ РЕЗУЛЬТАТА
Структурированный дизайн-документ из 7 разделов + схема кольца/раскладки ID.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Зафиксирована одна система; нагрузка/узлы заданы (своя оценка или допущение).
- [ ] Распределение через consistent hashing + vnodes; описано правило принадлежности и доля переезда при изменении узлов.
- [ ] Показано, как vnodes дают равномерность и веса под мощность.
- [ ] Выбрана схема ID под требования; обоснованы сортируемость/координация/размер.
- [ ] Для Snowflake-подобного — раскладка битов с проверкой под нагрузку/узлы.
- [ ] Учтён clock skew (если ID сортируемый).
- [ ] Показаны связи с модулями 09/12/13.

## ПОДСКАЗКА
Это синтез модуля и «клей» для предыдущих: consistent hashing соединяет кэш (09) и шарды (13), ID нужны каждой записи. Хорошая схема отвечает на: как распределяем ключи (CH+vnodes), что переезжает при росте, как генерируем уникальные сортируемые ID без координации и как защищены от clock skew.
