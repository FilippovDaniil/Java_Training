# Задача 03 — Модуль 15: Очереди и стриминг

## ЗАДАНИЕ
Выберите семантику доставки под требования и объясните, как достигается «эффект exactly-once».

## ВХОДНЫЕ ДАННЫЕ
- (А) обработка платежей: нельзя списать дважды, нельзя потерять платёж;
- (Б) сбор метрик-телеметрии: потеря единичных точек терпима, скорость важнее;
- (В) рассылка push-уведомлений: лучше доставить, чем потерять; дубль раздражает, но не критичен.

## ТРЕБОВАНИЯ
- Для каждого кейса выберите семантику (at-most-once / at-least-once) и обоснуйте.
- Объясните, почему «настоящий» exactly-once дорог/труднодостижим.
- Для (А) и (В) покажите, как at-least-once + идемпотентность даёт эффект exactly-once.
- Назовите, что произойдёт при at-least-once без идемпотентности.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: кейс → семантика → обоснование + объяснение «at-least-once + идемпотентность».

## КРИТЕРИИ ПРИЁМКИ
- [ ] (А) → at-least-once + идемпотентность (нельзя терять; дубли гасятся идемпотентностью).
- [ ] (Б) → at-most-once приемлем (скорость, потеря терпима).
- [ ] (В) → at-least-once (лучше доставить; дубль некритичен, но желательно гасить).
- [ ] Объяснено, почему exactly-once дорог (нужны end-to-end транзакционные гарантии, в общем случае недостижим дёшево).
- [ ] Показано: at-least-once + идемпотентный consumer = эффект exactly-once без его цены.
- [ ] Названо последствие at-least-once без идемпотентности (двойное списание/уведомление).

## ПОДСКАЗКА
Практический стандарт — at-least-once + идемпотентность. «Настоящий» exactly-once дорог, потому что требует атомарности «обработал И зафиксировал offset» через всю цепочку. Идемпотентность делает повтор безопасным, давая тот же результат.
