# Задача 02 — Модуль 17: Поиск, автодополнение и геосервисы

## ЗАДАНИЕ
Постройте inverted index для набора документов и покажите обработку запроса с ранжированием.

## ВХОДНЫЕ ДАННЫЕ
Документы:
- d1 = "scalable system design"
- d2 = "design patterns for systems"
- d3 = "system design interview"

Запрос: `system design`.

## ТРЕБОВАНИЯ
- Прогоните документы через конвейер: токенизация → нормализация → (упрощённый) стемминг → удаление стоп-слов.
- Постройте inverted index (term → список документов).
- Покажите обработку запроса (пересечение списков).
- Объясните на пальцах, как TF-IDF/BM25 отранжировал бы результаты (какой документ выше и почему).

## ФОРМАТ РЕЗУЛЬТАТА
Конвейер обработки + таблица inverted index + обработка запроса + объяснение ранжирования.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Конвейер применён: lowercase, "systems"→"system" (стемминг), стоп-слово "for" удалено.
- [ ] Inverted index построен: term → документы (system → [d1,d2,d3], design → [d1,d2,d3], ...).
- [ ] Запрос "system design" → пересечение списков → все три документа содержат оба терма.
- [ ] Объяснено ранжирование: документ с большей частотой термов / более коротким текстом (BM25-нормализация) ранжируется выше; редкие термы (IDF) весомее.
- [ ] Видно, что индекс отвечает «взять список по терму», а не «сканировать документы».

## ПОДСКАЗКА
Inverted index = «term → где встречается». Запрос = пересечение списков термов. Ранжирование: TF (частота в документе) × IDF (редкость по корпусу), BM25 нормализует по длине. Стемминг объединяет «systems/system», стоп-слова выкидываются.
