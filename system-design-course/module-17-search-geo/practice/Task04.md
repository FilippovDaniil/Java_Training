# Задача 04 — Модуль 17: Поиск, автодополнение и геосервисы

## ЗАДАНИЕ
Спроектируйте автодополнение (typeahead) через trie с топ-K предложениями.

## ВХОДНЫЕ ДАННЫЕ
Поисковая строка сайта: при вводе показывать до 5 самых популярных запросов, начинающихся с введённого префикса. Подсказки должны появляться быстрее, чем пользователь печатает следующий символ. Популярность запросов известна (частоты).

## ТРЕБОВАНИЯ
- Объясните, почему trie подходит для префиксного поиска.
- Опишите, как получить топ-5 по префиксу (наивно и оптимизированно).
- Предложите хранение топ-K в узлах trie (предрасчёт), чтобы не обходить всё поддерево.
- Объясните, как обновляются подсказки при изменении популярности.

## ФОРМАТ РЕЗУЛЬТАТА
Почему trie + получение топ-K (наивно vs предрасчёт) + хранение топ-K в узлах + обновление.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Объяснено: trie позволяет спуститься по префиксу за длину префикса и собрать продолжения.
- [ ] Наивно: обойти всё поддерево префикса и отсортировать → дорого для популярных коротких префиксов.
- [ ] Оптимизация: в каждом узле хранить предрассчитанный топ-K популярных предложений → подсказки мгновенно.
- [ ] Обновление: топ-K пересчитывают периодически/по событию при изменении частот (не на каждый запрос).
- [ ] Решение укладывается в latency-бюджет (десятки мс на символ).

## ПОДСКАЗКА
Trie = дерево префиксов: спуск по «c→a→r» даёт поддерево всех слов на "car". Но обходить поддерево на каждый символ дорого. Решение — заранее посчитать и хранить топ-K в узлах, обновляя их в фоне. Typeahead жёстко latency-критичен → предрасчёт обязателен.
