# Задача 07 — Модуль 17: Поиск, автодополнение и геосервисы

## ЗАДАНИЕ
Разберите geohash: выбор гранулярности и решение граничной проблемы.

## ВХОДНЫЕ ДАННЫЕ
Геопоиск на geohash. Два наблюдения: (1) при некоторой длине geohash в ячейку попадает то слишком много, то слишком мало объектов; (2) две близкие точки иногда «не находят» друг друга, хотя расстояние мало.

## ТРЕБОВАНИЯ
- Объясните, как длина geohash (число символов) влияет на размер ячейки и число кандидатов.
- Объясните граничную проблему: почему близкие точки могут иметь разные geohash-префиксы.
- Предложите решение граничной проблемы (проверка соседних ячеек).
- Дайте принцип выбора гранулярности под радиус запроса.

## ФОРМАТ РЕЗУЛЬТАТА
Влияние длины geohash → граничная проблема → решение (соседние ячейки) → принцип выбора гранулярности.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Объяснено: больше символов geohash → меньше ячейка → меньше кандидатов (но больше ячеек для покрытия радиуса), и наоборот.
- [ ] Граничная проблема: две близкие точки по разные стороны границы ячейки имеют разные префиксы → по своей ячейке не находятся.
- [ ] Решение: при запросе брать не только свою ячейку, но и 8 соседних, затем фильтровать по точному расстоянию.
- [ ] Принцип: выбирать длину geohash так, чтобы размер ячейки был соизмерим с радиусом запроса (баланс «лишние кандидаты ↔ много ячеек»).
- [ ] Финальная фильтрация по реальному расстоянию обязательна.

## ПОДСКАЗКА
Длина geohash = точность: длиннее → мельче ячейка. Главная ловушка — граница: соседние точки в разных ячейках. Лечится проверкой 8 соседних ячеек + фильтрацией по точному расстоянию. Гранулярность подбирают под радиус: ячейка ≈ радиус запроса.
