# Задача 01 — Модуль 18: OLTP vs OLAP, DWH

## ЗАДАНИЕ
Классифицируйте набор нагрузок как OLTP или OLAP и обоснуйте.

## ВХОДНЫЕ ДАННЫЕ
Запросы/операции:
- (А) «создать заказ и списать товар со склада»;
- (Б) «выручка по регионам и категориям за последние 3 года, по месяцам»;
- (В) «показать профиль пользователя по id»;
- (Г) «воронка конверсии: сколько дошло от просмотра до покупки за квартал»;
- (Д) «обновить статус платежа по id».

## ТРЕБОВАНИЯ
- Для каждой операции укажите OLTP или OLAP и обоснуйте (паттерн доступа, объём, метрика).
- Сгруппируйте: какие должны идти на прод-БД, какие — в аналитическое хранилище.
- Объясните, по каким признаках вы различали.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: операция → OLTP/OLAP → обоснование + группировка (прод vs DWH).

## КРИТЕРИИ ПРИЁМКИ
- [ ] (А), (В), (Д) → OLTP (точечные операции по ключу, низкая latency, прод-БД).
- [ ] (Б), (Г) → OLAP (агрегации по большим объёмам и периодам, throughput, DWH).
- [ ] Признаки различения названы: точечный доступ vs сканы/агрегации, мелкие транзакции vs тяжёлые запросы, latency vs throughput.
- [ ] OLAP-операции отнесены к аналитическому хранилищу, не к проду.
- [ ] Вывод увязан с разделением нагрузок.

## ПОДСКАЗКА
OLTP = «точечно по ключу, быстро, отдельные строки» (создать/прочитать/обновить запись). OLAP = «агрегации по миллионам строк за период» (выручка, воронки, тренды). Первое — на прод, второе — в DWH.
