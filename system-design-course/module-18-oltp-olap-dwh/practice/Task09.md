# Задача 09 — Модуль 18: OLTP vs OLAP, DWH

## ЗАДАНИЕ
Выполните dimensional modeling: разделите данные домена на факты и измерения.

## ВХОДНЫЕ ДАННЫЕ
Стриминговый видеосервис. Сырые данные: события просмотра (кто, что, когда, сколько минут, на каком устройстве, в каком регионе, по какой подписке). Нужно анализировать: просмотры и время по контенту, по регионам, по устройствам, по тарифам, во времени.

## ТРЕБОВАНИЯ
- Выделите fact table: какие метрики (измеримые числа) и какие ключи измерений.
- Выделите dimension tables и их атрибуты.
- Объясните принцип отнесения поля к факту или измерению.
- Приведите 2 аналитических запроса и покажите, какие части схемы они задействуют.

## ФОРМАТ РЕЗУЛЬТАТА
fact table + dimension tables + принцип разделения + 2 примера запросов.

## КРИТЕРИИ ПРИЁМКИ
- [ ] fact_views: метрики (минуты просмотра, счётчик просмотров) + ключи (date_id, content_id, device_id, region_id, plan_id).
- [ ] Измерения: dim_content (название/жанр), dim_device, dim_region, dim_plan, dim_date.
- [ ] Принцип: измеримые числа/события → факты; описательный контекст для группировки → измерения.
- [ ] Пример 1 (напр. «время просмотра по жанрам за месяц») и пример 2 (напр. «просмотры по устройствам и регионам») с указанием задействованных таблиц.
- [ ] Факты растут со временем (большая таблица), измерения небольшие.

## ПОДСКАЗКА
Спроси себя про каждое поле: «это число, которое я суммирую/считаю?» → факт; «это разрез, по которому я группирую/фильтрую?» → измерение. Минуты и счётчики — факты; жанр, устройство, регион, тариф, дата — измерения.
