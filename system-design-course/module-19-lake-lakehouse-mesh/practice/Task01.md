# Задача 01 — Модуль 19: Data Lake, Lakehouse, Data Mesh

## ЗАДАНИЕ
Выберите между DWH и Data Lake под кейсы и обоснуйте.

## ВХОДНЫЕ ДАННЫЕ
- (А) финансовая отчётность и BI-дашборды по структурированным данным, нужна высокая производительность и надёжность;
- (Б) хранение сырых логов, кликстрима, изображений для будущего ML, формат разнородный, объём огромный, бюджет важен;
- (В) исследовательская аналитика data scientists по сырым событиям, заранее неизвестно, какие признаки понадобятся.

## ТРЕБОВАНИЯ
- Для каждого кейса выберите DWH или Data Lake и обоснуйте (формат, схема, стоимость, потребители).
- Укажите главный риск выбранного варианта.
- Назовите неудачный выбор для каждого.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: кейс → DWH/Lake → обоснование → риск → анти-выбор.

## КРИТЕРИИ ПРИЁМКИ
- [ ] (А) → DWH (структурировано, BI, производительность, schema-on-write); риск — дороже/жёстче; анти-выбор — Lake (медленный BI без структуры).
- [ ] (Б) → Data Lake (любой формат, дёшево, ML, schema-on-read); риск — data swamp; анти-выбор — DWH (дорого, не для сырья/неструктуры).
- [ ] (В) → Data Lake (заранее неизвестна схема → schema-on-read); анти-выбор — DWH (требует схему заранее).
- [ ] Названы риски и анти-выборы.
- [ ] Выбор увязан с форматом/схемой/стоимостью/потребителями.

## ПОДСКАЗКА
DWH — структура, BI, schema-on-write, дорого. Lake — сырьё любого формата, дёшево, schema-on-read, ML. Если схему/использование знаешь заранее и нужен быстрый BI → DWH; если хранишь «всё на будущее» дёшево → Lake (но с governance).
