# Задача 09 — Модуль 19: Data Lake, Lakehouse, Data Mesh

## ЗАДАНИЕ
Выберите дата-платформу под конкретную организацию и обоснуйте.

## ВХОДНЫЕ ДАННЫЕ
- (А) стартап: структурированные данные, нужен быстрый BI, маленькая команда, нет ML;
- (Б) data-driven скейлап: и BI, и ML, не хочет держать два хранилища (Lake + DWH), растущие объёмы сырья;
- (В) корпорация: десятки автономных доменов, центральная дата-команда — узкое место, разные потребности у доменов.

## ТРЕБОВАНИЯ
- Для каждой организации выберите платформу (DWH / Lake / Lakehouse / Mesh) и обоснуйте.
- Учтите данные, зрелость, размер организации, потребности (BI/ML).
- Назовите, что было бы избыточным или недостаточным выбором.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: организация → платформа → обоснование → избыточный/недостаточный выбор.

## КРИТЕРИИ ПРИЁМКИ
- [ ] (А) → DWH (структурированный BI, простота, малая команда); избыточно — Mesh/сложный Lake.
- [ ] (Б) → Lakehouse (BI + ML без двух хранилищ, table format); недостаточно — только DWH (нет дешёвого сырья/ML) или только Lake (слабый BI).
- [ ] (В) → Data Mesh (организационный масштаб, автономные домены); недостаточно — централизованный монолит (узкое место).
- [ ] Выбор увязан с данными/зрелостью/размером/потребностями.
- [ ] Названы избыточные/недостаточные варианты.

## ПОДСКАЗКА
Идите от размера и потребностей: малый + только BI → DWH; нужны и BI, и ML без дублирования → Lakehouse; организационный масштаб с автономными доменами → Mesh. Не выбирайте «на вырост»: Mesh для стартапа избыточен, один DWH для ML-скейлапа недостаточен.
