# Задача 01 — Модуль 20: Конвейеры данных

## ЗАДАНИЕ
Выберите между ETL и ELT под кейсы и обоснуйте.

## ВХОДНЫЕ ДАННЫЕ
- (А) современный облачный DWH (Snowflake/BigQuery), нужно сохранить сырьё для будущих переобработок, трансформации меняются;
- (Б) загрузка в строго схематизированное хранилище со слабыми вычислительными ресурсами, схема фиксирована;
- (В) данные содержат чувствительные поля, которые нельзя хранить в сыром виде в целевом хранилище.

## ТРЕБОВАНИЯ
- Для каждого кейса выберите ETL или ELT и обоснуйте.
- Объясните, где происходит трансформация в каждом подходе.
- Назовите главное преимущество выбранного варианта для кейса.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: кейс → ETL/ELT → где трансформация → обоснование/преимущество.

## КРИТЕРИИ ПРИЁМКИ
- [ ] (А) → ELT (мощный DWH трансформирует на месте, сырьё сохраняется для переобработки/гибкости).
- [ ] (Б) → ETL (схема фиксирована, целевое хранилище слабое → трансформируем до загрузки).
- [ ] (В) → ETL (чувствительные данные обрабатываются/маскируются до загрузки, не хранятся сырыми).
- [ ] Указано, где трансформация: ELT — внутри хранилища; ETL — на промежуточном движке.
- [ ] Преимущество увязано с кейсом (гибкость/совместимость/безопасность).

## ПОДСКАЗКА
ELT — для мощных современных хранилищ: залей сырьё, трансформируй на месте, сохрани исходник. ETL — когда схема строгая, хранилище слабое или сырьё нельзя хранить (чувствительные данные). Где трансформируем — главное отличие.
