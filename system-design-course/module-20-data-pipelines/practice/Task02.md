# Задача 02 — Модуль 20: Конвейеры данных

## ЗАДАНИЕ
Выберите между batch и streaming под требования к свежести и обоснуйте.

## ВХОДНЫЕ ДАННЫЕ
- (А) ежемесячный финансовый отчёт по историческим данным;
- (Б) детекция мошенничества: решение нужно в момент транзакции;
- (В) дашборд «активные пользователи сейчас» с обновлением раз в несколько секунд;
- (Г) ночное наполнение витрин для BI, свежесть «вчерашний день» приемлема.

## ТРЕБОВАНИЯ
- Для каждого кейса выберите batch или streaming и обоснуйте через требуемую свежесть/latency.
- Назовите цену streaming (сложность) и когда она не оправдана.
- Для (Б)/(В) укажите, что именно требует streaming.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: кейс → batch/streaming → обоснование (свежесть/latency) + про цену streaming.

## КРИТЕРИИ ПРИЁМКИ
- [ ] (А), (Г) → batch (историческая/суточная свежесть достаточна, проще и дешевле).
- [ ] (Б), (В) → streaming (real-time: фрод в момент транзакции, живые метрики).
- [ ] Названа цена streaming: сложность (порядок, окна, состояние, эксплуатация).
- [ ] Подчёркнуто: не делать streaming там, где хватает batch (лишняя сложность).
- [ ] Выбор строго увязан с требуемой свежестью.

## ПОДСКАЗКА
Свежесть решает: исторические/суточные отчёты → batch (просто, дёшево); фрод/живые метрики → streaming (low latency). Streaming сложнее (окна, порядок, состояние) — не вводить ради «модно», только под реальную потребность в свежести.
