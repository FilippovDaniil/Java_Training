# Задача 05 — Модуль 20: Конвейеры данных

## ЗАДАНИЕ
Сделайте конвейер идемпотентным и спроектируйте backfill.

## ВХОДНЫЕ ДАННЫЕ
Дневной конвейер агрегирует события в витрину `daily_sales`. Сейчас он делает `INSERT` агрегатов за день. Проблемы: при повторном запуске (после сбоя/ретрая) данные дублируются; при исправлении логики невозможно пересчитать прошлые дни.

## ТРЕБОВАНИЯ
- Объясните, почему текущий `INSERT` не идемпотентен.
- Предложите идемпотентную загрузку (перезапись партиции/upsert по дате).
- Спроектируйте backfill: как пересчитать произвольный прошлый период.
- Объясните, почему backfill требует наличия сырья (связь с bronze, модуль 19).

## ФОРМАТ РЕЗУЛЬТАТА
Диагноз → идемпотентная загрузка → дизайн backfill → роль сырья.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Диагноз: голый `INSERT` при повторе дублирует агрегаты за день.
- [ ] Идемпотентность: перезапись партиции по дате (idempotent overwrite) или upsert — повторный запуск даёт тот же результат.
- [ ] Backfill: запустить конвейер за указанный прошлый период, перезаписав соответствующие партиции.
- [ ] Объяснено: backfill возможен только если сохранено сырьё (bronze-слой, модуль 19), из которого пересчитывают.
- [ ] Атомарность загрузки партиции (целиком или никак).

## ПОДСКАЗКА
Идемпотентность конвейера = та же, что у consumer'а очереди (модуль 15): retry неизбежен, повтор не должен дублировать. Решение — перезапись партиции по дате вместо накопительного insert. Backfill (пересчёт истории) работает только при сохранённом сырье (bronze).
