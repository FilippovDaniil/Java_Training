# Задача 01 — Модуль 21: CAP/PACELC

## ЗАДАНИЕ
Классифицируйте данные/системы как CP или AP и обоснуйте через цену несогласованности.

## ВХОДНЫЕ ДАННЫЕ
- (А) баланс банковского счёта и переводы;
- (Б) лента активности в соцсети;
- (В) остаток товара на складе при оформлении заказа;
- (Г) счётчик лайков под постом;
- (Д) распределённая конфигурация кластера (кто лидер, какие настройки).

## ТРЕБОВАНИЯ
- Для каждого случая выберите CP или AP и обоснуйте ценой несогласованности.
- Опишите, что произойдёт при partition в выбранном режиме (отказ vs устаревание).
- Сгруппируйте: что важнее — согласованность или доступность.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: данные → CP/AP → цена несогласованности → поведение при partition.

## КРИТЕРИИ ПРИЁМКИ
- [ ] (А) баланс → CP (двойное списание недопустимо; при partition лучше отказать).
- [ ] (В) остаток склада → CP (нельзя продать последний товар двоим).
- [ ] (Д) конфигурация/лидер → CP (рассогласованная конфигурация = split-brain).
- [ ] (Б) лента, (Г) лайки → AP (устаревание терпимо, важна доступность).
- [ ] Для каждого описано поведение при partition (CP — отказ/ошибка; AP — устаревший ответ).
- [ ] Выбор обоснован ценой несогласованности, а не «по привычке».

## ПОДСКАЗКА
Вопрос-фильтр: «что будет, если отдать устаревшее/несогласованное?». Опасно (деньги, остатки, конфиг) → CP (лучше отказать). Терпимо (лента, лайки) → AP (лучше доступность). Это тот же вопрос, что ACID vs BASE (модуль 11), но в момент сетевого сбоя.
