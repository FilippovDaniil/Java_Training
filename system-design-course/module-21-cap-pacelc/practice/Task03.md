# Задача 03 — Модуль 21: CAP/PACELC

## ЗАДАНИЕ
Для конкретной системы выберите поведение при сетевом разделении (CP или AP) и опишите, что происходит.

## ВХОДНЫЕ ДАННЫЕ
Система бронирования билетов: места ограничены, нельзя продать одно место дважды. Развёрнута в нескольких дата-центрах. Произошёл partition: дата-центры временно не видят друг друга, но каждый продолжает получать запросы на бронь.

## ТРЕБОВАНИЯ
- Выберите CP или AP для операции бронирования и обоснуйте.
- Опишите конкретно, что система делает с запросами на бронь во время partition в выбранном режиме.
- Объясните, что было бы при противоположном выборе (последствие).
- Уточните, можно ли часть данных этой системы оставить AP.

## ФОРМАТ РЕЗУЛЬТАТА
Выбор CP/AP + поведение при partition + последствие обратного выбора + про смешанные данные.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Для бронирования выбран CP (нельзя продать место дважды → лучше отказать).
- [ ] Поведение CP: при partition бронирование в «отрезанных» узлах отклоняется/блокируется до восстановления связи (или допускается только в кворумной части).
- [ ] Последствие AP здесь: два узла продали одно место → овербукинг (недопустимо).
- [ ] Отмечено: некритичные данные (просмотр расписания, описания) можно оставить AP (доступность важнее).
- [ ] Выбор обоснован ценой несогласованности именно для брони.

## ПОДСКАЗКА
Бронирование = деньги/ограниченный ресурс → овербукинг недопустим → CP (лучше «попробуйте позже», чем двойная продажа). Но не вся система обязана быть CP: чтение расписания может оставаться доступным (AP). Per-data-выбор.
