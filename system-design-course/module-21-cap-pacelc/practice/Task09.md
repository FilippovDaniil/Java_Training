# Задача 09 — Модуль 21: CAP/PACELC

## ЗАДАНИЕ
Выберите БД по CP/AP-профилю под требования подсистем.

## ВХОДНЫЕ ДАННЫЕ
- (А) хранилище финансовых транзакций: согласованность критична, готовы платить latency и отказом при partition;
- (Б) хранилище ленты/событий: нужна высокая доступность и запись при любых условиях, устаревание терпимо;
- (В) координация кластера (лидер, конфигурация, distributed locks): нужна строгая согласованность решений.

## ТРЕБОВАНИЯ
- Для каждой подсистемы определите нужный профиль (CP или AP) и подберите класс БД/хранилища.
- Обоснуйте через CAP/PACELC.
- Назовите неудачный выбор для каждой и почему.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: подсистема → CP/AP → класс БД (пример) → обоснование → анти-выбор.

## КРИТЕРИИ ПРИЁМКИ
- [ ] (А) → CP-хранилище (традиционный SQL / strongly-consistent), согласованность критична; анти-выбор — AP-БД (eventual для денег).
- [ ] (Б) → AP-хранилище (Cassandra/DynamoDB), доступность/запись важнее; анти-выбор — CP (отказ при partition недопустим для ленты).
- [ ] (В) → CP coordination (Zookeeper/etcd), строгие решения; анти-выбор — AP (рассогласованный лидер = split-brain, модуль 22).
- [ ] Выбор обоснован через CAP/PACELC и цену несогласованности.
- [ ] Названы анти-выборы с причинами.

## ПОДСКАЗКА
CP-нужды (деньги, координация/лидер) → strongly-consistent хранилища (SQL, etcd/Zookeeper). AP-нужды (лента, высокая доступность записи) → Cassandra/DynamoDB. Поставить AP под деньги или CP под ленту — классические анти-выборы.
