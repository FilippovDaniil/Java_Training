# Задача 02 — Модуль 22: Координация и консенсус

## ЗАДАНИЕ
Объясните кворум/majority: почему нечётное число узлов и сколько отказов переживает кластер.

## ВХОДНЫЕ ДАННЫЕ
Рассматриваются кластеры консенсуса размером N = 3, 4, 5. Нужно понять, сколько отказов каждый переживает и почему практики выбирают нечётные значения.

## ТРЕБОВАНИЯ
- Посчитайте размер большинства (кворума) для N = 3, 4, 5.
- Укажите, сколько одновременных отказов узлов переживает каждый кластер (продолжает принимать решения).
- Объясните, почему N=4 не лучше N=3 (та же отказоустойчивость, больше узлов + риск «ничьи»).
- Объясните, как кворум большинства предотвращает split-brain при partition.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: N → кворум → переживает отказов + объяснение нечётности + про split-brain.

## КРИТЕРИИ ПРИЁМКИ
- [ ] N=3 → кворум 2, переживает 1 отказ; N=4 → кворум 3, переживает 1 отказ; N=5 → кворум 3, переживает 2 отказа.
- [ ] Показано: N=4 переживает столько же отказов, что N=3, но дороже на узел.
- [ ] Объяснено: чётное N может дать «ничью» (2+2) при partition — ни у кого нет большинства.
- [ ] Объяснено: две непересекающиеся половины не могут обе иметь большинство → только одна сторона принимает решения → нет split-brain.
- [ ] Вывод: нечётное N оптимально по отказоустойчивости на узел.

## ПОДСКАЗКА
Кворум = ⌊N/2⌋+1. Переживает отказов = N − кворум. N=4 и N=3 переживают по 1 отказу, но N=4 дороже и допускает «ничью» 2+2. Нечётное N оптимально. Большинство исключает split-brain: две половины не могут обе иметь кворум.
