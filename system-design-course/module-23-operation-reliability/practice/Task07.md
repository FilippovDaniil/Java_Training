# Задача 07 — Модуль 23: Надёжность операций

## ЗАДАНИЕ
Скомбинируйте timeout, bulkhead и fallback для устойчивости сервиса с несколькими зависимостями.

## ВХОДНЫЕ ДАННЫЕ
Сервис главной страницы агрегирует данные из трёх зависимостей: каталог (критичный), рекомендации (некритичные), баннеры (некритичные). Проблемы: нет таймаутов (зависшая зависимость держит запрос вечно); медленные рекомендации съедают весь пул потоков, и страница не отдаётся вообще.

## ТРЕБОВАНИЯ
- Задайте timeout для вызовов зависимостей и объясните зачем.
- Примените bulkhead: изолируйте ресурсы (пулы) по зависимостям.
- Примените fallback/graceful degradation для некритичных зависимостей.
- Покажите итог: что происходит со страницей, если рекомендации зависли.

## ФОРМАТ РЕЗУЛЬТАТА
Timeout (+зачем) → bulkhead (изоляция пулов) → fallback для некритичных → итоговое поведение.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Timeout на каждый вызов зависимости: зависшая зависимость не держит запрос вечно.
- [ ] Bulkhead: отдельные пулы потоков/соединений на зависимость → медленные рекомендации не исчерпывают пул каталога.
- [ ] Fallback для некритичных: рекомендации/баннеры при сбое заменяются заглушкой/кэшем/пропускаются.
- [ ] Итог: при зависших рекомендациях страница всё равно отдаётся (с каталогом, без рекомендаций) — graceful degradation.
- [ ] Критичная зависимость (каталог) отделена от некритичных.

## ПОДСКАЗКА
Три инструмента вместе: timeout (не ждать вечно), bulkhead (изолировать пулы, чтобы одна зависимость не съела все ресурсы), fallback (некритичное деградирует, не роняя страницу). Итог — частичная функциональность вместо полного отказа.
