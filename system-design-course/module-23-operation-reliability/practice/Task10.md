# Задача 10 — Модуль 23 (мини-проект): Надёжная распределённая операция

## ЗАДАНИЕ
Спроектируйте надёжный многошаговый распределённый процесс с полным арсеналом надёжности. Синтез модуля и завершение Части 5.

## ВХОДНЫЕ ДАННЫЕ
Процесс на выбор (зафиксируйте один): (а) оформление заказа (платёж + резерв + доставка); (б) перевод средств между счетами в разных сервисах; (в) регистрация с подтверждением (создание аккаунта + письмо + начисление бонуса). Процесс распределён по нескольким сервисам с собственными БД.

## ТРЕБОВАНИЯ
Дизайн должен покрывать:
1. **Шаги процесса** — локальные транзакции и их границы.
2. **Транзакционность** — почему не 2PC; Saga (choreography/orchestration) + компенсации.
3. **Идемпотентность** — где и как (idempotency key/дедуп) при повторах.
4. **Dual-write** — Outbox для надёжной публикации событий.
5. **Resilience** — timeout, retry+backoff+jitter, circuit breaker, fallback для вызовов downstream.
6. **Согласованность** — какие промежуточные состояния возможны (eventual), как видны пользователю.
7. **Соразмерность** — что из стека действительно нужно этому процессу.

## ФОРМАТ РЕЗУЛЬТАТА
Структурированный дизайн-документ из 7 разделов + диаграмма процесса (Saga + компенсации).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Зафиксирован один процесс; шаги и границы локальных транзакций определены.
- [ ] Saga с компенсациями вместо 2PC; выбран и обоснован choreography/orchestration.
- [ ] Идемпотентность шагов (повторы безопасны).
- [ ] Outbox для надёжной публикации событий (dual-write решён).
- [ ] Resilience-паттерны на вызовах downstream (timeout/retry+jitter/circuit breaker/fallback).
- [ ] Описаны промежуточные eventual-состояния и их видимость пользователю.
- [ ] Стек соразмерен критичности процесса; решения связаны с модулями 21/22/15.

## ПОДСКАЗКА
Это синтез Части 5: надёжный процесс = Saga (вместо 2PC) + идемпотентность (повторы) + Outbox (события) + resilience (timeout/retry/circuit breaker). Saga даёт eventual consistency с компенсациями. Соразмерность: деньги — весь стек, тривиальное — минимум. Дальше — Часть 6: надёжность, производительность, масштаб.
