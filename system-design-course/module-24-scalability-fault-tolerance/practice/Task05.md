# Задача 05 — Модуль 24: Масштабируемость и отказоустойчивость

## ЗАДАНИЕ
Подберите уровень избыточности (N+1 / N+2 / 2N) под требование доступности и обоснуйте число серверов.

## ВХОДНЫЕ ДАННЫЕ
Под пиковую нагрузку нужно 8 серверов приложения (это N). Требование: система держит пик даже при (а) отказе одного сервера и (б) одновременном плановом обновлении одного узла. Отдельно рассмотрите критичный сценарий, где допускается одновременная потеря половины парка (отказ стойки/зоны).

## ТРЕБОВАНИЯ
- Определите минимальный уровень избыточности под требование (а)+(б) и число серверов.
- Определите уровень под критичный сценарий (потеря половины) и число серверов.
- Объясните, что переживает каждый уровень и какова цена.
- Отметьте требование независимости отказов (резерв не в той же точке отказа).

## ФОРМАТ РЕЗУЛЬТАТА
Расчёт по сценариям → таблица «уровень → серверов → переживает → цена» → вывод.

## КРИТЕРИИ ПРИЁМКИ
- [ ] (а)+(б): нужен запас на 2 одновременно недоступных узла (отказ + обновление) → N+2 = 10 серверов.
- [ ] Критичный сценарий (потеря половины) → 2N = 16 серверов.
- [ ] Для каждого уровня указано, сколько отказов переживает и линейная цена запаса.
- [ ] Отмечено: резерв должен быть в независимой точке отказа (иначе отказывает вместе с основным).
- [ ] N (=8) явно назван как «без запаса» (отказ любого → перегруз/деградация).

## ПОДСКАЗКА
Избыточность = запас сверх рабочей мощности под число одновременных отказов. Отказ + плановое обслуживание = 2 недоступных узла → N+2. Потеря половины (зона/стойка) → 2N. Запас бесполезен, если резерв в той же точке отказа — нужна независимость.
