# Задача 10 — Модуль 24 (мини-проект): Масштабируемая и отказоустойчивая архитектура

## ЗАДАНИЕ
Спроектируйте масштабируемую и отказоустойчивую архитектуру системы, доказав числами достижение целевой доступности. Синтез модуля.

## ВХОДНЫЕ ДАННЫЕ
Система на выбор (зафиксируйте одну): (а) e-commerce каталог + корзина + заказы; (б) API соцсети (лента + публикации); (в) платёжный шлюз. Дано: пик 20 000 RPS, целевая доступность 99.95%, рост ×5 за 2 года, ограниченный бюджет.

## ТРЕБОВАНИЯ
Дизайн должен покрывать:
1. **Масштабирование** — какие слои horizontal/vertical; что stateless и куда вынесено состояние (shared-nothing).
2. **Избыточность** — уровень (N+1/N+2/2N) по слоям + число узлов под пик с запасом.
3. **SPOF** — пройти по схеме, перечислить и устранить все единые точки отказа.
4. **Failover** — active-passive/active-active по компонентам (особенно БД).
5. **Доступность в числах** — расчёт по критическому пути (последовательная цепочка) и эффект резервирования; достигается ли 99.95%.
6. **Деградация** — что отключаем при частичном отказе (graceful degradation) и как ограничен blast radius.
7. **Рост ×5** — что меняется при пятикратном росте (шардирование/доп. ячейки).

## ФОРМАТ РЕЗУЛЬТАТА
Дизайн-документ из 7 разделов + диаграмма (компоненты, избыточность, точки failover) + расчёт доступности.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Зафиксирована одна система; сервисный слой stateless, состояние вынесено (shared-nothing).
- [ ] Уровень избыточности по слоям обоснован + число узлов под 20k RPS с запасом.
- [ ] Все SPOF перечислены и устранены (LB/БД/кэш/зона).
- [ ] Failover-схема по компонентам выбрана и обоснована (БД — active-passive).
- [ ] Доступность посчитана по критическому пути; показано, достигается ли 99.95% (и что добавить, если нет).
- [ ] Описана graceful degradation и ограничение blast radius.
- [ ] План на рост ×5 (шардирование/доп. ячейки/доп. реплики); решения связаны с модулями 8/12/13/23.

## ПОДСКАЗКА
Синтез модуля: stateless + scale-out (shared-nothing), redundancy N+1/2N под отказы, устранить все SPOF, failover (БД active-passive), и доказать доступность числами (последовательная цепочка перемножается → меньше звеньев + резерв 1−(1−a)ⁿ). Деградация и cells ограничивают blast radius. Дальше — Модуль 25: тот же подход на уровне регионов и DR.
