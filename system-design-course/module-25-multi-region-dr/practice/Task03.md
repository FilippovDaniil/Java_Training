# Задача 03 — Модуль 25: Multi-region и Disaster Recovery

## ЗАДАНИЕ
Сравните active-passive и active-active multi-region для системы и выберите топологию.

## ВХОДНЫЕ ДАННЫЕ
Система с сильной согласованностью записи (финансовые транзакции). Бизнес хочет: пережить отказ региона, по возможности — низкую latency для пользователей в двух регионах. Команда невелика, опыта с межрегиональными конфликтами записи нет.

## ТРЕБОВАНИЯ
- Опишите обе топологии применительно к этой системе (что с трафиком и данными).
- Назовите trade-off каждой (утилизация резерва, RTO, сложность данных/конфликты).
- Объясните, почему active-active запись здесь = межрегиональный multi-leader с конфликтами.
- Выберите топологию и обоснуйте с учётом согласованности и опыта команды.

## ФОРМАТ РЕЗУЛЬТАТА
Сравнение топологий → trade-off → выбор и обоснование.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Active-passive: трафик в один регион, второй — резерв; нет конфликтов записи, но резерв простаивает и есть RTO.
- [ ] Active-active: оба обслуживают, latency ниже и отказ почти без паузы, но запись в двух регионах → конфликты.
- [ ] Объяснено: active-active запись = multi-leader через WAN → конфликты (LWW/CRDT, модуль 12), что опасно для финансов и сложно команде.
- [ ] Выбор обоснован: для строгой согласованности + неопытной команды → active-passive (один primary, контролируемый failover).
- [ ] Отмечена WAN-latency как фактор для данных.

## ПОДСКАЗКА
Active-passive проще и без конфликтов (один пишущий регион), но резерв простаивает и есть RTO. Active-active — latency/отказ без паузы, но запись в двух регионах = multi-leader с конфликтами, рискованно для финансов. Для строгой согласованности и неопытной команды — active-passive.
