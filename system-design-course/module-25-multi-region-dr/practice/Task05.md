# Задача 05 — Модуль 25: Multi-region и Disaster Recovery

## ЗАДАНИЕ
Выберите DR-стратегию под заданные RPO/RTO и бюджет, обосновав место на спектре cost↔RPO/RTO.

## ВХОДНЫЕ ДАННЫЕ
Три системы с разными требованиями:
- S1: внутренний отчётный сервис, RTO 8 часов, RPO 24 часа, бюджет минимальный.
- S2: основной продукт SaaS, RTO 15 минут, RPO 1 минута, бюджет средний.
- S3: торговая площадка/платежи, RTO ≈0, RPO ≈0, бюджет высокий.

## ТРЕБОВАНИЯ
- Для каждой системы выберите DR-стратегию (backup&restore / pilot light / warm standby / hot или active-active).
- Обоснуйте соответствием RPO/RTO и стоимости.
- Объясните, что именно держится в резерве в каждой стратегии.
- Отметьте, почему нельзя дать S1 стратегию уровня S3 (и наоборот).

## ФОРМАТ РЕЗУЛЬТАТА
Таблица «система → стратегия → что в резерве → почему подходит».

## КРИТЕРИИ ПРИЁМКИ
- [ ] S1 → Backup & Restore (бэкапы в другом регионе; RTO часы, RPO до интервала бэкапа; дёшево).
- [ ] S2 → Warm Standby (уменьшенная живая копия, масштабируется при failover; RTO минуты, RPO секунды–минуты).
- [ ] S3 → Hot standby / multi-site active-active (полноразмерный резерв; RTO/RPO≈0; дорого).
- [ ] Для каждой названо, что в резерве (только бэкапы / ядро / уменьшенная копия / полная копия).
- [ ] Объяснён компромисс: нельзя обещать ≈0 на бэкапах и не нужно платить за hot там, где приемлемы часы.
- [ ] (Опц.) Pilot light упомянут как промежуточный вариант.

## ПОДСКАЗКА
Спектр: backup&restore (дёшево, RTO часы) → pilot light (ядро/реплика БД) → warm standby (уменьшенная живая копия, минуты) → hot/active-active (полный резерв, ≈0, дорого). Подбирайте под заданные RPO/RTO и бюджет: соразмерно цене простоя.
