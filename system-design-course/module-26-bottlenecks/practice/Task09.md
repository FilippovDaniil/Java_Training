# Задача 09 — Модуль 26: Анализ узких мест и оптимизация

## ЗАДАНИЕ
Примените закон Амдала, чтобы оценить предел ускорения от параллелизма и найти, что мешает масштабированию.

## ВХОДНЫЕ ДАННЫЕ
Пакетная обработка: 90% работы параллелится по узлам, но 10% — строго последовательная часть (общая блокировка на едином счётчике/координаторе, через который проходит каждая задача). Команда планирует нарастить кластер с 10 до 100 узлов, ожидая ускорение ×10.

## ТРЕБОВАНИЯ
- Оцените максимально достижимое ускорение по закону Амдала при доле последовательной части s=0.1.
- Объясните, почему рост числа узлов с какого-то момента почти не даёт прироста.
- Определите, что является настоящим ограничителем масштабирования здесь.
- Предложите, как поднять потолок (уменьшить последовательную долю).

## ФОРМАТ РЕЗУЛЬТАТА
Оценка предела (закон Амдала) → почему узлы не помогают → ограничитель → как поднять потолок.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Предел ускорения ≤ 1/s = 1/0.1 = 10× — независимо от числа узлов (даже при 100).
- [ ] Объяснено: при s=0.1 даже бесконечный параллелизм даёт максимум ×10; добавление узлов сверх — почти без эффекта (убывающая отдача).
- [ ] Ограничитель — последовательная часть (общая блокировка/единый координатор), а не число узлов.
- [ ] Предложение: устранить/сократить последовательную часть (шардировать счётчик, убрать глобальную блокировку, локальная агрегация) — это поднимает потолок.
- [ ] Вывод: масштабирование упирается в последовательную долю, её и надо атаковать.

## ПОДСКАЗКА
Закон Амдала: максимальное ускорение ≤ 1/s, где s — доля последовательной части. При s=0.1 потолок ×10 при любом числе узлов. Узкое место масштабирования — последовательная часть (общая блокировка/координатор). Поднять потолок = уменьшить s (шардировать счётчик, убрать глобальную блокировку).
