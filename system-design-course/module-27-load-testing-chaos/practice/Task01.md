# Задача 01 — Модуль 27: Нагрузочное тестирование и Chaos Engineering

## ЗАДАНИЕ
Подберите вид нагрузочного теста под каждую цель и обоснуйте.

## ВХОДНЫЕ ДАННЫЕ
Команда формулирует вопросы перед запуском нового сервиса:
- Q1: «Держим ли мы ожидаемые 5000 RPS с SLO p99 < 200 мс?»
- Q2: «При каком RPS система начнёт отказывать и как именно?»
- Q3: «Переживём ли внезапный наплыв в день распродажи (×8 за минуту)?»
- Q4: «Нет ли утечки памяти при работе сутки под нагрузкой?»
- Q5: «Сколько реально держим и какой запас под рост?»

## ТРЕБОВАНИЯ
- Для каждого вопроса выберите вид теста (load / stress / spike / soak / capacity).
- Кратко обоснуйте соответствие вида цели.
- Назовите ключевую метрику, на которую смотреть в каждом случае.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица «вопрос → вид теста → что меряем → обоснование».

## КРИТЕРИИ ПРИЁМКИ
- [ ] Q1 → load test (целевая нагрузка, проверка SLO p99).
- [ ] Q2 → stress test (за предел, точка отказа + характер деградации).
- [ ] Q3 → spike test (резкий всплеск ×8).
- [ ] Q4 → soak/endurance test (долго → утечки/деградация во времени).
- [ ] Q5 → capacity test (плавный рост до предела, запас/headroom).
- [ ] Для каждого названа адекватная метрика (p99/error rate/точка насыщения/память во времени/headroom).

## ПОДСКАЗКА
Каждый вопрос — свой тест: штатная нагрузка → load; за предел/точка отказа → stress; внезапный всплеск → spike; деградация/утечки во времени → soak; сколько держим и запас → capacity. Гонять только load — слепые зоны (нет точки отказа и поведения во времени).
