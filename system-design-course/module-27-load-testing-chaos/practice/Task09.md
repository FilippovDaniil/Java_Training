# Задача 09 — Модуль 27: Нагрузочное тестирование и Chaos Engineering

## ЗАДАНИЕ
Спланируйте game day — учение по отказу зоны/региона для проверки DR-готовности.

## ВХОДНЫЕ ДАННЫЕ
Система multi-region active-passive (модуль 25): основной регион A, резервный B (warm standby, async-реплика). Заявлено: RTO 15 мин, RPO 1 мин, failover частично автоматизирован, есть runbook. Failover ни разу не проверяли на практике. Бизнес хочет убедиться, что в реальную катастрофу команда и система справятся.

## ТРЕБОВАНИЯ
- Сформулируйте цель game day и гипотезу (steady state + ожидаемые RTO/RPO).
- Опишите сценарий: что именно «отказывает» и как это вносится.
- Перечислите, что проверяете во время учения (механика failover, время, потеря данных, действия команды).
- Укажите меры безопасности (объявить заранее, kill switch/откат, наблюдаемость).
- Опишите, что делаете с найденными расхождениями (runbook/RTO/RPO).

## ФОРМАТ РЕЗУЛЬТАТА
Цель+гипотеза → сценарий → что проверяем → меры безопасности → работа с результатами.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Цель: проверить DR на практике; гипотеза с измеримыми RTO≤15мин/RPO≤1мин и steady state (SLI).
- [ ] Сценарий: контролируемо имитировать отказ региона A (или его БД/сети), запустить процедуру failover на B.
- [ ] Проверяется: срабатывает ли failover, фактические RTO/RTO vs заявленные, потеря данных (RPO/lag), DNS TTL, действия команды по runbook.
- [ ] Меры безопасности: учение объявлено, есть план отката/failback, kill switch, включена наблюдаемость (модуль 28).
- [ ] Найденные расхождения → обновить runbook/конфиги/TTL и скорректировать заявленные RTO/RPO; повторить учение.
- [ ] Связь с модулем 25 (DR/RPO/RTO/failback) и принципом «непротестированный DR = иллюзия».

## ПОДСКАЗКА
Game day = плановое DR-учение по принципам chaos: гипотеза (RTO/RPO + steady state) → контролируемый отказ региона → запуск failover → замер фактических RTO/RPO и проверка действий команды → безопасность (объявить, откат, kill switch, наблюдаемость) → обновить runbook по находкам. Непротестированный failover в реальную катастрофу обычно не срабатывает.
