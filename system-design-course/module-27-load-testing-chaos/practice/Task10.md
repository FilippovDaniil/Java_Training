# Задача 10 — Модуль 27 (мини-проект): План нагрузочного тестирования + Chaos для системы

## ЗАДАНИЕ
Составьте комплексный план проверки системы под нагрузкой и в отказах. Синтез модуля.

## ВХОДНЫЕ ДАННЫЕ
Система на выбор (зафиксируйте одну): (а) API соцсети (лента/публикации); (б) платёжный сервис; (в) сервис бронирования с пиками. Дано: целевая нагрузка и SLO задайте сами реалистично; система отказоустойчива (избыточность + failover + resilience-паттерны из модулей 23–25), готовится к запуску.

## ТРЕБОВАНИЯ
Дизайн должен покрывать:
1. **Цели проверки** — что именно валидируем (capacity-расчёты, SLO, отказоустойчивость).
2. **Нагрузочное тестирование** — какие виды (load/stress/spike/soak/capacity) и зачем каждый.
3. **Профиль и среда** — реалистичный микс эндпоинтов, среда как прод, open-модель для перегруза, прогрев.
4. **Метрики и критерии** — throughput/p99/error rate/утилизация, точка насыщения, критерии успеха vs SLO.
5. **Chaos-эксперименты** — минимум 2–3 по циклу (steady state → гипотеза → отказ → проверка), что проверяют из 23–25.
6. **Blast radius и безопасность** — контроль радиуса, kill switch, наблюдаемость, прод vs staging.
7. **Game day** — учение (отказ зоны/DR) и работа с результатами.

## ФОРМАТ РЕЗУЛЬТАТА
План из 7 разделов + таблица «тест/эксперимент → цель → метрики/гипотеза → критерий».

## КРИТЕРИИ ПРИЁМКИ
- [ ] Зафиксирована одна система, заданы реалистичные целевая нагрузка и SLO.
- [ ] Виды нагрузочных тестов выбраны и обоснованы (минимум load + stress + soak).
- [ ] Профиль реалистичен (микс эндпоинтов), среда как прод, open-модель для перегруза, учтён прогрев.
- [ ] Метрики и критерии успеха привязаны к SLO; заложен поиск точки насыщения.
- [ ] ≥2 chaos-эксперимента по полному циклу (steady state + гипотеза + отказ + проверка), проверяющие failover/resilience из 23–25.
- [ ] Контроль blast radius (радиус + kill switch + наблюдаемость), различение staging/прод.
- [ ] Game day (DR-учение) с измеримыми RTO/RPO и работой над находками; связи с модулями 23–26, 28.

## ПОДСКАЗКА
Синтез модуля: цели (capacity/SLO/отказоустойчивость) → нагрузочные тесты (load+stress+soak, реалистичный профиль, open-модель, прогрев, найти knee) → chaos (steady state→гипотеза→отказ→проверка, blast radius+kill switch) → game day (DR). Всё опирается на наблюдаемость — это мост к модулю 28, которым завершается Часть 6.
