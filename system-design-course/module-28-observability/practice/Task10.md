# Задача 10 — Модуль 28 (мини-проект): Observability + SLO-стратегия системы

## ЗАДАНИЕ
Спроектируйте стратегию observability и SLO для системы. Синтез модуля и завершение Части 6.

## ВХОДНЫЕ ДАННЫЕ
Система на выбор (зафиксируйте одну): (а) платёжный сервис; (б) лента соцсети; (в) сервис бронирования. Дано: микросервисная архитектура, готовится к продакшену; нужно сделать систему наблюдаемой и управлять надёжностью измеримо.

## ТРЕБОВАНИЯ
Дизайн должен покрывать:
1. **Три столпа** — что собираете в metrics / logs / traces и как связываете (correlation/trace ID).
2. **RED/USE** — метрики для ключевых сервисов (RED) и ресурсов (USE).
3. **SLI** — 2–4 ключевых показателя на пользовательском пути (важное пользователю, не внутреннее).
4. **SLO** — целевые значения SLI за период; обоснование уровня (не 100%).
5. **SLA** — что обещаете наружу и почему он слабее SLO (буфер).
6. **Error budget** — расчёт из SLO + политика (фичи/заморозка), связь надёжность↔скорость.
7. **Алертинг** — на симптомы/SLO-burn (burn rate), против alert fatigue; что остаётся для диагностики.

## ФОРМАТ РЕЗУЛЬТАТА
Дизайн-документ из 7 разделов + таблица SLI/SLO/error budget + список алертов (симптом → порог → срочность).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Зафиксирована одна система; описаны три столпа и их связка через correlation/trace ID.
- [ ] RED для ключевых сервисов и USE для ресурсов заданы конкретно.
- [ ] 2–4 SLI на пользовательском пути (про пользователя, не CPU/очередь).
- [ ] SLO с конкретными значениями и периодом; уровень обоснован (< 100%).
- [ ] SLA слабее SLO (буфер) с пояснением.
- [ ] Error budget посчитан из SLO + политика использования (есть/исчерпан); связь надёжность↔скорость.
- [ ] Алертинг на симптомы/SLO-burn с burn rate; причинные метрики — для диагностики; связи с модулями 26/27.

## ПОДСКАЗКА
Синтез модуля: три столпа (+correlation ID) → RED/USE → 2–4 SLI про пользователя → SLO (<100%) → SLA слабее SLO → error budget (1−SLO) + политика фич → алерты на симптомы/SLO-burn (burn rate, против fatigue). Этим завершается Часть 6 — наблюдаемость замыкает цикл «спроектировал → нагрузил/сломал → увидел → улучшил». Дальше — Часть 7: безопасность и эксплуатация.
