# Задача 06 — Модуль 30: Защита API и противодействие атакам

## ЗАДАНИЕ
Спроектируйте защиту публичного API от brute-force и DDoS, комбинируя rate limiting и периметр.

## ВХОДНЫЕ ДАННЫЕ
Публичный API с эндпоинтом логина `/api/auth/login` и платными эндпоинтами с квотами по тарифу. Угрозы: перебор паролей на логине, перебор объектов по id, объёмные DDoS-атаки, превышение квот тарифа.

## ТРЕБОВАНИЯ
- Спроектируйте rate limiting: ключ лимита, разные лимиты для логина и платных эндпоинтов (связь с модулем 16).
- Опишите защиту от объёмного DDoS на периметре (связь с модулем 10).
- Опишите ответ при превышении (HTTP-код, заголовки).
- Разграничьте уровни: что на edge/периметре, что на gateway/приложении.

## ФОРМАТ РЕЗУЛЬТАТА
Rate limiting (ключ/лимиты) → DDoS на периметре → ответ при превышении → уровни защиты.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Rate limiting: ключ per-IP/per-user/per-API-key; строгий лимит на /login (против brute-force) и квоты по тарифу на платных (модуль 16).
- [ ] DDoS: поглощение на периметре — CDN/anycast/scrubbing, WAF режет вредоносные паттерны (модуль 10).
- [ ] Ответ при превышении: HTTP 429 + Retry-After (клиент знает, когда повторить).
- [ ] Уровни: объёмный DDoS — на edge/CDN; точные лимиты/квоты — на gateway/приложении.
- [ ] Отмечена связь rate limiting и BOLA-сканирования (ограничивает перебор id).

## ПОДСКАЗКА
Rate limiting (модуль 16): строгий лимит на /login (brute-force), квоты по тарифу, ключ per-user/IP/API-key; ответ 429 + Retry-After. Объёмный DDoS — на периметре (CDN/anycast/scrubbing/WAF, модуль 10). Уровни: объём → edge, точные лимиты → gateway/app. Rate limiting также тормозит перебор id (BOLA).
