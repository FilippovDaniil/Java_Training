# Задача 10 — Модуль 30 (мини-проект): Модель угроз и защита API системы

## ЗАДАНИЕ
Постройте модель угроз и спроектируйте многослойную защиту API системы. Синтез модуля.

## ВХОДНЫЕ ДАННЫЕ
Система на выбор (зафиксируйте одну): (а) финтех/платежи; (б) медицинский сервис с PHI; (в) e-commerce с пользовательскими данными. Публичный API + мобильные/веб-клиенты + партнёрские интеграции; есть чувствительные данные.

## ТРЕБОВАНИЯ
Дизайн должен покрывать:
1. **Поверхность атаки и модель угроз** — что выставлено, кто атакующий, какие активы защищаем.
2. **Defense in depth** — слои обороны от транспорта до данных.
3. **Авторизация на объект** — как закрываете BOLA (связь с модулем 29).
4. **Ввод и инъекции** — валидация (allowlist, сервер), параметризация, mass assignment.
5. **Нагрузка и периметр** — rate limiting + DDoS-защита (модули 16/10), TLS/CORS/CSRF.
6. **Секреты и данные** — secrets management, шифрование at-rest, минимизация/маскирование PII.
7. **Misconfiguration и эксплуатация** — least privilege, прод-hardening, обнаружение атак (модуль 28).

## ФОРМАТ РЕЗУЛЬТАТА
Дизайн-документ из 7 разделов + таблица «угроза → мера защиты → слой».

## КРИТЕРИИ ПРИЁМКИ
- [ ] Зафиксирована одна система; описаны поверхность атаки, активы и тип атакующего.
- [ ] Defense in depth по слоям (TLS→WAF/DDoS→gateway→сервис→БД→данные).
- [ ] Объектная авторизация против BOLA (серверная проверка владения).
- [ ] Валидация ввода (allowlist/сервер), параметризация, защита от mass assignment/excessive data exposure.
- [ ] Rate limiting + DDoS-защита + корректные TLS/CORS/CSRF.
- [ ] Secrets в vault, шифрование at-rest, минимизация/маскирование PII.
- [ ] Least privilege + прод-hardening (нет дефолтных кред/verbose-ошибок) + обнаружение атак (метрики/логи, модуль 28); связи с 16/29/10/25.

## ПОДСКАЗКА
Синтез модуля: модель угроз (что/кто/активы) → defense in depth по слоям → объектная authZ (BOLA) → валидация+параметризация → rate limiting/DDoS/TLS/CORS/CSRF → secrets в vault + шифрование + минимизация PII → least privilege + hardening + обнаружение (observability). Безопасность — свойство каждого слоя. Дальше — Модуль 31: как это безопасно доезжает до прода (деплой/CI-CD).
