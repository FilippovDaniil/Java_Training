# Задача 02 — Модуль 31: Развёртывание

## ЗАДАНИЕ
Выберите стратегию развёртывания (rolling / blue-green / canary) под каждое требование и обоснуйте.

## ВХОДНЫЕ ДАННЫЕ
Ситуации выката:
- R1: обычный релиз без простоя, ресурсы ограничены (нельзя держать двойную ёмкость).
- R2: критичный релиз, нужна возможность мгновенно откатиться, ресурсы есть.
- R3: рискованное изменение алгоритма ранжирования — хотим проверить на малой доле пользователей по метрикам, прежде чем катить всем.

## ТРЕБОВАНИЯ
- Для каждой ситуации выберите стратегию и обоснуйте.
- Назовите trade-off выбранной (простой, риск, скорость отката, стоимость).
- Для R3 укажите, на что опираться при решении «расширять/откатывать» (связь с модулем 28).

## ФОРМАТ РЕЗУЛЬТАТА
Таблица «ситуация → стратегия → обоснование → trade-off».

## КРИТЕРИИ ПРИЁМКИ
- [ ] R1 → rolling (без простоя, без удвоения ресурсов; откат постепенный, сосуществуют версии).
- [ ] R2 → blue-green (мгновенное переключение и мгновенный откат; цена — две полные среды).
- [ ] R3 → canary (малая доля → расширение по метрикам; самый безопасный для рискованного).
- [ ] Для каждой названы trade-off (ресурсы/скорость отката/совместимость версий).
- [ ] R3: решение о расширении/откате — по метрикам/SLO/observability (модуль 28), не «на глаз».

## ПОДСКАЗКА
Rolling — без простоя и без удвоения, но откат постепенный (нужна совместимость версий). Blue-green — мгновенный откат, но две среды (дорого). Canary — безопасен для рискованного: малая доля → расширение по метрикам/SLO (модуль 28). Подбирайте под простой/риск/откат/стоимость.
