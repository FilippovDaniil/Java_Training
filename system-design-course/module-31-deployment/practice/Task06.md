# Задача 06 — Модуль 31: Развёртывание

## ЗАДАНИЕ
Спроектируйте CI/CD-конвейер для системы: стадии от коммита до прода, включая security-скан.

## ВХОДНЫЕ ДАННЫЕ
Команда деплоит вручную: собирают на ноутбуке, тестируют выборочно, заливают на прод по SSH. Хочется автоматический конвейер с ранним обнаружением поломок, проверкой безопасности и надёжной доставкой по средам.

## ТРЕБОВАНИЯ
- Опишите стадии конвейера (build → test → security scan → package → deploy).
- Разграничьте CI и CD (что к чему относится).
- Встройте security-скан (что именно проверяется, связь с модулем 30).
- Опишите продвижение по средам (staging → prod) и роль автоматических тестов как «ворот».

## ФОРМАТ РЕЗУЛЬТАТА
Схема конвейера (стадии) → CI vs CD → security-скан → продвижение по средам.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Стадии: build → test (unit/integration) → security scan → package (образ) → deploy (staging → prod).
- [ ] CI: сборка + тесты при каждом коммите (раннее обнаружение интеграционных поломок). CD: доставка/выкат проверенного артефакта.
- [ ] Security-скан в конвейере: проверка зависимостей на уязвимости, скан образа, статический анализ (модуль 30) — до прода.
- [ ] Продвижение: один и тот же артефакт проходит staging → prod; провал тестов/скана = «ворота» блокируют выкат.
- [ ] Отмечена выгода: меньше ручных ошибок, повторяемость, частые мелкие безопасные релизы (связь с error budget).

## ПОДСКАЗКА
Конвейер: build → test → security scan → package(образ) → deploy(staging→prod). CI = сборка+тесты на коммит (ранние поломки); CD = доставка проверенного артефакта. Security-скан (зависимости/образ/SAST, модуль 30) — встроить в пайплайн. Тесты/скан — «ворота»: провал блокирует выкат. Один артефакт через среды.
