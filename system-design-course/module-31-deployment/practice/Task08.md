# Задача 08 — Модуль 31: Развёртывание

## ЗАДАНИЕ
Найдите ошибки в дизайне развёртывания и исправьте их.

## ВХОДНЫЕ ДАННЫЕ
Цитаты из дизайн-документа:
1. «Health checks не настраиваем — контейнер же запустился, значит готов».
2. «Секреты (пароль БД, ключи) зашиваем прямо в Docker-образ — удобно».
3. «Откат не продумывали — если что, разберёмся на месте».
4. «Конфиги для prod/staging зашиты в разные образы под каждую среду».
5. «На проде иногда заходим на сервер и правим конфиг/пакет руками».
6. «Деплой фичи сразу включает её для всех — флаги это лишнее».

## ТРЕБОВАНИЯ
- Для каждого пункта укажите ошибку и корректное решение.
- Свяжите с понятиями модуля (health checks/zero-downtime, секреты в деплое, rollback, 12-factor конфиг, immutable infra, feature flags).

## ФОРМАТ РЕЗУЛЬТАТА
Таблица «пункт → ошибка → исправление».

## КРИТЕРИИ ПРИЁМКИ
- [ ] (1) Нет readiness/liveness → трафик в неготовый инстанс/зависшие не рестартуют → настроить probes (zero-downtime).
- [ ] (2) Секреты в образе → утечка образа = утечка секретов → инъекция в рантайме (env/secret manager, модуль 30).
- [ ] (3) Нет rollback → долгий простой при сбое → готовить откат (предыдущий образ/blue-green/гашение canary).
- [ ] (4) Конфиг в образе под каждую среду → теряется переносимость → один образ, конфиг из env (12-factor).
- [ ] (5) Ручной патчинг прода → конфигурационный дрейф/«снежинки» → immutable infra + IaC (пересоздавать, не патчить).
- [ ] (6) Деплой = включение фичи → связывает риски → feature flags (разделить выкат и активацию).

## ПОДСКАЗКА
Классические ошибки деплоя: нет health checks (нет zero-downtime), секреты в образе, нет rollback, конфиг в образе под среду (а не env/12-factor), ручной патчинг прода (дрейф → immutable+IaC), деплой=активация фичи (нужны flags). Каждая — прямое нарушение принципов модуля.
