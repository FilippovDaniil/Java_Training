# Задача 10 — Модуль 31 (мини-проект): Топология развёртывания + CI/CD системы

## ЗАДАНИЕ
Спроектируйте топологию развёртывания, CI/CD-конвейер и стратегию выката системы. Синтез модуля и завершение Части 7.

## ВХОДНЫЕ ДАННЫЕ
Система на выбор (зафиксируйте одну): (а) микросервисный e-commerce; (б) финтех-платформа; (в) SaaS-продукт. Дано: высокая требуемая доступность, частые релизы, чувствительные данные, переменная нагрузка; возможно multi-region (модуль 25).

## ТРЕБОВАНИЯ
Дизайн должен покрывать:
1. **Упаковка и оркестрация** — контейнеры, оркестратор, что даёт (self-healing/HPA/discovery).
2. **CI/CD-конвейер** — стадии build→test→security scan→deploy; CI vs CD; «ворота».
3. **Стратегия выката** — rolling/blue-green/canary под требования (обоснование); feature flags.
4. **Zero-downtime** — health checks (readiness/liveness), draining.
5. **IaC и immutable** — воспроизводимость, паритет сред, нет дрейфа.
6. **Конфиг и секреты** — 12-factor (конфиг из env), секреты в рантайме (не в образе, модуль 30).
7. **Rollback и наблюдаемость** — критерий «катить/откатить» (SLO/error budget, модуль 28), готовый откат; (опц.) выкат по регионам (модуль 25).

## ФОРМАТ РЕЗУЛЬТАТА
Дизайн-документ из 7 разделов + диаграмма CI/CD-конвейера + схема выката (стратегия + точки отката).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Зафиксирована одна система; контейнеры + оркестратор с названными гарантиями (self-healing/HPA/discovery).
- [ ] CI/CD-конвейер со стадиями и security-сканом; разграничены CI и CD; тесты/скан как «ворота».
- [ ] Стратегия выката выбрана и обоснована под требования; feature flags для разделения выката/активации.
- [ ] Zero-downtime: readiness/liveness + connection draining.
- [ ] IaC + immutable infra + паритет сред.
- [ ] 12-factor конфиг из env; секреты инъектируются в рантайме (не в образе/гите).
- [ ] Rollback готов; критерий «катить/откатить» на SLO/error budget (модуль 28); связи с 24/25/27/30.

## ПОДСКАЗКА
Синтез модуля и Части 7: контейнеры + оркестратор (self-healing/HPA) → CI/CD (build→test→scan→deploy, security в пайплайне) → стратегия выката под риск (rolling/blue-green/canary + feature flags) → zero-downtime (readiness/liveness+draining) → IaC/immutable + паритет → 12-factor конфиг + секреты в рантайме → готовый rollback с критерием на SLO/error budget. Дальше — Часть 8: 14 референсных архитектур, где все 7 частей складываются в целые системы.
