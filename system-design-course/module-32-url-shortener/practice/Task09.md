# Задача 09 — Модуль 32: URL-shortener → Pastebin

## ЗАДАНИЕ
Адаптируйте дизайн URL-shortener под Pastebin (хранение и раздача текстовых фрагментов) — что переиспользуется, что меняется.

## ВХОДНЫЕ ДАННЫЕ
Pastebin: пользователь сохраняет текстовый фрагмент (от пары строк до больших файлов) и получает короткую ссылку; по ссылке фрагмент открывается. Фрагменты могут быть большими (мегабайты), читаются многократно.

## ТРЕБОВАНИЯ
- Укажите, что переиспользуется из URL-shortener (генерация кода, кэш, KV для метаданных).
- Определите, где хранить сам текст (маленький vs большой) и почему.
- Спроектируйте раздачу больших/популярных фрагментов.
- Отметьте, как меняется estimation (объём данных) по сравнению с URL-shortener.

## ФОРМАТ РЕЗУЛЬТАТА
Переиспользование → хранение текста (БД vs object storage) → раздача → отличие в estimation.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Переиспользуется: генерация кода (KGS/base62), кэш горячих, KV-БД для метаданных (code → ссылка на контент).
- [ ] Хранение текста: маленькие — в БД; большие — в object/blob storage (модуль 10), метаданные в KV-БД.
- [ ] Раздача: большие/популярные фрагменты через CDN (модуль 10), разгрузка origin.
- [ ] Estimation: объём данных существенно выше URL-shortener (текст ≫ URL) → object storage оправдан.
- [ ] Отмечено: скелет тот же (code→ресурс), меняется природа хранимого (URL → блоб).

## ПОДСКАЗКА
Скелет тот же: code→ресурс, KGS, кэш, KV для метаданных. Разница — храним текст: маленький в БД, большой в object storage (модуль 10), метаданные (code→ссылка на блоб) в KV. Раздача больших/популярных — через CDN. Объём данных ≫ URL-shortener.
