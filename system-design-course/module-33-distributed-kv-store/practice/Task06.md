# Задача 06 — Модуль 33: Distributed KV-store

## ЗАДАНИЕ
Спроектируйте членство кластера и обнаружение отказов: gossip, hinted handoff, anti-entropy.

## ВХОДНЫЕ ДАННЫЕ
Кластер без мастера (нет центрального координатора). Требуется: узлы знают о составе кластера и сбоях; система остаётся always-writeable при временной недоступности реплики; реплики не «дрейфуют» со временем.

## ТРЕБОВАНИЯ
- Опишите gossip-протокол и почему он подходит для бесмастерного кластера.
- Опишите hinted handoff и как он сохраняет доступность записи.
- Опишите anti-entropy с Merkle-деревьями и зачем оно.
- Объясните, как эти три механизма дают самовосстановление без центрального управления.

## ФОРМАТ РЕЗУЛЬТАТА
Gossip → hinted handoff → anti-entropy (Merkle) → самовосстановление без мастера.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Gossip: узлы периодически обмениваются информацией о членстве/состоянии → децентрализованно, без SPOF-координатора (модуль 22).
- [ ] Hinted handoff: при недоступной целевой реплике запись принимает сосед и хранит «подсказку», досылает позже → запись остаётся доступной (always-writeable).
- [ ] Anti-entropy: фоновая сверка реплик через Merkle-деревья — быстро находит расходящиеся диапазоны для синхронизации (не сравнивая все данные).
- [ ] Вместе: обнаружение состава (gossip) + доступность записи при сбое (hinted handoff) + фоновое схождение (anti-entropy) = самовосстановление без мастера.
- [ ] Отмечено отсутствие центрального координатора (нет SPOF).

## ПОДСКАЗКА
Gossip — эпидемический обмен членством/состоянием (без мастера/SPOF, модуль 22). Hinted handoff — сосед принимает запись за недоступную реплику и досылает позже (always-writeable). Anti-entropy — фоновая сверка через Merkle-деревья (быстро находит расхождения). Вместе = самовосстановление без центра.
