# Задача 02 — Модуль 34: Newsfeed

## ЗАДАНИЕ
Выполните estimation для ленты и покажите, как числа выводят на проблему fan-out.

## ВХОДНЫЕ ДАННЫЕ
Допущения: 300M DAU; в среднем 2 поста/день на пользователя; ~10 загрузок ленты/день на пользователя; средний пользователь имеет ~200 подписчиков; celebrity — до 50M подписчиков.

## ТРЕБОВАНИЯ
- Посчитайте записи постов (постов/с) и чтения ленты (чтений/с).
- Оцените fan-out среднего поста (число доставок) и поста celebrity.
- Сделайте вывод: где именно «боль» масштаба.
- Объясните, почему read-heavy + fan-out определяют архитектуру.

## ФОРМАТ РЕЗУЛЬТАТА
Записи/с → чтения/с → fan-out среднего vs celebrity → вывод о «боли».

## КРИТЕРИИ ПРИЁМКИ
- [ ] Записи постов: 300M×2 / 86400 ≈ ~7000 постов/с.
- [ ] Чтения ленты: 300M×10 / 86400 ≈ ~35000 чтений/с (read-heavy подтверждён).
- [ ] Fan-out среднего поста: ~200 доставок; celebrity: до 50M доставок на один пост.
- [ ] Вывод: «боль» — в fan-out, особенно celebrity (один пост → десятки млн доставок).
- [ ] Объяснено: read-heavy → кэш/предвычисление; огромный fan-out celebrity → нельзя чистый push.

## ПОДСКАЗКА
День ≈ 86400 с. Записи: 300M×2/86400 ≈ 7000/с. Чтения: 300M×10/86400 ≈ 35000/с (read-heavy). Fan-out: средний ~200 доставок, celebrity ~50M на пост — вот «боль». Числа диктуют: кэш/предвычисление для чтений + гибрид для celebrity.
