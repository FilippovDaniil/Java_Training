# Задача 03 — Модуль 34: Newsfeed

## ЗАДАНИЕ
Сравните fan-out on write (push) и fan-out on read (pull) и определите trade-off каждого подхода.

## ВХОДНЫЕ ДАННЫЕ
Нужно решить, как генерируется лента. Два чистых подхода: push (раскладывать пост в ленты подписчиков при публикации) и pull (собирать ленту из постов подписок при чтении). Профиль — read-heavy.

## ТРЕБОВАНИЯ
- Опишите, когда и где выполняется работа в push и в pull.
- Сравните стоимость чтения ленты и записи поста для каждого.
- Назовите главную «боль» каждого подхода.
- Объясните, почему ни один чистый подход не масштабируется на всех пользователей.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица «push vs pull: где работа / чтение / запись / боль» + вывод о невозможности чистого подхода.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Push: работа при публикации (вписать в ленты всех подписчиков); чтение быстрое (предвычислено), запись дорогая.
- [ ] Pull: работа при чтении (собрать из подписок); запись дешёвая (одна запись), чтение дорогое.
- [ ] Боль push: celebrity (миллионы доставок) + трата на неактивных. Боль pull: активный пользователь с тысячами подписок (дорогая сборка).
- [ ] Вывод: read-heavy благоприятствует push (быстрое чтение), но celebrity ломает чистый push → нужен гибрид.
- [ ] Trade-off сформулирован как «дорогая запись/быстрое чтение» (push) vs «дешёвая запись/дорогое чтение» (pull).

## ПОДСКАЗКА
Push: работа при публикации → быстрое чтение, дорогая запись; боль — celebrity + неактивные. Pull: работа при чтении → дешёвая запись, дорогое чтение; боль — активный с тысячами подписок. Чистый подход не масштабируется на всех (celebrity ломает push) → гибрид.
