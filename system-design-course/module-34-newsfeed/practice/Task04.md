# Задача 04 — Модуль 34: Newsfeed

## ЗАДАНИЕ
Решите celebrity problem, спроектировав гибридную стратегию генерации ленты.

## ВХОДНЫЕ ДАННЫЕ
В системе есть обычные пользователи (~200 подписчиков) и знаменитости (десятки миллионов подписчиков). Чистый push на посте celebrity вызывает всплеск из миллионов записей и задержку доставки. Нужно сохранить быстрое чтение ленты без этого всплеска.

## ТРЕБОВАНИЯ
- Объясните суть celebrity problem (почему push не работает для знаменитостей).
- Спроектируйте гибрид: какая стратегия для обычных, какая для celebrity.
- Опишите, как собирается итоговая лента читателя (push-часть + pull-часть).
- Укажите критерий, по которому пользователь считается celebrity.

## ФОРМАТ РЕЗУЛЬТАТА
Суть celebrity problem → гибрид (обычные/celebrity) → сборка ленты → критерий celebrity.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Celebrity problem: один пост знаменитости при push = миллионы записей в ленты → всплеск нагрузки и задержка.
- [ ] Гибрид: обычные → push (раскладывать в ленты подписчиков); celebrity → pull (не рассылать, подтягивать при чтении).
- [ ] Итоговая лента = предвычисленная часть (push от обычных) + свежие посты celebrity (pull при чтении), слитые и отранжированные.
- [ ] Критерий celebrity: число подписчиков выше порога (или вручную помеченные аккаунты).
- [ ] Сформулирован принцип: push для «хвоста» распределения, pull для «головы» (celebrity).

## ПОДСКАЗКА
Celebrity problem: push на знаменитости = миллионы записей (всплеск). Гибрид: обычные → push (быстрое чтение), celebrity → pull (не рассылать, подтянуть при чтении). Лента = push-часть + подмешанные свежие посты celebrity. Критерий — порог числа подписчиков. Принцип: push для хвоста, pull для головы.
