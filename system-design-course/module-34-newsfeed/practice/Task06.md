# Задача 06 — Модуль 34: Newsfeed

## ЗАДАНИЕ
Спроектируйте high-level архитектуру ленты: потоки публикации (push) и чтения (push+pull).

## ВХОДНЫЕ ДАННЫЕ
Известно: гибрид push/pull, fan-out через асинхронную очередь, feed cache в Redis. Нужно собрать компоненты в связную схему с двумя потоками.

## ТРЕБОВАНИЯ
- Перечислите компоненты (Post Service, очередь, Fan-out Service/воркеры, Feed Service, feed cache, БД постов, граф).
- Опишите поток публикации (push) по шагам, включая async fan-out.
- Опишите поток чтения (push-часть из cache + pull-часть celebrity).
- Покажите, где обеспечивается быстрый ответ на POST.

## ФОРМАТ РЕЗУЛЬТАТА
Компоненты → поток публикации → поток чтения → быстрый POST + диаграмма.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Компоненты: Post Service, очередь (модуль 15), Fan-out Service (воркеры), Feed Service, feed cache (Redis), БД постов, граф подписок.
- [ ] Публикация: POST → Post Service сохраняет пост → кладёт событие в очередь → Fan-out воркеры дописывают post_id в feed cache подписчиков (обычные авторы; celebrity исключены из push).
- [ ] Чтение: GET feed → Feed Service читает feed cache (push-часть) → подмешивает свежие посты celebrity (pull) → ранжирует → отдаёт.
- [ ] Быстрый POST: публикация не ждёт раскладки — fan-out в фоне через очередь (eventual доставка).
- [ ] Диаграмма с обоими потоками.

## ПОДСКАЗКА
Компоненты: Post Service→очередь→Fan-out воркеры→feed cache; Feed Service для чтения; БД постов+граф. Публикация: сохранить пост→событие в очередь→воркеры в фоне раскладывают (celebrity исключены). Чтение: feed cache (push) + подмешать celebrity (pull)→ранжировать. POST быстрый, т.к. fan-out асинхронный.
