# Задача 09 — Модуль 34: Newsfeed

## ЗАДАНИЕ
Найдите узкие места дизайна ленты при масштабе и спроектируйте их устранение.

## ВХОДНЫЕ ДАННЫЕ
Система выросла: посты знаменитостей вызывают всплески; на пиках доставка постов отстаёт (лента обновляется с задержкой в минуты); вирусные посты с медиа создают горячие точки; feed cache раздувается; сборка ленты с pull-частью медленная на хвосте.

## ТРЕБОВАНИЯ
- Перечислите узкие места (celebrity fan-out, fan-out lag, hot content, размер кэша, tail latency сборки).
- Для каждого дайте решение со ссылкой на модуль.
- Объясните, почему все они — проявления одной проблемы fan-out.
- Укажите, что из этого приемлемо оставить как есть (eventual).

## ФОРМАТ РЕЗУЛЬТАТА
Таблица «узкое место → решение → модуль» + вывод о fan-out + что приемлемо.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Celebrity fan-out → гибрид (pull для celebrity, не рассылать миллионам).
- [ ] Fan-out lag → масштабировать воркеры, приоритет активным пользователям; eventual допустима (модули 15/21).
- [ ] Hot content (вирусный пост/медиа) → кэш контента + CDN для медиа (модули 9/10).
- [ ] Размер feed cache → хранить окно N постов + пагинация из БД + TTL (модуль 9).
- [ ] Tail latency сборки pull-части → ограничить fan-out источников, hedged requests (модуль 26).
- [ ] Вывод: все узкие места — продолжение fan-out-проблемы; fan-out lag приемлемо оставить (eventual).

## ПОДСКАЗКА
Celebrity → гибрид; fan-out lag → больше воркеров/приоритет активным (eventual ок); hot content → кэш+CDN (9/10); размер кэша → окно N+пагинация (9); tail latency сборки → ограничить источники/hedged requests (26). Всё — проявления fan-out; lag приемлемо терпеть.
