# Задача 06 — Модуль 35: Чат / мессенджер

## ЗАДАНИЕ
Спроектируйте гарантию порядка сообщений в диалоге и объясните, почему глобальный порядок не нужен.

## ВХОДНЫЕ ДАННЫЕ
В одном диалоге сообщения не должны переставляться (ответ не должен показаться раньше вопроса). Система распределена, сообщения идут через разные серверы. Глобальный тотальный порядок дорог.

## ТРЕБОВАНИЯ
- Объясните, почему глобальный порядок дорог и не нужен.
- Спроектируйте порядок per-conversation (как нумеруются сообщения).
- Покажите, как партиционирование по conversation_id поддерживает порядок.
- Свяжите с принципом «порядок только в партиции» (модуль 15).

## ФОРМАТ РЕЗУЛЬТАТА
Почему не глобальный → порядок per-conversation → связь с партиционированием → принцип модуля 15.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Глобальный порядок дорог (координация между всеми узлами) и не нужен — важен порядок внутри диалога.
- [ ] Per-conversation: каждому сообщению — sequence number / логическая метка в рамках conversation_id; получатель сортирует по ней.
- [ ] Партиционирование по conversation_id: все сообщения диалога идут через одну партицию → естественный порядок.
- [ ] Связь: порядок гарантируется только в партиции (модуль 15), диалог = партиция.
- [ ] Отмечено: это «достаточная согласованность» — порядок там, где он виден пользователю.

## ПОДСКАЗКА
Глобальный порядок дорог (координация всех узлов) и не нужен. Порядок per-conversation: seq number в рамках conversation_id, сортировка у получателя. Партиция по conversation_id → все сообщения диалога через одну партицию → порядок (модуль 15: порядок только в партиции).
