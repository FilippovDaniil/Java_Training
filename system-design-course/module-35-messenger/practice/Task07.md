# Задача 07 — Модуль 35: Чат / мессенджер

## ЗАДАНИЕ
Спроектируйте хранилище сообщений: выбор типа БД, ключ партиционирования, хранение медиа.

## ВХОДНЫЕ ДАННЫЕ
Поток ~100k сообщений/с (write-heavy), петабайтная история. Основной запрос — «история диалога» (последние сообщения по conversation_id, хронологически). Сообщения могут содержать медиа (картинки/видео).

## ТРЕБОВАНИЯ
- Определите профиль нагрузки и паттерн доступа.
- Выберите тип БД и обоснуйте (write-heavy + доступ по диалогу).
- Спроектируйте ключ партиционирования и сортировку.
- Опишите хранение медиа (где, что в сообщении).

## ФОРМАТ РЕЗУЛЬТАТА
Профиль/паттерн → выбор БД → partition key + сортировка → хранение медиа.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Профиль: write-heavy (каждое сообщение пишется), огромный объём; паттерн — range по conversation_id (история).
- [ ] Выбор: wide-column (Cassandra-style) — высокая пропускная записи, доступ по партиции (модули 11/13); реляционная не тянет объём/запись.
- [ ] Partition key = conversation_id, clustering/сортировка = timestamp/seq → все сообщения диалога в одной партиции, отсортированы.
- [ ] Медиа: в object storage + CDN (модуль 10), в сообщении — ссылка (а не сам блоб).
- [ ] Отмечено: партиция по conversation_id даёт и порядок, и быстрый range-запрос истории.

## ПОДСКАЗКА
Write-heavy + доступ по диалогу → wide-column (Cassandra-style, модули 11/13). Partition key = conversation_id, сортировка по времени/seq → диалог в одной партиции (порядок + быстрая история). Медиа — в object storage + CDN (модуль 10), в сообщении только ссылка.
