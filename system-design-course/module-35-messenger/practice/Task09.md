# Задача 09 — Модуль 35: Чат / мессенджер

## ЗАДАНИЕ
Спроектируйте групповые чаты и перечислите ключевые узкие места мессенджера с решениями.

## ВХОДНЫЕ ДАННЫЕ
Группы бывают маленькие (несколько человек) и большие (тысячи участников). Система держит миллионы соединений, presence-обновления и поток сообщений. Нужно масштабировать соединения и обработать большие/активные группы.

## ТРЕБОВАНИЯ
- Опишите доставку в группу (fan-out членам) и разницу маленьких vs больших групп.
- Перечислите узкие места: масштаб соединений, presence fan-out, hot groups, порядок/durability.
- Для каждого дайте решение со ссылкой на модуль.
- Свяжите большие группы с celebrity problem (модуль 34).

## ФОРМАТ РЕЗУЛЬТАТА
Групповой fan-out (маленькие vs большие) → таблица «узкое место → решение → модуль».

## КРИТЕРИИ ПРИЁМКИ
- [ ] Группа: сообщение → fan-out членам; маленькие — push каждому; большие — отдельная стратегия (как celebrity, модуль 34).
- [ ] Масштаб соединений → много connection-серверов + LB по соединениям (модули 8, 24).
- [ ] Presence fan-out → ленивое обновление статуса (не push всем).
- [ ] Hot groups (большие/активные) → особая обработка fan-out (как celebrity).
- [ ] Порядок/durability → per-conversation seq + надёжное wide-column хранилище (модули 13, 11).
- [ ] Большие группы явно сопоставлены с celebrity problem (один пост/сообщение → много доставок).

## ПОДСКАЗКА
Группа: fan-out членам — маленькие push каждому, большие как celebrity (модуль 34). Узкие места: масштаб соединений → connection servers + LB (8/24); presence fan-out → ленивое обновление; hot groups → стратегия celebrity; порядок/durability → per-conversation seq + wide-column (13/11).
