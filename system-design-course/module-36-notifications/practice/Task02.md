# Задача 02 — Модуль 36: Система уведомлений

## ЗАДАНИЕ
Выполните estimation для системы уведомлений и оцените всплеск маркетинговой кампании.

## ВХОДНЫЕ ДАННЫЕ
Допущения: 100M пользователей, в среднем 5 уведомлений/день на пользователя; маркетинговая кампания охватывает 50M пользователей за окно ~10 минут; уведомления распределены по каналам push/SMS/email.

## ТРЕБОВАНИЯ
- Посчитайте средний поток уведомлений (уведомлений/с).
- Оцените пиковый throughput кампании (за окно 10 минут).
- Объясните, почему всплеск кампании на 1–2 порядка выше среднего.
- Сделайте вывод о необходимости очереди и приоритизации.

## ФОРМАТ РЕЗУЛЬТАТА
Средний поток → пик кампании → разница порядков → вывод (очередь+приоритеты).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Средний поток: 100M×5 / 86400 ≈ ~6000 уведомлений/с.
- [ ] Кампания: 50M / 600 с ≈ ~83000/с — на порядок-полтора выше среднего.
- [ ] Объяснено: кампания концентрирует огромный объём в короткое окно → всплеск ≫ ровного потока.
- [ ] Вывод: очередь сглаживает всплеск (load leveling), приоритеты защищают срочные транзакционные от застревания за кампанией.
- [ ] Расчёты прозрачны; разнесение по каналам учтено (у каждого свои лимиты).

## ПОДСКАЗКА
Средний: 100M×5/86400 ≈ 6000/с. Кампания: 50M/600с ≈ 83000/с — на порядок+ выше. Всплеск концентрирован в окне → нужна очередь (сглаживание) + приоритеты (OTP не за кампанией). Разнести по каналам (разные лимиты провайдеров).
