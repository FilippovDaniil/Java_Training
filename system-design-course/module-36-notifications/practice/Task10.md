# Задача 10 — Модуль 36 (мини-проект): Полный HLD системы уведомлений

## ЗАДАНИЕ
Соберите полный high-level design системы уведомлений по фреймворку design-интервью. Синтез модуля.

## ВХОДНЫЕ ДАННЫЕ
Условие интервью: «Спроектируйте систему уведомлений: каналы push/SMS/email, транзакционные (OTP) и маркетинговые кампании на миллионы, надёжная доставка, пользовательские настройки». 45–60 минут.

## ТРЕБОВАНИЯ
Дизайн-документ по шагам:
1. **Требования** — FR/NFR, два режима нагрузки.
2. **Estimation** — средний поток + всплеск кампании.
3. **Каналы и провайдеры** — push/SMS/email, адаптеры, ненадёжные зависимости.
4. **Архитектура** — notification service → очередь → воркеры по каналам → провайдеры.
5. **Надёжность** — at-least-once+дедуп, retry+DLQ, circuit breaker, fallback.
6. **Политики** — rate limiting (провайдер+пользователь), настройки/opt-out/compliance, приоритеты.
7. **Трекинг + узкие места** — статусы, отказы провайдеров, fan-out.

## ФОРМАТ РЕЗУЛЬТАТА
HLD из 7 разделов + диаграмма (producer → service → priority queues → channel workers → provider adapters → providers; tracking via webhooks).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Требования: каналы, надёжность, дедуп; два режима (транзакционные/кампании).
- [ ] Estimation: средний поток ~6000/с, пик кампании ~десятки тысяч/с.
- [ ] Каналы и провайдеры с адаптерами; дизайн вокруг ненадёжных зависимостей.
- [ ] Архитектура: очередь + воркеры по каналам (развязка, масштаб).
- [ ] Надёжность: at-least-once+дедуп (idempotency key), retry+backoff+DLQ, circuit breaker+fallback.
- [ ] Политики: rate limiting (провайдер+пользователь), проверка opt-out/compliance, приоритетные очереди.
- [ ] Трекинг (статусы/webhooks) + узкие места (отказы провайдеров/лимиты/fan-out); связи с модулями 15/16/23/34.

## ПОДСКАЗКА
По фреймворку: требования→estimation→каналы/провайдеры→архитектура (очередь+воркеры по каналам)→надёжность (at-least-once+дедуп+retry+DLQ+circuit breaker+fallback)→политики (rate limiting+opt-out+приоритеты)→трекинг+узкие места. Стержень — надёжность вокруг ненадёжных провайдеров и fan-out кампаний. Дальше — Модуль 37: видеостриминг.
