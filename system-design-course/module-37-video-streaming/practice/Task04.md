# Задача 04 — Модуль 37: Видеостриминг

## ЗАДАНИЕ
Спроектируйте транскодинг-конвейер: зачем несколько вариантов, почему async, как через очередь.

## ВХОДНЫЕ ДАННЫЕ
Загруженное видео нужно подготовить к воспроизведению на разных устройствах и сетях. Транскодинг CPU-heavy (минуты на видео). Поток загрузок переменный, бывают пики.

## ТРЕБОВАНИЯ
- Объясните, зачем кодировать в несколько разрешений/битрейтов/кодеков.
- Спроектируйте конвейер: загрузка → object storage → очередь → транскодеры → варианты → CDN.
- Обоснуйте async-обработку через очередь (модуль 15).
- Укажите, что видео нарезается на чанки (связь с ABR).

## ФОРМАТ РЕЗУЛЬТАТА
Зачем варианты → конвейер (шаги) → почему async/очередь → чанки.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Несколько вариантов: разные устройства (телефон/TV → разрешения), сети (4G/Wi-Fi → битрейты), кодеки (совместимость) → нужны для ABR.
- [ ] Конвейер: загрузка (chunked/multipart) → сырое в object storage → событие в очередь → транскодеры-воркеры → варианты в object storage → распространение в CDN.
- [ ] Async через очередь (модуль 15): загрузка завершается быстро, транскодинг в фоне, воркеры масштабируются под пик.
- [ ] Видео нарезается на чанки (сегменты) — основа adaptive bitrate.
- [ ] Отмечено: транскодинг CPU-heavy → нельзя синхронно в обработчике загрузки.

## ПОДСКАЗКА
Варианты нужны под устройства/сети/кодеки (основа ABR). Конвейер: загрузка→сырое в object storage→очередь→транскодеры→варианты→CDN. Async через очередь (модуль 15): быстрая загрузка, фоновый CPU-heavy транскодинг, масштаб воркерами. Видео нарезается на чанки.
