# Задача 07 — Модуль 37: Видеостриминг

## ЗАДАНИЕ
Обоснуйте, почему CDN — ядро доставки видео, и спроектируйте стратегию push vs pull / pre-positioning.

## ВХОДНЫЕ ДАННЫЕ
Глобальная аудитория. Egress из облачного origin дорог и даёт высокую latency для дальних пользователей. Часть контента предсказуемо популярна (новые релизы), часть — «длинный хвост».

## ТРЕБОВАНИЯ
- Объясните двойную роль CDN: latency и поглощение egress.
- Опишите pull CDN (по требованию) и push CDN (pre-positioning).
- Выберите стратегию для предсказуемо популярного vs хвоста.
- Свяжите CDN-стратегию со снижением egress-cost (модуль 7).

## ФОРМАТ РЕЗУЛЬТАТА
Двойная роль CDN → pull vs push → стратегия по типу контента → cost.

## КРИТЕРИИ ПРИЁМКИ
- [ ] CDN: (1) latency — отдача чанков с близкого edge; (2) поглощение egress — origin отдаёт лишь при miss → экономия (модули 10, 7).
- [ ] Pull CDN: edge тянет с origin при первом запросе и кэширует (хорошо для непредсказуемого/хвоста). Push CDN: контент заранее разложен по edge (pre-positioning).
- [ ] Предсказуемо популярное (новые релизы) → push/pre-positioning (заранее на edge). Хвост → pull (по требованию).
- [ ] Связь: CDN снижает дорогой egress из origin (главная статья) + улучшает latency глобально.
- [ ] Отмечено, что CDN — ядро, а не опция.

## ПОДСКАЗКА
CDN — двойная роль: latency (edge близко) + поглощение egress (origin платит лишь при miss, модули 10/7). Pull — тянуть по требованию (хвост); push/pre-positioning — заранее на edge (предсказуемо популярное, новые релизы). CDN снижает главную статью (egress) — это ядро, не опция.
