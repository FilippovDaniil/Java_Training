# Задача 09 — Модуль 37: Видеостриминг

## ЗАДАНИЕ
Найдите узкие места видеостриминга и сравните стратегии Netflix vs YouTube.

## ВХОДНЫЕ ДАННЫЕ
Эксплуатация: egress-расходы огромны; хранилище растёт безостановочно; на пиках транскодеры не успевают; вирусное видео создаёт горячую точку; синхронный счётчик просмотров не тянет нагрузку. Отдельно — вопрос: чем стратегия Netflix отличается от YouTube.

## ТРЕБОВАНИЯ
- Перечислите узкие места (egress, storage, transcoding throughput, viral, подсчёт просмотров).
- Для каждого дайте решение со ссылкой на модуль.
- Сравните Netflix (курируемый каталог) и YouTube (UGC) по транскодингу и CDN.
- Объясните, почему предсказуемость контента меняет стратегию.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица «узкое место → решение → модуль» + сравнение Netflix vs YouTube.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Egress → CDN (поглощает egress) + tiering (модули 10, 7).
- [ ] Storage → object storage + tiering hot/cold/archive (модули 10, 7).
- [ ] Transcoding throughput → масштаб воркеров + очередь + приоритет (модуль 15).
- [ ] Viral video → агрессивное кэширование + pre-positioning в CDN (модуль 10).
- [ ] Подсчёт просмотров → async-агрегация (не синхронный инкремент, модуль 26).
- [ ] Netflix: пред-кодирование курируемого каталога + агрессивный pre-positioning (Open Connect). YouTube: транскод UGC на загрузке, оптимизация под объём/разнообразие.
- [ ] Объяснено: предсказуемый контент (Netflix) → можно заранее кодировать и раскладывать по edge; UGC (YouTube) — нет.

## ПОДСКАЗКА
Узкие места: egress→CDN+tiering (10/7); storage→object+tiering (10/7); transcoding→воркеры+очередь (15); viral→pre-positioning CDN (10); просмотры→async-агрегация (26). Netflix (каталог) — пред-кодирование + агрессивный pre-positioning (Open Connect); YouTube (UGC) — транскод на загрузке. Предсказуемость контента меняет стратегию.
