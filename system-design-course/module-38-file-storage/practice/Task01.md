# Задача 01 — Модуль 38: Облачное файловое хранилище

## ЗАДАНИЕ
Соберите требования (FR/NFR) к облачному файловому хранилищу и определите, какое NFR доминирует.

## ВХОДНЫЕ ДАННЫЕ
Постановка: «спроектируйте Dropbox / Google Drive». Нужно зафиксировать scope и понять главную ось и главное нефункциональное требование.

## ТРЕБОВАНИЯ
- Сформулируйте ядро FR (загрузка/скачивание, синхронизация между устройствами, шеринг, версионирование).
- Сформулируйте NFR (durability, доступность, масштаб storage, latency sync, согласованность метаданных).
- Назовите главную ось (storage) и главное NFR (durability) с обоснованием.
- Объясните, почему потеря файла недопустима и как это влияет на дизайн.

## ФОРМАТ РЕЗУЛЬТАТА
FR → NFR → главная ось → почему durability главное → scope.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Ядро FR: загрузка/скачивание файлов, синхронизация между устройствами, шеринг/права, версионирование, папки.
- [ ] NFR: durability (потеря недопустима), доступность, масштаб storage (петабайты), разумная latency sync, согласованность метаданных.
- [ ] Главная ось — storage; главное NFR — durability (обосновано: потеря файла = потеря доверия).
- [ ] Отмечено, что метаданные требуют согласованности (file tree не должен рассыпаться).
- [ ] Зафиксирован scope.

## ПОДСКАЗКА
FR: upload/download, sync, шеринг, версии, папки. NFR: durability (главное!), доступность, storage петабайты, latency sync, согласованность метаданных. Главная ось — storage; durability доминирует, т.к. потеря файла = катастрофа доверия.
