# Задача 02 — Модуль 38: Estimation хранилища

## ЗАДАНИЕ
Сделайте back-of-the-envelope оценку storage и нагрузки, покажите эффект дедупликации.

## ВХОДНЫЕ ДАННЫЕ
Допущения (примите явно): десятки млн пользователей, сотни файлов на юзера, средний размер файла ~МБ, версии, replication factor. Часть файлов идентична у многих юзеров.

## ТРЕБОВАНИЯ
- Оцените «наивный» объём storage (пользователи × файлы × размер × версии × replication).
- Покажите, как дедупликация и compression снижают фактический объём.
- Оцените профиль нагрузки: тяжёлые операции (передача содержимого) vs лёгкие частые (sync-проверки метаданных).
- Сделайте обратную проверку порядка величины.

## ФОРМАТ РЕЗУЛЬТАТА
Допущения → расчёт наивного storage → эффект дедупа/compression → профиль нагрузки → проверка.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Явные допущения (юзеры, файлы, размер, версии, replication).
- [ ] Наивный объём посчитан → петабайты.
- [ ] Показано: дедупликация (идентичные блоки/файлы у многих) + compression кратно снижают реальный объём.
- [ ] Разделены тяжёлые операции (content) и лёгкие частые (метаданные/sync).
- [ ] Обратная проверка порядка.

## ПОДСКАЗКА
Наивно: users × files × size × versions × replication → ПБ. Дедуп (одинаковые блоки у тысяч юзеров) + compression снижают в разы. Нагрузка: передача содержимого тяжёлая и редкая; sync-проверки метаданных лёгкие, но частые.
