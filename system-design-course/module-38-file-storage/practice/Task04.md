# Задача 04 — Модуль 38: Chunking файла

## ЗАДАНИЕ
Обоснуйте разбиение файла на блоки фиксированного размера и перечислите, что это даёт.

## ВХОДНЫЕ ДАННЫЕ
Пользователь правит один байт в файле размером 1 ГБ; другой пользователь грузит большой файл по нестабильной сети. Нужно показать, зачем chunking.

## ТРЕБОВАНИЯ
- Опишите chunking: файл → блоки фикс. размера, метаданные = упорядоченный список хэшей.
- Объясните 4 выгоды: дельта-синхронизация, дедупликация, параллельная загрузка, resumable upload.
- На примере правки 1 байта в 1 ГБ покажите выигрыш дельта-sync.
- На примере обрыва связи покажите resumable upload.

## ФОРМАТ РЕЗУЛЬТАТА
Что такое chunking → 4 выгоды → пример дельта-sync (1 байт в 1 ГБ) → пример resumable.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Chunking описан: блоки фикс. размера, файл = упорядоченный список хэшей.
- [ ] Названы все 4 выгоды: дельта-sync, дедуп, параллелизм, resumable.
- [ ] Правка 1 байта в 1 ГБ → передаётся только изменённый блок, не весь файл.
- [ ] Обрыв связи → продолжение с недозагруженного блока, не с нуля.

## ПОДСКАЗКА
Chunking: файл → блоки (напр. 4 МБ), метаданные = список хэшей. Выгоды: дельта-sync (правка байта → один блок по сети), дедуп, параллельная загрузка, resumable (продолжить с места обрыва).
