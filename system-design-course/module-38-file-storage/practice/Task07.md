# Задача 07 — Модуль 38: Конфликты одновременного редактирования

## ЗАДАНИЕ
Спроектируйте разрешение конфликтов, когда два устройства правят файл оффлайн одновременно.

## ВХОДНЫЕ ДАННЫЕ
Устройства A и B оффлайн правят версию v3 одного файла. Оба, выйдя в сеть, пытаются создать v4. Нужно не потерять данные ни одного из них.

## ТРЕБОВАНИЯ
- Опишите сценарий конфликта (расхождение от общей версии).
- Покажите, почему last-write-wins недопустим (тихая потеря чужой правки).
- Предложите решение: версионирование и/или conflict copy (отдать решение пользователю).
- Свяжите с версионированием (старые версии = старые наборы блоков, дёшево через дедуп).

## ФОРМАТ РЕЗУЛЬТАТА
Сценарий конфликта → почему LWW плох → conflict copy/версии → связь с дешёвым версионированием.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Сценарий: A и B расходятся от v3, оба создают v4.
- [ ] Показано: LWW молча теряет одну из правок → недопустимо (главное правило — не терять данные тихо).
- [ ] Решение: conflict copy («conflicted copy») и/или хранение обеих версий, выбор за пользователем.
- [ ] Отмечено: версии дёшевы (неизменённые блоки переиспользуются через дедуп).

## ПОДСКАЗКА
A и B расходятся от v3 → конфликт. LWW тихо теряет правку → нельзя. Решение: conflict copy или обе версии, пользователь решает. Версии дёшевы: общие блоки переиспользуются (дедуп).
