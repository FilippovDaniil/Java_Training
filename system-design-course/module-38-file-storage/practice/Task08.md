# Задача 08 — Модуль 38: Шеринг и права доступа

## ЗАДАНИЕ
Спроектируйте шеринг файлов/папок и модель прав доступа.

## ВХОДНЫЕ ДАННЫЕ
Пользователь хочет: дать коллеге доступ на чтение к папке, соавтору — на запись к файлу, и сгенерировать публичную ссылку для внешнего человека.

## ТРЕБОВАНИЯ
- Опишите, где живут права (метаданные: ACL — кто читает/пишет).
- Покажите, как общая папка ссылается на те же блоки (без копирования содержимого).
- Опишите публичную ссылку (токенизированный/presigned URL — модуль 10).
- Отметьте, как шеринг сочетается с дедупом и версиями.

## ФОРМАТ РЕЗУЛЬТАТА
Модель прав (ACL в метаданных) → общая папка (ссылки на блоки) → публичная ссылка (presigned) → связь с дедупом.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Права хранятся в метаданных (ACL: read/write на файл/папку).
- [ ] Общая папка/файл ссылается на те же блоки — содержимое не дублируется.
- [ ] Публичная ссылка — токенизированный/presigned URL (модуль 10), без раздачи прав на аккаунт.
- [ ] Отмечено: шеринг не плодит копии содержимого (дедуп), версии сохраняются.

## ПОДСКАЗКА
Права = ACL в метаданных (read/write). Шеринг = ссылки на те же блоки (не копия). Публичная ссылка = presigned URL (модуль 10). Дедуп: общий файл = одни блоки для всех.
