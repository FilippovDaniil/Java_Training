# Задача 05 — Модуль 39: Матчинг / dispatch

## ЗАДАНИЕ
Спроектируйте матчинг rider'а с водителем, гарантируя «один водитель ↔ одна поездка».

## ВХОДНЫЕ ДАННЫЕ
Запрос поездки: нужно найти кандидатов рядом, предложить водителю, подтвердить. Опасности: два rider'а матчатся на одного водителя; клиент повторяет запрос (ненадёжная сеть).

## ТРЕБОВАНИЯ
- Опишите шаги dispatch: nearby-запрос → выбор кандидата → предложение → подтверждение.
- Решите гонку «два rider'а → один водитель»: атомарный захват статуса (lock/compare-and-set).
- Решите повтор запроса (retry): идемпотентность через idempotency key (модуль 23).
- Обработайте неответ водителя (таймаут → следующий кандидат).

## ФОРМАТ РЕЗУЛЬТАТА
Шаги dispatch → атомарный захват (один водитель ↔ одна поездка) → идемпотентность → таймаут/следующий.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Шаги: nearby → выбор (ближайший/ETA/рейтинг) → предложение → подтверждение.
- [ ] Гонка решена атомарным захватом статуса водителя (lock/CAS) — выигрывает один.
- [ ] Повтор запроса не плодит поездки → idempotency key (модуль 23).
- [ ] Неответ водителя → таймаут и переход к следующему кандидату.

## ПОДСКАЗКА
Dispatch: nearby → выбор → предложение → подтверждение. Один водитель ↔ одна поездка: атомарный захват (lock/CAS). Retry → idempotency key (модуль 23). Неответ → таймаут → следующий.
