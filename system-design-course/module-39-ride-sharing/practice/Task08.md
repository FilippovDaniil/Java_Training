# Задача 08 — Модуль 39: ETA и surge pricing

## ЗАДАНИЕ
Спроектируйте расчёт ETA и динамическое ценообразование (surge pricing).

## ВХОДНЫЕ ДАННЫЕ
Нужно показывать rider'у время прибытия и итоговую цену. В часы пик в популярных зонах спрос превышает предложение, обычная цена не балансирует рынок.

## ТРЕБОВАНИЯ
- Опишите ETA: расчёт по маршруту/трафику (сервис маршрутизации).
- Опишите surge pricing: множитель цены при дисбалансе спроса/предложения по гео-зонам.
- Покажите, как surge считается по зонам (тем же geo-ячейкам, что и матчинг).
- Обоснуйте, почему surge не считают синхронно на каждый запрос (агрегация/периодический пересчёт).

## ФОРМАТ РЕЗУЛЬТАТА
ETA (маршрут) → surge (множитель при дисбалансе) → расчёт по зонам → почему не синхронно.

## КРИТЕРИИ ПРИЁМКИ
- [ ] ETA — расчёт по маршруту/трафику отдельным сервисом маршрутизации.
- [ ] Surge — множитель цены при спрос ≫ предложение, балансирует рынок и стимулирует водителей.
- [ ] Surge считается по гео-зонам (соотношение запросов и свободных водителей в ячейке).
- [ ] Обосновано: периодическая агрегация по зонам, не синхронный пересчёт на каждый запрос (модуль 26).

## ПОДСКАЗКА
ETA — сервис маршрутизации (граф дорог/трафик). Surge — множитель при спрос ≫ предложение, по гео-зонам. Считать агрегацией по зонам периодически, не синхронно на запрос.
