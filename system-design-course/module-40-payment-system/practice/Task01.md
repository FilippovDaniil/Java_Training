# Задача 01 — Модуль 40: Платёжная система

## ЗАДАНИЕ
Соберите требования (FR/NFR) к платёжной системе и определите главное NFR, отличающее её от прочих систем.

## ВХОДНЫЕ ДАННЫЕ
Постановка: «спроектируйте платёжную систему / payment service». Нужно зафиксировать scope и понять, что здесь доминирует не масштаб, а корректность.

## ТРЕБОВАНИЯ
- Сформулируйте ядро FR (приём платежа, перевод, интеграция с PSP/банками, refund, история/баланс).
- Сформулируйте NFR (корректность денег, strong consistency/CP, durability, аудит, PCI DSS, доступность).
- Назовите главное NFR (корректность → strong consistency, CP-выбор) и обоснуйте.
- Объясните контраст с AP-системами (лента): почему здесь не eventual consistency.

## ФОРМАТ РЕЗУЛЬТАТА
FR → NFR → главное NFR (корректность/CP) → контраст с AP → scope.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Ядро FR: приём платежа, перевод средств, интеграция с PSP/банками, возвраты, история транзакций/баланс.
- [ ] NFR: корректность (не терять/не дублировать деньги), strong consistency/CP, durability, аудит/трассируемость, PCI DSS, доступность.
- [ ] Главное NFR — корректность → strong consistency, CP-выбор (модуль 21); обосновано.
- [ ] Контраст с лентой (AP/eventual): для денег eventual недопустима.
- [ ] Зафиксирован scope.

## ПОДСКАЗКА
FR: платёж/перевод/PSP/refund/история. NFR: корректность (главное!), strong consistency/CP, durability, аудит, PCI DSS. Деньги ≠ лента: CP, не AP; не eventual. Цена ошибки несоизмерима с масштабом.
