# Задача 02 — Модуль 40: Estimation

## ЗАДАНИЕ
Оцените нагрузку платёжной системы и покажите, почему главное здесь не throughput, а корректность.

## ВХОДНЫЕ ДАННЫЕ
Допущения (примите явно): десятки/сотни млн транзакций в день, пики при распродажах ×N, средний размер записи транзакции, требование к согласованности.

## ТРЕБОВАНИЯ
- Посчитайте TPS (транзакций/с) среднее и пиковое.
- Оцените объём хранения транзакций/ledger (записи append-only растут).
- Покажите, что объём/throughput умеренны по меркам курса, но цена ошибки несоизмерима.
- Сформулируйте требование к согласованности (strong, баланс сходится всегда).

## ФОРМАТ РЕЗУЛЬТАТА
Допущения → TPS (avg/peak) → объём ledger → вывод: главное не масштаб, а корректность → требование согласованности.

## КРИТЕРИИ ПРИЁМКИ
- [ ] TPS среднее и пиковое посчитаны (с peak factor).
- [ ] Объём ledger/транзакций оценён (append-only, растёт).
- [ ] Показано: throughput умеренный (не миллионы QPS/не петабайты), но каждая транзакция критична.
- [ ] Требование: strong consistency (баланс должен сходиться в любой момент).

## ПОДСКАЗКА
TPS = транзакции/день / 86400, пик ×N. Объём ledger растёт (append-only). Вывод: масштаб умеренный, но цена ошибки огромна → главное корректность, strong consistency.
