# Задача 04 — Модуль 40: Double-entry ledger

## ЗАДАНИЕ
Спроектируйте двойную бухгалтерскую запись (double-entry ledger) как источник правды.

## ВХОДНЫЕ ДАННЫЕ
Нужно хранить движения средств так, чтобы баланс всегда сходился, история была аудируемой, а ошибки — обнаружимыми и исправимыми без потери следов.

## ТРЕБОВАНИЯ
- Опишите двойную запись: каждая транзакция = дебет одного счёта + кредит другого (сумма = 0).
- Обоснуйте append-only (immutable) журнал: записи не правят/не удаляют.
- Опишите исправление ошибки через компенсирующую запись (reversal), а не правку.
- Покажите, что баланс = свёртка записей счёта, а не отдельное мутируемое поле.

## ФОРМАТ РЕЗУЛЬТАТА
Двойная запись (дебет/кредит) → append-only → reversal для исправлений → баланс = свёртка.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Каждая транзакция = две записи (дебет/кредит), сумма изменений = 0 → баланс сходится.
- [ ] Журнал append-only/immutable: записи не редактируют и не удаляют (аудит).
- [ ] Исправление = компенсирующая проводка (reversal), не правка записи.
- [ ] Баланс = агрегат (свёртка) записей счёта, а не мутируемое поле.

## ПОДСКАЗКА
Транзакция = дебет + кредит (сумма 0) → баланс сходится. Журнал append-only (нельзя править — аудит). Ошибка → reversal-проводка. Баланс = сумма записей, не отдельное поле. Ledger — источник правды.
