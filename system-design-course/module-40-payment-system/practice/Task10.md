# Задача 10 — Модуль 40 (мини-проект): Полный HLD платёжной системы

## ЗАДАНИЕ
Соберите полный high-level design платёжной системы по фреймворку design-интервью. Синтез модуля и завершение под-батча.

## ВХОДНЫЕ ДАННЫЕ
Условие интервью: «Спроектируйте платёжную систему: приём платежей и переводы, интеграция с банками/PSP, возвраты, корректный баланс и история; деньги нельзя терять/дублировать». 45–60 минут.

## ТРЕБОВАНИЯ
Дизайн-документ по шагам:
1. **Требования** — FR/NFR, главное NFR (корректность → strong consistency/CP).
2. **Estimation** — TPS, объём ledger; вывод: главное корректность, не масштаб.
3. **Идемпотентность** — idempotency key против двойного списания (модуль 23).
4. **Double-entry ledger** — дебет/кредит, append-only, источник правды.
5. **Согласованность** — strong/ACID/SQL, не eventual (CP — модуль 21).
6. **Saga + Outbox** — распределённая транзакция без 2PC (модуль 23).
7. **Интеграция PSP + reconciliation + безопасность/узкие места** — async/статусы/retry/circuit breaker, сверка; PCI DSS/токенизация; hot accounts.

## ФОРМАТ РЕЗУЛЬТАТА
HLD из 7 разделов + диаграмма (client → payment service (idempotency) → ledger (ACID) → PSP-адаптеры (Saga/Outbox) → reconciliation; очередь) + статусная модель.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Требования: главное NFR — корректность (strong consistency/CP); scope зафиксирован.
- [ ] Estimation: TPS/объём ledger; вывод — главное корректность, не throughput.
- [ ] Идемпотентность: idempotency key против двойного списания, at-least-once+дедуп=exactly-once (модуль 23).
- [ ] Double-entry ledger: дебет/кредит, append-only, источник правды, баланс = свёртка.
- [ ] Strong consistency (ACID/SQL), не eventual; CP-выбор (модуль 21); контраст с лентой.
- [ ] Распределённая транзакция через Saga + компенсации + Outbox, не 2PC (модуль 23).
- [ ] Интеграция PSP (async/статусы/retry/circuit breaker) + reconciliation; PCI DSS (токенизация, не хранить карты — модуль 30); узкие места (hot accounts/провайдеры); связи 21/23/30/13/11/28.

## ПОДСКАЗКА
По фреймворку: требования (корректность/CP) → estimation (не масштаб, а корректность) → идемпотентность → double-entry ledger → strong consistency/ACID → Saga+Outbox (не 2PC) → интеграция PSP+reconciliation+PCI DSS+узкие места. Стержень: корректность денег (strong/CP); идемпотентность против double-charge; append-only ledger — источник правды; Saga, не 2PC. Этим завершается под-батч c Части 8.
