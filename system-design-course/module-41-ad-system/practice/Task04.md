# Задача 04 — Модуль 41: Event ingestion pipeline

## ЗАДАНИЕ
Спроектируйте конвейер приёма событий показов/кликов (write-heavy путь).

## ВХОДНЫЕ ДАННЫЕ
Сотни тысяч событий показов/кликов в секунду нужно надёжно принять, сгладить пики (вирусная кампания) и подготовить к агрегации.

## ТРЕБОВАНИЯ
- Опишите путь: event collector → очередь (Kafka, модуль 15) → стрим-процессоры.
- Обоснуйте роль очереди (развязка, буфер/сглаживание пиков, retention, replay).
- Объясните, почему события НЕ пишут напрямую в аналитическую БД.
- Опишите роль стрим-процессоров (агрегация по окнам — модуль 20).

## ФОРМАТ РЕЗУЛЬТАТА
Путь ingestion → роль очереди → почему не прямая запись → роль стрим-процессоров.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Путь: event collector (лёгкий приём) → очередь Kafka (модуль 15) → стрим-процессоры.
- [ ] Очередь: развязка приёма от обработки, буфер/сглаживание пиков, retention, replay.
- [ ] Объяснено: прямая запись в БД не выдержит всплеска → буфер-очередь.
- [ ] Стрим-процессоры агрегируют по временным окнам (модуль 20).

## ПОДСКАЗКА
Collector → очередь Kafka (модуль 15) → стрим-процессоры. Очередь: развязка, буфер пиков, retention, replay. Прямая запись в БД не выдержит всплеска. Стрим агрегирует по окнам (модуль 20).
