# Задача 06 — Модуль 41: Exactly-once / дедуп подсчёта

## ЗАДАНИЕ
Спроектируйте подсчёт кликов, гарантирующий учёт ровно один раз, и защиту от накрутки.

## ВХОДНЫЕ ДАННЫЕ
Очередь даёт at-least-once доставку: событие клика может прийти дважды (retry). Двойной подсчёт = неверный биллинг. Плюс есть боты/накрутка кликов.

## ТРЕБОВАНИЯ
- Опишите проблему: at-least-once → дубли событий → двойной подсчёт → неверный биллинг.
- Спроектируйте дедуп по event_id (идемпотентность — модуль 23): клик считается один раз.
- Объясните связь at-least-once + дедуп = exactly-once-эффект.
- Опишите click fraud detection: фильтрация ботов/спам-кликов до подсчёта.

## ФОРМАТ РЕЗУЛЬТАТА
Проблема дублей → дедуп по event_id → at-least-once+дедуп=exactly-once → click fraud.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Проблема: at-least-once (модуль 15) → дубли → двойной подсчёт → неверный биллинг.
- [ ] Дедуп по event_id (идемпотентность, модуль 23): повторное событие игнорируется.
- [ ] Объяснено: at-least-once доставка + дедуп = exactly-once-эффект подсчёта.
- [ ] Click fraud detection: фильтрация ботов/накрутки до учёта.

## ПОДСКАЗКА
At-least-once → дубли → двойной подсчёт → неверный биллинг. Дедуп по event_id (идемпотентность, модуль 23). at-least-once + дедуп = exactly-once-эффект. Click fraud: фильтровать боты до подсчёта.
