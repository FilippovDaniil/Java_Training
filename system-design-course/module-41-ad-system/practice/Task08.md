# Задача 08 — Модуль 41: Бюджет и pacing кампании

## ЗАДАНИЕ
Спроектируйте контроль бюджета кампании и равномерное распределение показов (pacing).

## ВХОДНЫЕ ДАННЫЕ
У кампании ограниченный бюджет. Нельзя показать рекламы больше оплаченного; нежелательно потратить весь бюджет за первый час.

## ТРЕБОВАНИЯ
- Опишите near-real-time учёт расхода бюджета и остановку показа при исчерпании.
- Объясните, почему точный учёт расхода — на стыке быстрого приближённого счёта и точной агрегации.
- Спроектируйте pacing: равномерное растягивание показов по периоду кампании.
- Свяжите с фильтром в ad serving (не отдавать объявления исчерпанных кампаний).

## ФОРМАТ РЕЗУЛЬТАТА
Учёт расхода (near-real-time) → стоп при исчерпании → pacing → связь с serving-фильтром.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Near-real-time учёт расхода бюджета; показ останавливается при исчерпании.
- [ ] Объяснено: точный учёт — на стыке быстрого приближённого счёта и точной агрегации (Lambda/Kappa).
- [ ] Pacing: равномерное распределение показов по периоду (не весь бюджет за час).
- [ ] Ad serving фильтрует объявления исчерпанных кампаний.

## ПОДСКАЗКА
Near-real-time учёт расхода → стоп при исчерпании. Точность — на стыке приближённого real-time и точного батча. Pacing — растянуть показы по периоду. Serving фильтрует исчерпанные кампании.
