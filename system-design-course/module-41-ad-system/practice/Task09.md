# Задача 09 — Модуль 41: Узкие места

## ЗАДАНИЕ
Найдите узкие места рекламной системы на масштабе и предложите решения со ссылками на модули.

## ВХОДНЫЕ ДАННЫЕ
Система обрабатывает миллиарды событий/день; есть очень популярные кампании (hot ads), требования к точности биллинга и к свежести метрик.

## ТРЕБОВАНИЯ
Разберите минимум 4 узких места:
- event write throughput (сотни тыс/с);
- hot ads/кампании;
- точность vs latency подсчёта;
- exactly-once подсчёт на масштабе.
Для каждого — решение и ссылка на модуль.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: узкое место → причина → решение → модуль.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Event write throughput → очередь + стрим-обработка, не синхронная запись (модули 15, 20).
- [ ] Hot ads/кампании → партиционирование событий + агрегация по окнам (модули 13, 26).
- [ ] Точность vs latency → Lambda/Kappa (приближённо real-time + точно батчем, модуль 20).
- [ ] Exactly-once подсчёт → дедуп по event_id на масштабе (модуль 23).
- [ ] Ad serving latency → кэш кандидатов/таргетинга (модуль 9).

## ПОДСКАЗКА
Throughput → очередь+стрим (15/20). Hot ads → партиция+окна (13/26). Точность vs latency → Lambda/Kappa (20). Exactly-once → дедуп event_id (23). Serving → кэш (9).
