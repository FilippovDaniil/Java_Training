# Задача 01 — Модуль 42: Веб-краулер / поисковый индекс

## ЗАДАНИЕ
Соберите требования (FR/NFR) к веб-краулеру и выделите главное противоречие системы.

## ВХОДНЫЕ ДАННЫЕ
Постановка: «спроектируйте веб-краулер для поисковика: обойти миллиарды страниц и построить индекс». Нужно зафиксировать scope и увидеть конфликт масштаба и вежливости.

## ТРЕБОВАНИЯ
- Сформулируйте ядро FR (обход, парсинг/извлечение ссылок+контента, построение индекса, дедуп, robots.txt, свежесть/recrawl).
- Сформулируйте NFR (масштаб млрд страниц, politeness, свежесть, устойчивость к плохому HTML, расширяемость).
- Назовите главное противоречие: масштаб (высокий общий throughput) против politeness (лимит на домен).
- Объясните, что краулер питает поисковый индекс (модуль 17).

## ФОРМАТ РЕЗУЛЬТАТА
FR → NFR → противоречие масштаб vs politeness → связь с индексом → scope.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Ядро FR: обход страниц, парсинг/извлечение ссылок и контента, построение индекса, дедупликация, robots.txt, свежесть/recrawl.
- [ ] NFR: масштаб (млрд страниц), politeness (не перегружать сайты), свежесть, устойчивость к плохому HTML, расширяемость.
- [ ] Названо противоречие: высокий общий throughput против per-domain politeness.
- [ ] Отмечено: краулер питает inverted index (модуль 17).
- [ ] Зафиксирован scope.

## ПОДСКАЗКА
FR: обход/парсинг/индекс/дедуп/robots.txt/recrawl. NFR: масштаб млрд, politeness, свежесть, устойчивость. Противоречие: общий throughput vs per-domain politeness. Краулер → inverted index (модуль 17).
