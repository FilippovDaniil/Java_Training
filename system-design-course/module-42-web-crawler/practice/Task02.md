# Задача 02 — Модуль 42: Estimation

## ЗАДАНИЕ
Оцените скорость обхода и объём хранения; покажите ограничение politeness.

## ВХОДНЫЕ ДАННЫЕ
Допущения (примите явно): целевой объём (млрд страниц), срок полного обхода, средний размер страницы, recrawl-периодичность.

## ТРЕБОВАНИЯ
- Посчитайте требуемую скорость обхода (страниц/с) = объём / срок.
- Оцените storage: сырые страницы (blob — модуль 10) + индекс + граф ссылок.
- Покажите, как per-domain politeness ограничивает скорость на один сайт.
- Сделайте вывод: throughput достигается параллелизмом по доменам, не ускорением одного.

## ФОРМАТ РЕЗУЛЬТАТА
Допущения → скорость обхода → storage → ограничение politeness → вывод о параллелизме.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Скорость обхода = объём / срок → тысячи-десятки тыс. страниц/с.
- [ ] Storage оценён: сырые страницы (blob), индекс, граф ссылок → петабайты.
- [ ] Показано: politeness ограничивает запросы на ОДИН домен (нельзя долбить сайт).
- [ ] Вывод: высокий общий throughput — за счёт параллелизма по многим доменам.

## ПОДСКАЗКА
Скорость = млрд страниц / срок ≈ тыс-десятки тыс/с. Storage = страницы (blob, модуль 10) + индекс + граф → ПБ. Politeness лимитирует per-domain → общий throughput через параллелизм по доменам.
