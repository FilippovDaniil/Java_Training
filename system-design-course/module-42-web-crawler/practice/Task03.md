# Задача 03 — Модуль 42: URL frontier

## ЗАДАНИЕ
Спроектируйте URL frontier, балансирующий приоритет обхода и politeness.

## ВХОДНЫЕ ДАННЫЕ
Нужно обходить важные/свежие страницы раньше, но при этом не отправлять запросы к одному домену чаще допустимого.

## ТРЕБОВАНИЯ
- Объясните две цели frontier: приоритизация и politeness.
- Опишите двухуровневую схему: front queues (по приоритету) → back queues (по хосту, с задержкой).
- Покажите, почему frontier — не простой FIFO.
- Свяжите приоритет с метриками (свежесть, ранг страницы).

## ФОРМАТ РЕЗУЛЬТАТА
Две цели → двухуровневая схема (front/back) → почему не FIFO → приоритетные метрики.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Две цели: приоритет (важное раньше) + politeness (лимит на домен).
- [ ] Двухуровневая схема: front queues по приоритету → back queues по хосту с crawl-delay.
- [ ] Объяснено: frontier не FIFO, а приоритетно-вежливая очередь.
- [ ] Приоритет учитывает свежесть/ранг страницы.

## ПОДСКАЗКА
Frontier = приоритет + politeness. Front queues (приоритет) → back queues (по хосту, задержка между запросами). Не FIFO. Приоритет: свежесть/ранг.
