# Задача 04 — Модуль 42: Crawl loop (BFS)

## ЗАДАНИЕ
Опишите основной цикл обхода как BFS графа веба с проверкой виденных URL.

## ВХОДНЫЕ ДАННЫЕ
Страницы ссылаются друг на друга (граф). Без контроля обход зациклится на взаимных ссылках и захлебнётся дублями.

## ТРЕБОВАНИЯ
- Опишите цикл: frontier.next → robots-проверка → fetch → parse → store → добавить новые ссылки.
- Покажите проверку seen(url) (URL-дедуп) перед добавлением во frontier.
- Объясните, почему без seen-проверки обход зацикливается.
- Свяжите с BFS-обходом графа (страница=узел, ссылка=ребро).

## ФОРМАТ РЕЗУЛЬТАТА
Псевдокод цикла → роль seen() → почему зацикливается без него → BFS-модель.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Цикл: взять URL (frontier) → robots.txt → fetch → parse → store → новые ссылки во frontier.
- [ ] Перед добавлением — проверка seen(url) (URL-дедуп, bloom filter).
- [ ] Объяснено: без seen обход зацикливается на взаимных ссылках, захлёбывается дублями.
- [ ] Связь с BFS: страница — узел, ссылка — ребро графа веба.

## ПОДСКАЗКА
loop: url=frontier.next → allowed(robots)? → fetch → parse(content,links) → store → for link: if !seen → frontier.add. Без seen — зацикливание. Это BFS графа веба.
