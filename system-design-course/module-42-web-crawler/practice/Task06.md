# Задача 06 — Модуль 42: Дедупликация (URL и контент)

## ЗАДАНИЕ
Спроектируйте дедупликацию двух видов: URL и контента.

## ВХОДНЫЕ ДАННЫЕ
Миллиарды URL нельзя хранить точным множеством (память); разные URL дают одинаковый/похожий контент (зеркала, копии), который не нужно индексировать дважды.

## ТРЕБОВАНИЯ
- Спроектируйте URL-дедуп через bloom filter (модуль 9/14): компактный, вероятностный.
- Объясните семантику bloom filter («точно нет» / «возможно да») и экономию памяти.
- Спроектируйте контент-дедуп через checksums + simhash/shingles (near-duplicate detection).
- Покажите разницу: точные дубли (checksum) vs near-duplicates (simhash).

## ФОРМАТ РЕЗУЛЬТАТА
URL-дедуп (bloom filter) → семантика/экономия → контент-дедуп (checksum/simhash) → точные vs near-dup.

## КРИТЕРИИ ПРИЁМКИ
- [ ] URL-дедуп: bloom filter (модуль 9/14), компактный для млрд URL.
- [ ] Семантика: «точно не видели» / «возможно видели»; экономия памяти vs точный set.
- [ ] Контент-дедуп: checksums (точные дубли) + simhash/shingles (near-duplicates).
- [ ] Показана разница: зеркала/копии ловятся simhash, не только точным совпадением.

## ПОДСКАЗКА
URL-дедуп: bloom filter (модуль 9/14) — компактно, «точно нет/возможно да». Контент-дедуп: checksum (точные) + simhash/shingles (near-duplicates: зеркала/копии).
