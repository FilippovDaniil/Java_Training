# Задача 07 — Модуль 42: Crawler traps и устойчивость

## ЗАДАНИЕ
Спроектируйте защиту от ловушек (crawler traps) и устойчивость к плохим данным.

## ВХОДНЫЕ ДАННЫЕ
Некоторые сайты создают бесконечные пространства URL (календарь «вперёд» до бесконечности, динамические параметры), а HTML бывает битым.

## ТРЕБОВАНИЯ
- Опишите проблему spider trap (бесконечные пространства поглощают ресурсы).
- Спроектируйте защиту: лимит глубины, лимит URL на домен, детект паттернов, нормализация URL.
- Опишите устойчивость к плохому HTML/ошибкам (не падать на одной странице).
- Покажите, как нормализация URL снижает дубли и ловушки.

## ФОРМАТ РЕЗУЛЬТАТА
Spider trap → защита (лимиты/детект/нормализация) → устойчивость к плохому HTML → роль нормализации URL.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Описана проблема spider trap (бесконечные пространства URL).
- [ ] Защита: лимит глубины, лимит URL на домен, детект паттернов, нормализация URL.
- [ ] Устойчивость: краулер не падает на битом HTML/ошибке одной страницы (изоляция).
- [ ] Нормализация URL снижает дубли и помогает против ловушек.

## ПОДСКАЗКА
Spider trap = бесконечные URL (календарь/параметры). Защита: лимит глубины, лимит URL/домен, детект паттернов, нормализация URL. Устойчивость: не падать на битом HTML. Нормализация снижает дубли.
