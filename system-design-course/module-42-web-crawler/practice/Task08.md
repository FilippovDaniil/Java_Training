# Задача 08 — Модуль 42: Построение индекса и свежесть

## ЗАДАНИЕ
Спроектируйте переход от обхода к поисковому индексу и стратегию свежести (recrawl).

## ВХОДНЫЕ ДАННЫЕ
Извлечённый контент нужно сделать искомым; страницы со временем меняются, индекс должен оставаться актуальным.

## ТРЕБОВАНИЯ
- Опишите построение inverted index (модуль 17): токенизация контента, term → документы.
- Покажите, что краулер питает индекс (контент → indexer).
- Спроектируйте recrawl: периодический переобход по частоте изменений страницы.
- Объясните, почему частота recrawl различается (новости чаще, архив реже).

## ФОРМАТ РЕЗУЛЬТАТА
Inverted index (модуль 17) → краулер питает индекс → recrawl по изменчивости → дифференцированная частота.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Inverted index (модуль 17): токенизация, term → список документов.
- [ ] Показано: контент из краулера → indexer → поисковый индекс.
- [ ] Recrawl: периодический переобход для свежести.
- [ ] Частота recrawl зависит от изменчивости страницы (новости чаще, архив реже).

## ПОДСКАЗКА
Inverted index (модуль 17): токенизация, term→документы. Краулер питает индекс. Recrawl периодический; частота по изменчивости (новости часто, архив редко).
