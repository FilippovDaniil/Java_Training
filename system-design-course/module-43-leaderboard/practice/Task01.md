# Задача 01 — Модуль 43: Leaderboard / рейтинги

## ЗАДАНИЕ
Соберите требования (FR/NFR) к лидерборду и определите, какая ось доминирует.

## ВХОДНЫЕ ДАННЫЕ
Постановка: «спроектируйте лидерборд игры: топ игроков, ранг пользователя, рейтинги за период; миллионы игроков». Нужно зафиксировать scope и понять главную ось.

## ТРЕБОВАНИЯ
- Сформулируйте ядро FR (записать/обновить счёт, ранг пользователя, top-K, диапазон вокруг пользователя, временные лидерборды).
- Сформулируйте NFR (низкая latency чтений, real-time обновления, масштаб млн, точность рангов).
- Назовите главную ось: не storage (данные мелкие), а стоимость вычисления ранга.
- Объясните, почему ранг — дорогая операция.

## ФОРМАТ РЕЗУЛЬТАТА
FR → NFR → главная ось (стоимость ранга, не storage) → почему ранг дорог → scope.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Ядро FR: записать/обновить счёт, узнать ранг, top-K, ранг/счёт вокруг пользователя, временные лидерборды.
- [ ] NFR: низкая latency чтений, real-time обновления, масштаб (млн), точность рангов.
- [ ] Главная ось: стоимость вычисления ранга, а НЕ storage (данные мелкие).
- [ ] Объяснено: ранг = «сколько счетов выше моего» → наивно O(N).
- [ ] Зафиксирован scope.

## ПОДСКАЗКА
FR: счёт/ранг/top-K/вокруг/периоды. NFR: latency чтений, real-time, масштаб, точность. Ось — не storage (данные мелкие), а стоимость ранга (наивно O(N)).
