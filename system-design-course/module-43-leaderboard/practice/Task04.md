# Задача 04 — Модуль 43: Redis Sorted Set (ZSET)

## ЗАДАНИЕ
Спроектируйте хранение лидерборда на Redis Sorted Set и объясните его сложность.

## ВХОДНЫЕ ДАННЫЕ
Нужна структура, где и обновление счёта, и запрос ранга, и top-K дёшевы при миллионах элементов.

## ТРЕБОВАНИЯ
- Опишите ZSET: элементы с числовым score, отсортированы.
- Перечислите операции и сложность: ZADD (обновить), ZREVRANK (ранг), ZREVRANGE (top-K), ZSCORE.
- Объясните внутреннее устройство: skip list (O(log N), упорядоченность) + hash (O(1) доступ по элементу).
- Покажите, как это решает проблему O(N) из SQL.

## ФОРМАТ РЕЗУЛЬТАТА
ZSET (что это) → операции+сложность → skip list + hash → решение проблемы ранга.

## КРИТЕРИИ ПРИЁМКИ
- [ ] ZSET: элементы с числовым score, отсортированы по score.
- [ ] Операции: ZADD/ZREVRANK/ZREVRANGE — O(log N); ZSCORE — O(1).
- [ ] Внутри: skip list (упорядоченность, O(log N)) + hash (доступ по элементу O(1)).
- [ ] Показано: O(log N) против O(N) SQL → задача становится дешёвой.

## ПОДСКАЗКА
ZSET: score-сортированные элементы. ZADD/ZREVRANK/ZREVRANGE = O(log N), ZSCORE = O(1). Внутри: skip list + hash. Это решает O(N)-проблему SQL.
