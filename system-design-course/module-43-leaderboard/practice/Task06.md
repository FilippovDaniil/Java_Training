# Задача 06 — Модуль 43: Временные лидерборды

## ЗАДАНИЕ
Спроектируйте лидерборды за период (daily/weekly) с автоматической очисткой.

## ВХОДНЫЕ ДАННЫЕ
Нужны рейтинги за день, неделю и всё время одновременно; старые периодные лидерборды не должны копиться вечно.

## ТРЕБОВАНИЯ
- Опишите схему: отдельный sorted set на период (board:дата, board:неделя).
- Используйте TTL для автоистечения старых периодных лидербордов.
- Покажите, что событие счёта обновляет все актуальные периоды (daily + weekly + all-time).
- Объясните, почему периоды независимы и это масштабируемо.

## ФОРМАТ РЕЗУЛЬТАТА
Sorted set на период → TTL → обновление всех актуальных периодов → независимость периодов.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Отдельный sorted set на период (board:2026-06-28, board:2026-W26).
- [ ] TTL на периодные ключи → автоистечение старых лидербордов.
- [ ] Событие счёта обновляет все актуальные периоды (daily + weekly + all-time).
- [ ] Объяснено: периоды независимы, очистка автоматическая → масштабируемо.

## ПОДСКАЗКА
Отдельный ZSET на период (board:дата) + TTL (автоистечение). Счёт пишется во все актуальные периоды (daily+weekly+all-time). Периоды независимы.
