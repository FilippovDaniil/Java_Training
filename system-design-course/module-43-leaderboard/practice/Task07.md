# Задача 07 — Модуль 43: Шардирование и приближённые ранги

## ЗАДАНИЕ
Разберите, почему шардирование глобального ранга трудно, и предложите подходы для огромного масштаба.

## ВХОДНЫЕ ДАННЫЕ
Пользователей становится настолько много, что один ZSET не помещается на узел. Нужно сохранить top-K и ранги.

## ТРЕБОВАНИЯ
- Объясните, почему ранг — глобальная величина и потому труден к шардированию.
- Опишите подход «один большой ZSET, пока влезает» (часто достаточно — данные мелкие).
- Опишите партицию + merge для top-K и почему точный глобальный ранг при этом сложен.
- Предложите приближённые ранги (бакеты/перцентили — «вы в топ-5%») для огромного масштаба.

## ФОРМАТ РЕЗУЛЬТАТА
Почему ранг глобален/труден → один ZSET пока влезает → партиция+merge для top-K → приближённые ранги.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Объяснено: ранг глобален (нужны все счета) → шардирование точного ранга трудно.
- [ ] «Один большой ZSET, пока влезает» — часто достаточно (данные мелкие).
- [ ] Партиция + merge для top-K (top из шардов → объединить); глобальный ранг при шардировании сложен.
- [ ] Приближённые ранги (бакеты/перцентили) для огромного масштаба — осознанный trade-off точность/масштаб.

## ПОДСКАЗКА
Ранг глобален (нужны все счета) → шардировать трудно. Один ZSET пока влезает (данные мелкие). Top-K: партиция+merge. Огромный масштаб: приближённые ранги (перцентили «топ-5%»).
