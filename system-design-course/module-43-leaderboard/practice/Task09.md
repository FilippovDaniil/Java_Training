# Задача 09 — Модуль 43: Узкие места

## ЗАДАНИЕ
Найдите узкие места лидерборда на масштабе и предложите решения со ссылками на модули.

## ВХОДНЫЕ ДАННЫЕ
Один очень популярный глобальный лидерборд, высокая частота обновлений счёта, требования к durability и (иногда) к точному рангу при огромном числе игроков.

## ТРЕБОВАНИЯ
Разберите минимум 4 узких места:
- hot leaderboard (один популярный);
- write throughput обновлений счёта;
- шардирование рангов;
- durability ZSET.
Для каждого — решение и ссылка на модуль.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: узкое место → причина → решение → модуль.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Hot leaderboard → реплики ZSET для чтений + кэш top-K (модули 9, 12).
- [ ] Write throughput → пакетирование обновлений, реплика на запись (модуль 26).
- [ ] Шардирование рангов → top-K через merge; приближённые ранги для глобального (модуль 13).
- [ ] Durability ZSET → Redis-persistence + durable backing store (модуль 12).
- [ ] Отмечен trade-off точность vs масштаб (приближённые ранги/перцентили).

## ПОДСКАЗКА
Hot leaderboard → реплики чтений + кэш top-K (9/12). Write throughput → пакетирование (26). Шардирование → merge/приближённость (13). Durability → Redis-persistence + backing store (12).
