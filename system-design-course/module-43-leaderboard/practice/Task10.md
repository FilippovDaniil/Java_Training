# Задача 10 — Модуль 43 (мини-проект): Полный HLD лидерборда

## ЗАДАНИЕ
Соберите полный high-level design лидерборда по фреймворку design-интервью. Синтез модуля и завершение под-батча.

## ВХОДНЫЕ ДАННЫЕ
Условие интервью: «Спроектируйте лидерборд игры: топ игроков, ранг пользователя, соседи по рейтингу, рейтинги за день/неделю; миллионы игроков, real-time обновления». 45–60 минут.

## ТРЕБОВАНИЯ
Дизайн-документ по шагам:
1. **Требования** — FR/NFR, главная ось (стоимость ранга, не storage).
2. **Estimation** — обновления/чтения, объём (мелкий) vs стоимость ранга.
3. **Проблема ранга** — почему SQL O(N) не масштабируется.
4. **Redis Sorted Set** — O(log N), ZADD/ZREVRANK/ZREVRANGE, skip list + hash.
5. **Запросы** — top-K, ранг, диапазон вокруг, ties.
6. **Временные лидерборды** — отдельные ZSET + TTL.
7. **Шардирование/приближённость + персистентность + узкие места** — глобальный ранг труден → приближённые ранги; durable backing; hot leaderboard/write throughput.

## ФОРМАТ РЕЗУЛЬТАТА
HLD из 7 разделов + диаграмма (score service → Redis ZSET + durable store; чтения → ZSET) + список команд ZSET.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Требования: главная ось — стоимость ранга, не storage; scope зафиксирован.
- [ ] Estimation: обновления/чтения; данные мелкие, узкое место — ранг.
- [ ] Проблема ранга: SQL COUNT/ORDER BY = O(N), не масштабируется.
- [ ] Redis Sorted Set: O(log N), ZADD/ZREVRANK/ZREVRANGE, skip list + hash.
- [ ] Запросы: top-K, ранг, диапазон вокруг пользователя; обработка ties.
- [ ] Временные лидерборды: отдельные ZSET + TTL.
- [ ] Шардирование (глобальный ранг труден → приближённые ранги/перцентили); персистентность (durable backing + Redis-persistence); узкие места (hot leaderboard/write throughput); связи 9/26/13/12/11.

## ПОДСКАЗКА
По фреймворку: требования (стоимость ранга) → estimation → проблема ранга (SQL O(N)) → Redis ZSET (O(log N)) → запросы (top-K/ранг/вокруг) → временные лидерборды (TTL) → шардирование/приближённость + персистентность + узкие места. Стержень: ранг на масштабе труден; sorted set (skip list) решает за O(log N); глобальный ранг труден к шардированию → приближённость. Этим завершается под-батч d Части 8.
