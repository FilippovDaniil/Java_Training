# Задача 01 — Модуль 44: Распределённый планировщик задач

## ЗАДАНИЕ
Соберите требования (FR/NFR) к распределённому планировщику задач и определите главное NFR.

## ВХОДНЫЕ ДАННЫЕ
Постановка: «спроектируйте распределённый планировщик задач: разовые и cron-задачи, выполнение через воркеры, миллионы задач». Нужно зафиксировать scope и главное требование.

## ТРЕБОВАНИЯ
- Сформулируйте ядро FR (запланировать разовую/cron-задачу, выполнить через воркеры, retry, статус/история, отмена, приоритеты).
- Сформулируйте NFR (надёжность/at-least-once, масштаб, точность срабатывания, нет SPOF, durability определений).
- Назовите главное NFR (надёжность — задача обязана выполниться) и обоснуйте.
- Объясните, почему нагрузка неравномерна во времени (cron-пики).

## ФОРМАТ РЕЗУЛЬТАТА
FR → NFR → главное NFR (надёжность) → неравномерность во времени → scope.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Ядро FR: запланировать (разовая T / cron), выполнить через воркеры, retry, статус/история, отмена, приоритеты.
- [ ] NFR: надёжность/at-least-once, масштаб (млн задач), точность/низкая latency срабатывания, нет SPOF, durability определений.
- [ ] Главное NFR — надёжность (задача обязана выполниться); обосновано.
- [ ] Отмечено: нагрузка неравномерна (cron-задачи на «круглые» времена → пики).
- [ ] Зафиксирован scope.

## ПОДСКАЗКА
FR: запланировать (разовая/cron), выполнить, retry, статус, отмена, приоритеты. NFR: надёжность/at-least-once (главное!), масштаб, точность срабатывания, нет SPOF, durability. Нагрузка неравномерна — пики в полночь/начало часа.
