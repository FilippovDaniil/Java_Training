# Задача 04 — Модуль 44: Развязка планирования и выполнения

## ЗАДАНИЕ
Спроектируйте развязку scheduler'а и воркеров через очередь.

## ВХОДНЫЕ ДАННЫЕ
Если scheduler сам исполняет задачи, долгая задача блокирует планирование остальных. Нужно разделить «когда запускать» и «как исполнять».

## ТРЕБОВАНИЯ
- Опишите роль scheduler: найти due-задачу → поставить в очередь (не исполнять самому).
- Опишите роль workers: тянуть из очереди → исполнять → обновлять статус.
- Обоснуйте развязку через очередь (модуль 15): scheduler лёгкий, исполнение масштабируется отдельно.
- Покажите, почему scheduler не должен исполнять задачи сам.

## ФОРМАТ РЕЗУЛЬТАТА
Scheduler (найти→очередь) → workers (очередь→исполнить→статус) → зачем развязка → почему не исполнять в scheduler.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Scheduler: находит due → ставит в очередь, возвращается к планированию (не блокируется).
- [ ] Workers: тянут из очереди → исполняют → обновляют статус/историю.
- [ ] Развязка через очередь (модуль 15): scheduler лёгкий, исполнение масштабируется независимо.
- [ ] Объяснено: исполнение в scheduler → долгая задача блокирует планирование остальных.

## ПОДСКАЗКА
Scheduler: due → очередь (не исполняет). Workers: очередь → исполнить → статус. Развязка (модуль 15): scheduler лёгкий, воркеры масштабируются отдельно. Исполнение в scheduler блокирует планирование.
