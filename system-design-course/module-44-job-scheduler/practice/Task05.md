# Задача 05 — Модуль 44: Надёжность (at-least-once + идемпотентность)

## ЗАДАНИЕ
Спроектируйте надёжное выполнение задач с гарантией at-least-once и обработкой сбоев.

## ВХОДНЫЕ ДАННЫЕ
Задача обязана выполниться. Из-за retry и переотдачи после сбоя воркера она может выполниться повторно. Некоторые задачи стабильно падают.

## ТРЕБОВАНИЯ
- Обоснуйте выбор at-least-once execution (лучше дважды, чем потерять).
- Покажите следствие: повторное выполнение → задачи должны быть идемпотентными (модуль 23).
- Спроектируйте retry с backoff и DLQ для постоянно падающих («ядовитых») задач.
- Объясните, почему строгий exactly-once недостижим → at-least-once + идемпотентность.

## ФОРМАТ РЕЗУЛЬТАТА
At-least-once (почему) → идемпотентность задач → retry/backoff/DLQ → exactly-once недостижим.

## КРИТЕРИИ ПРИЁМКИ
- [ ] At-least-once execution обоснован (лучше выполнить дважды, чем потерять).
- [ ] Следствие: повтор/переотдача → задачи идемпотентны (модуль 23).
- [ ] Retry с backoff + DLQ для постоянно падающих задач.
- [ ] Объяснено: строгий exactly-once недостижим → at-least-once + идемпотентность.

## ПОДСКАЗКА
At-least-once (лучше дважды, чем потерять) → повтор → задачи идемпотентны (модуль 23). Retry+backoff, DLQ для ядовитых. Exactly-once недостижим → at-least-once + идемпотентность.
