# Задача 06 — Модуль 44: Отказ воркера (lease / visibility timeout)

## ЗАДАНИЕ
Спроектируйте обработку смерти воркера в процессе выполнения задачи.

## ВХОДНЫЕ ДАННЫЕ
Воркер взял задачу из очереди и упал, не завершив её. Задача не должна потеряться, но и не должна «застрять» навсегда у мёртвого воркера.

## ТРЕБОВАНИЯ
- Опишите lease/visibility timeout: задача «арендована» воркером на время T.
- Опишите heartbeat: воркер периодически продлевает аренду.
- Покажите переотдачу: нет heartbeat / истёк lease → задача переотдаётся другому воркеру.
- Свяжите с at-least-once и необходимостью идемпотентности (повтор после переотдачи).

## ФОРМАТ РЕЗУЛЬТАТА
Lease/visibility timeout → heartbeat → переотдача при истечении → связь с идемпотентностью.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Lease/visibility timeout: задача арендована на время T.
- [ ] Heartbeat: воркер продлевает аренду, пока жив.
- [ ] Нет heartbeat / истёк lease → задача переотдаётся другому воркеру (не теряется).
- [ ] Связь: переотдача → возможен повтор → идемпотентность обязательна (модуль 23).

## ПОДСКАЗКА
Lease/visibility timeout: аренда на время T + heartbeat. Истёк lease/нет heartbeat → переотдать другому воркеру. Переотдача → повтор → идемпотентность (модуль 23).
