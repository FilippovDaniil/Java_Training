# Задача 07 — Модуль 44: Scheduler как SPOF (leader election)

## ЗАДАНИЕ
Устраните единую точку отказа в планировщике, не допустив двойного запуска задач.

## ВХОДНЫЕ ДАННЫЕ
Один scheduler — SPOF (упал → ничего не планируется). Два scheduler'а наивно triggered бы каждую задачу дважды.

## ТРЕБОВАНИЯ
- Покажите проблему: один scheduler = SPOF; два наивно = двойной запуск.
- Спроектируйте leader election (модуль 22): только лидер триггерит задачи, при падении выбирается новый.
- Опишите альтернативу: партиционирование задач между scheduler'ами (каждый владеет частью).
- Сравните подходы (leader election vs партиция).

## ФОРМАТ РЕЗУЛЬТАТА
Проблема SPOF/двойной запуск → leader election (модуль 22) → партиция задач → сравнение.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Проблема: один scheduler — SPOF; два без координации — двойной запуск.
- [ ] Leader election (модуль 22): только лидер триггерит, failover при падении.
- [ ] Альтернатива: партиционирование задач (каждый scheduler владеет своей частью).
- [ ] Сравнение: leader election (простота, один активный) vs партиция (параллелизм планирования).

## ПОДСКАЗКА
Один scheduler — SPOF; два — двойной запуск. Leader election (модуль 22): лидер триггерит, failover. Или партиция задач между scheduler'ами. Сравнить: один активный vs параллелизм.
