# Задача 09 — Модуль 44: Узкие места

## ЗАДАНИЕ
Найдите узкие места планировщика на масштабе и предложите решения со ссылками на модули.

## ВХОДНЫЕ ДАННЫЕ
Миллионы задач, многие назначены на «круглые» времена (полночь, начало часа). Scheduler не должен быть SPOF, а поиск due-задач — узким местом.

## ТРЕБОВАНИЯ
Разберите минимум 4 узких места:
- scheduler как SPOF;
- поиск due-задач на масштабе;
- hot time (полночь, thundering herd);
- масштаб воркеров.
Для каждого — решение и ссылка на модуль.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: узкое место → причина → решение → модуль.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Scheduler SPOF → leader election / партиция задач (модуль 22).
- [ ] Поиск due-задач → индекс по next_run / timing wheel (не полный скан).
- [ ] Hot time (полночь) → джиттер/размазывание срабатываний + масштаб воркеров (модуль 16).
- [ ] Масштаб воркеров → горизонтальное масштабирование, очередь как буфер (модуль 15).
- [ ] Отмечено: строгий exactly-once недостижим → at-least-once + идемпотентность (модуль 23).

## ПОДСКАЗКА
Scheduler SPOF → leader election/партиция (22). Поиск due → индекс/timing wheel. Hot time → джиттер + воркеры (16). Масштаб воркеров → очередь-буфер (15). Exactly-once → at-least-once+идемпотентность (23).
