# Задача 01 — Модуль 45: Система метрик и мониторинга

## ЗАДАНИЕ
Соберите требования (FR/NFR) к системе метрик и определите её природу.

## ВХОДНЫЕ ДАННЫЕ
Постановка: «спроектируйте систему метрик и мониторинга (Prometheus/Datadog): приём метрик, хранение, запросы, алертинг, дашборды». Нужно зафиксировать scope и главную ось.

## ТРЕБОВАНИЯ
- Сформулируйте ядро FR (приём метрик от агентов, хранение time-series, запрос/агрегация, алертинг, дашборды).
- Сформулируйте NFR (огромный write-throughput, масштабируемое хранилище, быстрые запросы по диапазонам, retention/downsampling, HA).
- Назовите главную ось (write-heavy поток time-series) и почему не RPS-запросов.
- Отметьте, что это дизайн платформы (концепции SLI/SLO/RED/USE — в модуле 28).

## ФОРМАТ РЕЗУЛЬТАТА
FR → NFR → главная ось (write-heavy time-series) → связь с модулем 28 → scope.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Ядро FR: приём метрик, хранение time-series, запрос/агрегация, алертинг, дашборды.
- [ ] NFR: огромный write-throughput (млн метрик/с), масштабируемое хранилище, быстрые запросы по диапазонам, retention/downsampling, HA.
- [ ] Главная ось: write-heavy поток time-series (не RPS-запросов).
- [ ] Отмечено: это дизайн платформы; SLI/SLO/RED/USE — концепции модуля 28.
- [ ] Зафиксирован scope.

## ПОДСКАЗКА
FR: приём/хранение/запрос/алертинг/дашборды. NFR: write-throughput (главное!), масштаб storage, быстрые запросы по диапазонам, retention, HA. Ось — write-heavy time-series. SLI/SLO — модуль 28.
