# Задача 02 — Модуль 45: Estimation

## ЗАДАНИЕ
Оцените write-throughput метрик и storage, покажите эффект сжатия time-series.

## ВХОДНЫЕ ДАННЫЕ
Допущения (примите явно): число хостов, метрик на хост, частота сбора (раз в N секунд), размер точки, срок хранения.

## ТРЕБОВАНИЯ
- Посчитайте write QPS точек = хосты × метрик × частота.
- Оцените наивный storage = точки/с × срок × размер точки.
- Покажите, как сжатие time-series (delta-of-delta/XOR) кратно снижает реальный объём.
- Сделайте вывод: система write-heavy, главное — приём и эффективное хранение.

## ФОРМАТ РЕЗУЛЬТАТА
Допущения → write QPS → наивный storage → эффект сжатия → вывод write-heavy.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Write QPS = хосты × метрик × частота → порядка млн точек/с.
- [ ] Наивный storage = точки/с × срок × размер точки (терабайты).
- [ ] Показано: сжатие time-series (delta-of-delta + XOR) кратно снижает объём.
- [ ] Вывод: система write-heavy; ось — приём + эффективное хранение time-series.

## ПОДСКАЗКА
Write QPS = хосты × метрик × частота (напр. 10^5×10^3/10с=10^6/с). Наивный storage = QPS×срок×размер. Сжатие (delta-of-delta/XOR) снижает в разы. Write-heavy.
