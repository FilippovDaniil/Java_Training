# Задача 03 — Модуль 45: Модель данных time-series

## ЗАДАНИЕ
Спроектируйте модель данных time-series и понятие серии.

## ВХОДНЫЕ ДАННЫЕ
Нужно хранить измерения вида «http-запросов столько-то для сервиса api, метод GET, статус 200 в момент T» и уметь разбивать по измерениям.

## ТРЕБОВАНИЯ
- Опишите точку: metric_name + labels/теги + timestamp + value.
- Определите серию (series) как уникальную комбинацию (metric + labels).
- Покажите, как запрос выбирает серии по labels + временной диапазон и агрегирует.
- Отметьте, что labels дают многомерность, но являются источником cardinality (см. задачу 04).

## ФОРМАТ РЕЗУЛЬТАТА
Точка (metric+labels+ts+value) → серия → запрос (labels + диапазон + агрегация) → labels как источник cardinality.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Точка: metric_name + набор labels + timestamp + value.
- [ ] Серия = уникальная комбинация (metric + labels).
- [ ] Запрос: выбор серий по labels + временной диапазон → агрегация.
- [ ] Отмечено: labels дают многомерность, но порождают cardinality.

## ПОДСКАЗКА
Точка: metric{labels} ts value. Серия = metric + уникальный набор labels. Запрос: серии по labels + диапазон → агрегация. Labels = многомерность, но источник cardinality.
