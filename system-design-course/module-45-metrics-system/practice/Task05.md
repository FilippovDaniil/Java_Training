# Задача 05 — Модуль 45: Ingestion push vs pull

## ЗАДАНИЕ
Сравните push и pull модели приёма метрик и выберите под кейс.

## ВХОДНЫЕ ДАННЫЕ
Часть инфраструктуры — за NAT/firewall и эфемерные задачи; часть — стабильные сервисы с service discovery. Нужно понимать, кто «пропал».

## ТРЕБОВАНИЯ
- Опишите push (агенты шлют в коллектор): плюсы (NAT/firewall, ephemeral), минус (нет списка → «кто пропал?»).
- Опишите pull (сервер скрейпит таргеты, Prometheus): плюсы (явные таргеты, видно down, контроль частоты), минус (доступ + service discovery).
- Покажите роль буфера/очереди (модуль 15) перед TSDB.
- Выберите модель(и) под кейс с обоснованием.

## ФОРМАТ РЕЗУЛЬТАТА
Push (плюсы/минус) → pull (плюсы/минус) → буфер перед TSDB → выбор под кейс.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Push: + за NAT/firewall и ephemeral, − нет явного списка таргетов («кто пропал?»).
- [ ] Pull: + явные таргеты (видно down), контроль частоты, − нужен доступ + service discovery.
- [ ] Буфер/очередь (модуль 15) перед TSDB для сглаживания пиков.
- [ ] Выбор модели(ей) под кейс обоснован (часто гибрид).

## ПОДСКАЗКА
Push: +NAT/ephemeral, −нет списка (кто пропал?). Pull (Prometheus): +явные таргеты/видно down, −доступ+service discovery. Буфер (модуль 15) перед TSDB. Часто гибрид.
