# Задача 07 — Модуль 45: Downsampling и retention

## ЗАДАНИЕ
Спроектируйте downsampling старых данных и политику retention.

## ВХОДНЫЕ ДАННЫЕ
Хранить все сырые точки (секундное разрешение) вечно — слишком дорого. Но тренды за месяцы/год нужны.

## ТРЕБОВАНИЯ
- Опишите downsampling: raw → rollups (1m → 1h → 1d агрегаты) для старых данных.
- Спроектируйте retention tiers (свежее — высокое разрешение, старое — downsampled).
- Покажите, как это снижает storage при сохранении трендов.
- Приведите пример политики (raw 15 дней → 1m 90 дней → 1h год).

## ФОРМАТ РЕЗУЛЬТАТА
Downsampling (rollups) → retention tiers → снижение storage с сохранением трендов → пример политики.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Downsampling: raw → 1m → 1h → 1d агрегаты (min/max/avg по окну) для старых данных.
- [ ] Retention tiers: свежее в высоком разрешении, старое downsampled.
- [ ] Показано: резкое снижение storage при сохранении трендов.
- [ ] Приведён пример политики retention (например, raw 15д → 1m 90д → 1h год).

## ПОДСКАЗКА
Downsampling: старое → rollups (1m/1h/1d, min/max/avg). Retention tiers: свежее raw, старое downsampled. Пример: raw 15д → 1m 90д → 1h год. Storage ↓, тренды сохранены.
