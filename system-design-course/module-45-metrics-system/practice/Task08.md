# Задача 08 — Модуль 45: Запросы и алертинг

## ЗАДАНИЕ
Спроектируйте запросы по метрикам и систему алертинга с alert manager.

## ВХОДНЫЕ ДАННЫЕ
Нужны дашборды (агрегаты по времени и сериям) и алерты по условиям (error_rate > 5% за 5 мин). При инциденте не должно быть шквала одинаковых алертов.

## ТРЕБОВАНИЯ
- Опишите запросы: временной диапазон + агрегация по сериям (PromQL-подобный: rate, sum by).
- Спроектируйте алертинг: правила вычисляются по метрикам → алерт.
- Опишите alert manager: дедуп (один инцидент — один алерт), группировка, silencing, маршрутизация (модуль 36).
- Свяжите с alert fatigue (модуль 28).

## ФОРМАТ РЕЗУЛЬТАТА
Запросы (диапазон + агрегация) → правила алертинга → alert manager (дедуп/группировка/silencing) → связь с alert fatigue.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Запросы: временной диапазон + агрегация по сериям (PromQL-подобно: rate/sum by).
- [ ] Алертинг: правила по метрикам (порог за окно) → алерт.
- [ ] Alert manager: дедуп (один инцидент = один алерт), группировка, silencing, маршрутизация (модуль 36).
- [ ] Связь с alert fatigue (модуль 28): дедуп/группировка против шквала.

## ПОДСКАЗКА
Запросы: диапазон + агрегация по сериям (rate, sum by). Алертинг: правила → алерт. Alert manager: дедуп/группировка/silencing/маршрутизация (модуль 36). Против alert fatigue (модуль 28).
