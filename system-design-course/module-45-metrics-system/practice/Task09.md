# Задача 09 — Модуль 45: Масштаб и узкие места

## ЗАДАНИЕ
Спроектируйте масштабирование системы метрик и разберите узкие места.

## ВХОДНЫЕ ДАННЫЕ
Поток метрик растёт до миллионов точек/с, появляются длинные запросы (за месяцы) и высокая cardinality. Один узел TSDB не справляется.

## ТРЕБОВАНИЯ
- Опишите шардирование по серии/метрике (consistent hashing — модуль 14) и федерацию.
- Разберите узкие места: write throughput, high cardinality, запросы по длинным диапазонам, storage cost, hot metrics.
- Для каждого — решение и ссылка на модуль.

## ФОРМАТ РЕЗУЛЬТАТА
Шардирование (consistent hashing) + федерация → таблица узких мест (причина → решение → модуль).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Шардирование по серии/метрике (consistent hashing — модуль 14) + федерация.
- [ ] Write throughput → буфер/очередь + шардирование (модули 15, 14).
- [ ] High cardinality → ограничивать labels (не unbounded), cardinality-лимиты (модуль 28).
- [ ] Запросы по длинным диапазонам → downsampling/rollups; storage cost → сжатие + retention (модуль 18); hot metrics → шарды/реплики (модули 14, 12).

## ПОДСКАЗКА
Шард по серии (consistent hashing, 14) + федерация. Write throughput → буфер+шарды (15/14). Cardinality → ограничить labels (28). Длинные запросы → downsampling. Storage → сжатие+retention (18). Hot metrics → шарды/реплики (14/12).
