# Задача 02 — Модуль 46: Карта паттернов (архетип → блоки)

## ЗАДАНИЕ
Для заданных архетипов подберите строительные блоки из Частей 3–7.

## ВХОДНЫЕ ДАННЫЕ
Архетипы: read-heavy, write-heavy/ingestion, real-time/messaging, geo/proximity, correctness-critical.

## ТРЕБОВАНИЯ
- Для каждого архетипа перечислите ключевые блоки с указанием модуля.
- Объясните, почему именно эти блоки решают доминанту архетипа.
- Отметьте сквозные паттерны, встречающиеся почти везде (идемпотентность, кэш, шардирование, async-очередь).
- Покажите, как карта ускоряет дизайн (переиспользование, а не изобретение).

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: архетип → блоки (с модулями) → почему → сквозные паттерны.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Read-heavy → кэш (9) + реплики (12) + CDN (10) + fan-out (34).
- [ ] Write-heavy → очередь (15) + async-агрегация (20/26) + TSDB/OLAP (18/45).
- [ ] Real-time → WebSocket + connection servers (35); geo → geospatial index (17); correctness → идемпотентность (23) + ACID (21) + ledger (40).
- [ ] Отмечены сквозные паттерны (идемпотентность/кэш/шардирование/async-очередь); вывод о переиспользовании.

## ПОДСКАЗКА
Read-heavy→кэш+реплики+CDN (9/12/10). Write-heavy→очередь+async-агрегация+TSDB (15/20/45). Real-time→WebSocket (35). Geo→geospatial index (17). Correctness→идемпотентность+ACID+ledger (23/21/40). Сквозные: идемпотентность/кэш/шард/очередь.
