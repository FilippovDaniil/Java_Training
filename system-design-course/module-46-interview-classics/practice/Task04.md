# Задача 04 — Модуль 46: Применение 8-шагового фреймворка

## ЗАДАНИЕ
Составьте план ответа на классическую задачу по 8-шаговому фреймворку.

## ВХОДНЫЕ ДАННЫЕ
Задача: «спроектируйте typeahead/автодополнение поиска». 45–60 минут.

## ТРЕБОВАНИЯ
- Распишите ответ по 8 шагам: требования → estimation → API → модель данных → high-level → deep dive → bottlenecks → wrap-up.
- На шаге deep dive выберите 1–2 критичных компонента (для typeahead — trie + кэш популярных префиксов).
- Покажите think-aloud (что проговариваете) и где задаёте уточняющие вопросы.
- Отметьте архетип задачи (read-heavy + search/index) и какие блоки тянет.

## ФОРМАТ РЕЗУЛЬТАТА
8 шагов (тезисно) → выбор deep dive → think-aloud-точки → архетип + блоки.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Все 8 шагов покрыты тезисно (требования→...→wrap-up).
- [ ] Deep dive: 1–2 компонента (trie + кэш префиксов — модуль 17).
- [ ] Показаны think-aloud и уточняющие вопросы (требования первым).
- [ ] Архетип (read-heavy + search) и блоки (trie, кэш, inverted index — модуль 17) названы.

## ПОДСКАЗКА
8 шагов: требования→estimation→API→data→high-level→deep dive (trie+кэш префиксов)→bottlenecks→wrap-up. Архетип read-heavy+search → trie/кэш/inverted index (17). Think aloud + уточнения первым.
