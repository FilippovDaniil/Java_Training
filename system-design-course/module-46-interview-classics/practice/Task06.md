# Задача 06 — Модуль 46: Шпаргалка чисел под кейс

## ЗАДАНИЕ
Примените опорные числа (latency, порядки, peak) для быстрой оценки в кейсе.

## ВХОДНЫЕ ДАННЫЕ
Кейс: сервис с 100M DAU, read/write ratio 100:1, средний объект 1 КБ, хранение 5 лет. Нужна быстрая прикидка на интервью.

## ТРЕБОВАНИЯ
- Используйте порядки (день ≈ 86400 ≈ 10^5 с) для перевода DAU → RPS.
- Примените peak factor (2–10×) для пиковой нагрузки.
- Разложите read vs write по ratio 100:1.
- Прикиньте storage за 5 лет (с округлением до порядка).
- Приведите релевантные latency numbers (RAM/SSD/диск/сеть), если они влияют на решение.

## ФОРМАТ РЕЗУЛЬТАТА
DAU→RPS (порядки) → peak → read/write split → storage 5 лет → latency numbers по месту.

## КРИТЕРИИ ПРИЁМКИ
- [ ] DAU → RPS через 10^5 с/день (с разумным числом запросов на юзера).
- [ ] Применён peak factor (2–10×).
- [ ] Read/write разложены по 100:1.
- [ ] Storage за 5 лет прикинут до порядка (× replication при необходимости).
- [ ] Уместно приведены latency numbers (RAM ~100нс, SSD ~100мкс, диск ~10мс, сеть DC ~0.5мс).

## ПОДСКАЗКА
DAU/10^5 c → средний RPS; ×(2–10) пик. 100:1 → read доминирует. Storage = объекты × 1КБ × 5 лет × replication, округлить до порядка. Latency: RAM 100нс/SSD 100мкс/диск 10мс/сеть DC 0.5мс.
