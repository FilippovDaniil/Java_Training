# Задача 09 — Модуль 46: Сравнение задач одного архетипа

## ЗАДАНИЕ
Сравните две задачи одного архетипа: что общего (переиспользуемые блоки) и чем различаются.

## ВХОДНЫЕ ДАННЫЕ
Пара задач write-heavy/ingestion: metrics-system (модуль 45) и ad-aggregation (модуль 41).

## ТРЕБОВАНИЯ
- Назовите общий архетип и общие блоки (что переиспользуется).
- Покажите различия (специфика каждой задачи).
- Объясните, как распознавание общего архетипа ускоряет дизайн обеих.
- Сделайте вывод: архетип даёт каркас, специфика — детали.

## ФОРМАТ РЕЗУЛЬТАТА
Общий архетип → общие блоки → различия → вывод (каркас vs детали).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Общий архетип: write-heavy/ingestion.
- [ ] Общие блоки: очередь (15) + async-агрегация по окнам (20/26) + специализированное хранилище (TSDB/OLAP).
- [ ] Различия: metrics — time-series/TSDB/cardinality/downsampling; ad — exactly-once подсчёт/дедуп/click fraud/бюджет.
- [ ] Вывод: архетип = общий каркас (очередь+async-агрегация), специфика = детали хранилища и точности.

## ПОДСКАЗКА
Общий архетип write-heavy. Общее: очередь+async-агрегация по окнам+специализированное хранилище. Различия: metrics (TSDB/cardinality/downsampling) vs ad (exactly-once/дедуп/click fraud/бюджет). Архетип = каркас, специфика = детали.
