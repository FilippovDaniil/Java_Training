# Задача 03 — Модуль 47: Различение грейдов на одном вопросе

## ЗАДАНИЕ
Покажите, как ответы junior / mid / senior / staff различаются на одном и том же вопросе.

## ВХОДНЫЕ ДАННЫЕ
Вопрос: «спроектируйте newsfeed». Нужно описать, чем характерен ответ каждого грейда.

## ТРЕБОВАНИЯ
- Для каждого грейда опишите характерный ответ на этот вопрос.
- Junior — знает компоненты, не собирает целое; mid — полный дизайн + базовые trade-offs; senior — ведёт, квантификация, deep dive (fan-out push/pull, celebrity), failure modes; staff — несколько архитектур, cost/эволюция/организация.
- Подчеркните, что отличает каждый переход (junior→mid→senior→staff).
- Сделайте вывод: грейд — про глубину/проактивность/trade-offs, не про число компонентов.

## ФОРМАТ РЕЗУЛЬТАТА
Junior / mid / senior / staff ответы → что отличает переходы → вывод.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Junior: знает кубики (кэш/БД/LB), не собирает целостный дизайн, нет чисел/trade-offs.
- [ ] Mid: полный рабочий дизайн + базовые trade-offs, реактивен.
- [ ] Senior: ведёт, квантифицирует, deep dive (fan-out push/pull, celebrity → hybrid), failure modes, проактивен.
- [ ] Staff: несколько жизнеспособных архитектур, cost/эволюция/организация, лидирует; отмечено, что грейд — про глубину/проактивность, не число компонентов.

## ПОДСКАЗКА
Junior — кубики без целого. Mid — полный дизайн + базовые trade-offs, реактивен. Senior — ведёт+числа+deep dive (fan-out, celebrity→hybrid)+failure modes. Staff — несколько архитектур+cost/эволюция. Грейд = глубина/проактивность, не число компонентов.
