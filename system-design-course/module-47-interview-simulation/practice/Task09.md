# Задача 09 — Модуль 47: Что поднимает ответ с mid на senior

## ЗАДАНИЕ
Конкретно покажите, что превращает mid-ответ в senior-ответ на одном дизайне.

## ВХОДНЫЕ ДАННЫЕ
Mid-ответ на «спроектируйте rate-limiter»: рабочий дизайн (token bucket в Redis), базовый trade-off алгоритмов, но без чисел, без обсуждения распределённого случая и отказов, реактивно.

## ТРЕБОВАНИЯ
- Назовите конкретные добавления, поднимающие ответ до senior.
- Покажите: квантификация (лимиты под ёмкость downstream), проактивный deep dive (распределённый счётчик Redis, точность vs latency), trade-offs вслух, failure modes (что при недоступности лимитера).
- Подчеркните проактивность (вести, а не ждать вопросов).
- Сделайте вывод: mid→senior — это глубина + числа + trade-offs + проактивность, не больше компонентов.

## ФОРМАТ РЕЗУЛЬТАТА
Mid-ответ → конкретные senior-добавления → проактивность → вывод.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Названы senior-добавления: квантификация (лимит под ёмкость downstream), deep dive (распределённый счётчик Redis, точность vs latency), trade-offs вслух, failure modes.
- [ ] Подчёркнута проактивность (ведёт сам, углубляется без наводящих).
- [ ] Показано, как каждое добавление меняет сигнал.
- [ ] Вывод: mid→senior = глубина + числа + trade-offs + проактивность, не число компонентов.

## ПОДСКАЗКА
Senior-добавления: числа (лимит под ёмкость downstream), deep dive (распределённый Redis-счётчик, точность vs latency — модуль 16), trade-offs вслух, failure modes (лимитер недоступен → fail-open/closed). Проактивность. mid→senior = глубина+числа+trade-offs.
