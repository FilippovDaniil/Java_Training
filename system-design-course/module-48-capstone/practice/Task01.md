# Задача 01 — Модуль 48: Capstone — требования и синтез

## ЗАДАНИЕ
Соберите требования (FR/NFR) к системе онлайн-бронирования билетов и покажите, почему этот домен синтезирует весь курс.

## ВХОДНЫЕ ДАННЫЕ
Постановка: «спроектируйте систему продажи билетов на события (flash-sale ticketing): просмотр, выбор мест, бронь, оплата; популярные концерты — миллионы покупателей в момент старта продаж».

## ТРЕБОВАНИЯ
- Сформулируйте ядро FR (просмотр/поиск событий, выбор мест, hold/резерв, оплата, подтверждение).
- Сформулируйте NFR (нет оверселлинга=correctness, выдержать flash-sale спайк, read-heavy просмотр, fairness, доступность).
- Назовите архетипы, которые смешивает домен (correctness + read-heavy + контеншн + payment + search).
- Объясните, почему это хороший capstone (синтез блоков из разных частей).

## ФОРМАТ РЕЗУЛЬТАТА
FR → NFR → смешанные архетипы → почему синтезирует курс → scope.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Ядро FR: просмотр/поиск событий, выбор мест, hold/резерв, оплата, подтверждение.
- [ ] NFR: нет оверселлинга (correctness), выдержать flash-sale спайк (млн), read-heavy просмотр, fairness очереди, доступность.
- [ ] Названы смешанные архетипы: correctness-critical + read-heavy + экстремальная контеншн + payment + search.
- [ ] Объяснено: домен синтезирует блоки из разных частей → хороший capstone.

## ПОДСКАЗКА
FR: просмотр/поиск/выбор мест/hold/оплата. NFR: нет оверселлинга (главное), спайк, read-heavy, fairness. Архетипы: correctness+read-heavy+контеншн+payment+search. Синтез → capstone.
