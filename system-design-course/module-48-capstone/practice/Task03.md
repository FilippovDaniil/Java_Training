# Задача 03 — Модуль 48: Оверселлинг и жизненный цикл места

## ЗАДАНИЕ
Спроектируйте инвентарь мест, исключающий оверселлинг при тысячах претендентов на одно место.

## ВХОДНЫЕ ДАННЫЕ
Тысячи людей одновременно хотят забронировать одно и то же место. Нельзя продать его дважды. Покупателю нужно время на оплату, но бронь не должна висеть вечно.

## ТРЕБОВАНИЯ
- Опишите state machine места: available → held (TTL) → booked / released.
- Спроектируйте атомарный захват места (locking / CAS / SELECT FOR UPDATE) — один выигрывает.
- Объясните hold с TTL (резерв на время оплаты, авто-release при таймауте).
- Обоснуйте strong consistency/CP инвентаря (оверселлинг недопустим) и идемпотентность (модули 21/23).

## ФОРМАТ РЕЗУЛЬТАТА
State machine места → атомарный захват → hold с TTL → strong consistency + идемпотентность.

## КРИТЕРИИ ПРИЁМКИ
- [ ] State machine: available → held (TTL) → booked / released.
- [ ] Атомарный захват (locking/CAS/SELECT FOR UPDATE) — один покупатель выигрывает гонку.
- [ ] Hold с TTL (резерв на оплату, авто-release при таймауте — нет вечной заморозки).
- [ ] Strong consistency/CP инвентаря (оверселлинг недопустим — модуль 21) + идемпотентность (модуль 23, повтор не двоит бронь).

## ПОДСКАЗКА
State machine: available→held(TTL)→booked/released. Атомарный захват (locking/CAS/SELECT FOR UPDATE). Hold с TTL + авто-release. Strong consistency/CP (21) + идемпотентность (23).
