# Задача 05 — Модуль 48: Интеграция платежа

## ЗАДАНИЕ
Спроектируйте оплату брони, переиспользуя решения payment-модуля.

## ВХОДНЫЕ ДАННЫЕ
После hold места (TTL) пользователь оплачивает. Платёж проходит через внешний PSP, может повторяться (retry), и не должен ни задвоить списание, ни оставить место в неопределённом состоянии.

## ТРЕБОВАНИЯ
- Опишите связь hold → оплата → booked (или release при неоплате/таймауте).
- Переиспользуйте идемпотентность платежа (idempotency key против double-charge — модуль 40/23).
- Опишите распределённую транзакцию бронь+оплата через Saga + компенсации (модуль 23): неоплата → release места.
- Упомяните ledger/аудит платежей (модуль 40).

## ФОРМАТ РЕЗУЛЬТАТА
hold→оплата→booked/release → идемпотентность → Saga+компенсации → ledger/аудит.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Связь: hold (TTL) → оплата → booked; неоплата/таймаут → release места.
- [ ] Идемпотентность платежа (idempotency key против double-charge — модули 40/23).
- [ ] Распределённая транзакция бронь+оплата через Saga + компенсации (release при сбое — модуль 23).
- [ ] Упомянут ledger/аудит платежей (модуль 40).

## ПОДСКАЗКА
hold→оплата→booked / release при неоплате. Idempotency key против double-charge (40/23). Saga+компенсации: сбой оплаты → release места. Ledger/аудит (40).
