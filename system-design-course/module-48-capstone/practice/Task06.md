# Задача 06 — Модуль 48: Browsing и search слой

## ЗАДАНИЕ
Спроектируйте слой просмотра каталога и поиска событий (read-heavy часть).

## ВХОДНЫЕ ДАННЫЕ
До старта продаж и между ними пользователи массово просматривают каталог событий и ищут по названию/городу/дате. Это read-heavy нагрузка, отдельная от покупки.

## ТРЕБОВАНИЯ
- Спроектируйте просмотр каталога через кэш + CDN (модули 9/10) — разгрузка origin.
- Спроектируйте поиск событий через inverted index (модуль 17).
- Объясните, почему этот слой отделён от покупки/инвентаря (разные профили).
- Покажите, как кэш/CDN помогают пережить спайк просмотра перед on-sale.

## ФОРМАТ РЕЗУЛЬТАТА
Просмотр (кэш/CDN) → поиск (inverted index) → отделение от покупки → роль в спайке.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Просмотр каталога через кэш + CDN (модули 9/10).
- [ ] Поиск событий через inverted index (модуль 17).
- [ ] Объяснено: read-heavy слой отделён от correctness-критичной покупки (разные профили).
- [ ] Показано: кэш/CDN поглощают спайк просмотра перед on-sale.

## ПОДСКАЗКА
Просмотр → кэш+CDN (9/10). Поиск → inverted index (17). Отделить от покупки (read-heavy vs correctness). Кэш/CDN поглощают спайк просмотра.
