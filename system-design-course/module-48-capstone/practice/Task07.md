# Задача 07 — Модуль 48: Карта синтеза (блок → часть курса)

## ЗАДАНИЕ
Постройте карту синтеза: какая подсистема ticketing какой блок из какой части курса использует.

## ВХОДНЫЕ ДАННЫЕ
Capstone собирается из готовых блоков всего курса. Нужно явно показать переиспользование (а не изобретение).

## ТРЕБОВАНИЯ
- Для каждой подсистемы укажите блоки и модули: просмотр, поиск, спайк, инвентарь, оплата, estimation/cost, надёжность, эксплуатация.
- Покажите, что ничего не изобретается заново — всё из Частей 3–7 + Часть 8 (payment).
- Свяжите с методом распознавания архетипов (модуль 46).
- Сделайте вывод: capstone = синтез, а не новые изобретения.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: подсистема → блоки → часть/модуль → вывод о синтезе.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Просмотр→кэш+CDN (9/10); поиск→inverted index (17); спайк→очередь+rate limit (15/16).
- [ ] Инвентарь→ACID/locking/идемпотентность (21/23); оплата→ledger/Saga/idempotency (40/23).
- [ ] Estimation/cost (5–7); надёжность→redundancy/multi-region/шард (24/25/13); эксплуатация→observability/деплой (28/31).
- [ ] Вывод: всё переиспользовано из курса (модуль 46), ничего не изобретено заново.

## ПОДСКАЗКА
Просмотр→9/10; поиск→17; спайк→15/16; инвентарь→21/23; оплата→40/23; estimation/cost→5–7; надёжность→24/25/13; эксплуатация→28/31. Capstone = синтез, не изобретение (46).
