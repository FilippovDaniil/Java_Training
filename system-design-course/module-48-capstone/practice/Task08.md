# Задача 08 — Модуль 48: Узкие места и trade-offs

## ЗАДАНИЕ
Разберите узкие места ticketing и ключевые trade-offs.

## ВХОДНЫЕ ДАННЫЕ
Есть один сверхпопулярный концерт (hot event), тысячи претендентов на лучшие места (контеншн), и конфликт «не продать лишнего vs обслужить всех».

## ТРЕБОВАНИЯ
Разберите минимум 4 узких места/trade-offs:
- hot event (один популярный концерт);
- контеншн на популярных местах;
- оверселлинг vs availability;
- fairness waiting room.
Для каждого — решение/выбор и ссылка на модуль.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: узкое место/trade-off → решение/выбор → модуль.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Hot event → шардирование по событию (13) + waiting room (15) + кэш каталога (9).
- [ ] Контеншн на местах → атомарный захват + hold TTL + очередь (21/23/15).
- [ ] Оверселлинг vs availability → CP-выбор (корректность важнее — модуль 21).
- [ ] Fairness waiting room → честная позиция/токен + rate limit против ботов (16).

## ПОДСКАЗКА
Hot event → шард по событию (13)+waiting room (15)+кэш (9). Контеншн → атомарный захват+hold TTL (21/23). Оверселлинг vs availability → CP (21). Fairness → позиция/токен+rate limit (16).
