# Задача 09 — Модуль 48: Надёжность, безопасность, эксплуатация

## ЗАДАНИЕ
Добавьте к дизайну ticketing слой надёжности, безопасности и эксплуатации (нефункциональные части курса).

## ВХОДНЫЕ ДАННЫЕ
Capstone должен быть не только функционально полным, но и надёжным, безопасным и эксплуатируемым — как реальная продакшен-система.

## ТРЕБОВАНИЯ
- Надёжность/масштаб: redundancy/устранение SPOF (24), multi-region/DR при необходимости (25), шардирование по событию (13).
- Безопасность: защита платежа/PII (PCI DSS, модуль 40/30), rate limit/anti-bot (16/30).
- Эксплуатация: observability/SLO (28), стратегия деплоя (canary/blue-green — 31).
- Покажите, что эти слои — обязательная часть полного дизайна, не «довесок».

## ФОРМАТ РЕЗУЛЬТАТА
Надёжность → безопасность → эксплуатация → вывод (обязательность для полного HLD).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Надёжность: redundancy/SPOF (24), multi-region/DR (25), шард по событию (13).
- [ ] Безопасность: PCI DSS/PII платежа (40/30), rate limit/anti-bot (16/30).
- [ ] Эксплуатация: observability/SLO (28), деплой canary/blue-green (31).
- [ ] Объяснено: надёжность/безопасность/эксплуатация — обязательные слои полного дизайна.

## ПОДСКАЗКА
