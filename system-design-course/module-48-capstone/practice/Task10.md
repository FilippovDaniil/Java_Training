# Задача 10 — Модуль 48 (мини-проект): Полный независимый capstone

## ЗАДАНИЕ
Выберите СВОЙ новый домен (не из Части 8) и спроектируйте его полностью с нуля по фреймворку — финальный capstone курса.

## ВХОДНЫЕ ДАННЫЕ
Выберите домен, не разобранный в Части 8, например: e-commerce checkout, коллаборативный редактор (Google Docs), биржа/stock exchange, online judge, IoT-платформа, food delivery, hotel booking. 60+ минут самостоятельной работы.

## ТРЕБОВАНИЯ
Полный HLD по фреймворку:
1. **Требования** — FR/NFR, доминирующие архетипы (модуль 46).
2. **Estimation** — главная ось (модули 5–7).
3. **API + модель данных**.
4. **High-level** — компоненты, собранные из блоков по карте.
5. **Deep dive** — 1–2 критичных компонента.
6. **Узкие места / trade-offs**.
7. **Надёжность/безопасность/эксплуатация** + документация (C4/ADR — модуль 02).

## ФОРМАТ РЕЗУЛЬТАТА
Полный HLD-документ из 7 разделов + диаграмма (C4) + estimation + 1–2 ADR.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Выбран новый домен (не из Части 8); зафиксирован scope.
- [ ] Требования FR/NFR + определены доминирующие архетипы (модуль 46).
- [ ] Estimation с главной осью (модули 5–7).
- [ ] API + модель данных; high-level из блоков по карте (архетип → блоки).
- [ ] Deep dive в 1–2 критичных компонента.
- [ ] Узкие места/trade-offs; надёжность/безопасность/эксплуатация (24/25/28/30/31).
- [ ] Документация (C4 + ≥1 ADR — модуль 02); показан синтез блоков всего курса.

## ПОДСКАЗКА
Выбери новый домен → требования+архетипы (46) → estimation (5–7) → API+данные → high-level из блоков по карте → deep dive 1–2 → узкие места/trade-offs → надёжность/безопасность/эксплуатация (24/25/28/30/31) → документация C4/ADR (02). Это финальная демонстрация владения методом.
