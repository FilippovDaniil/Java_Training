# Задача 01 — Модуль 49: Два угла ИИ в System Design

## ЗАДАНИЕ
Разграничьте два угла ИИ в системном дизайне и объясните, почему ИИ не отменяет фундамент курса.

## ВХОДНЫЕ ДАННЫЕ
Постановка: «как ИИ связан с system design?». Нужно показать оба угла и место ИИ относительно фундамента.

## ТРЕБОВАНИЯ
- Опишите угол (A): проектировать системы, обслуживающие ML/LLM (inference, RAG, vector DB, feature store).
- Опишите угол (B): ИИ как помощник в дизайне (brainstorm/ревью/генерация) с проверкой.
- Объясните, почему ИИ-компонент проектируется теми же методами курса + новые блоки/ось (GPU cost).
- Сделайте вывод: ИИ дополняет, а не заменяет фундамент (estimation, кэш, очереди, надёжность).

## ФОРМАТ РЕЗУЛЬТАТА
Угол A → угол B → ИИ = те же методы + новые блоки → вывод (дополнение, не замена).

## КРИТЕРИИ ПРИЁМКИ
- [ ] Угол A: системы для ИИ (inference serving, RAG, vector DB, feature store).
- [ ] Угол B: ИИ-помощник в дизайне (brainstorm/ревью/ADR/диаграммы) с обязательной проверкой.
- [ ] Объяснено: ИИ-компонент проектируется методами курса + новые блоки + ось GPU cost.
- [ ] Вывод: ИИ дополняет фундамент (estimation/кэш/очереди/надёжность), не заменяет.

## ПОДСКАЗКА
Угол A: системы для ИИ (inference/RAG/vector DB/feature store). Угол B: ИИ-помощник (brainstorm/ревью), проверять. ИИ = те же методы + новые блоки + GPU cost. Дополнение, не замена.
