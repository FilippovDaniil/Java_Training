# Задача 02 — Модуль 49: Inference serving

## ЗАДАНИЕ
Спроектируйте обслуживание ML-модели под нагрузкой.

## ВХОДНЫЕ ДАННЫЕ
Сервис должен отдавать предсказания модели на запросы пользователей. Модель тяжёлая, работает на GPU, GPU — дефицитный дорогой ресурс.

## ТРЕБОВАНИЯ
- Разграничьте online inference (real-time, latency) vs batch inference (офлайн, пропускная).
- Опишите батчинг запросов (dynamic batching) для повышения throughput на GPU.
- Объясните, почему масштабирование осложнено (тяжёлая модель, медленный прогрев).
- Свяжите с обычным сервисом под нагрузкой (балансировка/узкие места — модули 8/24/26), но с GPU как bottleneck.

## ФОРМАТ РЕЗУЛЬТАТА
Online vs batch → батчинг на GPU → сложность масштабирования → связь с обычным сервисом + GPU bottleneck.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Разграничены online (real-time/latency) и batch (офлайн/пропускная) inference.
- [ ] Батчинг запросов (dynamic batching) для throughput на GPU, баланс с latency.
- [ ] Объяснено: масштабирование осложнено (тяжёлая модель, медленный прогрев, не stateless-мгновенно).
- [ ] Связь с сервисом под нагрузкой (8/24/26), GPU — дефицитное узкое место.

## ПОДСКАЗКА
Online (latency) vs batch (пропускная). Батчинг (dynamic batching) ↑ throughput на GPU. Масштабирование медленное (тяжёлая модель/прогрев). Как обычный сервис (8/24/26), но GPU = bottleneck.
