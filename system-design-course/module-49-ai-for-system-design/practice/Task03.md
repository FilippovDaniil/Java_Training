# Задача 03 — Модуль 49: RAG-архитектура

## ЗАДАНИЕ
Спроектируйте RAG-конвейер (retrieval-augmented generation) и обоснуйте, зачем он нужен.

## ВХОДНЫЕ ДАННЫЕ
Нужен ассистент, отвечающий на вопросы по корпоративной базе знаний (свежие, приватные документы). Переобучать модель на этих данных дорого.

## ТРЕБОВАНИЯ
- Опишите индексацию: документы → embeddings → vector DB.
- Опишите retrieval: запрос → embedding → поиск похожих векторов → релевантный контекст.
- Опишите generation: контекст + запрос → LLM → ответ по данным.
- Обоснуйте, зачем RAG (снижение галлюцинаций, свежие/приватные знания без переобучения).

## ФОРМАТ РЕЗУЛЬТАТА
Индексация → retrieval → generation → зачем RAG.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Индексация: документы → embeddings → vector DB.
- [ ] Retrieval: запрос → embedding → поиск похожих → контекст.
- [ ] Generation: контекст + запрос → LLM → ответ, опирающийся на данные.
- [ ] Обосновано: RAG снижает галлюцинации, даёт свежие/приватные знания без дорогого переобучения.

## ПОДСКАЗКА
Индексация: документы→embeddings→vector DB. Retrieval: запрос→embedding→похожие векторы→контекст. Generation: контекст+запрос→LLM. Зачем: снизить галлюцинации, свежие/приватные знания без переобучения.
