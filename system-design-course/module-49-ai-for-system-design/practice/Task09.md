# Задача 09 — Модуль 49: Узкие места ИИ-систем

## ЗАДАНИЕ
Разберите узкие места систем, обслуживающих ИИ, и предложите решения со ссылками на модули.

## ВХОДНЫЕ ДАННЫЕ
ИИ-фича выросла: высокая нагрузка inference, дорогой GPU, миллиарды векторов в vector DB, нужно оценивать качество модели.

## ТРЕБОВАНИЯ
Разберите минимум 4 узких места:
- inference latency;
- GPU cost;
- масштаб vector index;
- eval/качество модели.
Для каждого — решение и ссылка на модуль.

## ФОРМАТ РЕЗУЛЬТАТА
Таблица: узкое место → причина → решение → модуль.

## КРИТЕРИИ ПРИЁМКИ
- [ ] Inference latency → батчинг + кэш ответов + меньшая модель/квантизация (модуль 26).
- [ ] GPU cost → батчинг, autoscaling, right-sizing, кэш ответов (модуль 7).
- [ ] Масштаб vector index → ANN/HNSW + шардирование векторов (модуль 13).
- [ ] Eval/качество → оценка ML метриками качества, а не обычными pass/fail тестами (модуль 27).

## ПОДСКАЗКА
Inference latency → батчинг/кэш/квантизация (26). GPU cost → батчинг/autoscaling/right-sizing (7). Vector index → ANN/HNSW+шард (13). Eval → метрики качества, не pass/fail (27).
