## Ключ (эталонные ответы)

1. (A) Проектировать системы, обслуживающие ML/LLM (inference serving, RAG, vector DB, feature store); (B) использовать ИИ как помощника в самом дизайне (brainstorm, ревью, генерация ADR/диаграмм).
2. ML/LLM — это компонент системы; estimation, кэш, очереди, надёжность, cost-мышление работают как прежде. ИИ добавляет новые блоки и ось GPU cost, но не заменяет принципы.
3. Выдача предсказаний модели на запросы под нагрузкой — модель за сервисом, отвечающим в online или batch режиме.
4. Online — real-time, важна низкая latency на запрос; batch — офлайн, важна пропускная способность (обработка пачками).
5. Чтобы повысить throughput на GPU: несколько запросов обрабатываются в один проход (dynamic batching), балансируя пропускную и latency.
6. Модель тяжёлая (большая память, медленный прогрев), поэтому новый инстанс поднимается не мгновенно, в отличие от лёгкого stateless-сервиса; GPU — дефицитный ресурс.
7. Retrieval-augmented generation: извлечение релевантного контекста из ваших данных и подача его в промпт LLM. Нужен, чтобы заземлить ответ на данных без дорогого переобучения модели.
8. Индексация (документы → embeddings → vector DB), retrieval (запрос → embedding → поиск похожих → контекст), generation (контекст + запрос → LLM → ответ).
9. Ответ опирается на реально извлечённые документы, а не только на «память» модели; это уменьшает выдумывание фактов и даёт свежие/приватные знания.
10. Keyword (inverted index, модуль 17) ищет по точному совпадению термов; semantic ищет по смысловой близости embeddings (ловит синонимы и перефразировки).
11. Хранилище embeddings (векторов) с поиском по смысловой близости (косинусное расстояние), обычно с приближённым ANN-поиском.
12. Точный k-NN по миллиардам векторов слишком дорог; ANN (HNSW) даёт быстрый приближённый поиск, жертвуя капелькой точности ради скорости.
13. Централизованное хранилище ML-фич: offline (для обучения — батч, история) и online (для inference — низкая latency).
14. Расхождение между тем, как фича считается при обучении и в проде, ведущее к деградации модели. Избегается единым источником фич (feature store) для обоих путей.
15. Генерация LLM медленная; streaming отдаёт ответ по токенам по мере генерации, улучшая воспринимаемую latency/UX вместо ожидания всего ответа.
16. Стоимость считается за токены, поэтому длина промпта и ответа напрямую = деньги; это делает GPU/token cost значимой осью дизайна (модуль 7).
17. Контекст ограничен — нельзя подать всё; применяют RAG (извлечь только релевантное), чанкинг и суммаризацию.
18. Кэширование повторяющегося префикса промпта, чтобы не пересчитывать его каждый раз — дешевле и быстрее.
19. Вывод LLM недетерминирован и может быть некорректным/небезопасным; guardrails и валидация проверяют и ограничивают ответы.
20. LLM/модель — внешняя зависимость, которая может быть недоступна или медленна; fallback/timeout/circuit breaker (модуль 23) не дают ей уронить систему.
21. Inference на GPU/платном API дорог и масштабируется со стоимостью; GPU/token cost может доминировать в TCO, поэтому это отдельная дорогая ось (модуль 7) с оптимизациями (батчинг, кэш, квантизация).
22. ИИ полезен для brainstorm вариантов, ревью, генерации ADR/диаграмм/boilerplate, но галлюцинирует — выдаёт правдоподобное, но неверное. Правило: ИИ — ускоритель и второй взгляд, не источник истины; всё проверять.
23. Inference latency (→ батчинг/кэш/квантизация, модуль 26), GPU cost (→ батчинг/autoscaling/right-sizing, модуль 7), масштаб vector index (→ ANN/HNSW + шардирование, модуль 13), eval/качество (→ метрики качества, модуль 27).
24. ML-систему оценивают метриками качества (точность/релевантность/процент галлюцинаций) на наборах данных, а не бинарными pass/fail тестами обычного кода; качество вероятностно, а не детерминировано.
25. Проектировать ИИ-фичу как обычную распределённую систему + новые блоки: найти узкое место (GPU/inference), решить, нужен ли RAG (свежие/приватные знания → vector DB + semantic search), считать GPU/token cost (новая ось), добавить streaming/guardrails/fallback, оценивать качество (eval); переиспользовать фундамент (кэш/очереди/надёжность). ИИ дополняет, а не заменяет принципы курса.
